# sensitive-paths.txt — extra pre-push secret-guard patterns for THIS project.
# Copy to `sensitive-paths.txt` (drop the .example) to activate. One extended-regex (ERE) fragment per
# line; blank lines and #-comments ignored. These are OR-APPENDED to the built-in guard — they can only
# ADD protection, never remove it. If any line makes the combined regex invalid, the whole file is ignored
# and the base guard is used. Check a path with:  .harness/scripts/loop.sh --guard-selftest <path>
#
# Example (a project deployed on Vercel that also uses AWS):
# (^|/)\.vercel/
# (^|/)\.aws/
# aws-credentials

# local-jobs-specific additions (base guard already covers data/, .env*, chrome-profile,

# this repo's own .gitignore/CLAUDE.md privacy rules):

# .scratch/ — the general home for personal, non-public scratch content kept at the repo root
# (e.g. .scratch/plex-language-fix/, a personal Plex library-maintenance script, moved here
# 2026-07-15 from a dedicated ^plex-language-fix/ pattern — see that history in
# .harness/custom/CLAUDE.md's 2026-07-12 incident entry). ANCHORED to the repo root with ^ so it
# does NOT also match the legitimate tracked workflow at src/workflows/plex-language-fix/ (which
# the old (^|/) prefix false-positived, tripping the pre-push guard on every task touching that
# workflow). Also gitignored (belt-and-braces).
^\.scratch/

# Any gcp-*.json credential file, even one that doesn't literally contain "service-account"
# or "credentials.json" (the base guard's substring matches for those).
(^|/)gcp-.*\.json
