# T237 worklog

## What was changed

Two one-line fixes in `dashboard/app/backlog/page.tsx`:

1. `allDone` filter extended from `t.status === 'done'` to also include `t.done === true` — so needs-human tasks marked done via the overlay appear in the Done section.
2. `human` filter extended with `&& t.done !== true` — so tasks with the overlay `done=true` are excluded from the Needs a human section even though their TASKS.json `status` remains `pending`.

## Verification

- `npm --prefix dashboard run build` → green, no type errors.
- Logic: when `GET /api/backlog` returns a task with `done: true` (the human-done overlay), it now falls into `allDone` (and consequently `done` after filter), and is excluded from `human`. The `refresh()` call on `onMarkDone` was already present, so the re-sectioning is immediate after the API call completes.
