# T319 — enforced output template for projects-sync

Added an enforced markdown output contract for the `project-summarize` stage,
mirroring the `perfumes` workflow's template pattern.

- `src/jobs/projects-sync/project.template.md`: new template — YAML frontmatter
  (`name`, `full_name`, `url`, `language`, `topics`, `status`, `last_pushed`,
  `themes`, `domain`) + fixed `##` sections (`What It Is`, `Tech Stack`, `Status`,
  `Structure`, `Themes & Interests`, `Notable Technical Approaches`, `Sources`),
  designed for cross-project "second brain" querying via `themes`/`domain` and the
  Themes & Interests / Notable Technical Approaches sections.
- `config.ts`: added `templatePath`, overridable via `PROJECTS_SYNC_TEMPLATE_PATH`
  (mirrors perfumes' `PERFUMES_TEMPLATE_PATH`), documented in `.env.example`.
- `project-summarize.ts`: `buildSummaryPrompt` now embeds the template file
  (falling back to an inline `FALLBACK_TEMPLATE` if missing) and instructs Claude
  to follow it exactly. Content-gathering (catalog metadata + README text) is
  unchanged. Added `templateShapeViolations(md)` — checks for the leading `---`
  and every required heading; a violation throws with the missing pieces named,
  routing through the EXISTING catch → `markWorkItem(..., 'failed', ...)` path
  (no new failure mechanism).
- `project-summarize.test.ts`: updated the existing "clones + summarizes" test's
  stub response to be template-conformant (it previously used a bare 2-line
  string, which the new validation now correctly rejects), and added two new
  tests: accept path (conformant stub → success + markdown written) and reject
  path (missing sections → failed, no markdown written).
- Updated `CLAUDE.md`'s projects-sync entry and `README.md`'s projects-sync
  bullet to describe the enforced template contract and `PROJECTS_SYNC_TEMPLATE_PATH`.
