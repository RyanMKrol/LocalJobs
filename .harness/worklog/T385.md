# T385 — Add "Overall" tab to StageIoPanel, default-selected

## What changed

- `dashboard/app/lib/api.ts`: added `StageIoOverall` type (mirrors `StageIo` +
  `outputJobs`) and `api.workflowRunStageIoOverall(id)` hitting
  `GET /api/workflow-runs/:id/stage-io?overall=true` (T384's response shape).
- `dashboard/app/components/StageIoLists.tsx`:
  - New `StageIoOverallBlock` component — same `StageIoColumn`/`.stage-io-columns`
    layout as a per-stage block, fed by `workflowRunStageIoOverall`.
  - `StageIoPanel`'s tab bar now always renders an "Overall" chip (distinct from
    "All stages" and any per-stage chip), and `selectedJob` defaults to the new
    `OVERALL_TAB` sentinel on mount instead of `members[0]?.job_name`.
  - "All stages" chip is still only shown when `members.length > 1` (unchanged
    from before); "Overall" always shows even for a single-member workflow.
- `dashboard/scripts/_dashboard-harness.mjs`:
  - Added `stockDigestStageIoOverall` fixture (root-wave input =
    `stock-portfolio-snapshot`'s own output since it has no predecessor;
    terminal-wave output = `stock-digest-build`'s report) and wired
    `?overall=true` dispatch in the `stock-digest-run/stage-io` route handler.
  - Added a `stage-io-per-stage-tab` FLOW (clicks the `stock-portfolio-snapshot`
    chip) so a non-default state is still captured now that Overall is default;
    left the existing `stage-io-all-stages` flow as-is.
