# T448 — dedupe Overall tab Inputs/Outputs for single-wave workflows

## Fix

`src/api/server.ts`'s `GET /api/workflow-runs/:id/stage-io?overall=true` handler computed
`rootWave` and `outputJobs` independently and passed the RAW `rootWave` as the inputs job set to
`stageIoLists`. For any workflow whose DAG has only one wave (every member job independent, no
`dependsOn` edges — e.g. `plex-space-saver`, `claude-warmer`, `vercel-daily-redeploy`,
`listening-digest`), `rootWave` and `outputJobs` are literally the same job-name array, so the
Inputs and Outputs columns queried and rendered the byte-for-byte same ledger row twice.

Changed the handler to filter `outputJobs` out of the reported/queried input job set:

```ts
const inputJobs = rootWave.filter((j) => !outputJobs.includes(j));
const { inputs, outputs } = stageIoLists(outputJobs, inputJobs, run.id);
return json(res, 200, { inputs, outputs, predecessorJobs: inputJobs, outputJobs, job: '__overall__' });
```

For a single-wave workflow this yields `inputJobs = []`, so `inputs: []` — the dashboard's existing
`StageIoPanel` already renders `emptyText="No inputs recorded this run."` for an empty inputs array
(no dashboard component change needed — confirmed by reading `dashboard/app/components/StageIoLists.tsx`,
which already branches on an empty list; T448 is an API-only fix).

For every existing multi-wave workflow (places, perfumes, movie-recommendations, stock-digest, etc.)
`rootWave` and `outputJobs` are already disjoint job sets, so `inputJobs === rootWave` and behavior
is completely unchanged.
