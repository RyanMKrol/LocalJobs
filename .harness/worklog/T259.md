# T259 worklog

## Problem
The beige React Flow graph box was not evenly inset from the white panel border.
The `.dag-flow-wrap` had `margin: 0 18px 6px` — 18px left/right but 0px top and 6px bottom —
so the beige box was flush at the top and nearly flush at the bottom while having visible side margins.

## Fix
- Added `.dag-panel { padding: 18px; }` to `globals.css` — a scoped class that only applies
  to the DAG-wrapping panel divs, leaving all other `.panel` uses unchanged.
- Changed `.dag-flow-wrap` margin from `0 18px 6px` to `0` — the panel's padding now provides
  the even inset on all four sides.
- Added mobile override: `.dag-panel { padding: 8px; }` in the `@media (max-width: 640px)` block,
  and cleaned up the dag-flow-wrap mobile margin to `0` (was `0 4px 6px`).
- Added `dag-panel` class to the wrapping `<div className="panel dag-panel">` in both:
  - `dashboard/app/workflows/[name]/page.tsx` (workflow detail Graph section)
  - `dashboard/app/workflow-runs/[id]/page.tsx` (workflow-run DAG section)
- No change to `graphH` in `DagFlow.tsx` — the panel padding is outside the dag-flow-wrap so
  the dag-flow-wrap gets its full `graphH` height with no clipping.
