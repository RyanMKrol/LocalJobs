# T580 — re-attempt of T550 (widened scope to include page.tsx)

2026-07-15

## What was done

T550 was blocked (audit-FAILed 4x) because its scope omitted `dashboard/app/page.tsx`,
which had two bare `<span className={`badge ${status}`}>` spans using `statusLabel` — the
same pattern T550 converted elsewhere. This task widens scope to include `page.tsx` and
`workflows/page.tsx` (both listed in T580's SCOPE) so the whole-tree grep-clean Done-when
is actually achievable.

1. **`admin/page.tsx`** — replaced the bare `<span className={`pill ${r.status === 'reset' ? 'on' : 'off'}`}>{r.status}</span>`
   in the reset-results table with `<Pill kind={r.status === 'reset' ? 'on' : 'off'}>{r.status}</Pill>`,
   matching the existing `<Pill>` usage already in the same file (line 158).

2. **`ui.tsx`** — widened `StatusBadge`'s prop type from `{ status: RunStatus }` to
   `{ status: string }`. This was necessary (not just a `RunStatus | WorkflowRunStatus`
   union) because `StageIoLists.tsx`'s `StageIoItem.status` field is a plain `string`
   (work-item ledger statuses like `success`/`failed`/`skipped`/`ignored`, not typed as
   `RunStatus`) — `api.ts` is out of this task's scope, so that field's type couldn't be
   narrowed. `STATUS_LABELS`/`STATUS_EMOJI` are already `Record<string, string>` with a
   graceful fallback (unmapped values, e.g. `ignored`, render as their raw string with no
   emoji — same degradation the old bare spans had), so widening to `string` is safe and
   requires no cast anywhere. Removed the now-unused `RunStatus` import from `ui.tsx`.

3. **Converted bare badge spans to `<StatusBadge>`** at every remaining call site:
   - `page.tsx` (both the workflow-card last-run badge ~line 199 and the recent-runs-table
     badge ~line 275) — dropped the now-unused `statusLabel` import.
   - `workflows/[name]/page.tsx` (~line 296, the Runs table) — dropped the now-unused
     `statusLabel` import.
   - `workflows/page.tsx` (~line 132) — this one was a `<Link>` styled as a badge, not a
     bare `<span>`; wrapped `<StatusBadge>` inside the `<Link>` (`style={{ textDecoration:
     'none' }}`) so the click-through to the workflow run is preserved. Dropped the
     now-unused `statusLabel` import.
   - `components/StageIoLists.tsx` (~line 148, the per-item Inputs/Outputs pill) — now
     renders via `<StatusBadge status={item.status} />` instead of showing the raw string.

4. **Removed all three `as RunStatus`/similar casts**, now unnecessary since `StatusBadge`
   accepts `string` directly:
   - `workflow-runs/[id]/page.tsx` (~line 92, `run.status as RunStatus`) — dropped the
     now-unused `RunStatus` type import.
   - `workflow-runs/[id]/gates/[producer]/[key]/page.tsx` (two spots, ~lines 154/173,
     `gate.state as RunStatus`) — dropped the now-unused `RunStatus` type import.

5. Whole-tree grep sweep (`grep -rnE 'className=\{?`?(pill|badge) '` and the plain-string
   variant) across `dashboard/app` confirms it is now clean except the allowed exceptions:
   `ui.tsx`'s `StatusBadge` own definition, the static `badge queued` / bare `badge`
   "definition" label (non-status badges), and `Pill.tsx`'s doc-comment text.
