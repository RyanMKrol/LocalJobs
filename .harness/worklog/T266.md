# T266 worklog — Backlog: clickable dependency ids

## What was built

Made `dependsOn` task ids clickable on the Backlog page in two places:
1. **"needs:" pill** (Waiting section) — unmet dep ids are now `<button class="dep-id-link">` elements
2. **"depends on:" expanded body line** — all dep ids (met and unmet) are clickable links

When clicked, each dep link:
- Sets `openTaskId` state in `BacklogPage`
- Calls `openTask(id)` which after 50ms: walks DOM ancestors to expand any closed `<details>` section, then `scrollIntoView({ behavior: 'smooth', block: 'center' })`
- Passes `forceExpanded={openTaskId === t.id}` to the target row's `CollapsibleRow` (a `useEffect` triggers `setExpanded(true)`)
- Adds a brief blue highlight flash via `.task-row-highlight` CSS animation (1.6s, then cleared)

## CSS added (globals.css)
- `.dep-id-link` — inline button styled as an accent-coloured underlined mono link; inherits `font-family` from `--font-mono`; hover darkens underline
- `@keyframes task-highlight-flash` + `.task-row-highlight` — 1.6s fade from 25% accent background to transparent

## Harness changes (_dashboard-harness.mjs)
- Added `T001` (pending, Ready) so T040 has an unmet dep visible in the pill
- Updated T040 to depend on `['T001', 'T050']` — T001 unmet (pill), T050 done (body)
- Added `backlog-dep-click` FLOWS entry: clicks T001 in the pill, then expands T040 and clicks T050 in the body (Done section opens, T050 highlighted)
