# T348 — outputJob override for the unified Output section

## What I did
- Added `outputJob?: string` to `WorkflowDefinition` in `src/core/types.ts`, documented per the
  spec (opt-in, manifest-owned override for workflows whose DAG terminal stage never records
  work_items rows).
- `GET /api/workflows/:name/output-items` in `src/api/server.ts`: after computing `lastWave`,
  looks up `getWorkflowDefinition(name)`; if it declares `outputJob` and that name is an actual
  member of the workflow, `workflowTerminalItems` is called with `[def.outputJob]` instead of
  `lastWave`. A missing/misconfigured override silently falls back to `lastWave` — never throws.
- `src/jobs/stocks-sync/stocks-sync.workflow.ts`: set `outputJob: 'stocks-snapshot'` and extended
  the doc comment explaining why (stocks-notify is a pure notify-trigger with no ledger writes).
- CLAUDE.md: documented the `outputJob` override under the T205 convention bullet, and added a
  sentence to the stocks-sync paragraph noting its use.
- Added a unit test block (`src/api/server.test.ts`, T348 section) covering both branches:
  no-override behaves as before (terminal wave, empty here), and override routes to the named
  job's ledger rows even though the true terminal job has none.
