# T503 worklog

## Changes implemented

1. **Added `cacheTtlMs: 79_200_000` to finnhub service definition** (`src/services/finnhub.service.ts`)
   - Set the cache TTL to 22 hours (79,200,000 ms) for company-profile lookups
   - Industry classification data changes infrequently, making this an ideal candidate for caching

2. **Added per-ticker `cacheKey` to finnhub call site** (`src/workflows/stock-digest/stages/stock-sector-lookup.ts`)
   - Modified the `fetchProfile` function to pass a dynamic `cacheKey` of the form `finnhub:profile:${ticker}`
   - The cacheKey is now computed per ticker symbol, allowing repeated lookups of the same ticker within 22h to be served from `service_cache` without re-invoking the live Finnhub API
   - The cache key follows the established pattern from T502 (trading212-instruments caching)
