# T042 — More intuitive remote dashboard access (alias)

Investigated together on 2026-06-23. **Decision: no change — bookmark the existing URL.**

Current access is the raw tailnet IP + port: `http://100.112.137.1:4788/`. Options explored:

- **`tailscale serve --bg 4788`** → `https://ryans-mac-mini-2840.taildc9609.ts.net/`
  (no port, HTTPS, valid auto-cert, tailnet-only; persists across reboots). The README
  already documents this; it just hadn't been turned on (`tailscale serve` showed "No serve
  config").
- **Rename the machine** (free, any name, e.g. `jobs`) → `https://jobs.taildc9609.ts.net/`
  — the leading label is the part you actually read.
- **Rename the tailnet:** on a PERSONAL account you can only pick from Tailscale's
  auto-generated "fun names" (e.g. `giraffa-notothen.ts.net`), NOT a custom name — custom
  tailnet names need a domain/org account. It also locks on first cert use and is
  rate-limited, and changes every device's suffix. Not worth it.

Owner decided the nicer URL isn't worth the setup — will just **bookmark
`http://100.112.137.1:4788/`** and move on. If a port-less HTTPS URL is ever wanted, the
path is: rename the machine to `jobs`, then `tailscale serve --bg 4788` →
`https://jobs.taildc9609.ts.net/` (skip the tailnet rename).
