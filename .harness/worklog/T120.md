# T120: Services edit-limits — adopt workflow-run limit input styling

## What was done

The Services page edit-limits inputs were unstyled (no `type` attribute, just
`className="mono"` with inline `style={{ width: 70 }}`). The CSS base rules
`input[type="text"]` / `input[type="number"]` were already in place for the
dark-theme look — the services inputs just weren't picking them up because they
had no `type` attribute.

**Changes:**

1. `dashboard/app/globals.css`: Extracted a new shared `.limit-input` class
   (`width: 4.5rem`) that replaces the workflow-specific `.run-limit-input`. The
   layout classes (`.run-limit-control`, `.run-limit-label`) are unchanged since
   they're layout-only and not relevant to the services table context.

2. `dashboard/app/services/page.tsx`: Added `type="text"` to all three edit-limit
   inputs and replaced `className="mono"` + inline `style={{ width: 70 }}` with
   `className="mono limit-input"`. All existing behaviour preserved: blank = null,
   non-negative-integer validation, ✕ clear buttons, 'no limit' placeholder.

3. `dashboard/app/workflows/[name]/page.tsx`: Updated the run-limit input from
   `className="mono run-limit-input"` to `className="mono limit-input"` to use the
   shared class.
