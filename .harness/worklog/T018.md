# T018 — Dashboard: edit service rate limits

## Goal
Make a service's `rate_per_minute` / `daily_cap` / `monthly_cap` editable from the
Services page, persisted to the `services` table (overriding the code default), and
reconciled vs code-sync the SAME way the `enabled` toggle is (user override wins /
preserved across sync).

## Key realisation
`callService` enforces limits from the in-memory `ServiceDefinition`
(`getServiceDef`), NOT from the `services` DB row (the row was display-only). So a
persisted edit had to be wired into enforcement too, or it would never take effect.

## Implementation
- **schema.sql / db/index.ts**: added `limits_overridden INTEGER NOT NULL DEFAULT 0`
  to `services`, plus an additive `ALTER TABLE` migration (same pattern as the
  `runs.pipeline_run_id` migration) so existing DBs get the column.
- **store.ts**:
  - `upsertServiceStmt` (used by `syncService`) now preserves the three limit
    columns when `limits_overridden = 1` via `CASE WHEN … THEN <existing> ELSE
    excluded.<col> END`; `description`/`paid` stay code-owned and always refresh.
    When NOT overridden, code defaults still flow on every sync.
  - new `updateServiceLimits(name, {rate_per_minute, daily_cap, monthly_cap})` sets
    the columns + flips `limits_overridden = 1`; returns the updated row, or
    `undefined` if the service doesn't exist (no row created).
  - `ServiceRow` gained `limits_overridden`; added `ServiceLimits` type.
- **core/services.ts**: `effectiveLimits(def)` returns the DB override when
  `limits_overridden`, else the code default; `callService` uses it for the
  monthly/daily quota checks and the per-minute rate. `minIntervalMs`/`maxJitterMs`
  stay code-only.
- **api/server.ts**: `POST /api/services/:name/limits` — validates each field is a
  non-negative integer or null (`''`/null/undefined → null = "no cap/throttle"),
  404 if the service is unknown.
- **dashboard**: `api.updateServiceLimits` + `Service.limits_overridden` /
  `ServiceLimits` in `lib/api.ts`; Services page got per-row "✎ Edit limits" →
  inline number inputs (blank = no limit) with Save/Cancel, an "edited" pill when
  overridden, and client-side validation. Local draft state so the 3s poll doesn't
  clobber typing.

## Reconcile rule (mirrors `enabled`)
- Untouched service: every code-sync re-seeds limits from the def.
- After a UI edit: `limits_overridden = 1`; subsequent code-sync preserves the
  user's limits; `callService` enforces them immediately (no restart needed).

## Tests added (src/db/store.test.ts)
- persistence + flag flip; code-sync preserves override (incl. description/paid
  still refresh); null clears a limit; untouched service follows code default;
  update of unknown service is a no-op; `callService` enforces a tighter override
  (monthlyCap 1 over code default 100 → QuotaExceededError on the 2nd call).
