# T204 worklog

## What was built

Rebuilt the workflow graph (`dashboard/app/components/Dag.tsx`) to show true dependency edges
with accurate concurrency, replacing the wave-barrier layout.

### Problems fixed

1. **False wave barriers**: The old implementation rendered generic arrows between all nodes in
   consecutive waves, implying everything in wave N must finish before wave N+1 starts. This was
   wrong — e.g. in movie-recommendations, `franchise-gaps` is independent of `rec-merge` even
   though both are in the same topological wave.

2. **Wide-graph layout**: The boustrophedon snake layout (T181) was confusing and didn't work well
   for `missing-tv-seasons` at wide desktop widths.

### Approach

- **Topological column layout**: Each node is assigned a column = max(parent columns) + 1 using
  Kahn's algorithm, so column assignment is stable even with complex fan-in/fan-out.
- **SVG bezier edges**: Every `dependsOn` edge is drawn as an SVG cubic bezier from the producer
  node's right-center to the consumer node's left-center (desktop), or bottom/top-center (mobile).
  No generic inter-wave arrows — only actual dependency edges are shown.
- **Wide graphs**: The `.dag` outer container uses `overflow-x: auto` so any graph wider than the
  viewport scrolls horizontally. No more snake layout complexity.
- **Gate marks**: Placed as absolutely-positioned divs at each edge's bezier midpoint, preserving
  all gate links and state colouring.
- **Mobile**: `.dag-inner` switches to `flex-direction: column` on mobile (≤640px), and the
  SVG edge computation detects this and draws vertical bezier curves.

### No library added

The layout is simple enough that no library (dagre/elk/d3-dag) was needed. The topological column
assignment is O(n) via Kahn's algorithm, and node positions are measured via `getBoundingClientRect`
after render.
