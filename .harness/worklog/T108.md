# T108 — Plex new-seasons workflow: investigation + design (done)

Investigated + designed together 2026-06-23, using the owner's (public) PlexUtils repo as a
blueprint and **validating against the live Plex library + TMDB**. Implementation is tracked
as buildable task **T144**.

## Findings (validated live)
- Plex section **5 "TV shows" = 621 shows**; **100% have `tmdb://` GUIDs** → always match by
  GUID, never fuzzy title search. This kills silent mismatches AND the Futurama-class
  "sources disagree on the season count" problem, because Plex + TMDB then read the *same*
  entry with the *same* numbering.
- **TMDB has no hard rate limit**; the whole library (621 × ~2 calls) checks in **~35s** — so
  no daily rotation / per-run cap is needed; re-scan everything each run.
- A real GUID + complete-season check found **54 shows with complete missing seasons**
  (Futurama own S7 → missing S8–S10 confirmed; "Ended" anime like Demon Slayer / Hunter ×
  Hunter / Tokyo Ghoul flagged — so **"Ended" is NOT a safe skip**, revivals happen).
- The **complete-season filter works**: 4 shows excluded because their only new season is
  still airing.

## Design decisions
- Single section, env-overridable (default 5). Request JSON from Plex (items under
  `MediaContainer.Metadata`, not `Directory` — the XML shape PlexUtils uses).
- Match by `tmdb://` GUID; only flag **complete** seasons (every episode aired); **include
  ended shows**.
- **Weekly** schedule; **one digest push per run**; **first run = one big digest** of the
  current backlog; thereafter only newly-completed seasons. Dedup per `(show, season)` — the
  `work_items` ledger is a "**already-notified**" log, NOT a work-done log; snapshot/check
  recompute fresh each run.
- **Not "limitable"** (no `inputKeys()`) — inputs are discovered by scanning Plex, not a
  static file. Scheduled-only, unlimited. All stages key by tmdbId so `root_key = item_key`.
- **Code can be public** (PlexUtils is already a public repo; no secrets in code) → committed
  like places/perfumes, which is also what makes it loop-buildable. Tokens → `.env`, library
  data → `data/` (both gitignored).
