## Build (completed in 88550e2)

Implemented unified output section for all workflow detail pages:

- **`src/db/store.ts`**: Added `OutputItem` interface + `workflowTerminalItems(terminalJobNames)` —
  queries `work_items` for the DAG's final-wave job names with `status='success'`, ordered newest
  first. De-duped by `(job_name, item_key)` by construction.
- **`src/api/server.ts`**: Added `GET /api/workflows/:name/output-items` — resolves the workflow's
  DAG terminal wave, calls `workflowTerminalItems(lastWave)`, returns `{ items, terminalJobs }`.
  Also added `GET /api/workflows/:name/output?job=&key=` (workflow-scoped markdown artifact
  endpoint, same `safeOutputMarkdown` guard as the run-scoped endpoint).
- **`dashboard/app/lib/api.ts`**: Added `WorkflowOutputItem` + `WorkflowOutputItems` types,
  `workflowOutputItems(name)` and `workflowOutput(name, job, key)` client calls.
- **`dashboard/app/components/WorkflowOutputSection.tsx`**: New component — polls
  `workflowOutputItems`, renders a table of produced items with name/key/updatedAt, a "View"
  button for markdown artifacts (opens MarkdownModal with YAML frontmatter parsing). Hides itself
  when there are no items and no error.
- **`dashboard/app/workflows/[name]/page.tsx`**: Added `WORKFLOWS_WITH_SPECIFIC_MANAGERS` set
  (`movie-recommendations`, `missing-tv-seasons`); renders `<WorkflowOutputSection>` for all other
  workflows below the Runs table.
- **`CLAUDE.md`**: Documented the "Unified Output section (T205)" convention — terminal-stage
  work_items ledger IS the output list, markdown artifact workflows vs audit-style exclusions,
  how to add a new workflow.
