# T205 — Unified, de-duped output section on every workflow detail page

## Summary

The implementation was fully built in commit `88550e2` and remains intact on `main`.
The revert `c2f1603` only removed this worklog file, not any code.

## What was built

- **`GET /api/workflows/:name/output-items`** endpoint in `src/api/server.ts` (~line 1174):
  resolves the workflow's DAG terminal wave, queries `workflowTerminalItems(lastWave)` in
  `src/db/store.ts` to return de-duped items from the terminal stage's `work_items` ledger.

- **`workflowTerminalItems(terminalJobNames)`** in `src/db/store.ts` (~line 557):
  queries `work_items` for the given job names with `status='success'`, ordered newest first.
  Items are naturally de-duped by `(job_name, item_key)` (the ledger's UNIQUE constraint).
  Returns `OutputItem[]` with `jobName`, `itemKey`, `name`, `hasMarkdown`, `updatedAt`.

- **`WorkflowOutputSection`** component in `dashboard/app/components/WorkflowOutputSection.tsx`:
  polls `GET /api/workflows/:name/output-items`, renders a table with item name/key/updated,
  and a "View" button for markdown artifacts that opens a `MarkdownModal` (uses the existing
  `GET /api/workflows/:name/output?job=&key=` machinery from T110).

- **`WORKFLOWS_WITH_SPECIFIC_MANAGERS`** set in `dashboard/app/workflows/[name]/page.tsx`:
  excludes `movie-recommendations` and `missing-tv-seasons` (dedicated managers exist).
  Every other workflow gets `<WorkflowOutputSection workflowName={name} />` automatically.

- **CLAUDE.md** "Unified Output section" convention documented under Job conventions.

- **`api.workflowOutputItems(name)`** + types in `dashboard/app/lib/api.ts`.
