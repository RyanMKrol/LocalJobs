# T205 — Unified, de-duped output section on every workflow detail page

## Build summary

All checks pass green: `npx tsc --noEmit`, `npm test`, `npm --prefix dashboard run build`,
`node dashboard/scripts/mobile-check.mjs`.

## What was built

**Convention:** The terminal stage's `work_items` ledger (keyed by `(job_name, item_key)`) is the
output list. Items are de-duped by construction — the ledger has a UNIQUE constraint per pair, so
each produced item appears exactly once regardless of how many runs have processed it.

**Backend:**
- `src/db/store.ts`: `workflowTerminalItems(terminalJobNames)` — queries `work_items` for the
  DAG's final-wave job names with `status='success'`, parses `detail.name` and `detail.markdown`,
  orders newest first. `OutputItem` interface.
- `src/api/server.ts`: `GET /api/workflows/:name/output-items` — builds the workflow DAG, finds the
  last wave, returns `{ items, terminalJobs }`. `GET /api/workflows/:name/output?job=&key=` — reads
  the markdown artifact for one output item (same `safeOutputMarkdown` guard).

**Dashboard:**
- `dashboard/app/lib/api.ts`: `WorkflowOutputItem`, `WorkflowOutputItems` types; `workflowOutputItems(name)`
  and `workflowOutput(name, job, key)` client calls.
- `dashboard/app/components/WorkflowOutputSection.tsx`: Generic output section component. Shows an
  item table (name, key, updated date, optional View button for markdown artifacts). Includes a
  `MarkdownModal` with YAML frontmatter rendering. Hides itself when there's no data.
- `dashboard/app/workflows/[name]/page.tsx`: Renders `<WorkflowOutputSection>` for all workflows
  NOT in `WORKFLOWS_WITH_SPECIFIC_MANAGERS` (i.e. not `movie-recommendations` or
  `missing-tv-seasons`, which have their own richer managers).

**CLAUDE.md:** "Unified Output section on the workflow detail page (T205)" convention block added,
documenting the terminal-stage ledger convention, markdown artifact pattern, and how to add output
for a new workflow.
