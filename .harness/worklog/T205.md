## What was built

Implemented a unified **Output** section on every workflow detail page backed by the terminal-stage
`work_items` ledger.

### Backend (`src/db/store.ts`, `src/api/server.ts`)

- `workflowTerminalItems(terminalJobNames)` in `store.ts`: queries `work_items` for all `success`
  rows whose `job_name` is in the DAG's last wave, parsing `detail.name` and `detail.markdown` from
  each row. De-duped by `(job_name, item_key)` by construction (the ledger has a UNIQUE key per
  pair). Returns items newest-first.
- `GET /api/workflows/:name/output-items` in `server.ts`: builds the workflow DAG, identifies the
  last wave of job names, calls `workflowTerminalItems(lastWave)`, returns `{ items, terminalJobs }`.
  Read-only, DB-only, no paid/remote calls.
- `GET /api/workflows/:name/output?job=&key=` in `server.ts`: returns the markdown artifact content
  for one output item (not scoped to a specific run). Uses the same `safeOutputMarkdown` guard as
  the per-run endpoint — confined to `data/out/` tree, no traversal.

### Frontend

- `WorkflowOutputSection` component (`dashboard/app/components/WorkflowOutputSection.tsx`): polls
  `api.workflowOutputItems(workflowName)` every 10s, renders a table of output items. For items with
  a markdown artifact (`hasMarkdown=true`) shows a "View" button that fetches and opens the content
  in a `MarkdownModal` popover (YAML frontmatter parsing + `react-markdown` rendering). Hides itself
  entirely when there are no items and no error. Uses `.output-section` CSS class (theme-variable-
  driven, mobile-safe).
- `WORKFLOWS_WITH_SPECIFIC_MANAGERS` set in `page.tsx`: `movie-recommendations` and
  `missing-tv-seasons` have their own richer managers (MovieRecsManager/MovieGapsManager/
  MissingSeasonsManager). All other workflows render `<WorkflowOutputSection workflowName={name} />`.
- API client: `workflowOutputItems`, `workflowOutput` + types `WorkflowOutputItem`,
  `WorkflowOutputItems` added to `dashboard/app/lib/api.ts`.
- CSS: `.output-section` and `.output-section table` rules in `dashboard/app/globals.css` — all
  colours from CSS custom properties.
