## Convention

A workflow's "output items" = its TERMINAL stage's `work_items` ledger rows (`status='success'`),
keyed by `(job_name, item_key)`. The ledger's UNIQUE constraint on that pair means items are
**de-duped by construction** — each produced item appears exactly once regardless of how many runs
processed it. The stable per-item key is `item_key`.

## What was built

**Backend**
- `src/db/store.ts`: `workflowTerminalItems(terminalJobNames)` + `OutputItem` — selects the DAG's
  final-wave jobs' successful `work_items`, parses `detail.name` / `detail.markdown`, newest-first.
- `src/api/server.ts`: `GET /api/workflows/:name/output-items` (builds the DAG, finds the last wave,
  returns `{ items, terminalJobs }`) and `GET /api/workflows/:name/output?job=&key=` (reads one
  markdown artifact via the existing `safeOutputMarkdown` guard).

**Dashboard**
- `dashboard/app/lib/api.ts`: `WorkflowOutputItem(s)` types + `workflowOutputItems` / `workflowOutput`.
- `dashboard/app/components/WorkflowOutputSection.tsx`: generic de-duped output table (name, key,
  updated, View → markdown modal with YAML-frontmatter rendering); self-hides when empty.
- `dashboard/app/workflows/[name]/page.tsx`: renders `<WorkflowOutputSection>` for every workflow
  EXCEPT those with richer dedicated managers (`movie-recommendations`, `missing-tv-seasons`), which
  already present consolidated, de-duped output (recs by tmdbId, seasons by `show::season`).

**Docs:** CLAUDE.md gained the "Unified Output section" convention block (terminal-stage ledger +
markdown-artifact pattern + how to add output for a new workflow).
