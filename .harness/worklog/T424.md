# T424 — service-gate projects-sync GitHub calls per unit of work

## What changed

1. `src/workflows/projects-sync/stages/github-sync.ts`
   - `fetchAllRepos` now wraps EACH page's `fetch(url, { headers })` in its own
     `callService('github', ...)` call (added an injectable `ServiceCaller` param,
     default the real `callService`, for hermetic testing).
   - `runGithubSync`'s default `fetchRepos` now points straight at `fetchAllRepos`
     (the callService wrap that used to sit around the WHOLE function was removed
     since it's now per-page inside `fetchAllRepos` itself).

2. `src/workflows/projects-sync/stages/project-summarize.ts`
   - `cloneOrPullRepo` now wraps its git clone/fetch/reset operations in
     `callService('github', ...)` — reuses the same `github` service as the REST
     API calls (no new service file). Added an injectable `ServiceCaller` param
     (default real `callService`) so tests can spy on it without a real git
     network call.

3. Docs: updated `src/workflows/projects-sync/CLAUDE.md`'s two "known gap" notes
   to describe the fixed behaviour instead.

## Tests added

- `github-sync.test.ts`: `fetchAllRepos` test with a 3-page fixture (100, 100, 7
  repos) proving `callServiceFn` is invoked once per page (3 times), not once for
  the whole fetch.
- `project-summarize.test.ts`: `cloneOrPullRepo` test proving the git operation is
  routed through `callService('github', ...)` (spies on the injected
  `ServiceCaller`, does not perform a real git clone).

## Verification

- `npx tsc --noEmit` — clean.
- `npm test` — full suite green: 211 tests / 52 suites, 0 failures (includes the
  2 new tests above, confirmed passing individually).
- No live paid/network calls made during verification — pagination test stubs
  `globalThis.fetch`; the clone-gating test injects a `ServiceCaller` that never
  invokes the real git spawn.
