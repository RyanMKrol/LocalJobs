# T400 — fix brand link wrapping onto two lines

## What was wrong

T323 (Admin nav tab) gave `<nav>` an inline `width: '100%'`, which made it compete
with its `.brand` flex sibling for space inside `header.site .inner` (a flex row
with no `flex-wrap`) at every viewport width, not just mobile — starving `.brand`
and causing "⚙︎ Local Jobs" to wrap onto two lines at desktop widths.

## Fix

- `dashboard/app/globals.css`: added `flex-shrink: 0; white-space: nowrap;` to
  `header.site .brand`, so it always keeps its natural (shrink-to-fit) width and
  `<nav>` only takes the remaining space.
- `dashboard/app/layout.tsx`: corrected the stale inline comment above `<nav>` that
  incorrectly claimed "on desktop the six links already fit on one row so nothing
  wraps" — the real reason was the flex-sizing conflict between `.brand` and `nav`,
  not link-content overflow. The comment now explains the actual fix.
