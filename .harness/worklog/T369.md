# T369 — validation gate between stock-digest stages

## What was built
- `src/jobs/stock-digest/contracts.ts` — new `stockSectorsContract(file)` factory,
  key `stock-sectors`, validating `data/out/sectors.json` written by
  `stock-sector-lookup`. Mirrors the `places`/`perfumes` `fromChecks` idiom but is
  deliberately LENIENT: missing file → ok, `{}` → ok (both legitimate when
  `FINNHUB_API_KEY` is unset or nothing has resolved yet). Fails only on invalid
  JSON, a top-level array instead of an object, or a non-string/non-null value.
- Wired `produces: [stockSectorsContract()]` onto `stock-sector-lookup.job.ts` and
  `consumes: [stockSectorsContract()]` onto `stock-digest-build.job.ts`.
- `src/jobs/stock-digest/contracts.test.ts` — 6 contract-check cases (missing/
  empty/well-formed pass; invalid-JSON/array/bad-value-type fail) + 1
  `buildDag`/`deriveGates` test asserting exactly 1 gate for the 2-stage DAG.
