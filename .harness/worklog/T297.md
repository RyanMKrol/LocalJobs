# T297 — user-editable per-job timeoutMs

## What changed
- `src/db/schema.sql`: added `timeout_ms_overridden` to the `jobs` table.
- `src/db/index.ts`: additive migration adding `timeout_ms_overridden` for existing DBs.
- `src/db/store.ts`: `upsertJobStmt`/`syncJob` now preserve `timeout_ms` across a
  code re-sync when overridden (same CASE pattern as workflow `schedule`/
  `max_concurrency`); added `timeout_ms_overridden` to `JobRow`; added
  `updateJobTimeout(name, timeoutMs)`.
- `src/api/server.ts`: added `POST /api/jobs/:name/timeout { timeoutMs }` (ms,
  non-negative integer, 400/404 validated), behind the existing mutation guard.
- `dashboard/app/lib/api.ts`: added `api.updateJobTimeout(name, timeoutMs)`.
- `dashboard/app/jobs/[name]/page.tsx`: replaced the read-only ms display with an
  editable number input (seconds) + Save/Cancel, mirroring the workflow detail
  page's Schedule editor. Converts seconds ↔ ms at the API boundary.
