# T116 worklog — Workflow-run IO: 5 toggleable output line-item styles

## Attempt 1 — 2026-06-23

### What was built

Added five distinct output line-item styles for the workflow-run IO panel's Output column, with a persisted style selector (same pattern as T99 gate styles / T113 caret styles).

**Styles implemented:**

1. **Filename** — bare short filename + a separate "Preview" button
2. **Title** — file icon 📄 + document title only, entire cell clickable
3. **Excerpt** — title + short excerpt + filename (the original T110 style, now the default)
4. **Meta** — filename as an underlined link + file-size metadata (uses `out.bytes`)
5. **Chip** — compact inline chip (icon + filename); excerpt shown in `title` tooltip; click opens popover

**Files changed:**
- `dashboard/app/ui.tsx`: added `OUTPUT_STYLES` const, `OutputStyle` type, `useOutputStyle()` hook (localStorage key `localjobs.outputStyle`, default `excerpt`)
- `dashboard/app/workflow-runs/[id]/page.tsx`: imported new hook/types; replaced `OutputCell` with 5-branch renderer accepting `outputStyle` prop; updated `IoPanel` to host the style selector bar and pass `outputStyle` down; added `fmtBytes()` helper
- `dashboard/app/globals.css`: added `.output-style-bar/label/btn` selector classes; added `.out-filename`, `.out-title`, `.out-meta`, `.out-chip` style blocks

**All style branches:**
- Keep the click-to-preview popover working (`onOpen` callback → `MarkdownModal`)
- Handle no-artifact / still-loading fallback gracefully (rendered before the style branch)
- Mobile-responsive (no overflow at 402px — verified via `mobile-check.mjs`)
