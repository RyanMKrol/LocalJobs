# T150 worklog

## What was done

Fixed the final-status rollup in `src/core/workflow-executor.ts` so that a workflow run where
no member stage succeeded is recorded as `failed` rather than `partial`.

**Before:** `partial` was the catch-all for "not every stage succeeded", which incorrectly
lumped "some succeeded, some didn't" with "nothing succeeded at all".

**After:** three-way logic:
- all `success` → `success`
- at least one `success` but not all → `partial`
- no `success` (all failed / skipped / any mix) → `failed`
