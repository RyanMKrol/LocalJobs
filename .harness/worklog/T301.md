# T301 — stocks-snapshot: fetch the optional ISA account too

## What changed

- `src/jobs/stocks-sync/stages/stocks-snapshot.ts`:
  - `NormalizedPosition` gained `account: 'invest' | 'isa'`.
  - Added `positionKey(account, ticker)` — the composite ledger/lookup key, `${account}:${ticker}`.
  - `normalizePosition(pos, account = 'invest')` now tags the account.
  - `runStocksSnapshot` reads `TRADING212_ISA_API_KEY_ID` / `TRADING212_ISA_API_SECRET_KEY`; when
    BOTH are set it calls the same injectable `fetchPortfolio` a second time with the ISA
    credentials and tags the results `account: 'isa'`. When either is unset, behavior is unchanged
    from pre-T301 (Invest only, exactly one fetch call).
  - `markWorkItem` now keys on `positionKey(account, ticker)` instead of bare `ticker`.
  - `stocksSnapshotInputKeys()` returns composite keys.
  - `buildPortfolioMarkdown` adds an `Account` column (`Invest` / `ISA`).
- `src/jobs/stocks-sync/stages/stocks-watch.ts`: `BreachLine` gained `account`; the per-run check
  key and the `::notified` episode key are both now `positionKey(account, ticker)`, so the same
  ticker held in both accounts tracks independently. `formatBreachLine` labels ISA positions
  `TICKER (ISA)`.
- `src/jobs/stocks-sync/stages/stocks-notify.ts`: `formatBreachLine` updated to match (ISA label).
- Tests updated/added across all three `*.test.ts` files for the new `account` field, the composite
  key, the ISA-set/ISA-unset fetch paths, and a same-ticker-in-both-accounts non-collision case.
- `CLAUDE.md` (`stocks-sync` description) and `README.md` (`stocks-sync` bullet) updated to document
  the optional ISA account and its env vars (already present, undocumented in `.env.example`).
