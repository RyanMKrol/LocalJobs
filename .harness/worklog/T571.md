## 2026-07-15

### What I built
Added ONE combined `markWorkItem('success', ‚Ä¶)` visibility row per run to the snapshot,
each recommender branch, and the merge stage of BOTH the `movie-recommendations` (movies) and
`tv-recommendations` (tv-recs) workflows, so the run page's `StageIoPanel` (Input/Output panel)
shows what those 10 stages produced instead of being blank. Keyed by the run's ISO date
(`dayKey(now)`) ‚Äî a same-day manual re-run upserts the same row (stocks/stock-digest per-run-key
pattern). Purely additive: the notify-stage "have I recommended/ignored this?" ledger and the
non-limitable, re-scan-fresh idempotency model are untouched.

Detail shapes (per spec):
- snapshot ‚Üí `{ name, movies|shows: <count>, path: snapshot.json, format: 'json' }`
- each branch ‚Üí `{ name, suggestions: <n>, path: recs/<branch-id>.json }` (recorded under the
  branch's own job name)
- merge ‚Üí `{ name, balanced: <n>, path: recommendations.json, format: 'json' }`

Paths pass the natural absolute path; `markWorkItem`/`toStoredPath` normalizes them
workflows-root-relative (T447). Counts are read back from the file each stage already wrote.
