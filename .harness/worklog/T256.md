# T256 worklog ‚Äî projects-sync workflow

## What was built

Created `src/jobs/projects-sync/` as a tracked scheduled workflow (published like
`places`/`perfumes` examples). Created `src/services/github.service.ts` as the
token-authenticated GitHub rate-limit service.

### Files created

- `src/services/github.service.ts` ‚Äî GitHub REST API service definition; conservative
  limits (30 req/min, 200 daily, 3 000 monthly); reads `GITHUB_RATE_PER_MIN`,
  `GITHUB_DAILY_CAP`, `GITHUB_MONTHLY_CAP` from env.
- `src/jobs/projects-sync/projects-sync.workflow.ts` ‚Äî workflow manifest; daily 05:00
  schedule; single-stage `github-sync` job.
- `src/jobs/projects-sync/stages/github-sync.job.ts` ‚Äî job definition.
- `src/jobs/projects-sync/stages/github-sync.ts` ‚Äî core sync logic with injectable
  deps: `fetchAllRepos` (paginates GitHub, authenticated if `GITHUB_TOKEN` set),
  `filterAndSortRepos` (no forks, no archived, no private; sorted by pushed_at desc),
  `repoToTableItem` (maps to `ProjectsTableItem` with `repoId = String(repo.id)`),
  `runGithubSync` (orchestrates fetch‚Üífilter‚Üíupsert loop, marks items in work_items
  ledger, re-upserts every run so fields refresh).
- `src/jobs/projects-sync/stages/github-sync.test.ts` ‚Äî hermetic tests: no live
  GitHub API calls, no live AWS writes; stub fetcher + stub putter + scratch DB.

### Env vars added to `.env.example`

- `GITHUB_TOKEN` ‚Äî PAT with at least `public_repo` scope.
- `GITHUB_RATE_PER_MIN`, `GITHUB_DAILY_CAP`, `GITHUB_MONTHLY_CAP` (commented out).

## Design decisions

- **Re-upsert every run** (unlike workouts-sync which skips done items). Repos change
  (stars, description, language, topics) so refreshing fields on each run is the right
  idempotency shape. The work_items ledger still records each upsert (last-run wins).
- **Paginated fetch**: 100 per page (GitHub max), loops until a short page signals end.
- **No `data/` output files**: unlike places/perfumes, projects-sync is a pure
  DynamoDB pipeline with no local artifact ‚Äî the table IS the output.
