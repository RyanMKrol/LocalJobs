# T350

## What was built

Added generic price-pair rendering to the workflow-run IO panel
(`dashboard/app/workflow-runs/[id]/page.tsx`) for work-item `detail` blobs that carry
numeric `currentPrice`/`averageBuyPrice` (the shape `stocks-snapshot` now records per
T349) ‚Äî without hardcoding anything stocks-specific by name:

- `priceDetail(detail)` ‚Äî returns `{ current, avg }` when both fields are present as
  numbers, else `null`.
- `formatPriceDetail(p)` ‚Äî `"$198.32 (avg $150.00, +48.32 / +32.2%)"` style string.
- Wired into the input-cell render block (the row's `<td>` that already shows
  `itemLabel`/`detail.name`) ‚Äî renders as a sibling line whenever `priceDetail(row.inputDetail)`
  is non-null. This is the path that matters for stocks-sync: `stocks-snapshot` is the
  DAG's first wave, so its detail always lands on the INPUT side.
- Wired into `OutputCell`, in both the "no markdown path" branch and the
  markdown-preview branch, so a workflow where price-shaped detail lands on the OUTPUT
  side instead is also covered.
