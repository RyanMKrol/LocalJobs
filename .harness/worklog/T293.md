# T293 ŌĆö Workflows list page: group into category sections

## What changed
- `dashboard/app/workflows/page.tsx`: `category` is a manifest-owned field (T292) already returned
  by `GET /api/workflows` (spread straight from the DB row) but not yet added to the `Workflow`
  TS interface in `dashboard/app/lib/api.ts` (out of this task's scope, so left untouched). Added a
  local `WorkflowWithCategory = Workflow & { category?: string }` type and cast the polled data to
  it, rather than editing `api.ts`. Replaced the single flat table with a `CATEGORY_GROUPS`
  constant (`second-brain` ŌåÆ "Second brain", `recommendations` ŌåÆ "Recommendations",
  `regular-maintenance` ŌåÆ "Regular maintenance", `uncategorized` ŌåÆ "Uncategorized", in that order).
  Workflows are partitioned by `p.category || 'uncategorized'` into these groups; each non-empty
  group renders its own `.panel` with an `<h2>` heading and the EXACT same table markup/columns
  (Workflow / Enabled / Notifications / Stages / Schedule / Last run / Next / Run) as before ŌĆö same
  row content, same run button, same stuck-item popover trigger, same links. The Uncategorized group
  is only rendered when non-empty (per spec ŌĆö avoids permanent empty clutter while still catching a
  future manifest that omits `category`). The zero-workflows empty state ("No workflows yetŌĆ”") is
  preserved as its own panel.
- `dashboard/scripts/_dashboard-harness.mjs`: the `workflow()` fixture factory now defaults
  `category: 'second-brain'`. The `/api/workflows` fixture list now returns 5 workflows spanning all
  4 groups: `places`/`perfumes` (second-brain), `movie-recommendations` (recommendations),
  `workouts-sync` (regular-maintenance), `legacy-job` (uncategorized) ŌĆö so the workflows-list
  screenshot shows all 4 section headings with their workflows underneath.
