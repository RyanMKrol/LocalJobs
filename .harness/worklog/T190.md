# T190 worklog — Dark/Light/System mode picker

## What was built

Replaced the automatic time-of-day light/dark mode with an explicit **Dark / Light / System** picker in the 🎨 `ThemeControls`.

### Changes

**`dashboard/app/ui.tsx`**
- Added `ModeId` type (`'dark' | 'light' | 'system'`) and `MODES` constant.
- Added pure `resolveMode(stored, osPrefersDark)` helper: maps stored choice → effective `data-mode`. Default (null/system) follows OS.
- Added `useMode()` hook: reads `localjobs.mode` from localStorage on mount, sets `data-mode`, and attaches a `matchMedia('(prefers-color-scheme: dark)')` change listener so System mode reacts live. On update, writes localStorage and sets `data-mode`.
- Replaced `useTimeMode()` call in `ThemeControls` with `const [mode, setMode] = useMode()`.
- Added a "Mode" section in the picker UI (Dark / Light / System buttons).
- Removed the "Light/dark follows your local time of day" hint; added "System follows your OS dark/light preference." under the Mode section.

**`dashboard/app/layout.tsx`**
- Updated `PREPAINT` inline script: reads `localjobs.mode`; forces dark when `mo==='dark'`, light when `mo==='light'`, else reads `prefers-color-scheme`. Eliminates the `new Date().getHours()` time-of-day logic.

**`CLAUDE.md`**
- Updated the `data-mode` bullet to describe Dark/Light/System picker, `localjobs.mode` key, and `resolveMode` / `useMode` instead of `modeForHour` / `useTimeMode`.

## Verification

- `npx tsc --noEmit` → clean
- `npm test` → 4/4 pass
- `npm --prefix dashboard run build` → success
- `node dashboard/scripts/mobile-check.mjs` → all 9 pages pass (no overflow)

Note: `modeForHour` and `useTimeMode` are removed from the codebase. The pure `resolveMode` helper is straightforward to unit-test but the project's test setup is server-side Node (no DOM/localStorage); the logic is covered by its simplicity and the existing type system.
