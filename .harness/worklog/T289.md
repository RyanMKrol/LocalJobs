## API research (live, against the real Trading212 account)

Started at https://docs.trading212.com/api per the task instructions, but the docs
site (Redocly-hosted SPA) returned no fetchable endpoint content via WebFetch (title
only / login page). Fell back to reading the source of a well-known open-source
TypeScript client (`bennycode/trading212-api` on GitHub, MIT, actively maintained,
directly cites the same `t212public-api-docs.redoc.ly` docs in its source comments)
to confirm:

- Base URL: `https://live.trading212.com` (live account) ‚Äî the client also supports
  a `demo.trading212.com` practice base, unused here since this is the live account.
- Endpoint: `GET /api/v0/equity/portfolio` ‚Äî Personal Portfolio, all open positions.
- Response shape (per-position): `ticker`, `quantity`, `averagePrice`, `currentPrice`,
  `ppl`, `fxPpl`, `initialFillDate`, `frontend`, `maxBuy`, `maxSell`, `pieQuantity`.
  `currentPrice` is present directly on the position ‚Äî no separate market-data call
  needed, confirming the task's expectation.

**One real GET call was made** against the live account (credentials from the local,
gitignored `.env`) to confirm the exact shape and auth scheme before finalizing the
implementation:

- A bare `Authorization: <TRADING212_API_KEY_ID>` header (as the library's source
  literally does) returned `401 Unauthorized` against both the live and demo base
  URLs, and both env vars tried individually as that header.
- **What actually worked:** HTTP Basic auth, `TRADING212_API_KEY_ID` as the
  username and `TRADING212_API_SECRET_KEY` as the password
  (`Authorization: Basic base64(keyId:secretKey)`) returned `200 OK` with a real
  JSON array of open positions matching the shape above exactly (two real
  positions were returned: `AMD_US_EQ` and `MU_US_EQ`, both with sane
  quantity/averagePrice/currentPrice/ppl values). This is the auth scheme wired
  into `fetchPortfolio` in `stocks-snapshot.ts`.
- Confirmed the endpoint is GET-only in this implementation and in the service ‚Äî
  no POST/PUT/PATCH/DELETE call was made or is present anywhere in the new code
  (`grep -rniE "method:\s*['\"](post|put|patch|delete)['\"]" src/jobs/stocks-sync/
  src/services/trading212.service.ts` returns nothing, verified below).
- Only the one live call above was made; `runStocksSnapshot`/tests otherwise use an
  injected fake fetcher (no further live calls during test runs).

The real response was NOT committed anywhere (no data/ files are tracked); this is
recorded here as the observed verification evidence only.
