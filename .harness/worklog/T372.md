# T372 — openfigi service (ISIN -> ticker resolution)

## What I built

- `src/services/openfigi.service.ts`: default-exports a `ServiceDefinition` mirroring
  `finnhub.service.ts`'s shape (`name: 'openfigi'`, `category: 'api'`, `paid: false`,
  env-overridable `ratePerMinute`/`dailyCap`/`monthlyCap` — defaults 20/min, 2000/day,
  20000/month). Also exports `fetchOpenFigiTickers(isins, apiKey?)`, an injectable
  helper that POSTs `https://api.openfigi.com/v3/mapping` with an array body
  (`{ idType: 'ID_ISIN', idValue }` per ISIN), sets `X-OPENFIGI-APIKEY` only when an
  API key is provided, and maps the response array back to a `(string | null)[]` in
  input order — an unmapped ISIN (`{ warning: ... }`, no `data`) resolves to `null`
  rather than throwing.
- `.env.example`: documented `OPENFIGI_API_KEY` (optional placeholder) plus the three
  rate/cap overrides, in a new `--- openfigi service ---` block.
- `CLAUDE.md`: added `openfigi` to the services `category` controlled-values list and
  a one-line service mention (mirroring how other services are documented) ahead of
  the "Services (cross-job shared APIs)" convention paragraph.
