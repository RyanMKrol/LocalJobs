2026-07-12

## What happened

A prior attempt at this task landed in two commits (efb3950, 1f2daf6) and was audited PASS
(`.harness/worklog/T468.audit.md`), but a follow-up `git revert` of only the second commit
(b9ca96f) left `main` in a broken state: `movies.workflow.ts` still referenced the `franchise-gaps`
job, but its `.ts`/`.job.ts`/`.test.ts` files had been deleted (never restored by the revert),
and the `missing-movies` workflow didn't exist. `npm test` failed at `registry.test.ts` with
`orphan job(s): rec-merge, movie-gaps-notify, rec-auteur, ...` (movie-snapshot's downstream jobs
had no workflow membership at all once `movie-recommendations`'s own manifest referenced a
non-existent stage).

This session re-applied the full, previously-audited T468 diff (the tree state as of 1f2daf6)
onto the clean `main`, confirmed it's still correct against the current spec, and re-verified.

## Result

- `movie-recommendations`: `movie-snapshot` → 8 rec branches → `rec-merge` → `movie-recs-notify`
  (recs-only terminal notify, new job name `movie-recs-notify`).
- `missing-movies` (new workflow, own folder): its own `plex-movie-snapshot` → `franchise-gaps`
  (job name unchanged) → `movie-gaps-notify` (job name + `NOTIFY_JOB` ledger key unchanged — no
  `work_items` migration needed). Weekly schedule per the earlier design decision (`0 9 * * 1`).
- `RECS_JOB = 'movie-recs'` unchanged.
- Every DAG edge in both workflows has a produces/consumes gate (verified via
  `src/core/gate-coverage.test.ts`).
- `src/workflows/movies/config.ts` keeps a `gapsOut` compatibility re-export (pointing at
  `missingMoviesConfig.gapsOut`) and `stages/notify.ts` re-exports the legacy `NOTIFY_JOB`/`gapKey`
  constants, purely so `src/api/server.ts` (out of this task's scope) keeps resolving without a
  code change — documented inline and in both workflows' `CLAUDE.md`.
- Root `CLAUDE.md` "Shipped example workflows" table split into two rows (movie-recommendations:
  recs-only; missing-movies: franchise-gap audit only).
