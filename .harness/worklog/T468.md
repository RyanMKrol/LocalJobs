# T468 — split movie-recommendations into recs-only + missing-movies (gaps-only)

**Date:** 2026-07-12

## Starting state (important — this was NOT a clean baseline)

`main` HEAD was in a genuinely broken state before this attempt: two prior build+revert cycles for
this same task (see `git log --oneline` around 3918d8f/446c818/859ee3a/22603f8/b9ca96f/1f2daf6/efb3950)
left the tree with `movies.workflow.ts` still referencing a `franchise-gaps` job whose
`franchise-gaps.ts`/`.job.ts`/`.test.ts` files no longer existed on disk — a botched revert-of-a-revert.
`npx tsc --noEmit` passed (job wiring isn't type-checked), but `npm test`/CI failed hard: the registry
reported `franchise-gaps`'s dependents (`rec-merge`, `movie-gaps-notify`, all 8 rec branches,
`movie-snapshot`) as orphaned jobs. I recovered the last known-good `franchise-gaps.*` content from
commit `1ac5dfe` (pre-split) as the starting point for the moved files, rather than trying to
reconstruct/rebase the broken revert chain.

Root cause of the TWO prior CI failures (confirmed via `gh run view --log-failed` on the actual failed
runs): a test in the new `missing-movies` workflow threw `ENOENT` opening
`missing-movies/data/out/franchise-gaps.json` — a test that read/wrote the real `data/out/` path
instead of a tmpdir override. `data/` is gitignored, so a fresh CI checkout has no `data/` directory at
all; any test must use `mkdtempSync` + explicit path overrides (the established convention every other
job test in this repo already follows) rather than touching `moviesConfig`/`missingMoviesConfig`'s real
paths. All new tests here use `mkdtempSync(join(tmpdir(), ...))` for every input/output path — verified
by running each new/moved test file individually before running the full suite.

## What changed

- **New `src/workflows/missing-movies/` workflow**: own `config.ts`/`contracts.ts`/`lib.ts`/`CLAUDE.md`,
  own `plex-movie-snapshot` job (writes only `snapshot.json`, no `taste-profile.json` — per the T467
  design decision, verified `franchise-gaps.ts` never read one), the moved `franchise-gaps` job
  (unchanged job name), and a gaps-only `movie-gaps-notify` job (unchanged job name + `NOTIFY_JOB`
  ledger key `'movie-gaps-notify'`). DAG: `plex-movie-snapshot → franchise-gaps → movie-gaps-notify`,
  scheduled weekly (`'0 9 * * 1'`, matching `missing-tv-seasons`'s cadence per the design decision —
  deliberately more frequent than the old monthly cadence).
- **Trimmed `movie-recommendations`**: `movies.workflow.ts` drops the `franchise-gaps` member entirely;
  DAG is now `movie-snapshot → 8 rec branches → rec-merge → movie-recs-notify` (new terminal job name,
  recs-only). `contracts.ts` drops `franchiseGapsContract` (moved). `notify.ts`/`notify.job.ts`/
  `notify.test.ts` rewritten recs-only (`buildDigest`/`writeReport` no longer take a gaps param; report
  file renamed `recommendations.md` since the old shared `franchise-gaps.md` name now belongs to the
  gaps workflow's own report).
- **Pure logic (`movies.ts`) is NOT duplicated.** `missing-movies/stages/{snapshot,franchise-gaps}.ts`
  import `buildMovieSnapshots`/`buildOwnedSet`/`collectionGaps`/`collectionOwnedExample` from
  `movie-recommendations`'s `movies.ts` (cross-workflow reuse of pure helpers is an existing repo
  convention — `movies.ts` itself imports `extractTmdbId` from `missing-tv-seasons/plex.ts`). Types
  (`MovieSnapshotFile`, `FranchiseGap`, `FranchiseGapsFile`, `TmdbCollectionDetail`, `TmdbMovieDetail`)
  are likewise imported from `movies/types.ts` rather than redeclared, to avoid type drift between the
  two workflows.
- **Ledger continuity verified, zero migration**: `NOTIFY_JOB = 'movie-gaps-notify'`, `gapKey()`,
  `RECS_JOB = 'movie-recs'`, `recKey()`, and the `franchise-gaps` job name are all byte-identical to
  before the split.
- **Compat shims for out-of-scope `src/api/server.ts`** (per the T467 design record, T469 is the queued
  follow-up that finishes the relocation):
  - `moviesConfig.gapsOut` (in `movies/config.ts`) is aliased to `missingMoviesConfig.gapsOut` so
    `server.ts`'s direct `readFileSync(moviesConfig.gapsOut, ...)` reads for `/api/movie-gaps` keep
    resolving to the real (now-relocated) file.
  - `movies/stages/notify.ts` re-exports `NOTIFY_JOB`/`gapKey` from
    `missing-movies/stages/notify.ts`, so `server.ts`'s
    `import { NOTIFY_JOB as MOVIE_GAPS_JOB, gapKey } from '../workflows/movies/stages/notify.js'`
    still resolves without editing `server.ts`.
  - Documented both shims, and the fact that the franchise-gaps management UI still renders on the
    `movie-recommendations` dashboard page (not yet `missing-movies`) pending T469, in both workflows'
    `CLAUDE.md` and the root `CLAUDE.md`.
- **Docs**: rewrote `movies/CLAUDE.md` (recs-only), authored `missing-movies/CLAUDE.md` (new), updated
  the root `CLAUDE.md` workflow table (17 workflows now, two rows for movie-recommendations/
