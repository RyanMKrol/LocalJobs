# T468 — split movie-recommendations into recs-only + missing-movies (gaps-only)

## Starting state

Main HEAD (`c84014d`) was itself broken: four build+revert cycles had left `movies.workflow.ts`
referencing a `franchise-gaps` job whose files no longer existed anywhere in the tree (neither
`movies/stages/` nor a `missing-movies/` folder) — `npm test` failed at `registry.test.ts` with
"standalone jobs are not allowed … Orphaned job(s): rec-merge, movie-gaps-notify, rec-auteur, …".
`gh run list` confirmed **every** T468 attempt AND every revert of it failed CI on `main`.

## Root cause of the repeated CI failures (found via `gh run view --log`)

All 4 failed CI runs died identically: `scripts/run-tests.ts` reported `src/api/server.test.ts`
"test file threw: ENOENT … open '.../missing-movies/data/out/franchise-gaps.json'", then continued
importing every other test file (its own catch-and-continue behavior) but left `process.exitCode`
set — so the tail of the log looks all-green and misleads a quick read.

Traced to `src/api/server.test.ts`'s pre-existing (out-of-T468-scope) "movie-gaps endpoints (T145)"
test block:
```
const gapsPath = moviesConfig.gapsOut;       // aliased to missingMoviesConfig.gapsOut (a DIFFERENT dir)
mkdirSync(moviesConfig.outDir, { recursive: true });   // creates the WRONG (movies/) dir
writeFileSync(gapsPath, ...);                          // ENOENT: missing-movies/data/out/ never created
```
The prior attempts correctly introduced the `moviesConfig.gapsOut → missingMoviesConfig.gapsOut`
compat alias (needed since `server.ts` is out of scope), but this pre-existing test still
`mkdirSync`'d the OLD `moviesConfig.outDir` — which no longer contains the aliased path. This is
invisible on a dev box that has ever run the tests before (the target dir already exists from a
prior run) but deterministic in CI's always-fresh checkout, which is exactly the discrepancy that
made local `npm test` runs look clean before every push. Confirmed by isolating in a scratch git
worktree at the last failed attempt's commit, then confirming the fix from a truly clean directory
(deleted `missing-movies/data/`, reran `server.test.ts` alone — passed).

**Fix (in `src/api/server.test.ts`, a test file — always in scope):** `mkdirSync(dirname(gapsPath), …)`
instead of `mkdirSync(moviesConfig.outDir, …)`.
