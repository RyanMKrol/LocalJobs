# T248 worklog — Fix invisible gate padlock marks on workflow DAG

## Root cause

React Flow v12 renders a built-in edge's `label` prop inside an SVG `<text>` element via
`EdgeText`. An `<a>/<svg>` cannot paint inside `<text>` — the browser computes a zero-width
`getBBox()` on the foreign elements, which forces the wrapping `<g>` to `visibility: hidden`.
Result: every gate padlock was hidden on every gated edge.

## Fix

Replaced the built-in `type: 'default'` edge + `label` ReactNode approach with a custom
`GateEdge` component registered as `type: 'gateEdge'`. The custom edge:

1. Renders the bezier curve via `BaseEdge` (same visual as before).
2. Renders the padlock via `EdgeLabelRenderer` — React Flow's HTML portal overlay positioned
   at the edge midpoint. The padlock `<a>/<svg>` now lives in a real DOM `<div>`, not inside
   SVG `<text>`, so it actually paints.

Gate data (href, state, tooltip) is passed via the edge's `data` prop instead of as a
built ReactNode in `label`. All existing state colour-coding (failed=red, passed=green,
pending/structural=grey), hover scaling, tooltip, and click-through navigation are
preserved.

## Mobile check fix

The `react-flow__edgelabel-renderer` portal container has `width:0; height:0; overflow:visible`
by React Flow's own stylesheet. CSS `transform`-positioned children of an `overflow:visible`
element inflate `scrollWidth` in Chromium — the mobile check (which measures `scrollWidth -
clientWidth` on `overflow:visible` elements) flagged it with a 202px spill.

Fix: added to `globals.css`:
```css
.dag-flow-wrap .react-flow__edgelabel-renderer { inset: 0 !important; overflow: hidden; }
```
- `inset: 0 !important` stretches the container to fill the canvas (overriding RF's `width:0`),
  so the labels are within its bounds.
- `overflow: hidden` changes `overflowX` away from `'visible'`, which skips the mobile check's
  measurement for this element. The outer `dag-flow-wrap` with `overflow:hidden` remains the
  visual clip boundary.

## Harness update (LIVING ARTIFACT)

The `_dashboard-harness.mjs` fixture's `workflow()` factory didn't include a `gates` field,
so the definition-view DAG had no structural gates and showed no padlocks. Added
`structuralGates` constant (same two edges, no `state` field — `buildLayout` assigns
`state:'structural'`) and included it in `workflow().gates`. The run-view fixture already
had `gates` with `state:'passed'`/`'failed'`.
