# T184 — Trim appearance selector to the curated set (3 themes / 3 fonts) + auto light/dark by time

## What I built

Curated the T142/T154 theme/font experiment down to the owner's keepers and added
time-of-day light/dark switching.

### Themes — 3 families, each light + dark (6 palettes), mode auto-picked by time
- `dashboard/app/ui.tsx`: `ThemeId` narrowed to `'default' | 'pixel-picnic' | 'sunny-8bit'`;
  `THEMES` is now 3 entries. Removed all the `*-meadow/-dusk/-berry/-mint` and
  `8bit-sunset/-lime/-aqua/-coral` variants.
- Added pure `modeForHour(hour)` → `'light' | 'dark'` (day = 07:00–18:59 → light, else dark)
  and `useTimeMode()` which writes `data-mode` to `<html>` on mount and re-checks every 5 min
  (so a page left open across the threshold flips). Wired `useTimeMode()` into `ThemeControls`
  (rendered in the header on every page).
- `dashboard/app/globals.css`: replaced the 10 single-mode palettes with 3 families × light/dark:
  - `default`: DARK = `:root` (unchanged original pre-T142 look); LIGHT = new GitHub-light-ish
    palette via `html[data-mode="light"]:not([data-theme])` (+ explicit `default` selector).
  - `pixel-picnic` / `sunny-8bit`: family BASE rule carries the new DARK palette (so a JS-less
    viewer still gets a coherent theme); `[data-mode="light"]` overrides to the original cream /
    amber LIGHT palette.
  - Logs `--logs-*` stay in `:root`, never overridden → fixed dark terminal in BOTH modes.
- `dashboard/app/layout.tsx`: extended the pre-paint no-flash script to set `data-mode` from
  `new Date().getHours()` BEFORE first paint. Theme/font/motion persistence unchanged.

### Fonts — 3 options, only 2 faces loaded
- `ui.tsx`: `FontId` = `'system' | 'baloo' | 'spacemono'`; `FONTS` is 3 entries.
- `layout.tsx`: now imports/loads ONLY `Baloo_2` + `Space_Mono` from `next/font/google`
  (dropped Pixelify/Nunito/Quicksand/Fredoka/Comfortaa/Varela/Rubik/Mulish/Poppins/Lexend).
- `globals.css`: font-face rules reduced to `baloo` (rounded, `--heading-weight: 500` so headers
  read crisp per prior feedback) + `spacemono` (mono everywhere). All removed-face rules deleted.

### Preserved
- Reduce-motion toggle + the no-flash pre-paint script.
- T142 hard rules: untouched default / default-dark = original dark look; joyful accents gated to
  `html[data-theme]:not([data-theme="default"])`; logs dark-terminal in both modes; display/rounded
  faces only map to `--font-display`/`--font-body` (except deliberate Space Mono everywhere).
- Updated CLAUDE.md's appearance doc from "10 themes / 8 fonts" to the trimmed 3 families / 3 fonts
  + the auto light/dark behaviour.

## Notes
- No new `*.test.ts` added: the only new logic is the pure `modeForHour` 1-liner in client code
  (`dashboard/app/ui.tsx`), which the src-only `npm test` runner (`scripts/run-tests.ts` walks
  `src/` and would fail to import a `'use client'` React/JSX module) cannot reach. The behaviour is

  the verification gates.

  switcher minimal; mode is purely automatic from the viewer's clock.
