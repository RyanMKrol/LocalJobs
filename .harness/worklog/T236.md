# T236 — Run button accent

## What was done

Introduced `--run` / `--on-run` CSS variables in `dashboard/app/globals.css`:

- `:root` defaults: `--run: var(--accent)` and `--on-run: var(--on-accent)` — so the default theme (dark + light) is completely unchanged (accent is already blue there).
- `pixel-picnic` dark override: `--run: var(--human-blue); --on-run: #fff` — stable blue replaces green accent on the button.
- `pixel-picnic` light override: `--run: #1a6db5; --on-run: #fff` — darker blue for adequate contrast on cream background (#fbf6ea).
- `sunny-8bit` dark override: `--run: var(--human-blue); --on-run: #fff` — stable blue replaces amber on dark warm background.
- `sunny-8bit` light override: `--run: #1a6db5; --on-run: #fff` — darker blue for adequate contrast on warm yellow background (#fff3d6).

Added `.btn.btn-run { background: var(--run); color: var(--on-run); }` so the class is purely opt-in and other `.btn` uses are unaffected.

Applied `btn btn-run` to the Run/Running… buttons in both:
- `dashboard/app/workflows/page.tsx` (list page)
- `dashboard/app/workflows/[name]/page.tsx` (detail page)

Disabled/Running… state preserved (the class is the same regardless of disabled state; opacity .5 from `.btn:disabled` still applies).

## Verification

- `npm --prefix dashboard run build` — green, no errors.
- `node dashboard/scripts/mobile-check.mjs` — all 8 pages pass.
- Default theme dark/light: button color unchanged (--run inherits --accent which is the same blue).
- Joyful themes: button renders blue (#58a6ff / #1a6db5) contrasting against green and amber backgrounds.
