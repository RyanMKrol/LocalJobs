# T419 — fail movie-gaps-notify run when the digest push fails

## What changed
- `src/workflows/movies/stages/notify.ts`: `runNotify` now checks `res.ok` after calling
  `pushFn(...)`. On failure it throws a summarizing `Error` (naming the push error and how many
  gaps/recs would have been marked) BEFORE marking any `newGaps`/`newRecs` in their ledgers or
  calling `appendHistory` — so a failed digest no longer poisons the "already notified" ledger.
  The next run will see the same items as still-new and retry.
- On success (`res.ok === true`) behaviour is unchanged: mark ledgers, append history, log summary.
- The "nothing new to notify" early-return path is untouched (still a legitimate no-op success).
- `rec-merge` (merge.ts) was not touched — out of scope per the task spec.

## Tests
- Added a new case to `src/workflows/movies/stages/notify.test.ts`: an injected `push` stub
  returning `{ ok: false, error: 'network down' }`. Asserts `runNotify` rejects, the gap/rec are
  NOT marked done in their ledgers (still resurface as "new" on the next call), and the history
  file is not written.
- Ran `notify.test.ts` directly (`npx tsx --test`) — all 14 assertions pass, including the new one.
