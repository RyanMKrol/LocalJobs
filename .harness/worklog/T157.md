# T157 — Lazy output column (no mount-time fetch)

## What was done

Refactored `OutputCell` in `dashboard/app/workflow-runs/[id]/page.tsx` to eliminate
the per-row mount-time fetch that caused N heavy markdown fetches on first render plus
a visible layout shift as each cell grew after loading.

### Phase 1 — instant render from existing row data

`OutputCell` now derives the display info directly from `row.outputDetail`:
- `row.outputDetail.markdown` (string path) → `basename` is the displayed filename
- `row.outputDetail.name` (string) → label/heading

No `useEffect`, no `useState`, no fetch on mount. The component renders its final
`out-meta` layout (filename link + "click to preview") on first paint — same DOM
structure as before, so no column reflow.

- If `!row.outputKey` → renders `—` (unchanged)
- If `outputKey` present but no `markdown` path → shows key + name, no preview button

### Phase 2 — lazy fetch on preview open

When the user clicks "click to preview", `OutputCell` fires
`api.workflowRunOutput(...)` and passes the Promise to `onOpen`. The `IoPanel`
receives `(title, contentPromise)`, immediately sets modal to `{ loading: true }`,
then resolves the Promise to fill in the content.

`MarkdownModal` gained a `loading` prop — when true it shows "Loading…" in the
modal body instead of rendering content.

### Removed

- The `out.bytes` / `fmtBytes` size display was dropped (required the fetch; spec
  says to drop it rather than reintroduce a mount fetch just for size).
- The `WorkflowRunOutput` import removed from the page (no longer used in state).
