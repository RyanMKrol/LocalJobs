# T166 worklog

## What was done

The run detail page (`dashboard/app/runs/[id]/page.tsx`) had its `StatusBadge` as a direct
sibling of the title `<div>` inside a `.row` flex container with `align-items: center`. Because
the title `<div>` contained both the `<h1>` (job name) and a subtitle `<span>` (the word "run"),
the badge was vertically centred against the whole two-line block, landing it below the job-name
line.

**Fix**: moved the `StatusBadge` inside the title column `<div>`, into a new inner
`display: flex; align-items: center` wrapper alongside the `<h1>`. The subtitle remains below that
inner flex row. The outer `.row` now uses `align-items: flex-start` so the run-ID span to the
right also aligns at the top rather than mid-block.

The workflow-run detail page (`dashboard/app/workflow-runs/[id]/page.tsx`) was checked — its
header has the h1 directly in the `.row` with no subtitle in the same block, so `align-items: center`
is fine there; no change needed.
