# T148 worklog — IO panel height cap + scrollable table

## What was done

- Wrapped the `<table>` in `dashboard/app/workflow-runs/[id]/page.tsx`'s `IoPanel` in a
  `<div className="io-table-scroll">` scroll container.
- Added row count to the heading: "Input → Output mapping · N items".
- Added CSS in `dashboard/app/globals.css`:
  - `.io-table-scroll { max-height: 480px; overflow-y: auto; overflow-x: auto; }` — caps height
    and scrolls both axes inside the container.
  - `.io-table-scroll thead th { position: sticky; top: 0; background: var(--panel); z-index: 1; }`
    — keeps column headers visible while scrolling.
- The empty-state `<p>` (no items recorded) is NOT inside the scroll container and remains uncapped.
- The `.panel:has(> table)` mobile rule no longer fires for this panel (the panel now contains a
  `<div>` not a `<table>` directly), but horizontal scrolling is preserved by `overflow-x: auto`
  on `.io-table-scroll` itself.
