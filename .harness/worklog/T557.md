# T557 — remove fs.service.timeoutMs knob

## What was done
Removed the unused `timeoutMs: 5_000` field and its justifying comment from the fs service definition (`src/services/fs.service.ts`). This field was a dashboard-editable knob that had no effect because nothing reads `effectiveServiceTimeoutMs('fs', ...)` — only the plex and claude-cli services have readers for that function.

Updated the fs service unit test (`src/services/fs.service.test.ts`) to assert `timeoutMs === undefined` instead of `=== 5_000`, since the field was removed.

## Verification
- `npx tsc --noEmit` — PASS (no type errors)
- `npm test` — PASS (all 99 test files pass, including the updated fs.service.test.ts)
- `grep -n 'timeoutMs' src/services/fs.service.ts` — returns nothing (field fully removed)
- Confirmed `effectiveServiceTimeoutMs` in `src/core/services.ts` still reads plex and claude-cli correctly and compiles without errors

## Impact
- fs service definition is now cleaner — one fewer misleading dashboard-editable knob
- No runtime behavior change; nothing was using this field anyway
- All scope constraints met (only modified src/services/fs.service.ts and its test file)
