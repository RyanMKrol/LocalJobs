# T408 — places: fix identity-only success detail on places-ingest and places-enrich

## What changed

- `src/workflows/places/stages/ingest.ts` (`recordIngestLedger`): success `detail`
  now records `{ name, url, lists }` — `url` is the derived `?cid=` Maps URL
  (`p.cidUrl`), `lists` is the saved-list membership (`p.lists`) — instead of just
  `{ name }`.

  the `ptype` computation to precede the `markWorkItem` call so it can be included.
