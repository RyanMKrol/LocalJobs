# T584 — Collapse the four managers into a generic RecsManager + GroupedManager (re-attempt of T543)

2026-07-15

## What was built

Re-ran T543's fully-verified refactor with `CLAUDE.md` correctly added to scope this time, so the
doc update the spec always required can land in the same commit.

- Added `dashboard/app/components/RecsManager.tsx` — a generic `RecsManager<T extends RecItem>`
  parameterized by `{ heading, noun, tmdbPath, fetchData, ignore, unignore, unignoreBulk }`,
  replacing `TvRecsManager.tsx` and `MovieRecsManager.tsx` (deleted).
- Added `dashboard/app/components/GroupedManager.tsx` — a generic
  `GroupedManager<T, GK extends string | number, D>` parameterized by `{ heading, description,
  fetchData, getGeneratedAt, getItems, isIgnored, itemKey, groupBy, renderGroupLabel,
  activeColumns, ignoredColumns, renderCells, emptyBeforeGenerated, summaryLine, noActiveMessage,
  ignoredSubtitle, ignoreItem, unignoreItem, ignoreGroup, unignoreGroup }`, replacing
  `MovieGapsManager.tsx` and `MissingSeasonsManager.tsx` (deleted). `renderGroupLabel`/
  `renderCells` take an extra `ignoredSide` boolean so the active/ignored tables can render their
  (slightly different — e.g. no TMDB column, no "You own:" example, no notified badge) shapes
  from one function.
- `dashboard/app/workflows/[name]/page.tsx`: the four call sites became thin config objects
  (`MOVIE_RECS_CONFIG`, `TV_RECS_CONFIG`, `MOVIE_GAPS_CONFIG`, `MISSING_SEASONS_CONFIG`) rendering
  `<RecsManager config={...} />` / `<GroupedManager config={...} />`. `groupByCollection` /
  `groupByShow` moved into `page.tsx` alongside their configs (`groupByShow` now groups full item
  arrays instead of just season numbers, since the generic component's `renderCells` operates on
  items directly).
- Reused the existing `SortTh`, `IgnoredSection`, and the R05-a `useAction` hook — no new
  state-management pattern introduced. Updated `useAction.ts`'s doc comment (stale reference to
  the four deleted managers).
- `dashboard/scripts/_dashboard-harness.mjs` did NOT need any changes — confirmed the rendered
  DOM/URL surface is unchanged (same routes, same CSS classes: `.panel`, `.movie-gaps-scroll`,
  `.table-group-header`, `.ignored-section`, `.sort-th`, etc.), only the component internals were
  consolidated. All existing FLOWS referencing these pages/selectors (`movie-recs-sorted-by-lens`,
  `tv-recs-default-sort`, `tv-recs-empty`) still pass.
- **`CLAUDE.md` updated in this same commit** (the corrected part vs T543): rewrote the four
  places that named the deleted files by path — the "Audit-style workflows" bullet in the Output
  section conventions, the "Bulk collection/show dismiss (T210)" bullet, the `<IgnoredSection>`
  entry, and replaced the old `<TvRecsManager>`/`<MovieRecsManager>`/`<MovieGapsManager>`/
  `<MissingSeasonsManager>` entry with new `<RecsManager<T>>`/`<GroupedManager<T, GK, D>>` entries
  describing their config-object parameterization and call sites. The two remaining mentions of
  the old names in `CLAUDE.md` are historical prose ("replacing the former separate
  `TvRecsManager`/`MovieRecsManager`…") — not file-path references — which is accurate.

One FLOW (`movie-recs-sorted-by-lens`, which clicks the "Lens" column header) produced a
screenshot that still shows the default TMDB-descending order rather than a lens-sorted one.
Investigated: this is a pre-existing Playwright click-target quirk in the shared, UNMODIFIED
`SortTh.tsx` component (the `<button>` inside the `<th>` doesn't fill the cell, so
`page.click('table th.sort-th:has-text("Lens")')` can land outside the button) — confirmed by
diffing `handleSort`/`SortTh` wiring in the deleted `MovieRecsManager.tsx` (via `git show
HEAD:...`) against the new `RecsManager.tsx`: byte-identical sort logic. Not a regression
introduced by this refactor and out of scope for T584 (SortTh itself is unmodified); not fixed
here.
