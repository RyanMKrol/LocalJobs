# T131 ‚Äî move task do/doneWhen into per-task Markdown specs

## What I built (all six coupled parts, single pass ‚Äî fit one context window)

1. **Migration.** Wrote a throwaway `node` migration that, for every one of the 131 tasks,
   created `.harness/tasks/TNNN.md` with `## Do` + the verbatim `do` text and `## Done when` +
   the verbatim `doneWhen` text, added a `spec: ".harness/tasks/TNNN.md"` field (in the slot the
   `do` field occupied, preserving key order), and removed the `do`/`doneWhen` fields. Output was
   `JSON.parse`-validated before write. Ran it, verified (`131 migrated`, 131 MD files, 0 tasks
   still carrying `do`/`doneWhen`), then deleted the script (not committed).

2. **Loop.** `.harness/loop.sh`:
   - new helper `task_spec_rel <id>` reads the JSON `spec` path;
   - `prompt()` now appends the spec MD's full text verbatim (`cat "$ROOT/$rel"`) under a
     `--- Task <id> spec (<rel>) ---` header (warns if the file is missing);
   - prompt instructions updated: orchestration fields from `jq`, do/done-when from the spec MD
     (also embedded);
   - added a `LOOP_SOURCE_ONLY=1` source-guard (`return 0` before the dry-run/main loop) so tests
     can source just the helpers.

3. **Backlog API.** `src/api/server.ts`: new exported `readTaskSpec(specRel, baseDir)` ‚Äî resolves
   the repo-relative `spec` path, confined to `<.harness>/tasks/*.md` (via `isWithin`), `.md`-only,
   local read (no network/paid). `readBacklog` now inlines each task's spec as `specContent`.

4. **Backlog page.** `dashboard/app/backlog/page.tsx`: new `<TaskSpec>` renders `specContent` via
   `react-markdown` (XSS-safe, no rehype-raw ‚Äî reused the T115 renderer + `.md-body` styles). Both
   the buildable/needs-human card and the done-row expansion use it; flat `do`/`doneWhen` removed.
   `dashboard/app/lib/api.ts`: `BacklogTask` drops `do`/`doneWhen`, adds `spec?`/`specContent?`.

5. **Docs.** HARNESS.md ¬ß8 schema + new authoring sub-section; CLAUDE.md harness section (new
   bullet on the spec MD + authoring = JSON object + MD); README.md backlog description; two new
   LIMITATIONS.md ¬ßHarness trade-offs (two-file split; bash/jq dependency of the prompt test).

6. **Tests** (`src/api/server.test.ts`): `readTaskSpec` happy-path + null cases (absent/non-string/
   non-md/traversal/missing); `GET /api/backlog` inlines `specContent` and omits it when the file
   is missing (and no flat do/doneWhen); `loop.sh prompt()` embeds the `## Do`/`## Done when` from
   `.harness/tasks/T001.md` (sources loop.sh with `LOOP_SOURCE_ONLY=1`, self-skips if `jq` absent).
