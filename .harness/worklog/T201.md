# T201 worklog

## What was built

End-to-end unlimited concurrency (no cap) for workflows via sentinel value `0`.

### Changes
- **`src/core/workflow-executor.ts`**: Added `UNLIMITED_CONCURRENCY_SENTINEL = 0` export. Updated `effectiveWorkflowConcurrency` to map `0 → Infinity` so `executeDag` skips the `inflight.size >= concurrency` check.
- **`src/db/store.ts`**: Relaxed `updateWorkflowConcurrency` validation to accept `0` (the unlimited sentinel) alongside ≥1, rejecting negatives/non-integers.
- **`src/api/server.ts`**: Relaxed `/concurrency` endpoint validation to match. Fixed the `effective_max_concurrency` in the GET response to return `0` (the sentinel) instead of `Infinity` (which can't be JSON-serialised).
- **`dashboard/app/workflows/[name]/page.tsx`**: Added `concUnlimited` state. The editor now shows an "Unlimited" checkbox; when checked, passes `0` to the API. Read view shows "Unlimited" when `effective_max_concurrency === 0`.
- **`dashboard/app/lib/api.ts`**: Updated `effective_max_concurrency` comment to document the `0` sentinel.
- **`CLAUDE.md`**: Added "Unlimited (no cap, T201)" paragraph to the `maxConcurrency` section.
- **`src/core/workflow-executor.test.ts`**: Added 4 tests for T201: effectiveWorkflowConcurrency with sentinel, store round-trip, invalid value rejection, and wall-time overlap proof for unlimited runs.
- **Test files updated**: `src/api/server.test.ts` (removed `0` from bad-values list, added explicit 200 test for `0`), `src/db/store.test.ts` (removed obsolete `throws(0)` assertion).
