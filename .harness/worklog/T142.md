# T142 ‚Äî Joyful dashboard themes + fonts switcher (evaluation aid)

## What I built

A live, mix-and-match appearance switcher across ALL dashboard pages, behind a compact
header **üé® Theme** control ‚Äî mirroring the existing chooser‚Üíreview‚Üíhardcode pattern.
Three persisted (localStorage) axes applied to `document.documentElement`:

1. **Theme** (`data-theme`, 6 themes) ‚Äî full palette/texture/accent var-sets in `globals.css`:
   - **2 dark:** `default` (Default Dark = the current look, the untouched default) +
     `midnight-neon` (synthwave dark, neon purple accent, sparkle/emoji).
   - **4 bright:** `pixel-picnic` (BASKET-like cream + dot-grid texture + soft green),
     `candy-bright` (white + pink/teal/purple), `sunny-8bit` (amber arcade + scanline texture),
     `soft-pastel` (airy periwinkle pastels).
   - Reconciliation of the spec's "2 dark = Joyful Dark + Midnight Neon": I made the two darks
     **Default Dark + Midnight Neon** so the count is exactly 6 AND the untouched default stays
     the current look (spec allows "refine freely"). Midnight Neon carries the joyful-dark vibe.

2. **Font** (`data-font`, 8 display+body pairs) ‚Äî loaded once each via `next/font/google` in
   `layout.tsx` (self-hosted, `display:swap`, no blocking FOUC), exposing CSS vars on `<html>`;
   the switcher remaps `--font-display`/`--font-body`/`--font-mono`:
   `system` (default), `pixelify-nunito`, `silkscreen-quicksand`, `vt323-fredoka`,
   `pressstart-baloo`, `fredoka-nunito`, `spacemono`, `jetbrains-quicksand`.
   **Pixel/retro faces only ever map to `--font-display`** (brand/headings) ‚Äî never body/table/log.

3. **Reduce motion / minimise emoji** (`data-motion="reduced"`) ‚Äî defaults to the OS
   `prefers-reduced-motion`; kills animations/transitions + hides emoji when on.

**Joyful accents** (gated to non-default themes, dampened by reduce-motion): the signature
**yellow-spark flash** on a succeeding run (box-shadow glow tinted `--spark` + ‚úÖ emoji pop),
hover-lift on cards/rows/DAG nodes, animated progress-fill sheen, a CSS entrance "pop" on stat
numbers, emoji status badges (`StatusBadge` renders a hidden `.badge-emoji` span revealed only on
joyful themes), and a friendlier empty state ("Nothing stuck ‚Äî nice ‚ú®").

**No-flash:** an inline pre-paint `<script>` in `layout.tsx` sets the three html attributes from
localStorage before first paint; `<html suppressHydrationWarning>` avoids the hydration mismatch.

## Key design decisions / constraints honoured

- **Default untouched = no regression.** Joyful CSS is gated by
  `html[data-theme]:not([data-theme="default"])`; with no attributes the page is byte-for-byte the
  pre-T142 dark/system look. Confirmed by `mobile-check.mjs` still passing (it sets no localStorage).
- **Density / legibility.** Logs keep a FIXED dark-terminal palette (`--logs-bg/text/err/warn/ts`
  in `:root`, never overridden by a theme) so streaming logs stay readable on bright backgrounds.
  Tables/DAG/stat-tiles use the theme's contrasting `--text`/`--panel` vars.
- **Mobile.** The switcher is a fixed-position modal (never widens the header) with a label that
  drops at ‚â§640px; popover grid collapses to one column. The success "spark" is a box-shadow glow
  (NOT an overflowing `::after`) precisely because an absolutely-positioned `‚ú®` at `right:-7px`
  spilled the badge box and tripped the 402px boundary rule (caught in spot-check, then fixed).

## Notes / follow-ups

- **T143 (paired review task) is referenced by the spec but NOT yet in `.harness/TASKS.json`.** The
  spec's chooser‚Üíreview‚Üíhardcode convention says a chooser needs a paired `needs-human` review task
  (T143) that records the owner's choice and unblocks a hardcode follow-up. Authoring it means
  editing TASKS.json, which the autonomous builder must NOT do ‚Äî flagging for the owner/loop to add.
  Docs (README/CLAUDE.md) reference T143 as the gating follow-up, consistent with the spec.
- **Stat count-ups** are a CSS entrance pop, not JS number animation ‚Äî the stat tiles live in
  `page.tsx` (out of T142 scope). See `.harness/LIMITATIONS.md`.
- Scope respected: only `dashboard/app/{globals.css,ui.tsx,layout.tsx}` touched in `dashboard/`
  (no `package.json` change needed ‚Äî `next/font` ships with `next`), plus docs.
