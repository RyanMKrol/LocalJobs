# T389 — remove the dead old joined `IoPanel` mechanism

2026-07-10

## What was deleted

`StageIoPanel` (T386/T387/T472/T473) has replaced the old joined `IoPanel` as the
default Inputs & Outputs view for every workflow's run page. This task removed the
now-dead old mechanism entirely:

- **`dashboard/app/workflow-runs/[id]/page.tsx`** — removed the `IoPanel` component
  and its exclusively-used helpers (`itemLabel`, `priceDetail`, `formatPriceDetail`,
  `mdPreview` (already fully dead/uncalled), `OutputCell`, `rowEffectiveState`,
  `sortIoRows`, `IO_FILTER_STATES`, `IoFilterState`, `IoSortCol`, `ModalState`), plus
  the now-unused imports (`MarkdownModal`, `SortTh`/`SortDir`, `IoRow`/`WorkflowIo`,
  `useCallback`).
- **`src/db/store.ts`** — removed `workItemIoRows`, its `IoRow`/`IoResult` types, and
  its private `toIoRow` helper. Kept `LedgerRow` (still used by `stageIoLists`/
  `toStageIoItem`) and `workflowHasRunLinkage` (not mentioned in the task spec for
  removal; it has its own direct test coverage in `store.test.ts` beyond the deleted
  `/io` endpoint, so left in place as a still-tested, if now less-consumed, utility).
- **`src/api/server.ts`** — removed the `GET /api/workflow-runs/:id/io` handler and
  the `workItemIoRows`/`workflowHasRunLinkage` imports (the latter had no other call
  site in this file once the endpoint was gone).
- **`dashboard/app/lib/api.ts`** — removed the `IoRow`/`WorkflowIo` types and the
  `workflowRunIo` client function.
- **`dashboard/app/globals.css`** — removed `.io-footnote`, `.io-table-scroll`
  (+ its two descendant rules), and `.io-state-arrow` — confirmed via grep these were
  ONLY used by the deleted `IoPanel` markup. Deliberately did **NOT** remove
  `.io-filter-bar`/`.io-filter-chip*`/`.io-filter-count` as the task spec suggested:
  grepping `dashboard/app/` first showed `dashboard/app/logs/page.tsx` (the Logs
  page's level-filter chip bar) also uses these classes — removing them would have
  broken a live, unrelated page. This is exactly the kind of drift the task's
  "verify before deleting" step exists to catch; the spec's file list for this bullet
  was written when those classes were assumed IoPanel-only, which is no longer true.
- **`dashboard/scripts/_dashboard-harness.mjs`** — removed the `ioRow`/`workflowIo`/
  `workflowIoScopedToEnrich`/`workflowIoSkipped`/`workflowIoStocks` fixtures and the
  `/io`-suffix routing branches (`/skipped/io`, `/movie-recs-run/io`, `/stocks/io`,
  and the generic `pathname.endsWith('/io')` catch-all). Kept `stocksMembers`/
  `stocksWorkflowRun`/`stocksRunJobs` — confirmed via grep they're still shared with
  the live `/stage-io` fixtures and the `/workflows/stocks-sync` route. Renamed the
  `io-panel-job-scoped` FLOWS entry to `stage-io-job-scoped-places` and updated its
  comment — investigation showed this flow was NOT dead: it exercises
  `StageIoPanel`'s own `.io-job-filter-chip` chip-scoping on `/workflow-runs/1`
  (places), just under a stale name left over from before T386 repurposed it. No
  `io-panel-sorted-by-state` entry existed (already removed in an earlier sweep).
- **`src/db/store.test.ts`** / **`src/api/server.test.ts`** — removed the T139/T313
  `workItemIoRows`/`GET .../io` test blocks. In `store.test.ts` kept the
  `work_item_runs` linkage-recording assertions and the `workflowHasRunLinkage`
  assertions (trimmed to a smaller, focused setup) since those exercise
  `markWorkItem`'s linkage recording and `workflowHasRunLinkage` itself, not the
  removed join function.
- **`CLAUDE.md`** (root) — updated the `<StageIoPanel>` entry under "Shared
  dashboard UI components" and the T139 `work_item_runs` paragraph to describe
  `StageIoPanel`/`stageIoLists` as the only mechanism, not an alternative to a
  still-existing generic `IoPanel`/`workItemIoRows`.

other hits were exactly the definitions/call-sites being removed. Proceeded with the
deletion as planned.
