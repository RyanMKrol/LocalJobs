# T090 ‚Äî DAG: gate chips on the connecting arrow, between the two jobs

## Goal
Validation-gate chips previously rendered UNDER the consumer (second) job node,
making a gate look like a property of that one job. A gate bridges a
producer‚Üíconsumer boundary, so move its chip ON/BELOW the connecting arrow BETWEEN
the two waves it guards ‚Äî for both the run view (`GateChips`) and the
structure-only definition view (`StructuralGateChips`), at both desktop (`‚Üí`) and
phone (`‚Üì`) widths.

## Placement logic (linear example workflows)
Both example workflows (`places`, `perfumes`) are strictly linear, so every gate's
producer is the wave immediately before its consumer ‚Üí `ci === pi + 1` ‚Üí every chip
lands on its inter-wave arrow. The fallback only triggers for a hypothetical
non-adjacent edge, which the example data doesn't contain.
