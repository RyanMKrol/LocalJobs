# T343 — fix theme icon flicker on full-page nav

## What changed

`useMode()` in `dashboard/app/ui.tsx` previously initialized state with a
hardcoded default: `useState<ModeId>('dark')`. Since `dashboard/app/layout.tsx`'s
nav links are plain `<a>` tags (full page loads, not client-side transitions),
every navigation fully remounted `useMode()` from scratch. The pre-paint inline
script in `layout.tsx` correctly sets `data-mode` on `<html>` before React
mounts, so the CSS was always right, but React's first render used the
hardcoded `'dark'` default — producing the wrong icon (🌙 vs ☀️) for one frame
whenever the real mode was `'light'`, until the `useEffect` corrected it.

Fix: changed the initializer to a lazy `useState(() => ...)` that reads
`document.documentElement.getAttribute('data-mode')` synchronously at mount
time (falling back to `'dark'` under SSR where `document` is undefined), so
the first render already matches what the pre-paint script set — no flicker.
The existing `useEffect` is unchanged and still reconciles with localStorage
and listens for OS `prefers-color-scheme` changes.

`ModeId` (`'dark' | 'light'`), `resolveMode`, the pre-paint script, and the
toggle button's markup/click behaviour are all untouched.
