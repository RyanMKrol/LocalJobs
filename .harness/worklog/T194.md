# T194 worklog

## Root cause

The gap between the last IO table row's hover highlight and the footnote came from two sources:

1. `.io-footnote` had `margin: 10px 0 0` — 10px of empty space between the table scroll container and the footnote's border-top, which was visually exposed when the last row was highlighted.
2. The last `td` had `border-bottom: 1px solid var(--border)` (from the global `th, td` rule), which placed a border line right below the highlighted row, making the gap even more noticeable.

The top row has no corresponding gap above it because the `thead th` border-bottom sits flush against the first `tbody tr` with no margin.

## Fix

`dashboard/app/globals.css`:
- Changed `.io-footnote` `margin: 10px 0 0` → `margin: 0`. The footnote's `border-top` + `padding-top: 8px` already provide sufficient visual separation.
- Added `.io-table-scroll tbody tr:last-child td { border-bottom: none; }` so the last row ends without a bottom border, allowing the footnote's `border-top` to serve as the sole divider — symmetric with the header.
