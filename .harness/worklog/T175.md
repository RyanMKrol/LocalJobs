# T175 worklog ‚Äî skip-wave DAG edge connectors

## What was done

Updated `dashboard/app/components/Dag.tsx` to draw non-adjacent (skip-wave) producer‚Üíconsumer
edges as visible SVG curved paths instead of silently falling back to a gate chip under the
consumer node with no connector.

### Changes

**Dag.tsx:**
- Added `useRef`/`useState`/`useEffect` imports.
- Identified skip edges: any producer‚Üíconsumer pair where `consumerWave > producerWave + 1`.
- `useEffect` + `ResizeObserver`: after each render measures producer/consumer node positions
  relative to the `.dag` container and computes an SVG cubic-bezier path for each skip edge.
  - Desktop (horizontal flex): routes below the intermediate waves (`M x1,y1 C cx,baseY cx,baseY x2,y2`).
  - Phone (vertical flex, detected via `getComputedStyle`): routes to the right with a horizontal bulge.
- Renders an `<svg>` overlay (absolutely positioned, `overflow:visible`, `pointer-events:none`)
  with a `<path class="dag-skip-path">` per skip edge.
- Renders absolutely-positioned `<GateMarks>` at each path midpoint for any gate chips that
  belong to a skip edge.
- Skip-edge gates are excluded from the per-node fallback chip rendering so they don't appear twice.
- Adjacent-wave arrows and their gate chips are completely unchanged.
- Added `data-dag-job` attribute on each node wrapper for element lookup.
- Added `position: relative; zIndex: 1` on node wrappers so the SVG (z-index 0) renders behind
  the interactive node content.

**globals.css:**
- Added `.dag-skip-path` rule: dashed grey stroke, 1.5px width, 0.6 opacity ‚Äî visually distinct
  from the solid adjacent-wave arrows.

The `franchise-gaps ‚Üí movie-gaps-notify` skip edge (wave 1 ‚Üí wave 3, bypassing `rec-merge` at
