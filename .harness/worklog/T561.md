# T561 — extract shared recommender pipeline

2026-07-14

## What was done

Extracted the near-duplicate `movies`/`tv-recs` recommender implementation into a generic,
domain-parameterized pipeline under `src/core/recommender/`:

- `types.ts` — `RecommenderDomain<M,P>`, `BranchContext<M,P>`/`BranchSpec<M,P>`, and the
  shared artifact shapes (`RawSuggestion`, `BranchOutputFile`, `Recommendation`,
  `RecommendationsFile`, `RecsHistoryFile`).
- `pure.ts` — the byte-identical pure helpers (`mulberry32`, `seededShuffle`,
  `stratifiedSample`, `normTitle`, `dedupeRawByTitleYear`, `balanceByGenre`, `mergeLens`,
  `topGenres`, `thinGenres`) plus a new shared `ownedAtLeast` (movies'
  `directorsOwnedAtLeast` / tv-recs' `creatorsOwnedAtLeast` are now thin wrappers over it)
  and `buildOwnedSet`/`recKey`.
- `branch.ts` — the generic branch runner (`runBranch`, `collectBranchSuggestions`,
  `makeBranchJob`, `parseSuggestions`, `allHistoryTitles`, `ignoredSuggestionTitles`).
- `merge.ts` — the generic merge stage (`runMerge`): TMDB verify, dedup, quality bar,
  genre balance, bounded top-up loop, the `searchFailed` throw-on-error guard.
- `notify.ts` — the generic notify stage (`runRecsNotify`): dedup vs ledger, digest,
  report, history append, the push-ok-then-mark guard (B11, T527).

Both workflows now keep only what's genuinely domain-specific:
- `stages/branches.ts` — the 8 branches' lens PROMPT TEXT (byte-identical content, just
  `ctx.movies`/`ctx.shows` renamed to the generic `ctx.items`) + a `moviesDomain`/`tvDomain`
  wiring object (TMDB endpoint + field mapping, genre table, digest wording, push
  job/tags, report heading/filename; tv-recs' domain also supplies `extraNotifyDetail` for
  its ledger `tmdbUrl` field).
- `recs.ts` — thin re-exports of the shared pure helpers + `RECS_JOB` + the
  director/creator-completion wrapper + (movies only) the TMDB movie-genre table.
- `stages/recommend.ts`, `stages/merge.ts`/`tv-rec-merge.ts`, `stages/notify.ts`/
  `tv-recs-notify.ts` — thin wrappers preserving every existing exported name
  (`runBranch`, `makeBranchJob`, `runMerge`/`runTvRecMerge`, `SearchMovieFn`/`SearchTvFn`,
  `runNotify`/`runTvRecsNotify`, `buildDigest`) so the untouched `*.job.ts` wrappers and
  every pre-existing test keep working unchanged.
- `contracts.ts` and `config.ts` are untouched / per-workflow as before.

`tv-recs/config.ts`'s `recsModel` default changed from `claude-sonnet-4-6` to
`claude-sonnet-5`, aligning it with `movies` per the owner's 2026-07-12 decision recorded in
the spec.

Job names, ledger keys, DAG shape, `*.job.ts`/`*.workflow.ts`/`contracts.ts` are all
byte-unchanged (none were in scope and none needed changes) — `dashboard/scripts/
_dashboard-harness.mjs` needs no update.

## Tests

- Added `src/core/recommender/pure.test.ts` (consolidated the duplicated pure-helper
  assertions from both workflows' old `recs.test.ts`), `merge.test.ts`, and `notify.test.ts`
  (golden-file assertions: two synthetic domains with distinct genre tables / digest
  wording / job namespaces run through the SAME shared `runMerge`/`runRecsNotify` and
  produce exact, hardcoded-expected `recommendations.json` / digest / recs-history rows —
  proving the extraction is domain-agnostic and deterministic).
- Trimmed `movies/recs.test.ts` and `tv-recs/recs.test.ts` to their domain-specific pieces
  only (`genreNameFromIds`/`directorsOwnedAtLeast`, `creatorsOwnedAtLeast`).
- Updated `movies/stages/recommend.test.ts`'s two literal `BranchContext` constructions
  from `movies:` to the new generic `items:` field.
- Updated `tv-recs/stages/tv-recs-notify.test.ts`'s failed-push regex to match the new
  unified throw message (`digest push failed (...)`, same wording both domains now share).
