# T370: hard-enforce validation-gate coverage across every workflow edge

## What was built

Added `src/core/gate-coverage.test.ts`, a whole-repo invariant test (sibling to
`dag.test.ts`/`contracts.test.ts`) that:

1. Loads `jobs`/`workflows` from `src/jobs/registry.js` (real, auto-discovered
   arrays — no hardcoded workflow list).
2. For each workflow, builds its `Dag` via `buildDag`, collects each member job's
   `produces`/`consumes` keys, and calls `deriveGates`.
3. Asserts every DAG edge (producer → consumer) has a matching derived gate. A
   miss fails with a message naming the workflow, producer, and consumer
   explicitly.
4. Shares the edge/gate-diff logic via a `missingGateEdges(dag, produces,
   consumes)` helper, reused by two synthetic unit tests:
   - a bare `{ ok: true }` stub producer/consumer pair passes (trivial minimum
     bar — proves a rich shape-check isn't required).
   - a producer/consumer pair with no `produces`/`consumes` at all is reported as
     exactly one missing edge (regression guard for the checker itself).
