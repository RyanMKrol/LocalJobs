# T367 — projects-sync validation gate

## What was built

Added a real `ArtifactContract` at the `github-sync → project-summarize` boundary of
the `projects-sync` workflow (previously ungated), mirroring the `places`/`perfumes`
pattern.

- `src/jobs/projects-sync/contracts.ts` — `projectsCatalogContract(file = projectsSyncConfig.catalogPath)`,
  key `projects-catalog`. Validates `data/out/projects.json`: exists + parses as JSON,
  top-level is an array, array is non-empty, and every entry has a non-empty string
  `repoId` and `name`. Declares a `shape` (summary/format/expectations) and returns
  per-expectation `checks[]` + a `sample` from `check()`.
- `src/jobs/projects-sync/stages/github-sync.job.ts` — added `produces: [projectsCatalogContract()]`.
- `src/jobs/projects-sync/stages/project-summarize.job.ts` — added `consumes: [projectsCatalogContract()]`.
- `src/jobs/projects-sync/contracts.test.ts` — synthetic-fixture tests (temp dir, no
  live GitHub API / clone) covering pass + all fail cases (missing file, invalid JSON,
  non-array, empty array, entry missing repoId/name), plus a `buildDag`/`deriveGates`
  regression test asserting exactly 1 gate derives for the `github-sync→project-summarize`
  edge — the direct guard for the bug the owner observed (this workflow's run page
  showing no gate).

## Verification observed

- `npx tsc --noEmit` — clean, no errors.
- `npm test` — full suite green (146 tests, 41 suites, 0 failures). The new
  `contracts.test.ts` file's 7 tests all pass (confirmed via targeted grep of the
  test run output).
- No live GitHub API call or repo clone made anywhere in the new code or tests —
  all fixtures are synthetic JSON written to a `mkdtempSync` temp dir.
- No dashboard change, so `npm --prefix dashboard run build` was not required.
