# T274 — Migrate Run page to shared component library

## Finding: nothing applicable to migrate

Reviewed `dashboard/app/runs/[id]/page.tsx` against the shared components from T268
(`dashboard/app/components/`: `RunButton`, `Pill`, `DagFlow`, `WorkflowOutputSection`):

- **`RunButton`** wraps `.btn.btn-run` for workflow run-trigger actions (with an
  "already running" disabled state). The Run page has no run-trigger button at all —
  it's a read-only run detail view (status, metadata, logs).
- **`Pill`** wraps `.pill` chips. Grepped the page for `pill`/`badge`/`btn-run` — none
  present. The only status rendering is `StatusBadge` (from `../../ui`), which the
  page already imports and uses at the top of the run header.
- **`DagFlow`** / **`WorkflowOutputSection`** are workflow-run-graph / output-panel
  components; the Run page has neither (those live on the workflow-run detail page).
- The log-level filter buttons (`all`/`info`/`warn`/`error`) are plain `.btn`/`.btn
  secondary` toggle buttons — a multi-select filter control, not a run-trigger or a
  status chip, so wrapping them in `RunButton` or `Pill` would misrepresent their
  semantics and isn't a like-for-like swap.

Conclusion: the page already uses the one applicable shared piece (`StatusBadge`) and
has no other UI surface matching `RunButton`/`Pill`/`DagFlow`/`WorkflowOutputSection`.
No code change made — this mirrors the T273 (Job page) finding.
