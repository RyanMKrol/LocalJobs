# T066 — Workflow-run DAG: fix the graph layout on mobile/narrow widths

## Approach

CSS-only change in `dashboard/app/globals.css` inside the `@media (max-width: 640px)` block.

The existing mobile rules only adjusted `padding` and removed `min-width` — they didn't change the
flex axis, so the DAG remained left-to-right with vertical centring misalignment at narrow widths.

**What changed:**

```css
/* Before */
.dag { padding: 14px; gap: 8px; }
.dag-node { min-width: 0; }

/* After */
.dag { flex-direction: column; align-items: stretch; padding: 14px; gap: 8px; }
.dag-wave { flex-direction: row; flex-wrap: wrap; justify-content: center; }
.dag-node { min-width: 0; }
.dag-arrow { font-size: 0; text-align: center; }
.dag-arrow::after { content: '↓'; font-size: 22px; color: var(--grey); }
```

- `.dag` switches from row (horizontal) to column (vertical) — waves stack top-to-bottom
- `.dag-wave` switches from column to row with wrap — nodes within a wave flow left-to-right
- `.dag-arrow` text (`→`) is hidden (font-size:0) and replaced via `::after` with `↓`
- Desktop layout (> 640px) is entirely unchanged

No changes to `Dag.tsx` or any other files.
