## This attempt (final)

### Problem

The two pages diverged:
- Gate page: `← {workflow_name} run · {shortId}` ✓
- Run/job page: `← {raw-full-UUID}` ✗

### Solution

1. **`dashboard/app/ui.tsx`** — Added `WorkflowRunBackLink` component (exported)
   right after `backFrom`. Renders `<p className="muted">` with:
   - When both `workflowRunId` and `workflowName` are known:
     `← {workflowName} run · {shortId}` with `href=/workflow-runs/{id}` and
     `title={fullId}` tooltip.
   - Otherwise: a plain `← {label}` link using the `fallback` prop.

2. **Gate page** (`workflow-runs/[id]/gates/[producer]/[key]/page.tsx`) — Replaced
   the inline `<p className="muted"><a ...>` back link with `<WorkflowRunBackLink>`
   using the already-fetched `run.workflow_name` and the route param `id`.
   Fallback is `← workflows`.

3. **Run/job page** (`runs/[id]/page.tsx`) — Added a second `usePoll` call:
   ```tsx
   const { data: wfRunData } = usePoll(
     () => run?.workflow_run_id ? api.workflowRun(run.workflow_run_id) : Promise.resolve(null),
     5000,
     [run?.workflow_run_id],
   );
   const wfRun = wfRunData?.run;
   ```
   Then replaced the old `backFrom`-based back link with `<WorkflowRunBackLink>`.
   Fallback is `← {job_name}` (the job page). Removed unused `backFrom` and
   `useSearchParams` imports.
