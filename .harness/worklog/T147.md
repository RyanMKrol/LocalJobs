# T147 worklog

## What was done

Changed the back-navigation label on the run-scoped validation-gate detail page
(`dashboard/app/workflow-runs/[id]/gates/[producer]/[key]/page.tsx`).

The label was hardcoded as `← workflow run` regardless of which workflow the run
belonged to. Changed to `← {run.workflow_name} run` so it identifies the specific
workflow (e.g. `← places run`), using the `run` object the page already polls.
Fallback when `run` is not yet loaded remains `← workflows` linking to `/workflows`.

The definition-level gate page (`dashboard/app/workflows/[name]/gates/[producer]/[key]/page.tsx`)
already showed `← {name}` (the workflow name from the URL param), so no change was
needed there.
