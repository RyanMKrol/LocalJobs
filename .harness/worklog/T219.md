# T219 worklog — TV recs dashboard surfacing

## What was built

Mirrored the movie-recs surfacing pattern (T209) for the TV recommendations workflow.

### API (`src/api/server.ts`)
- Added imports for `tvRecsConfig`, `TV_RECS_JOB` (as `RECS_JOB` from tv-recs/recs.js), `tvRecKey`, and `TvRecommendationsFile`
- `GET /api/tv-recs` — reads `tvRecsConfig.recsOut` (recommendations.json), overlays `ignored`/`notified` per rec from the `tv-recs` work_items ledger, returns empty list (not error) when file absent
- `POST /api/tv-recs/:tmdbId/ignore` — calls `ignoreSurfacedItem('tv-recs', tvRecKey(tmdbId))`, behind the existing loopback/token mutation guard

### Client (`dashboard/app/lib/api.ts`)
- Added `TvRec` and `TvRecs` interfaces mirroring `MovieRec`/`MovieRecs`
- Added `tvRecs()` and `ignoreTvRec(tmdbId)` to the `api` object

### UI (`dashboard/app/workflows/[name]/page.tsx`)
- Added `TvRec` to the import
- Added `'tv-recommendations'` to `WORKFLOWS_WITH_SPECIFIC_MANAGERS`
- Added `TvRecsManager` component mirroring `MovieRecsManager` (TMDB links use `/tv/` path, column header says "Show" not "Film")
- Wired `{name === 'tv-recommendations' && <TvRecsManager />}` in the render section

### Tests (`src/api/server.test.ts`)
- Added 4 tests mirroring the movie-recs T209 test block:
  - `GET /api/tv-recs` overlays notified + ignored status
  - `POST /api/tv-recs/:id/ignore` suppresses a rec
  - `POST /api/tv-recs/:id/ignore` rejects non-numeric id (400)
  - `GET /api/tv-recs` returns empty list when file absent
