# T306 — Group Services dashboard page by category

## What changed
- `dashboard/app/lib/api.ts`: added `category: string` to the `Service` interface (mirrors
  the `category` field T305 added to `ServiceDefinition`/`GET /api/services`).
- `dashboard/app/services/page.tsx`: added `CATEGORY_GROUPS` (`cli-tool` → "CLI tools",
  `website-scrape` → "Website scrapes", `api` → "APIs", `uncategorized` → "Uncategorized").
  Split `services` into per-category groups (partition + existing paid-then-name sort applied
  within each group), rendering the existing table markup once per non-empty group inside its
  own `.panel` with an `<h2>` heading, mirroring `workflows/page.tsx`'s T293 grouping. All
  per-row behavior (inline limit editing, consumers popover, paid/free/edited pills) is
  untouched — only grouping/sectioning changed.
- `dashboard/scripts/_dashboard-harness.mjs`: added `category: 'api'` to the `service()`
  fixture default, and expanded the `/api/services` fixture list to include `fragrantica`
  (website-scrape), `claude-cli` (cli-tool), and `legacy-service` (uncategorized) alongside
  the existing `google-places`/`gemini` (api) — now covers all 4 groups including the
  "no category" fallback.
