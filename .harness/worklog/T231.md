# T231 — Show the worklog in the expanded Backlog task cell

## What was built

Added `worklogContent` inlining to `GET /api/backlog`, mirroring the existing `specContent` pattern:

- **`src/api/server.ts`**: added `readWorklogContent(id, baseDir)` — reads `.harness/worklog/<id>.md`, confined to the worklog directory (validates id is alphanumeric/hyphen only, no slashes, no traversal), returns null when absent. Wired into `readBacklog` alongside `specContent`.
- **`dashboard/app/lib/api.ts`**: added `worklogContent?: string` to `BacklogTask`.
- **`dashboard/app/backlog/page.tsx`**: in the expanded row, added a collapsible `<details>` "Build log" section rendered via `ReactMarkdown` when `worklogContent` is present. Tasks with no worklog show nothing extra.
- **`src/api/server.test.ts`**: added a T231 block testing `readWorklogContent` (valid read, missing file, traversal/invalid ids) and `GET /api/backlog` inlining (worklogContent present when file exists, absent otherwise).
