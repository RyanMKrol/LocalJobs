# T064 — Validation gates: reframe producer/consumer as input/output

## What was done

Relabeled all user-visible "producer" / "consumer" gate terminology to "output" / "input" language. Data model field names (`gate.producer`, `gate.consumer`, `produces`, `consumes`) are unchanged.

### Files changed

**`src/core/workflow-executor.ts`**
- `gateAssertions`: now emits `output (from <stage>): <desc>` and `input (to <stage>): <desc>` instead of `producer asserts:` / `consumer asserts:`.
- `enforceGate`: side labels in violation messages changed from `'producer'` / `'consumer'` to `'output'` / `'input'`.

**`src/api/server.ts`**
- Gate `description` field in `gatesForWorkflow`: changed from `produces — <desc> · consumes — <desc>` to `output: <desc> · input: <desc>`.

**`dashboard/app/workflow-runs/[id]/gates/[producer]/[key]/page.tsx`**
- Logs table: "Producer — stage" → "Output (from stage)"; "Producer not run yet" → "Output stage not run yet".
- "Consumer — stage" → "Input (to stage)"; "Consumer not run yet" → "Input stage not run yet".
- Gate-violation label kept as "Gate violation — stage" (unchanged, already clear).

**`src/core/workflow-executor.test.ts`**
- Updated two assertions to match the new log format.
