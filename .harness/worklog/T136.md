# T136 ŌĆö Durable reviews: owner-owned `.harness/reviews.json`, commit+push under the loop lock

## What was built

Moved the human-owned `reviewed` flag OUT of `.harness/TASKS.json` into its own
owner-owned committed file `.harness/reviews.json`, and made the daemon commit +
push that file immediately under the SAME lock the autonomous loop uses ŌĆö so a
review is durable (survives a daemon restart + a working-tree reset), reaches
GitHub, and can never conflict with the loop (disjoint git paths).

### 1. New shared lock helper ŌĆö `src/core/repo-lock.ts`
- `resolveRepoPaths(cwd)` derives `root` (`git rev-parse --show-toplevel`),
  `gitCommonDir` (`--git-common-dir`, made absolute), and the lock dir
  `<git-common-dir>/<basename(root)>-loop.lock` ŌĆö **byte-identical** to loop.sh's
  `acquire_lock` derivation.
- `acquireRepoLock()` is the mkdir-based lock + `pid` file with the same stale-pid
  reclaim protocol as loop.sh (dead PID ŌåÆ reclaim), async-polling for a live holder
  up to a timeout, returning an idempotent PID-checked release.
- Added a cross-reference comment in `.harness/loop.sh acquire_lock` documenting the
  two derivations must stay in sync.

### 2. Owner-owned reviews store + overlay ŌĆö `src/api/server.ts`
- `REVIEWS_PATH` / `reviewsPathFor()`; `readReviews()` (absent/empty ŌåÆ `{}`).
- `readBacklog()` now OVERLAYS `reviewed = reviews[id]?.reviewed ?? false` (tasks no
  longer carry the field); keeps `specContent` inlining.
- Replaced `setTaskReviewed`/`writeTaskReviewed` (TASKS.json editors) with
  `setReviewEntry` (pure field-scoped reviews-map editor, stamps `at`) +
  `writeReviewEntry` (atomic temp+rename).
- `migrateReviewsOut(tasksRaw, at)` ŌĆö pure one-time migration: strips `reviewed`
  from every task, seeds reviews from any `reviewed:true` task.
- `commitReviewsFile()` ŌĆö acquires the repo lock, stages ONLY `.harness/reviews.json`
  explicitly, commits `reviews: <id> reviewed=<bool> [skip ci]`, then
  fetch+rebase+push with a bounded 3-try retry; best-effort push (failed ŌåÆ non-fatal
  `warning`); lock released in `finally`; whole phase bounded by a timeout. Uses
  `execFile` with arg arrays (never shell strings). `defaultCommitReviews` no-ops
  outside a git repo.
- `POST /api/backlog/:id/reviewed`: now validates a `T\d+` id (400 otherwise),
  writes reviews.json atomically + commits+pushes, serialized by an in-process
  mutex (`serializeReview`), returns `{ ok, id, reviewed, committed, pushed, warning? }`.

### 3. Migration applied to the real files
- Seeded `.harness/reviews.json` = `{}` (no task was `reviewed:true` ŌĆö all 136 were

  committed).
