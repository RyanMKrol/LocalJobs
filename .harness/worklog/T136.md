## What was built

Moved the human-owned `reviewed` flag OUT of `.harness/TASKS.json` into its own
owner-owned committed file `.harness/reviews.json`, and made the daemon commit +
push that file immediately under the SAME lock the autonomous loop uses ÔÇö so a
review is durable (survives a daemon restart + a working-tree reset), reaches
GitHub, and can never conflict with the loop (disjoint git paths).

### 1. Shared lock helper ÔÇö `src/core/repo-lock.ts`
- `resolveRepoPaths(cwd)` derives `root` (`git rev-parse --show-toplevel`),
  `gitCommonDir` (`--git-common-dir`, made absolute), and the lock dir
  `<git-common-dir>/<basename(root)>-loop.lock` ÔÇö **byte-identical** to loop.sh's
  `acquire_lock`.
- `acquireRepoLock()` is the mkdir-based lock + `pid` file with the same stale-pid
  reclaim protocol as loop.sh (dead PID Ôćĺ reclaim), async-polling for a live holder
  up to a timeout, returning an idempotent PID-checked release.
- Added a cross-reference comment in `.harness/loop.sh acquire_lock` documenting that
  the two derivations must stay byte-for-byte in sync.

### 2. Owner-owned reviews store + overlay ÔÇö `src/api/server.ts`
- `REVIEWS_PATH` / `reviewsPathFor()`; `readReviews()` (absent/empty/unparseable Ôćĺ `{}`).
- `readBacklog()` OVERLAYS `reviewed = reviews[id]?.reviewed ?? false` (tasks no
  longer carry the field); keeps `specContent` inlining.
- Replaced `setTaskReviewed`/`writeTaskReviewed` (TASKS.json editors) with
  `setReviewEntry` (pure field-scoped reviews-map editor, stamps `at`) +
  `writeReviewEntry` (atomic temp+rename).
- `migrateReviewsOut(tasksRaw, at)` ÔÇö pure one-time migration: strips `reviewed`
  from every task, seeds reviews from any `reviewed:true` task.
- `commitReviewsFile()` ÔÇö acquires the repo lock, stages ONLY `.harness/reviews.json`
  explicitly (`git add -- <rel>`), commits `reviews: <id> reviewed=<bool> [skip ci]`,
  then fetch+rebase+push with a bounded 3-try retry; best-effort push (failed Ôćĺ
  non-fatal `warning`); lock released in `finally`; whole phase bounded by a timeout.
  Uses `execFile` with arg arrays (never shell strings). A no-op when nothing is
  staged (idempotent re-write). `defaultCommitReviews` no-ops cleanly outside a git repo.
- `POST /api/backlog/:id/reviewed`: validates a `T\d+` id (400 otherwise), writes
  reviews.json atomically + commits+pushes, serialized by an in-process mutex
  (`serializeReview`), returns `{ ok, id, reviewed, committed, pushed, warning? }`.

## Notes
- The push is best-effort by design; the local commit is the durability guarantee.
- loop.sh holds the loop lock for its WHOLE run, so a daemon review-commit during a
  build simply waits up to its timeout then defers with a non-fatal warning ÔÇö the
  next click (or the next loop push that touches an unrelated file) carries it out.
  This is acceptable per the spec (bounded git phase, lock never pinned).
