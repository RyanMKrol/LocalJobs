### What changed

Single file: `dashboard/app/globals.css` (lines 144–146 area).

**Before:**
```css
.section-heading-summary { ... border-radius: 6px; transition: background 0.15s; }
details .section-heading-summary:hover { background: rgba(255,255,255,.06); }
```

**After:**
```css
.section-heading-summary { ... /* border-radius and transition removed */ }
details:has(> summary.section-heading-summary) { border-radius: 8px; transition: background 0.15s; padding-bottom: 6px; }
details:has(> summary.section-heading-summary:hover) { background: rgba(255,255,255,.06); }
```

The `:has()` approach drives the highlight off the header hover but applies the background to the whole `<details>`, so:
- **Open section**: hover tints header + all expanded task cards as one unit.
- **Collapsed section**: hover tints just the header (details = summary only), unchanged behaviour.

The summary's negative left margin (`-8px`) was intentionally preserved — the caret content still
starts at the details left edge (margin -8px + padding 8px = 0 net offset), so the tinted background
aligns correctly with the caret without needing to adjust the margin.

`padding-bottom: 6px` on details gives slight breathing room at the section bottom.
`border-radius: 8px` gives rounded corners on the tinted background when hovering.
`transition: background 0.15s` moved from summary to details (smooth in AND out).

The `marginTop: 28` inline styles in `backlog/page.tsx` remain untouched and continue to
visually separate the three sections.
