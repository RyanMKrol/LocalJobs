# T096 worklog — Gate detail: neutral 'Produced →' / '→ Consumed' labels

## What was done

Replaced the frame-of-reference-ambiguous "Output"/"Input" labels on the gate
detail page with neutral directional labels: **"Produced →"** (producer side)
and **"→ Consumed"** (consumer side), as decided in the 2026-06-22 interview.

### Changes

**`src/api/server.ts`**
- Renamed the gate inspection response fields from `output`/`input` to
  `produced`/`consumed` (line ~442).
- Updated the gate description string builder from `output: ${pd}` / `input: ${cd}`
  to `produced: ${pd}` / `consumed: ${cd}` for consistency.
- Updated the inline comment describing the response shape.

**`dashboard/app/lib/api.ts`**
- Renamed `GateInspection.output: GateSide | null` → `produced`
- Renamed `GateInspection.input: GateSide | null` → `consumed`
- Updated the `GateSide` interface comment.

**`dashboard/app/workflow-runs/[id]/gates/[producer]/[key]/page.tsx`**
- Changed `SideCard` `role` type from `'Output' | 'Input'` to
  `'Produced →' | '→ Consumed'`.
- Added `logLabel: string` prop so the "view … logs →" foot link still reads
  "view producer logs →" / "view consumer logs →" (the old `role.toLowerCase()`
  would have produced "view produced → logs →").
- Updated both `<SideCard>` call sites: roles, logLabels, and `side` props
  (`inspect?.produced`, `inspect?.consumed`).
- Updated the flow comment.

**`CLAUDE.md`**
- Updated the gate-page description: "upstream **Output** → **Gate** →
  downstream **Input**" → "**Produced →** → **Gate** → **→ Consumed**".
- Updated the API response field descriptions.
