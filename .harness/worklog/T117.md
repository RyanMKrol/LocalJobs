# T117 — Workflow-run header: show 'X of N stages · % · time'

## What changed

`dashboard/app/workflow-runs/[id]/page.tsx` line ~380:

- Removed `{run?.progress_msg}` (which carried the last-succeeded stage name, e.g. `perfumes-build`).
- Replaced with `${completedStages} of ${totalStages} stages` computed from:
  - `totalStages = workflow?.jobs.length ?? latestRuns.length` — prefers the structural definition count; falls back to the distinct stages seen in member runs if the workflow definition hasn't loaded yet.
  - `completedStages = latestRuns.filter(r => r.status !== 'queued' && r.status !== 'running').length` — counts only terminal states (success, failed, timeout, cancelled, skipped) from the latest run per stage (via the existing `latestByStage` helper), consistent with the T081 completed-stages progress roll-up.
- Percentage and duration are unchanged.
