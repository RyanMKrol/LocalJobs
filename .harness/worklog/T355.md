# T355: Logs page — level bracket tags + distinct colour/background per level

## What changed
- `dashboard/app/logs/page.tsx`: added a `<span className="log-lvl-tag log-lvl-tag--${l.level}">[{l.level.toUpperCase()}]</span>`
  immediately before the message text, after the existing source link/span (real text content, not CSS generated content).
- `dashboard/app/globals.css`:
  - Added `--logs-info: #58a6ff` to the fixed `--logs-*` `:root` block (alongside `--logs-err`/`--logs-warn`), not theme-overridden.
  - Added `.logs .lvl-info { color: var(--logs-info); }` (previously no rule existed for `.lvl-info` at all).
  - Gave `.lvl-error`/`.lvl-warn`/`.lvl-info` a shared `display:block; padding:1px 8px; margin:0 -8px; border-radius:4px;` rule plus
    per-level low-opacity `background: rgba(...)` tints (~8-10% alpha) so each level reads as a full-width row band, following the
    same tinting approach as `.io-filter-chip--success.active` / `.io-filter-chip--failed.active`.
  - Added `.log-lvl-tag { font-weight: 600; margin-right: 6px; color: inherit; }`.
- Left `dashboard/app/runs/[id]/page.tsx` and `dashboard/app/workflow-runs/[id]/page.tsx` untouched (out of scope) — they inherit
  the improved colour/background from the shared CSS only, as expected.
