# T080 worklog — Mobile workflow DAG: centre the down arrow between stacked node boxes

## Attempt 1 (2026-06-22)

### Root cause
In the `@media (max-width: 640px)` block in `dashboard/app/globals.css`, the `.dag-arrow` rule
overrides `font-size: 0` (to hide the desktop `→` text) and adds `text-align: center`, but did
not set `justify-content: center`. Since `.dag-arrow` has `display: flex` inherited from the
desktop rule (line 123), the `::after` pseudo-element that renders `↓` was flex-positioned at
the start (left) instead of centred, even though the adjacent `.dag-wave` nodes were centred
via `justify-content: center` (line 274).

### Fix
Added `justify-content: center` to the mobile `.dag-arrow` override (line 277):
```css
/* before */
.dag-arrow { font-size: 0; text-align: center; }

/* after */
.dag-arrow { font-size: 0; text-align: center; justify-content: center; }
```

This centres the `↓` horizontally within its flex row, aligning it with the centred node
boxes above and below it. The desktop rule is unchanged (it doesn't set `justify-content`, so
the `→` text remains left-aligned in the horizontal DAG flow).
