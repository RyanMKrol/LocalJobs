# T447 — Fix framework-wide stale absolute output-artifact paths in work_items.detail

## What changed

1. **`src/db/store.ts`** — added `WORKFLOWS_ROOT` (mirrors `src/api/server.ts`'s) and
   an exported `toStoredPath(absPath)` helper: given an absolute path under the
   workflows root, returns it relative to that root; anything else is left
   unchanged. `markWorkItem` now runs `opts.detail.markdown`/`opts.detail.path`
   through this before `JSON.stringify` — zero changes needed at any job call site.

2. **`src/api/server.ts`** — `safeOutputMarkdown`/`safeOutputFile` now resolve a
   *relative* candidate against the freshly-computed `WORKFLOWS_ROOT` (not
   `process.cwd()`); an absolute candidate (legacy rows, or anything genuinely
   outside the tree) resolves exactly as before. `resolveOutputForm` needed no
   change — it just returns whatever string is stored; the guard functions do the
   resolution.

3. **`scripts/backfill-fix-stale-output-paths.ts`** (new) — one-time, idempotent
   repair for rows written before this fix. Two repair strategies, tried in order:
   - Plain root-rename repair: try each path segment as a candidate suffix against
     the current `WORKFLOWS_ROOT` (handles the `src/jobs` → `src/workflows` rename).
   - Workflow-folder-rename repair: for jobs whose OWNING WORKFLOW FOLDER was also
     renamed (e.g. `plex` → `missing-tv-seasons`, T098), resolve the job's current
     workflow directory via `workflow_jobs` + a `*.workflow.ts` scan/import (mirrors
     `findWorkflowDataOut` in `server.ts`), then re-join the stale path's `data/...`
     suffix against that directory.
   - **Found + fixed a real bug while building this**: my first version's directory
     walk for `*.workflow.ts` files did NOT skip `data/` folders — it picked up the
     stale nested self-clone under `src/workflows/projects-sync/data/repos/LocalJobs/`
     (the exact incident documented in root `CLAUDE.md`'s "Never let data/ folders be
     scanned for code" gotcha) and its `Map.set(name, dir)` let the clone's path win
     over the real one. Fixed by skipping any directory literally named `data` during
     the walk, same as the real registry does.

4. **Docs** — added the storage-convention note to root `CLAUDE.md` (under the
   `detail.markdown` T110 bullet) and a short standalone rule to
   `src/workflows/CLAUDE.md`.
