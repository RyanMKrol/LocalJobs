# T284 — pre-loop-checkin slash command

## What was built

Added `.claude/commands/pre-loop-checkin.md` — a read-only, advisory-only runbook mirroring
`.claude/commands/loop-recover.md`'s house style (YAML frontmatter, guardrails section, numbered
`bash`/`jq` steps, consolidated final report) but that never mutates anything.

Four checks implemented:

1. Needs-human tasks that block buildable (`pending`, `gate:null`) work, cross-referenced against
   `.harness/human-done.json` to distinguish "about to auto-reconcile" from "owner action needed now".
2. Session hygiene: working tree status, ahead/behind vs `origin/main`, and whether a
   `loop.sh`/`supervise.sh` process or the repo lock (same path as `loop.sh`'s `acquire_lock` /
   `src/core/repo-lock.ts`) is currently held.
3. Dependency short-circuits: `dependsOn` edges whose target is already `done` (informational only).
4. Per-task definition quality: missing/invalid `facets.layer`/`facets.workType` (against
   `.harness/facets.json`'s vocabulary), missing spec file, missing `## Do`/`## Done when` sections,
   empty `scope` array — checked for every `pending`, non-`needs-human` task.

Ends with one consolidated GO / GO-with-notes / NO-GO report.
