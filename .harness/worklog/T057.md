# T057 — Validation gates: clickable detail + comprehensive logging

## Goal
Make every validation gate on a workflow run inspectable: log each gate check
(assertion + result) to the run's framework logs, expose per-gate detail
(contract key + description + state) via the API, and make EVERY gate chip on the
run DAG (passed/pending/failed, not just failed) clickable to a detail view.

## What I built
1. **Executor logging** (`src/core/workflow-executor.ts`)
   - New `gateAssertions(gate)`: pure lookup of the producer's `produces[key]` and
     consumer's `consumes[key]` contract `description`s ("producer asserts: …",
     "consumer asserts: …").
   - Before each gate check, log `⛒ checking gate [producer → consumer] artifact
     "key" · <assertions>`; after, the existing `✓ gate ok …` / `⨯ Gate violation
     …` result line now also carries the assertion suffix. The recorded
     `recordGateFailure` detail format (the `gateFailurePrefix` that `classifyGates`
     matches on) is unchanged — no breaking change to gate-state classification.

2. **API per-gate detail** (`src/api/server.ts` `gatesForWorkflow`)
   - Each derived gate is enriched with a `description` combining the producer's
     `produces[key].description` and consumer's `consumes[key].description`
     ("produces — … · consumes — …"). Flows through `classifyGates` (which spreads
     `...gate`) into the `gates[]` returned by `GET /api/workflow-runs/:id`.
   - `Gate` interface in `src/core/dag.ts` gained an optional `description?` field;
     `deriveGates`/`classifyGates` logic is otherwise untouched (deriveGates only
     sees keys, so description is enriched by the API which has the job defs).

3. **Dashboard detail view**
   - `dashboard/app/components/Dag.tsx`: exported `gateAnchor(g)` (stable DOM id);
     `GateChips` now renders EVERY gate as a link to `#<gateAnchor>` (was: only
     failed gates linked, to the failure run).
   - `dashboard/app/workflow-runs/[id]/page.tsx`: new **Validation gates** panel
     (`GatePanel`) — one anchored row per gate showing state badge, producer →
     consumer, artifact key, the asserts/description, and links to the
     producer/consumer/violation run logs (consumer run id from the page's
     existing `runIdByJob`, or `failureRunId` when failed).
   - `dashboard/app/lib/api.ts`: `GateStatus.description?`.
   - `globals.css`: `.badge.passed` (green) + generic `a.dag-gate:hover`.

## Tests added
- `src/core/workflow-executor.test.ts`: new described-contract gate members
  (`g-prod-desc`/`g-cons-desc`) + test asserting the executor logs the "checking"
  line, both assertions, and the gate-ok result with the assertion suffix.
- `src/core/dag.test.ts`: test that `classifyGates` preserves a gate's
  `description` (detail) into the classified status.
- `dashboard/scripts/mobile-check.mjs`: fixture gates now carry `description` so
  the new panel column is exercised at phone width.
