# T138 — Gate page: collapse identical producer/consumer sides; keep + prove asymmetric (fan-in) support

## What I built

1. **`shapesIdentical(a, b)` — pure, exported, unit-tested** (`src/core/dag.ts`).
   Deep-compares two DECLARED `ArtifactShape`s: `summary`, `format`, and the
   `expectations` array (each by `label` + `detail`, order-sensitive, treating an
   absent `detail` as `''`). A missing/absent shape on EITHER side → `false` (we
   never claim a match we can't confirm). Based on declared shapes only, never live
   actuals.

2. **`identical: boolean` on BOTH gate inspection endpoints** (`src/api/server.ts`):
   - `GET /api/workflow-runs/:id/gates/:producer/:key` (run-scoped, live actuals)
   - `GET /api/workflows/:name/gates/:producer/:key` (run-agnostic, shapes only)
   Both compute it via `shapesIdentical(producedShape, consumedShape)`. Mirrored in
   the dashboard API types (`dashboard/app/lib/api.ts`: `GateInspection`,
   `StructuralGateDetail`).

3. **Collapse the page when identical** — both gate detail pages
   (`workflow-runs/[id]/gates/...` and `workflows/[name]/gates/...`). Extracted a
   shared `ShapeBody` building block (reuses `pairExpectations`/`Mark`) so the
   collapsed single panel and the two-sided `SideCard`s render the same content. When
   `identical` → ONE consolidated panel: centre **Gate** block (key + description +
   state badge + violation link) + a single **Contract** card (shared shape, run-side
   actual ✓/✗ + sample, producer+consumer log links in the footer). When NOT identical
   → the existing two-sided `Produced → | Gate | → Consumed` layout, unchanged. Reuses
   `.gate-flow` (already column-on-mobile); added only a tiny `.gate-collapsed-note`
   rule to `globals.css`.

4. **Asymmetric / fan-in support confirmed — no framework change needed.**
   `deriveGates` is edge-scoped (one gate per producer→consumer edge) and
   `enforceGate` checks `produces[key]` and `consumes[key]` independently, so
   asymmetry already worked. Proved with synthetic fixtures:
   - `dag.test.ts`: fan-in (a, b → c, distinct keys) derives one gate per edge;
     plus the `shapesIdentical` unit tests.
   - `server.test.ts`: identical shapes → `identical:true` (both endpoints, both
     sides still returned); the existing asymmetric `gate-wf` → `identical:false`.
   - `workflow-executor.test.ts`: an asymmetric gate (producer shape ≠ consumer
     shape, separate checks + counters) — both checks run; a consumer-side drift
     fails the gate independently of the ok producer side.
