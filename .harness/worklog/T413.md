# T413 — stocks-sync: add stocks-resolve-names stage

## What was built

- `src/services/trading212.service.ts`: added optional `name?: string` to `NormalizedPosition`
  (additive only — no existing exports touched).
- `src/workflows/stocks-sync/config.ts`: added `namedPositionsJsonPath`.
- `src/workflows/stocks-sync/contracts.ts`: added `stocksNamedPositionsContract()` (key

  resolution logic is untouched (deliberate interim duplication, to be removed by T414).
- `stocks-sync.workflow.ts`: 5-job DAG (`stocks-fetch → stocks-resolve-names → stocks-snapshot →
  stocks-watch → stocks-notify`), doc comment updated.
- `src/workflows/stocks-sync/CLAUDE.md` + root `CLAUDE.md`-adjacent doc: updated stage list/count
  (root `CLAUDE.md` needed no stocks-sync-specific edit — its stocks-sync mentions are all
  high-level/thin-index and unaffected).
- Tests: new `stocks-resolve-names.test.ts` (name resolved / soft-miss warn / zero-input skip /
  ledger detail shape / static check that no isin-field or OpenFIGI function is referenced);
  `contracts.test.ts` extended with a `stocks-named-positions` block + updated shape/keys tests +
  DAG-gate regression test now asserts 5 jobs / 4 gates including the new boundary.
  `stocks-snapshot.test.ts` needed no changes (all its raw-positions reads are dependency-injected).
