# T179 worklog — Missing seasons view on the TV-seasons workflow page

## What was built

1. **API** (`src/api/server.ts`):
   - `GET /api/missing-seasons` — reads `missing-seasons.json`, overlays each `(tmdbId, season)` pair with `notified`/`ignored` state from the `plex-seasons-notify` ledger.
   - `POST /api/missing-seasons/:tmdbId/:season/ignore` — calls `ignoreSurfacedItem('plex-seasons-notify', pairKey(tmdbId, season))`. Guarded by the global loopback/token mutation check.

2. **notify stage** (`src/jobs/plex/stages/notify.ts`):
   - Imported `ignoredItemKeys` from `db/store.ts`.
   - `writeReport` now filters out ignored `(tmdbId, season)` pairs before building the markdown report, mirroring the movies notify T145 pattern.

3. **API client** (`dashboard/app/lib/api.ts`):
   - Added `MissingSeason` and `MissingSeasons` types.
   - Added `api.missingSeasons()` and `api.missingSeasonsIgnore(tmdbId, season)` methods.

4. **Dashboard** (`dashboard/app/workflows/[name]/page.tsx`):
   - Added `MISSING_SEASONS_WORKFLOWS = new Set(['plex'])` — declarative gate, rename-safe (T151 updates the set).
   - Added `MissingSeasonsManager` component mirroring `MovieGapsManager`: polls `/api/missing-seasons`, groups by show, per-item Ignore button, scrollable panel (reuses `.movie-gaps-scroll`), themed buttons, tidy empty states.
   - Rendered conditionally: `{MISSING_SEASONS_WORKFLOWS.has(name) && <MissingSeasonsManager />}`.

5. **Tests** (`src/api/server.test.ts`):
   - `GET /api/missing-seasons` overlays notified + ignored status.
   - `POST /api/missing-seasons/:tmdbId/:season/ignore` suppresses a gap and reflects in GET.
   - `POST` rejects bad ids (400).
   - `runNotify` writeReport excludes ignored pairs from both report and notifications.
