# T600: Per-suggestion ledger rows for recommender branches

## Summary
Extended recordBranchLedgerRow to record ONE row per raw suggestion instead of one count-only row per branch run. The Output section on workflow run pages now shows each actual recommendation the branch proposed, not just a count.

## What was implemented

### File changes
1. **src/workflows/movies/stages/recommend.ts** (lines 71-106): Rewrote `recordBranchLedgerRow` to:
   - Loop through `file.suggestions` array
   - Create a ledger row for EACH suggestion with key `dayKey::index`
   - Record suggestion details (title, year, reason, lens) in each row's detail blob
   - Gracefully handle error/skip cases (zero rows, not a placeholder)

2. **src/workflows/tv-recs/stages/recommend.ts** (lines 80-96): Identical logic for TV domain

3. **src/core/recommender/branch.test.ts**: Updated regression test to verify per-suggestion rows (was checking for one per-branch row)

### Test coverage
- **src/workflows/movies/stages/recommend.test.ts**: Added two new T600 test blocks:
  - One checking 3 suggestions → 3 ledger rows with full detail fields
  - One checking error case (zero suggestions → zero rows, not crash)
  
- **src/workflows/tv-recs/stages/recommend.test.ts**: New file with same test pattern as movies

### Design
- Keys: `${dayKey(now)}::${index}` — stable, deduped across same-day re-runs, branch-local
- Detail shape: `{ title, year, reason, lens }` per CLAUDE.md convention
- Error handling: zero rows on skip/error (legitimate outcomes, no crash, no placeholder)
- Backward compat: zero rows per skip is cleaner than one empty-suggestions row

## Dashboard impact
The run page's Input/Output (StageIoPanel) now shows each individual suggestion as a row instead of one row with `{ name, suggestions: N }`.
