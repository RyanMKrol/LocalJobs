## Edge case: no prior anchor

Read `toggleOne`: the shift-click branch is only taken when
`shiftKey && lastClickedId != null && visibleIds.includes(lastClickedId)`. On first load
`lastClickedId` is `null`, so a shift-click with no prior plain-click anchor in the session falls
straight to the `else` branch — a normal single toggle of the clicked row, no crash, no
silently-wrong selection. Same fallback applies if the anchor row got filtered out of
`unreviewedVisible` (e.g. the review-status filter changed, or the anchor got marked reviewed) since
`visibleIds.includes(lastClickedId)` would then be false.
