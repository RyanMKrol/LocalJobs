# T549 — fmtDuration hours tier + fmtRelative ISO/clock-skew robustness

Date: 2026-07-14

## What I did
- `dashboard/app/ui.tsx`:
  - `fmtDuration`: added an hours tier. Durations >= 60 minutes now render as
    `Xh Ym` instead of overflowing minutes (e.g. `125m 12s` -> `2h 5m`).
  - `fmtRelative`: reused `fmtTime`'s `includes('Z') || includes('T')` guard so an
    already-ISO input (ending in `Z`) parses correctly instead of double-appending
    `Z` and producing `NaN` -> `NaNs ago`. Also clamped a negative diff (a future
    timestamp under clock skew) to return `'just now'` instead of `-2s ago`.
- `dashboard/app/ui.test.ts`: added
  - `fmtDuration: durations over an hour render as "Xh Ym"` (2h5m12s -> `2h 5m`).
  - `fmtRelative: an already-ISO (Z-suffixed) input parses without NaN`.
  - `fmtRelative: a future timestamp (clock skew) clamps to "just now"`.
