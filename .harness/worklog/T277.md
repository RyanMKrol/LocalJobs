# T277 — Migrate gate detail pages to shared component library

## What changed
- `dashboard/app/workflow-runs/[id]/gates/[producer]/[key]/page.tsx`: replaced the two
  hand-rolled `<span className={`badge ${gate.state}`}>{statusLabel(gate.state)}</span>`
  usages with the shared `<StatusBadge status={gate.state as RunStatus} />` (ui.tsx). Same
  markup/CSS class (`.badge <state>`), same emoji-gating behaviour — no visual change.
  `GateStatus.state` ('passed'|'failed'|'pending') isn't part of the `RunStatus` union but
  `STATUS_LABELS`/`STATUS_EMOJI` in ui.tsx already cover all three, so the cast is safe.
- `dashboard/app/workflows/[name]/gates/[producer]/[key]/page.tsx`: replaced the hand-rolled
  `<p className="muted"><a href={...}>← {name}</a></p>` back-link with
  `<WorkflowRunBackLink workflowRunId={null} fallback={{ href: ..., label: name }} />` — its
  no-run fallback branch renders byte-identical markup.
- The "definition" badge and the gate-center "Gate" role labels were left as-is: they're not
  status values (no mapping in StatusBadge's label/emoji tables) and aren't `.pill` semantics
  (different CSS class/look from `.badge`), so swapping them would be a visual/behavioural
  change, which the spec explicitly rules out.
- `dashboard/scripts/_dashboard-harness.mjs`: added `gateInspection` / `structuralGateDetail`
  fixtures, routed `/api/workflow-runs/*/gates/*` and `/api/workflows/*/gates/*` (ahead of the
  generic catch-alls), and added `gate-run-scoped` / `gate-definition-scoped` PAGES entries so
  both gate pages are covered by mobile-check + visual-check going forward.
