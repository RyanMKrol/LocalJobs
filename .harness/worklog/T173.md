# T173 worklog

## Root cause

Each `CollapsibleRow` had `borderBottom: 1px solid var(--border)` set as an inline style on its
inner `.done-row` div. This border was applied uniformly to ALL rows, including the last one. The
last row's border was flush against the panel's `border-bottom` (0 panel bottom-padding), creating
a visual double-border / extra-gap effect at the bottom of each section list: row content → 5px
padding → 1px row border → panel border. Compared to the top (just 5px padding from first row),
this bottom looked "thicker" — and the double-line at the very edge of the panel looked unbalanced.

## Fix

- Moved `borderBottom` logic from inline style to a CSS class in `globals.css`.
- Added `.done-row { border-bottom: 1px solid var(--border); }` (separator between rows).
- Added `.done-row.expanded { border-bottom: none; }` (expanded rows already had no border inline).
- Added `.panel > div:last-child .done-row { border-bottom: none; }` — removes border from the last
  CollapsibleRow in each panel, so the bottom inset is just 5px (matching the top inset and the
  per-row padding), with no decorative line pressed against the panel edge.
- In `page.tsx`: removed `borderBottom` from inline style; added `expanded` class when expanded.
- Applies to all three section lists (Harness-buildable, Needs-a-human, Done) since all use `CollapsibleRow`.
