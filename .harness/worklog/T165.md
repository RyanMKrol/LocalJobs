# T165 worklog

## What was done

Removed `tbody tr:hover` from the `transform: translateY(-2px)` rule in
`dashboard/app/globals.css` (was lines 648-651, grouped with `.statcard:hover`
and `.dag-node:hover`).

The hover-lift transform stays on `.statcard` and `.dag-node` where it renders
cleanly. Table rows (`<tr>`) don't compose `transform` cleanly — the sub-pixel
reflow as the cursor enters causes visible jitter/shift of row content.

Replacement for table rows under joyful themes: a smooth `box-shadow: inset 3px 0 0 var(--accent)` left-accent stripe on hover, with a `transition: background .12s, box-shadow .12s` so it fades in instead of popping. The base `background: var(--panel-2)` highlight (line 88, default theme) is unchanged and still applies.
