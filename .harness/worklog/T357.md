# T357 — remove `.movie-gaps-scroll` internal scroll cap

## Changes

- `dashboard/app/globals.css`: `.movie-gaps-scroll` no longer sets `max-height`/`overflow-y` —
  it's now just `display: flex; flex-direction: column; gap: 12px`, so content displays in
  full and the page itself scrolls. Rewrote the stale comment on `.movie-gaps-scroll .panel`
  (was explaining the now-removed max-height/scroll interaction) to instead just note
  `flex-shrink: 0` guards against unwanted compression in a flex column. Added a one-line
  note above `.movie-gaps-scroll` itself flagging the class name as a historical misnomer
  now that it isn't a scroll container — left the rename out of scope per the spec (would
  touch `page.tsx` x2 + `WorkflowOutputSection.tsx` for a cosmetic-only change).
- `dashboard/scripts/_dashboard-harness.mjs`: removed the two T333 `FLOWS` entries
  (`movie-recs-scrolled`, `tv-recs-scrolled`) that drove a mouse-wheel scroll over
  `.movie-gaps-scroll` and asserted `scrollTop > 0` — that behavior no longer exists. Kept
  the `movieRecs`/`tvRecs` fixture data at ~15 rows (still useful realistic data, and now
  exactly what lets the baseline `workflow-movie-recs`/`workflow-tv-recs` `PAGES` screenshots
  show the full uncapped list).
