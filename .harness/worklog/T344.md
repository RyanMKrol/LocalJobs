# T344 — Reintroduce System mode to the dashboard light/dark control

## Interaction shape (decision record)

Same single compact header button as T308 (no dropdown/popover). Clicking it
CYCLES through three states in this fixed order:

  Dark (🌙) → Light (☀️) → System (🖥️) → Dark (🌙) → …

- Dark: forces `data-mode="dark"`.
- Light: forces `data-mode="light"`.
- System: `data-mode` follows the OS `prefers-color-scheme` live (a `matchMedia`
  change listener is attached only while in this state).

A fresh/default load (nothing in `localStorage.localjobs.mode`) resolves to
`'system'` — matching the pre-T142/T190 "untouched = follow OS" convention and
the task spec's explicit instruction to default the `useMode()` lazy-init to
`'system'` when the stored key is absent.

## Implementation

- `dashboard/app/ui.tsx`:
  - `ModeId` is tri-state again: `'dark' | 'light' | 'system'`.
  - `resolveMode` explicitly handles `stored === 'system'` (same branch as
    `null`) for clarity, per the spec.
  - `useMode()`'s lazy `useState` initializer reads `localStorage` synchronously
    (not `data-mode`, since that attribute never holds `'system'`), falling back
    to `'system'` when absent.
  - `cycle()` replaces `toggle()`: advances via a `MODE_CYCLE` lookup table and
    persists the literal choice (including `'system'`) to `localStorage`.
  - A `useEffect` keyed on `mode` sets `data-mode` directly for `dark`/`light`,
    or — for `system` — attaches/detaches a `matchMedia('(prefers-color-scheme:
    dark)')` change listener that keeps `data-mode` in sync with the OS live.
  - `ThemeControls` renders 🌙/☀️/🖥️ per current mode with an `aria-label`/
    `title` stating the current state and what the next click does (e.g.
    "System mode (follows OS) — click for dark mode").
- `dashboard/app/layout.tsx` (`PREPAINT`): verified, no change needed. Its
  existing logic (`dark = mo==='dark' || (mo!=='light' && OS dark)`) already
  treats any non-`'light'`, non-`'dark'` stored value — including the new
  `'system'` — as "follow OS", which is exactly correct for all three states.
- `dashboard/scripts/_dashboard-harness.mjs`: added a new `FLOWS` entry,
  `overview-theme-system`, that clicks `.theme-trigger` (up to 3 times, using
  the actual current icon rather than a hardcoded click count, since a fresh
  load already starts in System state) until the button shows 🖥️, then asserts
  it via `waitForFunction`.
- `CLAUDE.md`: rewrote the stale `data-mode` bullet (still describing the old
  T190 tri-state picker, since it was apparently never updated for T308) to
  describe the current T344 cycle-on-click tri-state control accurately.
