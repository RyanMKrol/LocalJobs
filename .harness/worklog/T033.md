# T033 ‚Äî Dismissed vs ignored: park ignored items, show only on overview

## Goal
Unify the manual-park concept on ONE name ("ignored"). An ignored item must
stay ignored across re-runs, never count as stuck, never appear on the pipelines
tab or pipeline/job detail, and surface ONLY on a new clickable overview
**Ignored** tile that filters to them. Unit tests cover persistence +
exclusion-from-stuck.

## Root cause / framing
The code already had a working manual park, but under TWO names: the store used
`dismissed` while the dashboard button said "Ignore" ‚Äî the "two concepts" the
task calls out. Unified everything on `ignored`.

## Changes
- **src/db/store.ts** ‚Äî `WorkStatus` `dismissed`‚Üí`ignored`; `isWorkItemDone`
  treats `ignored` as done; renamed `dismissWorkItem`‚Üí`ignoreWorkItem` (parks a
  `failed` row as `ignored`); added `ignoredItems()` (overview list, StuckItem
  shape). `stuckItems`/`stuckCount` unchanged ‚Äî they query `status='failed'`, so
  ignored is never counted as stuck.
- **src/db/schema.sql** ‚Äî status comment updated to `ignored`.
- **src/db/index.ts** ‚Äî idempotent data migration on startup:
  `UPDATE work_items SET status='ignored' WHERE status='dismissed'`.
- **src/api/server.ts** ‚Äî `POST /api/stuck/ignore` (was `/dismiss`) ‚Üí
  `ignoreWorkItem`; new `GET /api/ignored` (optional `?job=`) ‚Üí `ignoredItems`.
- **dashboard/app/lib/api.ts** ‚Äî `dismiss`‚Üí`ignore`; added `ignored()`.
- **dashboard/app/page.tsx** ‚Äî added 6th clickable **Ignored** tile; new
  `'ignored'` filter shows ONLY the ignored-items table (hides pipelines / stuck
  / runs); stuck-section "Ignore" button now calls `api.ignore`.
- **src/db/store.test.ts** ‚Äî rewrote the park test (`t-ignore`): parks as
  `ignored`, leaves the stuck list + count, shows on `ignoredItems()`, stays
  ignored / not-stuck across a re-run cycle, idempotent, distinct from unstick.
- Docs: CLAUDE.md (work-ledger + stuck-vs-ignored convention, migration note),
  README.md (six tiles incl. Cancelled which T032 left stale, + Ignored).

Pipelines/pipeline-detail/job-detail pages needed no change: they show
`stuck`/`stuckCount` only, which already exclude `ignored`.
