# T351 — Integrations page action-column spacing/clipping fix

## What I did
- `dashboard/app/services/page.tsx`: rebalanced the `<colgroup>` widths (Service 40%→38%,
  Rate/min|day|month 16%→15% each, action column 12%→17%).
- Replaced the edit-mode action `<td style={{ whiteSpace: 'nowrap' }}>` (which forced Save/Cancel
  onto one line and caused "Cancel" to clip as "Cal…") with a plain `<td>` wrapping both buttons in
  a `display: inline-flex; flexWrap: 'wrap'; justifyContent: 'flex-end'` span.
- `dashboard/app/globals.css`: added `.services-table td:last-child { text-align: right; }` and
  `.services-table td:last-child .btn { white-space: normal; text-align: center; }` right after the
  existing `.services-table { table-layout: fixed; }` rule, so action-column buttons can wrap
  instead of overflowing at in-between viewport widths.
- `dashboard/scripts/_dashboard-harness.mjs`: added a `services-edit-limits-row` FLOWS entry that
  clicks the first "Edit limits" button on `/services` and waits for "Save" to appear, so the
  edit-mode row is captured by visual-check going forward.
- No route rename (`services-url-rename`) had landed, so the file stayed at
  `dashboard/app/services/page.tsx` (nav label already reads "Integrations").
