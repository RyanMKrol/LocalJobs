# T028 — Validation gates: surface on the DAG graph + failure logs

## Goal
Render validation gates on the pipeline-RUN DAG: a gate marker on each
producer→consumer boundary, failed gates visually distinct (red) + clickable to
their failure logs, gate state exposed via the API. Gates must NOT show on the
structure-only `/pipelines/[name]` view.

## What I built
- **`src/core/dag.ts`** (pure, testable):
  - `gateFailurePrefix(gate)` — the single source of truth for the leading text
    `recordGateFailure` writes (`Gate violation [producer → consumer] artifact "key"`).
    The executor now builds its detail from this helper too, so the writer and the
    reader can never drift apart.
  - `classifyGates(gates, runs)` → `GateStatus[]` with `state: passed|failed|pending`
    and `failureRunId?`. Logic keys off the consumer's **latest** run (DB returns
    member runs `ORDER BY started_at`): latest is a gate-failure for THIS gate →
    `failed` (link it); latest is skipped or a gate-failure for a *different* inbound
    gate → `pending`; consumer otherwise executed → `passed`; no consumer run → `pending`.
  - Added `GateState`, `GateStatus`, `GateRunRef` types.
- **`src/core/pipeline-executor.ts`** — detail string now uses `gateFailurePrefix(gate)`.
- **`src/api/server.ts`** — new `gatesForPipeline(name)` derives the pipeline's gate
  structure (buildDag + deriveGates over members' produces/consumes); the
  `GET /api/pipeline-runs/:id` endpoint classifies it against that run's member runs
  and returns `gates[]`. Structure-only `/api/pipelines/:name` is untouched.
- **`dashboard/app/lib/api.ts`** — `GateStatus` interface; `pipelineRun()` response
  now includes `gates`.
- **`dashboard/app/components/Dag.tsx`** — optional `gates?` prop. A `GateChips`
  subcomponent renders one chip per inbound gate beneath its consumer node; failed
  chips are `<a href="/runs/{failureRunId}">` (red). No `gates` prop → no chips, so
  the structure-only view stays clean.
- **`dashboard/app/pipeline-runs/[id]/page.tsx`** — threads `gates={data.gates}` into `<Dag>`.
- **`dashboard/app/globals.css`** — `.dag-gates` / `.dag-gate` (+ passed/pending/failed).
- Docs: README pipeline section + CLAUDE.md gate convention updated; trade-off
  (chips-not-edge-lines + prefix coupling) added to `.harness/LIMITATIONS.md`.

## Tests
Added 7 `classifyGates` cases to `src/core/dag.test.ts` covering failed+link,
passed, real-consumer-failure-still-passed, not-run pending, skipped pending,
one-of-two-inbound-gates pending, and latest-failure-wins-across-cycles.
