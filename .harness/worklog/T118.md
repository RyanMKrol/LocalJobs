# T118 — Reusable stuck-items popover with per-item + bulk unstick/ignore

## Attempt 1 — 2026-06-23

### What was built

**Backend (`src/db/store.ts`)**
- Added `BulkStuckScope` type: `{ type: 'all' } | { type: 'job'; jobName } | { type: 'workflow'; jobNames[] }`
- Added `bulkUnstickItems(scope, minAttempts?)` — deletes all currently-`failed` ledger rows in scope
- Added `bulkIgnoreItems(scope, minAttempts?)` — marks all currently-`failed` ledger rows `ignored`
- Both act ONLY on `status='failed'` rows (mirrors the single-item semantics exactly)
- Empty `jobNames` on workflow scope is a safe no-op (returns 0)

**API (`src/api/server.ts`)**
- Added `resolveBulkScope(body)` helper that converts the HTTP request body to a `BulkStuckScope`
  - Workflow scope resolves the workflow name → member job names via `getWorkflowJobs`
  - Unknown workflow name → returns `null` → caller responds 400
- Added `POST /api/stuck/unstick-bulk` — returns `{ ok, unstuck: number }`
- Added `POST /api/stuck/ignore-bulk` — returns `{ ok, ignored: number }`
- Both use the existing global POST mutation guard (loopback/token)

**API client (`dashboard/app/lib/api.ts`)**
- Added `BulkScope` type mirroring the backend scope
- Added `scopeBody()` helper converting `BulkScope` to request body
- Added `api.unstickBulk(scope?)` and `api.ignoreBulk(scope?)`

**UI (`dashboard/app/ui.tsx`)**
- Added `StuckPopover` component: takes `items`, optional `scope`, `onClose`, `onAction`
- Per-item ↻ Unstick (no confirmation) and ✕ Ignore (with confirmation)
- Bulk "Unstick all" (confirmation) and "Ignore all" (strongly-worded confirmation)
- After any action: calls `onAction()` so the parent can force a data re-poll
- Reuses existing `.db-modal-overlay` / `.db-modal` CSS chrome; only added `.stuck-popover` and `.stuck-popover-bulk`

**CSS (`dashboard/app/globals.css`)**
- `.stuck-popover` — wider than the DB row modal (760px)
- `.stuck-popover-bulk` — footer action row with gap + border-top

**Overview page (`dashboard/app/page.tsx`)**
- Added `stuckPollTick` state to force re-poll after bulk actions
- "Stuck items" tile: if stuck items exist, click opens the popover instead of filtering
- Added "Manage all…" button next to the stuck-items `<h2>` heading
- `StuckPopover` rendered when `stuckPopoverOpen === true`
- Existing per-row ↻ Unstick / ✕ Ignore buttons in the inline table untouched

**Docs**
- `CLAUDE.md` — extended the "Stuck vs ignored" section with bulk operations + `StuckPopover` details
- `README.md` — updated Overview description to mention the modal and bulk actions
