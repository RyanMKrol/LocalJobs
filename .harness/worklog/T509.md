# T509: Cache trading212 portfolio fetch via cacheKey (22h TTL)

## What was done

Implemented 22-hour caching for the Trading212 portfolio fetch endpoints to avoid redundant API calls during daily scheduled runs and intra-day re-runs.

### Changes made:

1. **`src/services/trading212.service.ts`**: Added `cacheTtlMs: 79_200_000` (22 hours in milliseconds) to the trading212 service definition. This sets the cache TTL for all portfolio fetch calls routed through this service.

2. **`src/workflows/stocks-sync/stages/stocks-fetch.ts`**: 
   - Modified to call `callService('trading212', ..., { cacheKey: 't212:portfolio:invest' })` for invest account fetches
   - Modified to call `callService('trading212', ..., { cacheKey: 't212:portfolio:isa' })` for ISA account fetches (when configured)
   - Added logic to disable caching when tests inject a mock `fetchPortfolio` function (to prevent test pollution from cached results)

3. **`src/workflows/stock-digest/stages/stock-portfolio-fetch.ts`**: 
   - Applied identical caching logic as stocks-sync for the separate stock-digest workflow
   - Uses the same account-based cache keys: `t212:portfolio:invest` and `t212:portfolio:isa`
   - Same test pollution prevention logic

### How caching works:

- When `runStocksFetch` or `runStockPortfolioFetch` call the real `fetchPortfolio` function (not test-injected), the results are cached via `callService`
- Repeated calls to the same account within 22 hours return the cached result without making a live API call
- Different accounts (invest vs ISA) have separate cache keys, so each is independently cached
- Test-injected functions bypass caching to prevent cross-test pollution in the scratch database

## Notes

- The 22-hour TTL ensures that daily scheduled runs (which are 24+ hours apart) always fetch fresh data
- Intra-day re-runs or manual runs within 22 hours of the last scheduled run get the cached result
- Read-only requirement maintained: only portfolio GET requests are cached, never any mutating calls (broker APIs are strictly read-only per repo policy)
- No other services affected; this is specific to trading212 portfolio fetches
