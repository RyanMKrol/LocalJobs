# T146 ‚Äî movies 8-branch Claude recommendation layer (TMDB-verified merge)

Built on top of T145's `movies` workflow (snapshot ‚Üí franchise-gaps ‚Üí notify). Added the
subjective half: 8 Claude recommender branches ‚Üí TMDB-verified merge ‚Üí folded into the same
monthly digest, separate from the franchise gaps.

## What was built

**Shared Claude CLI helper** ‚Äî `src/services/claude.ts` (`runClaude` / `extractJsonObject`),
gated through the existing `claude-cli` service, self-contained (reads `LOCALJOBS_CLAUDE_BIN`/
`_TIMEOUT_MS` from env). This is the "extract, don't duplicate" the spec asked for, placed in the
shared services layer (the cross-job home). Perfumes' own `perfumes/claude.ts` was left untouched
(out of scope ‚Äî scope = movies/services/.env.example/CLAUDE.md), so the spawn logic is briefly
duplicated; recorded in LIMITATIONS.md as a follow-up.

**8 recommender branches** (a true DAG fan-out off the snapshot, one `*.job.ts` each via a
`makeBranchJob` factory):
- 3 stratified-random serendipity branches (`rec-random-1/2/3`) ‚Äî each a different seeded
  stratified slice of the owned library.
- 5 targeted: `rec-auteur` (depth ‚Äî directors owned ‚â•3), `rec-canon` (depth ‚Äî top-genre canon),
  `rec-thin-genre` (breadth), `rec-older-era` (breadth ‚Äî pre-1980), `rec-world-cinema` (breadth ‚Äî
  non-English).
- Each builds its prompt from the taste profile + a STRATIFIED (balanced, not proportional)
  library sample (`recs.ts::stratifiedSample`), asks Claude for ~5 diverse un-owned films, and
  writes raw `{title, year, reason, lens}` to `data/out/recs/<branch>.json`. A junk reply / Claude
  error / rate-limit / no-target branch writes an EMPTY file + error and returns normally ‚Äî never
  fails the run. Recent recommendations (history file) are fed into every prompt so months vary.

**Merge stage** (`rec-merge`, `merge.ts`) ‚Äî code enforces correctness: pool all branches ‚Üí cheap
title/year dedup ‚Üí TMDB-search each (`/search/movie`, via the `tmdb` service) ‚Üí must resolve to a
real id, must NOT be owned, must NOT be in the `movie-recs` ledger (already recommended/ignored) ‚Üí
dedupe across branches by resolved tmdb id (merging lenses) ‚Üí balance per genre
(`balanceByGenre`, cap 3, ~12 total). Resilient: per-item search failures skipped, TMDB quota stops
gracefully, always writes a (possibly empty) `recommendations.json` and succeeds (so the gaps
digest still fires).

**Notify** (`movie-gaps-notify`, extended) ‚Äî now consumes BOTH `franchise-gaps` and
`recommendations` contracts. ONE combined monthly digest + ONE markdown report with two SEPARATE
sections (Franchise gaps grouped by collection; Recommendations with lens + reason + TMDB link).
Recs dedupe per recommended tmdb id (`movie-recs` ledger), are appended to the history file, and
support ignore-to-suppress (`ignoreSurfacedItem('movie-recs', ‚Ä¶)`) exactly like gaps. `buildDigest`
made recs-aware while staying back-compatible (gaps-only title unchanged).

**Contracts/DAG** ‚Äî `recommendationsContract()` (SHAPE-only, allows empty so an empty recs run
doesn't block the gaps digest). Branch‚Üímerge edges deliberately have NO gate (an empty branch is
legitimate). Workflow: snapshot ‚Üí {franchise-gaps, 8 branches} ; 8 branches ‚Üí rec-merge ;
{franchise-gaps, rec-merge} ‚Üí notify. `maxConcurrency: 1` keeps Claude CLI sequential.

## Tests (hermetic ‚Äî NO live Claude/TMDB; injected fns + scratch DB + synthetic fixtures)
- `recs.test.ts` ‚Äî stratified sampler is BALANCED not proportional (500/100 ‚Üí ~20/20; thin stratum
  drained fully before over-sampling the dominant one; diverges by seed yet deterministic);
  `balanceByGenre` cap+round-robin; `dedupeRawByTitleYear`; `genreNameFromIds`; profile slicing.
- `stages/merge.test.ts` ‚Äî drops hallucinated (no TMDB match) / owned / previously-recommended /
  ignored; keeps a valid novel one; cross-branch dedup merges lenses; per-genre cap enforced.
- `stages/recommend.test.ts` ‚Äî `parseSuggestions` valid + rejects junk; runBranch writes
  suggestions on a valid reply; junk/no-JSON and Claude-error/rate-limit handled WITHOUT failing;
  a no-target (auteur, no directors) branch skips and never calls Claude.
- `stages/notify.test.ts` (extended) ‚Äî combined/recs-only digest titles; recs Recommendations
  section in the report; dedup per tmdb id; ignore-to-suppress excludes a rec from digest+report.
  Existing gaps tests unchanged & still green.
