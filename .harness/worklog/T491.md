# T491: Single-click "Run all workflows" button

## Summary
Collapsed the two-step confirmation flow for the admin page's "Run all workflows" button into a single click. The button now dispatches `confirmRunAll()` directly on click without requiring an intermediate inline confirm/cancel panel.

## Changes Made

### 1. dashboard/app/admin/page.tsx
- **Removed state**: `runAllConfirming` (line 19) — no longer needed
- **Updated confirmRunAll()**: Removed the `setRunAllConfirming(false)` call at the end (line 45) since the state no longer exists
- **Replaced conditional rendering**: 
  - Removed the `{!runAllConfirming && ...}` block that rendered the initial button (lines 87-101)
  - Removed the `{runAllConfirming && ...}` block that rendered the inline yellow-bordered confirm panel (lines 103-122)
  - Added a single always-rendered button that calls `confirmRunAll()` directly on click (lines 85-91)
- **Updated button text**: Shows 'Starting…' when `runAllBusy` is true, 'Run all workflows' otherwise
- **Simplified input disabled condition**: Changed from `disabled={runAllBusy || runAllConfirming}` to `disabled={runAllBusy}` (line 80)

### 2. dashboard/scripts/_dashboard-harness.mjs
- **Removed FLOWS entry**: Deleted the `admin-run-all-confirm` flow entry (original lines 1009-1021)
  - This flow was driving the interaction test that clicked the button expecting an inline confirm panel
  - Since the panel no longer exists, the flow is no longer needed
  - The baseline 'admin' PAGES entry still screenshots the page's default state with the single always-visible button

## Visual Confirmation
The change simplifies the UI by removing the two-step interaction:
- **Before**: User clicks button → panel appears → user clicks Confirm/Cancel
- **After**: User clicks button → dispatches immediately (single step)

The admin page's baseline screenshot still captures the page's default state, which now shows only the single Run all workflows button with no leftover confirm/cancel panel, confirming the visual intent of the task.

## Notes
- The task spec called the modification "non-destructive" since running all workflows doesn't delete anything
- This aligns with the explicit description on the page copy: "This mirrors clicking 'Run now' on every workflow's own page" — each workflow's individual Run button also dispatches immediately without a confirm step
- The change improves UX by reducing unnecessary modal/panel interactions while maintaining the same dispatch behavior
