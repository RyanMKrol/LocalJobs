# T210 — Dismiss entire collection/show at once (bulk-ignore current items only)

## What was built

Added a **"✕ Ignore all"** button at the header of each grouped output panel on the
workflow detail page: movie franchise gaps grouped by collection (`MovieGapsManager`)
and missing TV seasons grouped by show (`MissingSeasonsManager`).

### Backend
- `src/db/store.ts`: Added `ignoreSurfacedItems(jobName, itemKeys[])` — a
  transactional batch of the per-item `ignoreSurfacedItem` upsert. Returns total rows
  affected. Empty-array short-circuits to 0 without a DB round-trip.
- `src/api/server.ts`: Added two bulk endpoints behind the global loopback/token
  mutation guard:
  - `POST /api/movie-gaps/ignore-bulk { tmdbIds: number[] }`
  - `POST /api/missing-seasons/ignore-bulk { items: { tmdbId, season }[] }`

### Frontend
- `dashboard/app/lib/api.ts`: Added `api.ignoreMovieGapBulk(tmdbIds)` and
  `api.missingSeasonsIgnoreBulk(items)` client methods.
- `dashboard/app/workflows/[name]/page.tsx`: Added `ignoreCollection` handler +
  `busyCollection` state to `MovieGapsManager`; `ignoreShow` handler + `busyShow`
  state to `MissingSeasonsManager`. Each group header now has a flex row with the
  title on the left and the "✕ Ignore all" button (disabled while in-flight) on the
  right.

### Semantics
CRITICAL: the bulk dismiss is NOT a standing rule. Only the exact item keys passed
are upserted to `ignored`. A new item key appearing in a later run for the same
collection/show is unaffected — it will surface fresh.

## Tests

Added `T210 ignoreSurfacedItems` unit test in `src/db/store.test.ts` that:
- Verifies 3 items bulk-ignored return `affected = 3`
- Confirms each is `isWorkItemDone` / `ignoredItemKeys`
- Confirms a NEW 4th key (`col:4`) is NOT treated as done (resurface semantics)
- Confirms idempotency (re-bulk-ignoring same keys = still 3 changes)
- Confirms empty array returns 0
