# T262 worklog

## What was built

Added backend support for declared output forms in `src/api/server.ts`:

- **`safeOutputFile`** — new exported guard like `safeOutputMarkdown` but without
  the `.md` restriction. Same `data/out/` tree confinement + realpath/symlink checks.
- **`resolveOutputForm`** — reads `detail.format` + `detail.path` (or `detail.markdown`
  for the default markdown form) from the work item via `getWorkItem` (imported from store).
- Both output endpoints (`/api/workflow-runs/:id/output` and `/api/workflows/:name/output`)
  now dispatch on `detail.format`: `"markdown"` → `safeOutputMarkdown`, any other form →
  `safeOutputFile`. Response includes `format` so T282 can dispatch the renderer.
- **Backward compat**: `detail.format` is optional; items with only `detail.markdown`
  (places, perfumes) behave byte-identically — `resolveOutputForm` defaults to `"markdown"`.
- **CLAUDE.md** updated with the Output-form convention section documenting how to add a
  new form without touching the endpoint plumbing.

## Tests added (server.test.ts, T262 block)

- `safeOutputFile`: accepts json file in `data/out/`
- `safeOutputFile`: rejects null / traversal / outside `data/out/` / missing file
- Output endpoint: serves markdown form byte-identically (backward compat, `format: 'markdown'`)
- Output endpoint: serves declared non-markdown form (json item, `format: 'json'`)
- Output endpoint: path-safety guard blocks unsafe declared path → `found: false`
