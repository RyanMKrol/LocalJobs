# T377 — gate padlock state background

## What changed

`GateEdge` in `dashboard/app/components/DagFlow.tsx`:
- Added `lockBg` — same three-way `lockColor` (red/green/muted), mixed at 24% (bumped up
  from the icon's existing 18% internal-fill ratio, `lockFill`) since 18% read too faint for
  the pending/grey state against the dotted graph background once actually rendered.
- Passed `lockBg` into the padlock's `<a>`/`<span>` wrapper (the element already carrying the
  `dag-gate-lock`/`--failed`/`--passed` classes) via an inline `--lock-bg` CSS custom property,
  rather than onto the separate outer `EdgeLabelRenderer` div — so the badge scales/transitions
  together with the icon on hover instead of the icon "escaping" a static background.

`dashboard/app/globals.css`'s `.dag-gate-lock` rule:
- `background: var(--lock-bg, transparent)` + `border-radius: 7px` + `padding: 3px 4px` (up
  from the icon's own zero padding) to read as a rounded-square badge with breathing room
  around the padlock glyph.
- Added `background-color` to the existing `transition` list so state changes (pending → passed
  once a run completes) animate consistently with the existing opacity/transform transitions.
- Left `--failed`/`--passed` opacity + the failed drop-shadow untouched — this only adds the
  background layer underneath the existing icon styling.

No changes to padlock SVG geometry, `lockSize`, `strokeWidth`, or the `lockColor`/`isFailed`/
`isPassed` derivation.
