# T414 — remove redundant ISIN/OpenFIGI resolution from stocks-snapshot

Step 2 of 2 (T413 was step 1): `stocks-snapshot` no longer independently calls Trading212's
instruments-metadata endpoint or the `openfigi` service — that resolution work now lives solely in
`stocks-resolve-names`. `stocks-snapshot` is now a pure report builder: reads
`named-positions.json`, writes `portfolio.json`/`portfolio.md`.

## Changes

- `src/workflows/stocks-sync/stages/stocks-snapshot.ts`: removed the `TRADING212_API_KEY_ID`/
  `_SECRET_KEY` credential check, the `fetchInstrumentsMetadata`/`resolveOpenFigiTickersBatched`/
  `resolveTickers` imports and the whole ISIN-resolution try/catch. Renamed
  `readRawPositions`/`RawPositionsReader` → `readNamedPositions`/`NamedPositionsReader`.
  `runStocksSnapshot`'s opts are now `{ readNamedPositions?, writePortfolio?, now? }`.
  `buildPortfolioMarkdown` now renders a "Company name" column (`position.name ?? '—'`) instead of
  "Real ticker". Dropped `resolvedCount` from the ledger detail.
- `src/workflows/stocks-sync/stages/stocks-snapshot.job.ts`: updated `description` to describe the
  narrower pure-report-builder responsibility. `consumes`/`produces` unchanged (already correct
  from T413).
- `src/workflows/stocks-sync/stages/stocks-snapshot.test.ts`: removed the ISIN-resolution test
  block and the credential-missing throw tests (this stage no longer touches Trading212 credentials
  at all); renamed fixture helpers to match; added a company-name-column markdown test; updated the
  ledger-detail test to not expect `resolvedCount`.
- `src/workflows/stocks-sync/CLAUDE.md`: rewrote the stage-3 section to describe the final
  behaviour (no resolution, company-name column) and removed the note about the interim duplicate
  instruments-metadata call.
