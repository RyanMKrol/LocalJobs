# T115 worklog — Workflow-run IO: render output markdown as markdown

## Decision: react-markdown

Chose `react-markdown@^10.1.0` (focused, well-maintained, actively developed).
- XSS-safe by default: does NOT parse or execute raw HTML embedded in content
  — raw HTML tags are escaped, not rendered. No `rehype-raw` plugin added.
- Pure CSS styling (no theme plugin dependency).
- Minimal bundle addition (~44 kB gzip for the workflow-run page).

Alternatives considered:
- `marked` + manual sanitisation: less safe by default, requires explicit
  DOMPurify/sanitize-html pass; more code.
- `micromark`: lower-level, no React integration out of the box.

## YAML frontmatter handling

`parseFrontmatter()` strips the leading `--- ... ---` block before passing
body to `<ReactMarkdown>`. The parsed key-value pairs render as a compact
`<dl>` (`.md-fm`) header inside the modal — tidier than dumping raw `---`
lines into the markdown renderer (which would show them literally).
