# T100 — Polish the run-limit control on the workflow Run UI

## What I did

### Problem
The run-limit input added by T094 was a bare `<input type="number">` sitting awkwardly next to the "▶ Run now" button:
- No visible label (only a tooltip visible on hover)
- Rendered as a bright browser-default white box on the dark theme — visually jarring
- On the workflow-run detail page, the `run_limit` badge had no background/border (`className="badge"` with no status class) making it look like plain text

### Changes

**`dashboard/app/globals.css`** — Added:
1. Base `input[type="number"], input[type="text"]` styling: dark panel-2 background, var(--border) border, proper padding + focus ring. This makes all inputs look site-native instead of browser-default white boxes on the dark theme.
2. `.run-limit-control`, `.run-limit-label`, `.run-limit-input` classes for the grouped label+input widget.

**`dashboard/app/workflows/[name]/page.tsx`** — Wrapped the bare input in a `.run-limit-control` div with a visible `<label>` ("LIMIT" in uppercase muted style matching table headers). Moved the width to the CSS class `run-limit-input`.

**`dashboard/app/workflow-runs/[id]/page.tsx`** — Changed `className="badge"` → `className="badge queued"` on the run_limit badge, giving it the grey pill background/border. Also tightened the label from "limited · N inputs" → "N inputs limit" (more naturally reads as metadata).
