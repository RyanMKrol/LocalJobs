## What was built

New single-stage workflow `src/workflows/plex-profiles/` â€” writes one markdown profile per Plex
title (movie + TV show) to `data/out/movies/` / `data/out/shows/`, sourced purely from the Plex
API (list endpoints + a new-to-this-repo `GET /library/metadata/<ratingKey>` detail fetch) â€” no
LLM call anywhere in this stage (phase 2, an optional Claude-narrated layer, is explicitly
deferred as a separate future task, per the spec).

Files added:
- `config.ts` â€” reuses `PLEX_MOVIE_SECTION`/`PLEX_TV_SECTION`; adds `PLEX_PROFILES_RUN_LIMIT`
  (default 0 = unlimited).
- `types.ts` â€” `PlexGuid`, `PlexTag`, `PlexRating`, `PlexStream`, `PlexPart`, `PlexMedia`,
  `PlexEpisodeMeta`, `PlexListItem`, `PlexMovieDetail`, `PlexShowDetail`.
- `lib.ts` â€” pure helpers: `extractGuidIds` (generalizes `extractTmdbId` to also pull imdb/tvdb
  ids), `tags`, `formatBytes`, `itemBytes` (both copied verbatim from `plex-space-saver/lib.ts`
  per the repo's small-per-workflow-helper convention), `slugFileName` (reuses Plex's own `slug`
  field), `buildMovieProfileMarkdown`, `buildShowProfileMarkdown` (fixed YAML frontmatter + four
  `##` sections: Summary, Cast & Crew, Ratings, Technical, Source).
- `lib.test.ts` â€” 13 unit tests covering every pure helper, including frontmatter/heading
  presence assertions against synthetic `PlexMovieDetail`/`PlexShowDetail` fixtures, and the
  `original_title` omit/include edge case for shows.
- `stages/build.ts` + `stages/build.job.ts` â€” job `plex-profiles-build`. Fetches the movie list,
  show list, and flat TV episode list (`?type=4`, for show byte totals) via `plexGet`; declares
  `inputKeys()` (every `movie:<ratingKey>`/`show:<ratingKey>`); per-key idempotency via a stored
  `detail.updatedAt` marker (mirrors `projects-sync/project-summarize.ts`'s `pushedAt` idiom) â€”
  unchanged titles are skipped without a detail fetch; applies `ctx.rootAllowed` +
  `PLEX_PROFILES_RUN_LIMIT`; fetches full detail per title needing (re)build, writes the markdown,
  and calls `markWorkItem(..., 'success', { detail: { name, markdown, updatedAt } })`; per-item
  `ctx.progress`/`ctx.log`; throws a summarizing error if any item failed this run (per the
  item-loop-must-fail-its-own-run convention).
- `plex-profiles.workflow.ts` â€” workflow `plex-profiles`, one member, `category: 'second-brain'`,
  weekly schedule `'0 5 * * 6'` (Saturday 05:00 â€” offset from `plex-space-saver`'s Sunday 06:00).
- `CLAUDE.md` â€” full current-state doc: confirmed Plex fields, the `updatedAt` marker idiom, the
  template shape, the "no DAG edge, no gate needed" reasoning (mirroring `plex-space-saver`), and
  an explicit note that phase 2 (a Claude-narrated layer) is deferred, not built here.
