# T252 — Shared dynamodb service

## What was built

- `src/services/dynamodb.service.ts`: self-contained `ServiceDefinition` named `'dynamodb'`,
  lazy-initialising a `DynamoDBDocumentClient` (from `@aws-sdk/lib-dynamodb` over
  `@aws-sdk/client-dynamodb`). Exposes five helpers: `dynamoGet`, `dynamoPut`, `dynamoQuery`,
  `dynamoDelete`, `dynamoBatchWrite`.
- Creds/region from env (`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_REGION`).
- Conservative free-tier limits: `ratePerMinute=30`, `monthlyCap=50_000` (daily = 1666),
  all env-overridable via `DYNAMODB_RATE_PER_MIN`, `DYNAMODB_MONTHLY_CAP`, `DYNAMODB_DAILY_CAP`.
  50k/month << 200M free-tier ceiling.
- `_resetClient()` export allows test injection of a mock client.
- `package.json`: added `@aws-sdk/client-dynamodb` + `@aws-sdk/lib-dynamodb` ^3.840.0.
- Unit test in `src/services/dynamodb.service.test.ts`: 10 subtests, all mocked (no live AWS),
  covering shape, free-tier math, all helpers, and the batch-25 guard.
