# T575 — env-overridable stocks-watch breach threshold

2026-07-15

## What I did
- `src/workflows/stocks-sync/config.ts`: added `stocksSyncConfig.breachThresholdPct` — a
  function (not a static value, since `stocksSyncConfig` is only evaluated once at module
  load) that reads `process.env.STOCKS_WATCH_BREACH_PCT`, falling back to 30 when unset,
  non-numeric, or ≤ 0.
- `src/workflows/stocks-sync/stages/stocks-watch.ts`: turned the hardcoded
  `export const BREACH_THRESHOLD_PCT = 30` into `export function BREACH_THRESHOLD_PCT()`
  delegating to `stocksSyncConfig.breachThresholdPct()`. `isBreaching`, the run's log line,
  the `gain >= threshold` check, and both `markWorkItem` detail computations now read the
  resolved threshold (captured once per run into a local `breachThreshold` at the top of
  `runStocksWatch`) instead of the old fixed constant.
- `src/workflows/stocks-sync/stages/stocks-watch.test.ts`: added two new cases —
  (g) sets `STOCKS_WATCH_BREACH_PCT=15` and confirms a +20% gain (which does not breach the
  default 30% bar) now breaches, and confirms the unset-env default is still 30%; (h) sets
  the env to `'not-a-number'`, `'0'`, and `'-10'` and confirms each falls back to the 30%
  default (a +20% gain never breaches under any of them).
- `.env.example`: documented `STOCKS_WATCH_BREACH_PCT` under the stocks-sync section with a
  one-line comment and a commented-out `=30` placeholder, matching the existing
  `TRADING212_*` override convention right above it.
- `src/workflows/stocks-sync/CLAUDE.md` (stage 4 section) and `README.md` (stocks-sync
  bullet): updated the fixed "30%"/"≥30%" wording to note the threshold is now
  env-overridable via `STOCKS_WATCH_BREACH_PCT` (default 30).

## Checks run
