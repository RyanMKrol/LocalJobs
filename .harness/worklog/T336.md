# T336 — Delete stale listens-backfill/listens-sync + orphaned spotify service

## What was built

- `src/db/store.ts`: three new transaction-wrapped admin helpers —
  `deleteWorkflowCompletely`, `deleteJobCompletely`, `deleteServiceCompletely` —
  each deleting a definition row and everything under it across the related
  tables (workflow_run_logs/workflow_runs/workflow_jobs/workflows;
  run_logs/runs/work_item_runs/work_items/jobs; service_usage/service_consumers/services),
  idempotent (no-op if the name is absent), following the existing
  `resetWorkflowOutput` transaction/prepared-statement style.
- `scripts/cleanup-listens-spotify.ts`: one-off admin script (mirrors
  `backfill-service-usage.ts`'s header-comment style) that calls the three
  helpers for `listens-backfill`, `listens-sync`, `lastfm-backfill`,
  `lastfm-sync`, `spotify` in that order, printing a before/after summary.
- `src/db/store.test.ts`: new T336 test block seeding a synthetic target
  workflow/job/service + a sibling of each, asserting full cross-table
  removal, sibling rows untouched, and idempotent no-op on a second call
  (plus a call against a never-existed name).
