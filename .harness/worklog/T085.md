# T085 — Human-readable exit codes

## Approach

Added `exitCodeLabel(code: number | null): string` to `dashboard/app/ui.tsx` mapping:
- `null` → "no exit code"
- `0` → "OK"
- `1` → "failed — exception"
- `137` → "killed (SIGKILL)"
- `143` → "terminated (SIGTERM)"
- other → "unexpected exit"

Updated `dashboard/app/runs/[id]/page.tsx` to use the helper. Display shows `label (code)` when exit_code is non-null (e.g. "OK (0)", "failed — exception (1)"). For null exit codes, shows the label only when the run is in a terminal state (success/failed/timeout/cancelled/skipped/partial), otherwise shows '—' (still running/queued).

The exit code is the only place in the dashboard that shows raw exit codes (confirmed by grep). The helper is exported from `ui.tsx` for future reuse if needed.
