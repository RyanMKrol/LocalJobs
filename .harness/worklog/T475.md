# T475 — overrides-audit workflow + `_overridden_at` staleness tracking

2026-07-10

## What I did

1. Added a nullable `_overridden_at TEXT` timestamp column matching each of the 5 existing
   `_overridden` flags: `services.limits_overridden_at`, `workflows.schedule_overridden_at`,
   `workflows.max_concurrency_overridden_at`, `workflows.notify_enabled_overridden_at`,
   `jobs.timeout_ms_overridden_at`. Added to `schema.sql` (fresh DBs) and as additive
   `ALTER TABLE` statements in `src/db/index.ts` (existing DBs), following the T098-safe
   pattern (no index on any of these columns).
2. Updated `updateServiceLimits`, `updateWorkflowSchedule`, `updateWorkflowConcurrency`,
   `updateWorkflowNotifyEnabled`, and `updateJobTimeout` in `src/db/store.ts` to stamp the
   matching `_overridden_at` column with `datetime('now')` whenever they set the
   corresponding `_overridden` flag to 1. There is no "reset to code default" action
   anywhere in the codebase today, so there's nothing that needs to clear these back to
   NULL yet — left a comment in `src/db/index.ts` noting where a future reset path would
   clear the pair together.
3. Added `listStaleOverrides(minAgeMs)` in `src/db/store.ts` — walks every row across
   services/workflows/jobs with an `_overridden = 1` flag and returns it if
   `_overridden_at IS NULL` (unknown age, always report) or older than `minAgeMs`.
4. Built `src/workflows/overrides-audit/` (manifest + `overrides-audit-scan` stage),
   mirroring `plex-space-saver`'s single-stage report-only shape: calls
   `listStaleOverrides(14 days)`, writes a JSON report to `data/out/stale-overrides.json`
   (declared output form `json`, T262 convention), idempotent per ISO week via the
   work_items ledger. Weekly cron, Sundays 07:00, category `regular-maintenance`. No push
   notification, no IDEAS.jsonl write, no automated code patching — report only.
5. Wrote `src/workflows/overrides-audit/CLAUDE.md`; added the workflow's row to root
   `CLAUDE.md`'s "Shipped example workflows" table (+ bumped the "16 worked-example
   workflows" count and the two other lists that enumerate every shipped folder) and to
   `README.md`'s workflow list.
6. Added the required Conventions-section sentence to root `CLAUDE.md`: a dashboard
   override is provisional, not a place to retire a value — fold it into the manifest's
   code default and clear the flag once stable for 2+ weeks; this workflow's weekly report
   is the reminder mechanism.
