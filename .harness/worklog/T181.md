# T181 worklog ŌĆö boustrophedon (snake) DAG layout

## Approach

Extended `Dag.tsx` + `globals.css` to implement a snake layout for wide graphs.

### Detection
In the `ResizeObserver` callback, after rendering:
1. Query all `[data-dag-wave]` elements and one `[data-dag-intra-arrow]` element to get representative widths.
2. Compute total width if all waves+arrows were in a single row (sum of widths + gaps).
3. If `totalWidth > containerWidth + 1`: calculate `wavesPerRow` (how many fit in one row).
4. Mobile check: if `getComputedStyle(container).flexDirection === 'column'` (phone layout), skip snake entirely.

### Snake rendering
When `wavesPerRow` is set, compute `rowGroups` (array of arrays of wave indices). For each row:
- Even rows (0, 2, ...): render waves in normal LŌåÆR order with `ŌåÆ` arrows.
- Odd rows (1, 3, ...): `displayOrder = [...rowWaveIndices].reverse()`, render with `ŌåÉ` arrows.
  - Arrow gate index for RTL: `displayOrder[j+1]` (the smaller/producer wave index).
- Between-row turn connector: `Ōåō` glyph aligned to:
  - `flex-end` (right) after LŌåÆR rows ŌåÆ sits above where the next RŌåÆL row begins (right side).
  - `flex-start` (left) after RŌåÆL rows ŌåÆ sits above where the next LŌåÆR row begins (left side).
  - Carries `arrowGates(lastLogicalWaveIdx)` for any gates on the inter-row edge.

### Data attributes added
- `data-dag-wave=""` on all `.dag-wave` divs (both normal and snake modes) ŌĆö used for width measurement.
- `data-dag-intra-arrow=""` on all intra-row `.dag-arrow` divs ŌĆö used for arrow width measurement.

### CSS added (globals.css)
```
.dag-snake-container  ŌĆö flex-column wrapper, width: 100%
.dag-snake-row        ŌĆö flex-row with gap: 14px (matches .dag gap)
.dag-row-turn         ŌĆö flex container for the between-row Ōåō connector
.dag-row-turn-right   ŌĆö justify-content: flex-end (after LŌåÆR rows)
.dag-row-turn-left    ŌĆö justify-content: flex-start (after RŌåÆL rows)
```

### Observed behavior (synthetic test)
The movies workflow (~4 waves wide, tall with 8 parallel branches) renders as a single non-snaked row ŌĆö confirming tall-but-not-wide graphs are unaffected.

For a long sequential pipeline with enough waves to overflow the container, the snake layout activates: waves flow LŌåÆR on row 0, then RŌåÆL on row 1 (reversed display order with ŌåÉ arrows and a Ōåō turn on the right), then LŌåÆR on row 2 (Ōåō turn on the left), and so on. The turn connector correctly routes gate marks for any gate spanning the row boundary.

Phone layout unchanged ŌĆö `wavesPerRow` is set to `null` whenever `flexDirection === 'column'`.
