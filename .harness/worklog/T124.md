# T124 — human-reviewed flag on backlog tasks

## Goal
Let the owner mark `done` tasks as personally reviewed: a `reviewed` boolean on each
TASKS.json task, surfaced + toggled from the dashboard Backlog page, persisted via the
FIRST dashboard→harness-file write (otherwise the dashboard is strictly read-only).

## What I built
1. **DATA.** Backfilled `reviewed: false` onto all 131 tasks in `.harness/TASKS.json`
   via `jq '.tasks |= map(if has("reviewed") then . else . + {reviewed:false} end)'`.
   Verified: 0 tasks missing the field, 0 non-false, all 131 present, 112 `done`
   statuses preserved, 2-space indent + trailing `}\n` unchanged.
2. **API (`src/api/server.ts`).**
   - `readBacklog(path = BACKLOG_PATH)` now normalises each task to default
     `reviewed: false` when absent (defensive), and takes an optional path (test seam).
   - `setTaskReviewed(raw, id, reviewed)` — pure, exported: parses the JSON text, sets
     ONLY the matched task's `reviewed`, returns re-serialised JSON (2-space + newline)
     or `null` for an unknown id.
   - `writeTaskReviewed(path, id, reviewed)` — reads, applies the pure edit, validates
     it parses, then writes via temp-file + `renameSync` (atomic; can't clobber the
     loop's concurrent `jq` status-write, which is itself field-scoped).
   - `GET /api/backlog` returns `reviewed`; new `POST /api/backlog/:id/reviewed`
     `{ reviewed: bool }` — guarded by the existing `authoriseMutation` loopback/token
     check; 400 on non-boolean body, 404 on unknown id.
   - `createApiServer` gained `opts.backlogPath` so tests drive a throwaway fixture file.
3. **UI (`dashboard/app/backlog/page.tsx` + `lib/api.ts` + `globals.css`).**
   `markReviewed` client method + `reviewed?` on `BacklogTask`. Each Done row shows a
   green **Reviewed** / muted **Not reviewed** pill and a "Mark as reviewed" /
   "Mark not reviewed" toggle (stops row-expand propagation), with an immediate refetch
   (refresh nonce) so the pill flips at once. Done list gained a Reviewed /
   Not-reviewed / All filter; header shows `N done (M reviewed)`. New CSS:
   `.pill.reviewed`, `.pill.unreviewed`, `.review-toggle`, `.review-filter-bar`.
4. **DOCS.** CLAUDE.md (dashboard-is-read-only line + harness backlog rule now note the
   `reviewed` exception), HARNESS.md §8 (schema + a dedicated `reviewed` paragraph),
   README Backlog bullet, and a new LIMITATIONS.md entry (no cross-process file lock →
   theoretical last-writer-wins on the whole file in a ms-wide window; atomic, so never
   corrupt, only a dropped edit).
5. **TESTS (`src/api/server.test.ts`).** Pure `setTaskReviewed` (sets only target,
   preserves nested/sibling/top-level fields, null on unknown id); GET surfaces
   `reviewed` (absent→false); POST atomically sets only the target on disk preserving
   the rest; 404 unknown id + 400 non-boolean. Driven against a `mkdtempSync` fixture.
