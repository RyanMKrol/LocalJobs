# T362 — stock-digest: persist raw facts JSON + soft ticker cross-check

## Context
Investigation into a suspected YNDX "hallucination" confirmed it was a real Trading212 holding
(logged by `stock-digest-build` before Claude was ever called). No hallucination occurred. This task
is a forward-looking observability improvement so a genuine future drift is caught automatically.

## Changes
- `src/jobs/stock-digest/config.ts`: added `factsPathFor(weekKey, outDir?)`, mirroring
  `reportPathFor` — resolves to `data/out/stock-digest-facts-<weekKey>.json`.
- `src/jobs/stock-digest/stages/stock-digest-build.ts`:
  - Writes `JSON.stringify(facts, null, 2)` to `factsPathFor(key, outDir)` right after
    `mkdirSync(outDir)`, BEFORE the Claude call, and logs the path.
  - Added exported pure helpers `extractCandidateTickers(text)`, `findUnknownTickers(candidates,
    facts)`, and the `KNOWN_NON_TICKER_TOKENS` stoplist (ISA, USD, GBP, EUR, ETF, N/A, ID, URL, API).
  - After the Claude call, runs `findUnknownTickers(extractCandidateTickers(result.text), facts)` and
    logs one `warn:` line if non-empty — soft signal only, never throws, never skips the write, never
    changes the `markWorkItem` outcome.
- `src/jobs/stock-digest/stages/stock-digest-build.test.ts`: added tests for
  `extractCandidateTickers`, `findUnknownTickers`, an end-to-end facts-JSON-write assertion, and two
  end-to-end runs asserting the warn line appears (unknown ticker) / doesn't appear (no unknown
  ticker) without throwing or altering the work-item outcome.
- `CLAUDE.md`: updated the `stock-digest` paragraph to describe both additions.
