# T375 — Portal the cron-tooltip so it escapes `.panel` clipping

## What I did

- Confirmed the repro first: with the harness's existing `workflow()` fixture schedule
  (`'0 3 * * 1-5'`) `cronToEnglish` doesn't recognise the weekday-range pattern, so `CronBadge`
  fell back to plain text with **no icon at all** on every existing screenshot — meaning the bug
  was never actually exercised by the harness before this task. Fixed the fixture (step 4 below)
  first, confirmed the icon renders, then confirmed clipping via a throwaway hover script (deleted
  before committing — not present in the final diff).
- `dashboard/app/ui.tsx` — `CronBadge`: added a `triggerRef` on the `.cron-help` span, a
  `useLayoutEffect` (keyed on `open`) that computes the trigger's document-relative position via
  `getBoundingClientRect()` + `window.scrollX/Y`, and renders the `.cron-tooltip` via
  `createPortal(..., document.body)` instead of as an inline child — escaping every ancestor
  `.panel`'s `overflow: hidden` entirely. Positioned with inline `position: absolute` +
  `transform: translate(-50%, -100%)` (document-relative, not viewport-relative `fixed`, so it
  scrolls naturally with the page) plus a 6px gap above the trigger, matching the old CSS's visual
  offset. `react-dom` has no installed `@types/react-dom` in this project (would need a
  `package.json`/lockfile change, out of this task's scope), so the import is annotated
  `// @ts-expect-error` rather than editing dependencies.
- `dashboard/app/globals.css` — removed the now-stale `position: absolute; bottom: calc(100% + 6px);
  left: 50%; transform: translateX(-50%);` declarations from `.cron-tooltip` (positioning is now
  driven entirely by the portal's inline styles); kept every purely-visual rule (background, border,
  radius, padding, font, colors, width/max-width, box-shadow, z-index, pointer-events, text-align,
  font-family, font-weight) untouched.
- `dashboard/scripts/_dashboard-harness.mjs`:
  - Changed the shared `workflow()` fixture's `schedule` from `'0 3 * * 1-5'` to `'0 3 * * *'` (a
    `cronToEnglish`-recognised daily pattern, "At 03:00, every day") so the "ⓘ" icon actually
    renders in the existing `PAGES` screenshots.
  - Added two new `FLOWS` entries: `cron-tooltip-workflow-detail` (`/workflows/places`, hovers
    `.cron-help`, waits for `.cron-tooltip` visible) and `cron-tooltip-workflows-list`
    (`/workflows`, same interaction) — both use `.hover()` only (not `.click()`, since `onClick`
    toggles `open` and Playwright's click already fires `mouseenter` first, which would immediately
    toggle it back off).
