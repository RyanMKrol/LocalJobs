# T097 — Schedules: human-readable cron tooltip on hover

## Approach

Implemented a small in-repo `cronToEnglish(expr: string): string` helper in
`dashboard/app/ui.tsx` rather than reaching for a library (e.g. cronstrue).
Rationale: the app uses a small set of cron patterns (all observed schedules are
daily-at-HH:MM); hand-rolling coverage of the common shapes keeps the repo
dependency-light and the helper is deliberately conservative — it falls back to
the raw expression for any pattern it can't confidently describe, so a wrong
description is structurally impossible.

## Patterns covered

- `* * * * *` → Every minute
- `*/N * * * *` → Every N minutes
- `0 * * * *` → Every hour
- `0 */N * * *` → Every N hours
- `M * * * *` → At M minutes past every hour
- `M H * * *` → At HH:MM, every day  ← the pattern actually in use
- `M H * * D` → At HH:MM on <DayName>  (single-digit DOW)
- Anything else → raw expression unchanged (safe fallback)

Month-specific patterns (`month !== '*'`) are not handled and fall back to raw.

## Render sites updated

Three places where schedules are shown in the dashboard:
1. `dashboard/app/workflows/page.tsx` — Schedule column in the workflows table
2. `dashboard/app/workflows/[name]/page.tsx` — Schedule row in the detail kv panel
3. `dashboard/app/page.tsx` — Inline schedule text in workflow cards on the overview

`null` schedules (manual-only workflows) render as before with no tooltip.
