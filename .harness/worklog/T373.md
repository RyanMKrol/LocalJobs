# T373 — resolve ISIN + real-world ticker for stocks-sync positions

## What was built

- Added `fetchInstrumentsMetadata` (GET `https://live.trading212.com/api/v0/equity/metadata/instruments`,
  same HTTP Basic auth pattern as `fetchPortfolio`) in `src/jobs/stocks-sync/stages/stocks-snapshot.ts`,
  called AT MOST ONCE per stage run (not per position), routed through `callService('trading212', ...)`.
- Added `resolveTickers` + `resolveOpenFigiTickersBatched`, which build a ticker->ISIN lookup map from
  the instruments-metadata response and resolve each ISIN to a real-world ticker via OpenFIGI
  (`fetchOpenFigiTickers` from T372's `src/services/openfigi.service.ts`), batching ISINs at 10/request
  without `OPENFIGI_API_KEY` or 100/request with one, routed through `callService('openfigi', ...)`.
- `NormalizedPosition` gains two optional fields: `isin?: string`, `resolvedTicker?: string`. A miss at
  either lookup step logs a `warn:` line naming the unresolved ticker/ISIN and leaves both fields
  `undefined` for that position — never throws, never fails the whole snapshot. A failure fetching
  instruments-metadata itself (e.g. transient API error) is also caught and soft-skipped (logs a `warn:`
  and continues without isin/resolvedTicker for any position), so a Trading212-side hiccup on this
  secondary endpoint can't break the whole snapshot.
- `writePortfolio`/`buildPortfolioMarkdown` updated: `data/out/portfolio.json` includes the new fields
  when present (no schema change needed — it just serializes whatever's on `NormalizedPosition`);
  `data/out/portfolio.md` gained a "Real ticker" column, showing `—` when a position has no
  `resolvedTicker`.
- `ticker` field is untouched — still the Trading212-native join key.
- Root `CLAUDE.md`'s `stocks-sync` section updated to document the new fields + soft-skip behavior.

## Tests

Added 3 unit tests in `src/jobs/stocks-sync/stages/stocks-snapshot.test.ts` (all inject fake fetchers —
no live API calls in the suite):
- `resolves ISIN + real-world ticker end-to-end` — confirms instruments-metadata is fetched exactly
  once, and `isin`/`resolvedTicker` land on the normalized position.
- `leaves isin/resolvedTicker undefined and logs a warn when the ticker is absent from
  instruments-metadata` — a Trading212-ticker miss.
- `leaves resolvedTicker undefined (but isin populated) on an OpenFIGI resolution miss` — an
  OpenFIGI-side miss.

`npx tsc --noEmit` and `npm test` both green (153 top-level tests, 0 failures, including the 3 new ones).
