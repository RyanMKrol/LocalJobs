# T019 worklog ‚Äî Dashboard: read-only DB browser

## Attempt 1 ‚Äî 2026-06-21

### Goal
Add a read-only SQLite table viewer to the dashboard: list tables, page through
rows, via a strictly read-only API path (no write SQL, no arbitrary mutation),
respecting the existing read-only-client model.

### Implementation (within scope)
- **`src/db/store.ts`** ‚Äî added `listDbTables()` (queries `sqlite_master`,
  excludes `sqlite_*` internal tables), an internal `isKnownTable()` whitelist
  guard, and `browseTable(table, limit, offset): TablePage | null`. Read-only by
  construction: rejects any table not in the live schema (‚Üí null), clamps `limit`
  to [1,500] (bad/0 ‚Üí 50), `offset` ‚â• 0, runs only `SELECT *`/`PRAGMA table_info`,
  orders by `rowid` for stable paging. Table name is whitelisted *before* it's
  interpolated as a quoted identifier, so injection can't reach the query.
- **`src/api/server.ts`** ‚Äî two GET routes (no POST):
  `GET /api/db/tables` ‚Üí `{ tables }`; `GET /api/db/tables/:name?limit=&offset=`
  ‚Üí the `TablePage` (404 if `browseTable` returns null).
- **`dashboard/app/lib/api.ts`** ‚Äî `TablePage` type + `dbTables()` / `dbTable()`.
- **`dashboard/app/db/page.tsx`** ‚Äî new client page: table chips to select,
  Prev/Next paging (50/page), auto-refresh via `usePoll` keyed on `[table, offset]`.
  Cells render verbatim (null ‚Üí ‚àÖ, objects ‚Üí JSON), truncated with ellipsis + full
  value in `title`.
- **`dashboard/app/layout.tsx`** ‚Äî added "Database" nav link.
