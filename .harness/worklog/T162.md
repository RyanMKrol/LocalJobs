# T162 — Movies recs: quality bar (TMDB ≥7.0 w/ vote floor) + target ≥15

## What I built
Two improvements to the movies recommendation engine (`src/jobs/movies`), preserving
the T146 stratified-sample + per-genre-balance design.

### 1. Quality threshold in the merge
- `merge.ts` `verifyInto()` (refactored out of the old inline loop): after a suggestion
  TMDB-resolves and passes the owned / already-recommended checks, it must also clear a
  quality bar — `vote_average >= minRating` **AND** `vote_count >= minVotes` — else it's
  dropped (counted as `dropLowQuality`). The vote floor stops a fluke high rating on a
  handful of votes from sneaking through.
- Added `vote_count?: number` to `TmdbSearchResult` (`types.ts`).
- New env-overridable config: `recsMinRating` (`MOVIES_RECS_MIN_RATING`, default **7.0**),
  `recsMinVotes` (`MOVIES_RECS_MIN_VOTES`, default **50**).

### 2. Target ≥15 via larger asks + a bounded top-up loop
- Per-branch ask raised: `recsPerBranchAsk` (`MOVIES_RECS_PER_BRANCH_ASK`, default **9**),
  threaded into `BranchContext.ask`; `branches.ts` `RULES` → `rules(ask)` interpolates it
  and gained an `excludeBlock()` for the top-up exclusions + a "favour well-regarded films"
  rule. `recsTarget` default bumped 12 → **15**.
- New bounded top-up loop in `runMerge`: after the first verify pass, while
  `balanced.length < target` and `round < topUpRounds` and no TMDB quota hit, it calls a
  `TopUpFn` (default = fan out over all 8 branch specs IN-MEMORY via the new
  `collectBranchSuggestions()` in `recommend.ts`, excluding everything already
  considered/owned/recommended this run), keeps only genuinely-new titles (cross-round
  `seen` set), verifies + merges, re-balances. Stops early when target reached, when a
  round returns no new titles, or at the round cap (`recsTopUpRounds` /
  `MOVIES_RECS_TOPUP_ROUNDS`, default **3**). Logs a ⚠ when it can't reach the target.
- All thresholds/target/cap also overridable via `MergeOpts` (for tests).

## Tests (hermetic — no live LLM/TMDB)
- New `stages/merge-quality.test.ts`: drops sub-7.0 rating + too-few-votes, keeps ≥7.0
  well-voted incl. exact boundary; top-up requests more under target & stops the moment
  target is hit; bounded — stops at round cap; stops when a round yields no new titles;
  per-genre cap preserved across top-up; thresholds/target are config/opts-driven (not
  hardcoded — overriding `minRating` flips a 6.5 film in/out).
- Updated existing `stages/merge.test.ts`: TMDB result helper now carries `vote_count: 100`
  (so the existing valid picks clear the new floor), and injects `topUp: async () => []`
  so it stays hermetic (target is now 15).
