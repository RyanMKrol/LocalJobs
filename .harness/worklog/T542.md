### What was built (verified working, but see BLOCKED note below — not committed)

- Added `dashboard/app/components/useAction.ts`: a generic `useAction<K>(setErr)` hook
  that owns the `setBusy(key) -> setErr(null) -> await fn -> optional refetch ->
  catch(setErr) -> finally(setBusy(null))` block, with an `aliveRef` guard against
  setState-after-unmount.
- Added `dashboard/app/components/recSort.ts`: the shared `RecSortCol` type +
  `sortRecs()` helper used by both rec managers (previously duplicated inline in
  page.tsx).
- Extracted the four manager components verbatim (same JSX/markup/behaviour) into
  their own files, each now using `useAction` for its busy/err handling instead of
  a hand-rolled try/catch/finally per action:
  - `dashboard/app/components/TvRecsManager.tsx`
  - `dashboard/app/components/MovieRecsManager.tsx` (imports `recSort`)
  - `dashboard/app/components/MovieGapsManager.tsx` (keeps `groupByCollection`)
  - `dashboard/app/components/MissingSeasonsManager.tsx` (keeps `groupByShow`)
- `dashboard/app/workflows/[name]/page.tsx` now just imports the four managers;
  dropped from 1,144 lines to 326 lines (~3.5x smaller).
- Replaced ~14 duplicated `setBusy/setErr/try/catch/finally` blocks across the four
  managers with `useAction` instances (one per independent busy "slot" — e.g.
  MovieGapsManager has 3: per-item, per-collection ignore, per-collection un-ignore
  — all sharing one `err` state per manager, matching the pre-refactor behaviour
  where every handler in a manager wrote to the same `err`).
