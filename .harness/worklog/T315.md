## Root-cause note for whoever picks up T308/T309 next
Measured contrast in the `sunny-8bit`-in-light-mode palette (now the app's ONLY
palette, per the note above): `--accent` (`#e8821f`) against `--panel` (`#fff9ec`) is
~2.6:1; `--muted` (`#9c7e44`) against `--panel` is ~3.65:1. Both are under the WCAG AA
floor of 4.5:1 for body text. This task fixed the ONE component it was scoped to
(`MissingSeasonsManager`) by overriding to `var(--text)` locally (the new `.pill.info`
kind + an inline style on the show-title link) ‚Äî it deliberately did NOT touch the
theme's `--accent`/`--muted` values themselves (out of scope: a global theme concern).
Now that sunny-8bit is hardcoded as the only theme, this contrast gap is app-wide and
permanent until the palette itself is fixed ‚Äî every other place still using bare
`<a>` (inheriting `--accent`) or `.muted`/`className="muted"` text on a panel
background likely has the same problem (e.g. `MovieGapsManager`'s own show/collection
links and `<td className="muted">` cells, and the "Ignored" table's links in this same
component, left untouched here since they're out of this task's scope).
