# T241 worklog

## What was done

Audited all back-navigation labels in the scoped dashboard files for ID truncation.

Found one place truncating: `WorkflowRunBackLink` in `dashboard/app/ui.tsx` computed
`const shortId = workflowRunId.split('-')[0]` and rendered `← {shortId}` with the full id
only in a `title` tooltip. Changed to render the full `workflowRunId` directly (removed the
tooltip since the text IS the id now).

The `backFrom` helper (also in `ui.tsx`) extracts `segs[1]` from the URL path — already the
full run ID — so no change needed there.

The gate pages (`workflow-runs/[id]/gates/...` and `workflows/[name]/gates/...`) both use
`WorkflowRunBackLink`, so they inherit the fix automatically. No other truncation found.
