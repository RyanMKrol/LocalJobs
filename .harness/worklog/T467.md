# T467 — Decide the missing-movies split design

**Outcome:** Decision record complete. No code changed (decision-only task). This record is the
authoritative spec for the buildable follow-up **T468** (split the movie-recommendations DAG into
`movie-recommendations` recs-only + `missing-movies` gaps-only). A fresh agent building T468 should
need no further judgment calls.

## Verified in code before deciding (2026-07-11)

- `src/workflows/movies/stages/franchise-gaps.ts` reads **only** `snapshot.json`
  (`moviesConfig.snapshotOut`); it never opens `taste-profile.json`.
- `src/workflows/movies/stages/franchise-gaps.job.ts` declares `consumes: [movieSnapshotContract()]`
  — the snapshot only, confirming the gate contract too.
- `taste-profile.json` is written by `snapshot.ts` and read **only** by `recommend.ts` (the 8
  recommender branches). → Building it in a gaps-only snapshot would be dead work.
- `missing-tv-seasons` (the cadence/category being mirrored): `schedule: '0 9 * * 1'`,
  `category: 'recommendations'`.
- Ledger constants: `NOTIFY_JOB = 'movie-gaps-notify'` + `gapKey(tmdbId)` in
  `movies/stages/notify.ts`; `RECS_JOB = 'movie-recs'` + `recKey(tmdbId)` in `movies/recs.ts`.
- `src/api/server.ts` imports `NOTIFY_JOB as MOVIE_GAPS_JOB, gapKey` from
  `'../workflows/movies/stages/notify.js'` and `RECS_JOB, recKey` from `'../workflows/movies/recs.js'`.

## Decisions

### 1. New workflow identity
- **Name:** `missing-movies`
- **Schedule:** `0 9 * * 1` (Monday 09:00) — **matches `missing-tv-seasons` exactly**. This is a
  deliberate cadence CHANGE from franchise-gaps' current monthly cadence inside movie-recommendations
  → more frequent gap alerts.
- **Category:** `recommendations` (matching the other Plex-recs-style workflows).
- **Folder:** new `src/workflows/missing-movies/` (its own `config.ts`, `contracts.ts`, `CLAUDE.md`,
  `data/`, `stages/`). `src/workflows/movies/` is trimmed to recs-only.

### 2. Snapshot — duplicate, own job, SKIP taste-profile
- Duplicate the Plex movie snapshot into its **own job** for `missing-movies` (do NOT share
  `movie-snapshot`'s output across two workflows), mirroring `tv-snapshot` vs `plex-tv-snapshot`.
- **New snapshot job name: `plex-movie-snapshot`** (mirrors `plex-tv-snapshot`). `movies/` keeps
  `movie-snapshot`.
- **`plex-movie-snapshot` SKIPS building `taste-profile.json`** — it writes only `snapshot.json`
  (into `missing-movies`'s own `data/out/`). Verified safe above: `franchise-gaps` is the only
  consumer in this workflow and never reads the taste profile. The snapshot gate contract for
  `missing-movies` should point at that workflow's own `data/out/snapshot.json`.

### 3. Which jobs move vs stay
- **Move to `missing-movies`:** `franchise-gaps`, `movie-gaps-notify`. Their `JobDefinition.name`
  strings stay **exactly** `'franchise-gaps'` and `'movie-gaps-notify'`.
- **Stay in `movies` (recs-only):** `movie-snapshot`, the 8 `rec-*` branches, `rec-merge`. `movies`
  no longer contains a franchise-gaps stage or a `movie-gaps-notify` terminal.
- `missing-movies` DAG: `plex-movie-snapshot → franchise-gaps → movie-gaps-notify` (a linear
  3-stage workflow, gates on each edge).
- `movies` DAG after trim: `movie-snapshot → (8 rec branches) → rec-merge`. Its `outputJob`/terminal
  shape may need revisiting since `movie-gaps-notify` (the old terminal) is leaving — the build task
  should confirm `rec-merge` (or the recs digest) is the recs workflow's terminal/output stage.

### 4. Ledger continuity — NO migration needed (the crux; getting it wrong orphans ignore decisions)
The `work_items` / notified / ignored ledger is keyed by **`(job_name, item_key)` only** — never by
workflow membership. So every existing ignored/notified row keeps resolving **with zero data
migration**, provided:
- `NOTIFY_JOB` stays literally `'movie-gaps-notify'` and `gapKey()` is unchanged (the gaps
  notified/ignored ledger).
- `RECS_JOB` stays literally `'movie-recs'` and `recKey()` is unchanged (the recs
  never-re-recommend / ignore ledger — this one stays in `movies`).
- The `franchise-gaps` job's `name` stays `'franchise-gaps'`.

**Path change ≠ key change.** Physically moving `notify.ts` from
`movies/stages/notify.ts` → `missing-movies/stages/notify.ts` changes **import paths** in
`src/api/server.ts` (the `MOVIE_GAPS_JOB`/`gapKey` import) and in tests — those imports MUST be
updated to the new file location — but the runtime constant VALUE `'movie-gaps-notify'` is
unchanged, so the ledger rows are untouched. `RECS_JOB`/`recKey` import path is unaffected (recs
stays in `movies`).

### 5. Dashboard split — IN SCOPE for the build pass (not deferred)
`MovieGapsManager` (the franchise-gaps management section, incl. the `/api/movie-gaps/*` ignore
endpoints and its detail-page UI) moves to the **`missing-movies`** workflow detail page. The
`MovieRecsManager` / `/api/movie-recs/*` recs section stays on the `movie-recommendations` page.
Update `WORKFLOWS_WITH_SPECIFIC_MANAGERS` in `dashboard/app/workflows/[name]/page.tsx` accordingly
(both workflows keep a dedicated manager, so both stay excluded from the generic Output section).
