# T397 — Add "Run all workflows" action to the Admin page

## What was built

Added a new "Run everything" section to `dashboard/app/admin/page.tsx`, additive to (not
replacing) the existing "Danger zone" delete-all-output section:

- Explains what it does in plain language (starts a manual run of every workflow, capped to N
  originating inputs where the workflow supports limiting; not destructive but does trigger real
  work governed by existing per-service spend caps).
- A `Limit (originating inputs, where supported)` number input, default 3.
- Clicking "Run all workflows" reveals an INLINE confirm panel (not a native `confirm()`, no
  typed-phrase gate) with "Confirm"/"Cancel" buttons — Cancel collapses with no API call, Confirm
  calls `api.runAllWorkflows(limit)` (the T396 client helper) and shows a busy state.
- On success, renders the `results[]` as a table (workflow name, `<Pill>` started/skipped status,
  limit-used or skip-reason) plus aggregate `startedCount`/`skippedCount`/`totalWorkflows`, and a
  sentence pointing to the Workflows page to watch progress (since "started" only means dispatched).
- On error, shows the message inline.

Reused `<Pill>` (kind="on"/"off") for the status chips, matching the delete-all section's
`pill on`/`pill off` idiom. No new CSS was needed — reused `.panel`, `.btn`, `.mono`, existing
inline-style patterns already used in this file, and `.panel:has(> table)` (globals.css) for the
results table's mobile scroll.

## Harness updates (same change)

- `dashboard/scripts/_dashboard-harness.mjs`: added a synthetic fixture for
  `POST /api/workflows/run-all` (3 workflows: one started+limited, one started+unlimited, one
