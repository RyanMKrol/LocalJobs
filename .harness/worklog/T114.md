# T114 worklog — ntfy 429 backoff

## Attempt 1 (2026-06-23)

### What was done

Added exponential backoff with a hard cap to the ntfy notification sender in `src/core/notifier.ts`.

**config.ts:**
- Added `ntfyBackoffBaseMs` (default 30 000ms, env `LOCALJOBS_NTFY_BACKOFF_BASE_MS`)
- Added `ntfyBackoffCapMs` (default 600 000ms = 10 min, env `LOCALJOBS_NTFY_BACKOFF_CAP_MS`)

**notifier.ts:**
- Added module-level `ntfyBackoffUntil` and `ntfyBackoffMs` state
- Extracted `_sendNtfyToUrl(url, ...)` — testable inner function that:
  - Skips the call (returns suppressed error) if `Date.now() < ntfyBackoffUntil`
  - On 429: reads `Retry-After` header; falls back to `backoffMs * 2 || base`; caps at `ntfyBackoffCapMs`
  - On 200: resets both `ntfyBackoffUntil` and `ntfyBackoffMs` to 0
  - Uses injectable `_fetch` (swappable for tests)
- `sendNtfy` (private, config-aware) delegates to `_sendNtfyToUrl` — all other callers unchanged
- Exported test helpers: `_resetNtfyBackoff`, `_ntfyBackoffState`, `_setFetchForTest`, `_resetFetchForTest`
- macOS notification path (`sendMacNotification`) is untouched — not rate-limited

**notifier.test.ts:**
- Added 5 new async tests covering:
  1. 429 sets a cooldown
  2. Subsequent sends within cooldown are skipped without calling fetch
  3. Retry-After header overrides computed cooldown
  4. Exponential doubling on consecutive 429s
  5. Successful send resets backoff to zero
- Changed test runner to sequential chain (shared mutable state requires this)

- `npx tsc --noEmit` — clean
- `npm test` — all tests passed (12 notifier, all others green)
