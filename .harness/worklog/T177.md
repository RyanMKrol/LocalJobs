# T177 worklog ‚Äî Copy logs button

## What was built

Added a shared `CopyLogsButton` component to `dashboard/app/ui.tsx` that:
- Takes a `LogLine[]` prop and formats each line as `<timestamp> [LEVEL] message`
- Writes the full text to the clipboard via `navigator.clipboard.writeText`
- Falls back to a hidden `<textarea>` + `execCommand('copy')` when the clipboard API is unavailable
- Shows "Copied!" for 1.5 s after a successful copy, then reverts to "Copy logs"
- Is disabled when the log array is empty

The button uses the existing `.btn.secondary` class so it's automatically themed (dark/joyful).

## Where it appears

- **Job run detail** (`dashboard/app/runs/[id]/page.tsx`): button in the header row beside the level-filter buttons, copies ALL logs (not just the filtered subset).
- **Workflow-run framework logs** (`dashboard/app/workflow-runs/[id]/page.tsx`): button in a header row above the framework log panel.
