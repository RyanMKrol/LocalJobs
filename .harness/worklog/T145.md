# T145 — movies workflow: deterministic franchise-gap audit + monthly digest + ignore-to-suppress

## What was built (single context, no split)

A new public example workflow `movies` (`src/jobs/movies/`), reusing the Plex + TMDB
foundation from T144 (`plex`) rather than duplicating it.

### Foundation reuse
- `movies/config.ts` REUSES `plexConfig` (same Plex host/token, same TMDB Bearer token) via
  getters; only `PLEX_MOVIE_SECTION` (default 4) + the data paths are movie-specific.
- The stages import `plexGet` / `tmdbGet` from `../plex/client.js` directly (generic, path-based).
- GUID extraction reuses `../plex/plex.js` `extractTmdbId` (re-exported from `movies.ts`).

### Stages (DAG: movie-snapshot → franchise-gaps → movie-gaps-notify; schedule `0 9 1 * *`, monthly)
1. **movie-snapshot** — `GET /library/sections/4/all?includeGuids=1`; per movie capture
   title/year/tmdbId/ratingKey + taste metadata (Genre/Director/Country/audienceRating/rating).
   Writes `data/out/snapshot.json` + `data/out/taste-profile.json` (per-genre/director/decade/
   country counts for T146). Builds the owned set = all tmdbIds. Re-scans fresh every run.
2. **franchise-gaps** — DETERMINISTIC detector via TMDB Collections API. Pass 1: `GET /movie/{id}`
   → `belongs_to_collection`, collecting DISTINCT collection ids (deduped). Pass 2:
   `GET /collection/{id}` → `parts[]`, keep RELEASED parts (release_date ≤ today); a gap = a
   released part not in the owned set. NO quality filter, NO skip heuristics — every factual gap;
   `tmdbRating` (vote_average) is context only. Routes every call through the `tmdb` service;
   graceful `QuotaExceededError` stop. Writes `data/out/franchise-gaps.json`. Injectable fetchers
   (`fetchMovie`/`fetchCollection`) for hermetic tests.
3. **movie-gaps-notify** — monthly markdown report grouped by collection (TMDB link + rating,
   recorded via `detail.markdown`), ONE digest push of newly-detected gaps (first run = whole
   backlog). Ledger keyed `(movie-gaps-notify, "<tmdbId>")` — a "notified" log; dedups so a gap is
   announced once. Excludes owner-ignored gaps from BOTH report AND notifications.

### Framework shape (same inversion as plex/TV)
- No member declares `inputKeys()` → not limitable, scheduled-only, recomputes each run; ledger
  lives only in notify. Same-key stages → `root_key = item_key` (no lineage args).

### Ignore-to-suppress (load-bearing) — implemented fully, NOT split
- `ignoreSurfacedItem(jobName, itemKey)` in `store.ts`: UPSERTS a ledger row to `ignored`
  (creating it if absent), unlike `ignoreWorkItem` which requires a `failed` row.
- `ignoredItemKeys(jobName)` returns the ignored key-set; notify filters the report by it, and
  `isWorkItemDone` already treats `ignored` as done so it's never re-notified.
- API: `GET /api/movie-gaps` (overlays notified/ignored on franchise-gaps.json, file+DB only, no
  paid calls) and `POST /api/movie-gaps/:tmdbId/ignore` (loopback/token-guarded, 400 on bad id).
- Dashboard: new `Movie gaps` page (nav link added) — gaps grouped by collection, TMDB link +
  rating + ✕ Ignore button, plus an Ignored section. Reuses `.panel`/`table` (mobile-safe).
