# T444 — plex-language-no-track-flag stage

## What was built

Added a third member to the `plex-language-fix` workflow: `plex-language-no-track-flag`
(`dependsOn: ['plex-language-scan']`, runs in parallel with `plex-language-apply`). It reads the
existing `data/out/language-scan.json` (no re-scan, no extra TMDB/Plex spend) and picks out every
`FileEntry` the scan already marked `status: 'no-match'` — a file with zero audio candidates in any
of the title's candidate languages, a probable wrong-release signal distinct from
`plex-language-apply`'s "switch which existing track plays by default" job.

- Tracking is per FILE/EPISODE (`itemRatingKey::part<partId>`), not per show — matches the spec's
  Hunter x Hunter example exactly (confirmed live: 148 flagged episodes).
- Modeled directly on `missing-tv-seasons`'s `plex-seasons-notify` "have I already flagged this?"
  ledger pattern: only actionable (no-match) files get ledger rows; first run digests the whole
  current backlog in ONE push; a re-run with nothing new sends nothing.
- Writes `data/out/reports/no-track.md` grouped by show/movie, 🆕 marker on newly-flagged files.
- Wired the generic `ignoreSurfacedItem`/`unignoreSurfacedItem` store functions through two new
  endpoints: `POST /api/plex-no-track/:itemKey/ignore` and `/unignore` (itemKey is the URL-encoded
  ledger key). No dashboard UI was added for these (out of scope's file list has no dashboard/ path;
  the store-level call path + API endpoints are wired so a follow-up can add the UI with zero
  backend work, per the spec's fallback instruction).
- Reused `plexLanguageScanContract()` as this stage's `consumes` (same factory as
  `plex-language-apply`'s `consumes` — the gate-coverage collapse convention).
- Added `reportDir` to `plexLanguageFixConfig` (mirroring `missing-tv-seasons`'s `config.ts`).
