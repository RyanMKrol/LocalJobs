# T134 worklog — hardcode "key" gate style, remove toggle + 4 unused styles

## What was done

Hardcoded the `key` pill as the sole validation-gate display style on both graph
views (workflow-run DAG and structure-only workflow graph), removing the live
gate-style chooser and the four unused styles.

### Changes

**`dashboard/app/components/Dag.tsx`**
- Removed imports of `GATE_STYLES`, `useGateStyle` (fn), `GateStyle` (type) from `../ui`.
- Replaced `gateContent(style, g)` + style-branching with inline `g.key`.
- Simplified `GateMarks` to take only `{ gates }` (no `style` param), always renders
  `dag-gates gs-key` container with `dag-gate gs-key <state>` marks.
- Deleted `GateStyleBar` component entirely.
- Removed `const [style, setStyle] = useGateStyle()` from `Dag`.
- Removed `connector`-specific arrow-colouring branch (the `style === 'connector' && ag.length > 0`
  block that rendered `<a class="dag-arrow-glyph gate-conn …">`).
- Removed `{hasGates && <GateStyleBar … />}` render site and the `hasGates` variable.
- All gate marks still clickable (`g.href` → `<a>` vs `<span>`) and state-coloured.
- Non-adjacent consumer-node fallback (`nodeGates`) unchanged and still renders key pills.

**`dashboard/app/ui.tsx`**
- Removed `GATE_STYLES`, `GateStyle` type, `GATE_STYLE_KEY`, `DEFAULT_GATE_STYLE`,
  and `useGateStyle` — verified only `Dag.tsx` imported them.

**`dashboard/app/globals.css`**
- Removed: `.gate-style-bar`, `.gate-style-label`, `.gate-style-btn`, `:hover`, `.active` rules.
- Removed: `.dag-gate.gs-icon`, `.dag-gate.gs-lock`, `.dag-gate.gs-dot` (+ all dot state variants),
  `a.dag-gate.gs-dot:hover`.
- Removed: `.dag-arrow-glyph.gate-conn*` rules (desktop and mobile `::after` variants).
- Kept: `.dag-gates`, `.dag-gate`, `.dag-gate.gs-key`, and all gate state colour rules
  (`.passed`, `.failed`, `.pending`, `.structural`).
