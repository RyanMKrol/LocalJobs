# T103 ŌĆö Backlog DAG: render as a real directed graph (node shapes + drawn edges)

## Goal
Redesign `BacklogDag` (the `/backlog` DAG view) so it reads as an actual directed
acyclic graph: every task wrapped in a clearly-bounded, status-coloured node shape,
and REAL directed edges (with arrowheads) from each task to each of its dependents ŌĆö
not a single `ŌåÆ` between waves. Keep topological order, node select/inspect,
horizontal scroll, and phone-width responsiveness.

## Approach (dependency-light, no graph library)
Honoured the repo's dependency-light convention ŌĆö NO dagre/d3. Layout is computed
in JS from the existing topological waves with fixed node dimensions, so edge
endpoints are deterministic (no DOM measurement, no layout pass):

- `dashboard/app/components/Dag.tsx` ŌĆö rewrote `BacklogDag`:
  - Position map: x = topological wave index (`computeBacklogWaves`, unchanged),
    y = stacked within the wave, each wave vertically centred against the tallest.
    Fixed geometry constants (`BL_NODE_W/H`, `BL_COL_GAP`, `BL_ROW_GAP`, `BL_PAD`).
  - Nodes are absolutely-positioned `<button>` cards (id + status badge header +
    2-line-clamped title), status-coloured (done / needs-human / pending),
    selectable (toggle), with a highlight ring when selected.
  - Edges: one SVG cubic-bezier path per `dependsOn` entry, from the producer's
    right-centre to the consumer's left-centre, with an arrowhead marker. Edges
    incident to the selected node are highlighted (accent colour + bigger arrow).
    Two `<marker>` defs (grey default, accent highlighted) since SVG markers don't
    reliably inherit stroke colour cross-browser.
  - Container sized exactly to the graph; the parent panel (`overflowX: auto` in
    `backlog/page.tsx`, unchanged) provides horizontal scroll on narrow/phone
    widths ŌĆö so on mobile the graph scrolls inside its panel rather than the page
    overflowing.
- `dashboard/app/globals.css` ŌĆö added `.backlog-graph`, `.backlog-graph-edges`,
  `.backlog-edge(.hi)`, `.backlog-gnode` (+ state/hover/selected) and a 2-line
  clamp on the in-graph title. Removed the now-dead `.backlog-node` /
  `.backlog-dag-scroll` rules (the old flex-card layout) and the dead mobile
  `.backlog-node { max-width }` override; kept the shared
  `.backlog-node-head/-id/-badge/-title` rules (still used inside the node).
- `dashboard/app/backlog/page.tsx` ŌĆö unchanged API (`tasks`/`selectedId`/`onSelect`);
  the panel wrapper already supplies horizontal scroll.
