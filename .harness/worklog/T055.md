# T055 worklog — Backlog view: show each task's escalation path

## Attempt 1 — 2026-06-22

### Changes made

1. **`src/api/server.ts`** — `readBacklog()` now passes through `defaults` from TASKS.json alongside `tasks`:
   ```ts
   function readBacklog(): { tasks: unknown[]; defaults?: unknown; error?: string }
   ```

2. **`dashboard/app/lib/api.ts`** — added two new types:
   - `EscalationRung { model?, effort? }` — one rung of the ladder
   - `BacklogDefaults { model?, effort?, escalation? }` — the top-level `defaults` block
   - Added `escalation?: EscalationRung[]` to `BacklogTask`
   - Updated `api.backlog()` return type to include `defaults?: BacklogDefaults`

3. **`dashboard/app/backlog/page.tsx`** — resolution logic:
   - `resolveRung(model, effort, defaults)`: resolves per-rung model/effort falling back to defaults, strips `claude-` prefix
   - `escalationPath(t, defaults)`: builds the full ladder string: rung0 (per-task model/effort else defaults) + each escalation rung
   - `TaskCard` receives `defaults` prop; renders `ladder` in the meta row instead of the old model/effort display
   - The literal word "default" never appears — all inheritance is resolved to the actual value
