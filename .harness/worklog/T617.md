# T617 ‚Äî size-table bar-chart renderer for plex-space-saver Output

Date: 2026-07-16

## What I did
- Added `SizeTableOutputBody` to `dashboard/app/components/OutputRenderer.tsx`, registered under
  `OUTPUT_RENDERERS['size-table']` (the T458 "add a new form's renderer HERE" convention). It
  `JSON.parse`s the content into a `SizeBreakdownFile`-shaped `{ items: [...] }`, and renders ONE
  proportional horizontal bar per item, preserving the biggest-first order the file already carries
  (no re-sort, no top-N truncation ‚Äî every item renders). Bar width = `bytes / maxBytes` (guarded
  against a `maxBytes === 0` divide-by-zero ‚Üí 0% width). Each row shows `title` + `human`; per the
  owner-confirmed visual style, no year and no movie/show color distinction (single bar color).
  On a JSON parse failure or an empty/missing `items` array, falls back to `RawOutputBody` (mirrors
  `JsonOutputBody`'s existing try/catch fallback pattern) so it never throws or renders blank.
- Added `.size-bar-list`/`.size-bar-row`/`.size-bar-label`/`.size-bar-track`/`.size-bar-fill`/
  `.size-bar-value` to `dashboard/app/globals.css`, following the existing `.md-*` idiom (no new
  styling system). Added a mobile rule in the existing `@media (max-width: 640px)` block that drops
  each row's grid to a single column so the label/track/value stack instead of squeezing.
- Updated `dashboard/scripts/_dashboard-harness.mjs` (LIVING ARTIFACT rule):
  - Added `plexSpaceSaverOutputFixture` ‚Äî a synthetic `SizeBreakdownFile` JSON string (4 items,
    varied `bytes`, already biggest-first) returned as the `content` for both the run-scoped
    `GET /workflow-runs/:id/output?job=plex-space-saver-scan&...` handler and the workflow-definition
    `GET /api/workflows/plex-space-saver/output` handler (both previously fell through to the
    generic markdown fixture, which is what the raw-JSON-dump bug looked like before this task).
  - Added a `/api/workflows/plex-space-saver/output-items` case so the workflow-definition Output
    section has a real row to click "View" on.
  - Added two FLOWS entries: `workflow-output-section-size-table` (clicks the Output section's View
    button on `/workflows/plex-space-saver`, waits for `.size-bar-list`) and `stage-io-size-table`
    (clicks the Stage I/O popover's item link on `/workflow-runs/plex-space-saver-run`, same wait) ‚Äî
    both capture the interactive popover state, per the "capture interactive states" rule.
