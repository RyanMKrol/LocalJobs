# T589 — 2026-07-16

## What I did
- Added `fmtAbsolute(t: string | null): string` to `dashboard/app/ui.tsx`, next to
  `fmtTime`/`fmtRelative`. Reuses the same UTC-parse guard as the other two
  (`t.includes('Z') || t.includes('T') ? t : t.replace(' ', 'T') + 'Z'`), returns
  `'—'` for a null/falsy input, and formats via explicit
  `Intl.DateTimeFormat('en-GB', { day: 'numeric', month: 'short', year: 'numeric',
  hour: '2-digit', minute: '2-digit', hour12: false })` — a fixed, locale-independent
  `D MMM YYYY, HH:mm` string (e.g. `15 Jul 2026, 16:00`), not a bare `toLocaleString()`.
  `fmtTime`/`fmtRelative` are untouched and still exported/used elsewhere.
- Replaced `fmtRelative(...)` with `fmtAbsolute(...)` at all 6 named "Started"/"When"/
  last-run-time call sites:
  - `dashboard/app/page.tsx` — Overview "Recent workflow runs" table, "Started" column.
  - `dashboard/app/workflows/page.tsx` — Workflows list, inline last-run-time span.
  - `dashboard/app/workflows/[name]/page.tsx` — workflow-detail "Runs" table, "Started".
  - `dashboard/app/workflow-runs/[id]/page.tsx` — "Member runs" table "When" column
    AND the nested earlier-attempts sub-rows (both call sites).
  - `dashboard/app/jobs/[name]/page.tsx` — job-detail "Run history" table, "Started".
  - Left the `s.updated_at` (Stuck items "WHEN" column) `fmtRelative` calls in
    `page.tsx`/`ui.tsx` unchanged — those are not Started/run-start columns.
- Added unit tests for `fmtAbsolute` in `dashboard/app/ui.test.ts` (null → em dash,
  fixed-pattern match for both a space-separated SQLite timestamp and an
  already-ISO/Z-suffixed one).
- No changes needed to `dashboard/scripts/_dashboard-harness.mjs` — its fixtures use
  a fixed `NOW = '2026-06-22 09:00:00'` timestamp already; the visual/mobile checks
  automatically show the new absolute rendering with no fixture edits.
