# T489 — Ignore/un-ignore: instant list update + drop confirm() popups

2026-07-10

## What I did
- `dashboard/app/ui.tsx`: extended `usePoll<T>` to also return `refetch()` — a
  function that immediately re-invokes the poll's `fn` and updates
  `data`/`error` outside the interval timer. Implemented via a `tickRef` that
  captures the current `tick` closure each effect run; `refetch` is a stable
  `useCallback` that calls `tickRef.current()`. Purely additive — existing
  `usePoll(...)` call sites (e.g. the workflow detail page's own
  `usePoll(() => api.workflow(name), 3000, [name])`) are untouched and still
  compile, since they just ignore the extra return field.
- `dashboard/app/workflows/[name]/page.tsx`: for all 4 managers
  (`TvRecsManager`, `MovieRecsManager`, `MovieGapsManager`,
  `MissingSeasonsManager`):
  - Destructured `refetch` from each manager's own `usePoll` call.
  - Removed every `if (!confirm(...)) return;` guard gating an
    ignore/unignore/bulk mutation (13 lines total across the 4 managers).
  - Added `await refetch();` immediately after each successful mutation
    (`await api.*(...)`) inside the `try` block, so a thrown error still
    skips it and the existing `catch`/`finally` (busy-state reset, error
    display) is unchanged.
  - Removed now-unused local `count`/`label` variables that existed only to
    build the removed confirm() message text.
  - Left the unrelated `resetOutput()` confirm() (~line 870, danger-zone
    "Clear output data") untouched, as instructed.
