## Design notes

- trackId = `${artist.toLowerCase().trim()}::${track.toLowerCase().trim()}` — stable identity
- Work-item ledger key = `${trackId}::${scrobbledAt}` — unique per scrobble
- DynamoDB item keys exactly match schema spec: `trackId` (S, PK), `scrobbledAt` (N, SK)
- Spotify enrichment is fully optional — missing SPOTIFY_CLIENT_ID/SECRET → skipped silently
- Fetches up to `LASTFM_MAX_PAGES` pages (default 2 × 200 = 400 tracks) per run
- All I/O injectable → hermetic tests with zero network or AWS calls
