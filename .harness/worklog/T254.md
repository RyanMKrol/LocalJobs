## Key decisions

- Job name `lastfm-sync` (not `listens-sync`) to avoid collision with workflow name `listens-sync`
- `trackId` = `"{artist.lower()}::{track.lower()}"` as stable partition key (matches spec)
- `scrobbledAt` = Unix epoch seconds as sort key (matches spec)
- work_items key = `{trackId}::{scrobbledAt}` — unique per scrobble
- 5-hour lookback window on a 4-hour schedule — overlapping to never miss a scrobble
- Spotify enrichment is fully optional (skipped if creds absent), wrapped in callService('spotify')
- now-playing tracks filtered out (no `date` field)
- Duplicate scrobbles from Last.fm deduplicated before ledger check
