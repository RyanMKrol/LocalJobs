# T006 — Validation gates between pipeline stages

## Goal
Typed-artifact validation gates between dependent pipeline stages: a job declares
what it `produces`/`consumes`; the DAG/pipeline executor validates the contract at
each boundary so an external-format drift (Takeout CSV layout, Fragrantica page
structure) fails LOUD at the exact gate instead of passing bad data downstream.
A failed gate is a first-class surfaced failure (run status + notification).

## What I built
- **`src/core/types.ts`** — new `ArtifactContract { key, description?, check() }`
  and `GateResult { ok, violations?, detail? }`. `JobDefinition` gains optional
  `produces?: ArtifactContract[]` and `consumes?: ArtifactContract[]`.
- **`src/core/dag.ts`** — `Gate { key, producer, consumer }` + pure `deriveGates(dag,
  produces, consumes)`: for every DAG edge, each key the upstream produces and the
  downstream consumes becomes a gate. Edge-scoped — a consumed key with no producing
  upstream is an external input, not a gate.
- **`src/db/store.ts`** — `recordGateFailure(jobName, pipelineRunId, error)`: inserts
  a terminal **failed** run row (mirrors `recordSkippedRun`) so the violation is a
  first-class failed run carrying the exact drift in `error`.
- **`src/core/pipeline-executor.ts`** — builds produces/consumes maps from member
  defs, derives gates, groups inbound gates by consumer. In `runOne`, BEFORE spawning
  a consumer, every inbound gate is enforced via `enforceGate` (runs the producer's
  `produces[key].check()` AND the consumer's `consumes[key].check()`; throw → violation).
  On violation: log the exact drift, `recordGateFailure`, return `'failed'` — the
  consumer never spawns, `onSettle` fires the stage notification, and executeDag's
  existing cascade skips the consumer's own dependents. Start log now reports gate count.

## Surfacing (first-class failure)
- Run status: a violated consumer gets a real **failed** run row (NOT skipped), error
  = `Gate violation [producer → consumer] artifact "key": <drift>`.
- Notification: the existing per-stage `notifyStage` fires for the failed status.
- Aggregate: pipeline finishes `partial`; cascade skips downstream.

## Tests (added, all green)
- `dag.test.ts` (+5): deriveGates — basic gate, no-producer external input, key
  mismatch, non-adjacent producer skipped, diamond join gates both producers.
- `pipeline-executor.test.ts` (+5): satisfied contract → consumer runs/success +
  gate-ok logged; consumer-side drift → consumer failed (not skipped) with drift in
  error, status partial; producer-side drift → same; a `check` that throws → treated
  as violation not crash; gate failure cascades (downstream dependent skipped).
