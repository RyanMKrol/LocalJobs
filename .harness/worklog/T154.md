# T154 ÔÇö Joyful dashboard iteration 2: 10 themes + narrowed rounded fonts

Iteration 2 of the joyful-dashboard chooser (T142 built it Ôćĺ owner reviewed in T045 Ôćĺ
this is the next narrow). Reused the EXISTING live/persisted switcher infra from T142
(`data-theme`/`data-font` on `<html>`, the `useHtmlPref` hooks, the pre-paint script) ÔÇö
just swapped the option sets. No new switcher mechanism.

## THEMES ÔÇö narrowed to the two winning families (10 total)
- Kept `default` (dark baseline) untouched.
- Kept `pixel-picnic` and `sunny-8bit` exactly as built in T142.
- Added 4 in-family variants of each (palette warmth/cool, accent hue, dot-grid /
  scanline intensity, contrast, card radius):
  - Pixel Picnic: `pixel-picnic-meadow` (cool green, denser dots), `-dusk` (dusty rose,
    larger soft dots, rounder cards), `-berry` (plum accent), `-mint` (minty cool,
    tighter crisp dots, small cards).
  - Sunny 8-bit: `-sunset` (orange/red), `-lime` (lime/green), `-aqua` (cyan, small
    cards), `-coral` (coral/pink, rounder cards).
- Removed the other T142 themes: `midnight-neon`, `candy-bright`, `soft-pastel`
  (the spec also named `joyful-dark`, which never existed in the T142 code ÔÇö only
  `midnight-neon` was the dark joyful theme; removed it).
- Per-variant card rounding via a new `--card-radius` var consumed by the joyful
  `.panel`/`.statcard` rule (default 14px, so behaviour for the originals is unchanged).

## FONTS ÔÇö narrowed to ROUNDED + a Space Mono cluster
- Loaded via `next/font/google` (layout.tsx): kept Pixelify_Sans (DISPLAY only),
  Nunito, Quicksand, Fredoka, Baloo_2, Space_Mono; ADDED Comfortaa, Varela_Round,
  Rubik, Mulish, Poppins, Lexend. REMOVED Silkscreen, VT323, Press_Start_2P,
  JetBrains_Mono.
- Options: 10 self-paired rounded faces (Fredoka, Nunito + 8 more: Quicksand, Baloo 2,
  Comfortaa, Varela Round, Rubik, Mulish, Poppins, Lexend); 3 thin-header variants
  (`fredoka-light`/`nunito-light`/`poppins-light` dropping `--heading-weight` to
  500/400/400); 2 Pixelify-display + rounded-body pairings (Pixel Picnic vibe);
  Space Mono cluster (`spacemono` all-mono, `spacemono-fredoka` = Fredoka display +
  Space Mono body, `spacemono-light` = lighter heading + tighter tracking).
- Removed the non-rounded body faces (VT323/JetBrains-as-body) per the narrow.
- Header weight: new `--heading-weight` var (default 700 = original) applied to
  `.brand`/`h1`/`h2`. Light variants set it to 400ÔÇô500 so headings read crisp not
  chunky. Loaded the needed weights (Poppins 300ÔÇô700 explicit; variable faces cover
  the range; Varela Round is 400-only so it sets heading-weight 400). NOTE: deliberately
  did NOT add `--heading-weight` to `.section-heading-summary` ÔÇö a `<summary>` defaults
  to normal weight, so forcing 700 there would have changed the untouched default look
  (caught + reverted during the build).
- Logs/code keep the system `--font-mono` for every rounded option (legibility); only
  the Space Mono cluster remaps `--font-mono`.

## Guarantees preserved
- Untouched default (no html attributes) renders identically to pre-T142: accents still
  gated by `html[data-theme]:not([data-theme="default"])`, `--heading-weight` default 700.
- Dense data legible, mobile-safe.
