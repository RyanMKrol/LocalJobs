# T164 worklog — cronToEnglish monthly pattern fix

## Root cause confirmed

`cronToEnglish` in `dashboard/app/ui.tsx` had an early-return guard:
```ts
if (month !== '*') return expr; // don't attempt month-specific patterns
```
This correctly gates month-specific expressions (e.g. `0 9 1 1 *`), but it also
blocked the **monthly day-of-month** pattern `0 9 1 * *` where `month='*'` and
`dom=1`. The check fired BEFORE reading `dom`, so any non-wildcard `dom` fell
through to the raw-expr fallback — no hint for movies.

## Fix

Moved the monthly-day-of-month branch BEFORE the `month !== '*'` guard:

```ts
// Monthly at HH:MM on day D: M H D * *  (month=*, dow=*)
if (isNum(min) && isNum(hour) && isNum(dom) && month === '*' && dow === '*') {
  const h = toInt(hour), m = toInt(min), d = toInt(dom);
  if (h >= 0 && h < 24 && m >= 0 && m < 60 && d >= 1 && d <= 31)
    return `At ${pad2(h)}:${pad2(m)} on the ${ordinal(d)} of each month`;
}
```

Added an `ordinal()` helper (1→"1st", 2→"2nd", 3→"3rd", 4→"th", etc.) to
produce natural-language output like `"At 09:00 on the 1st of each month"`.

The `month !== '*'` guard is kept for all other branches (they're already
month=`*`-only patterns); the monthly branch explicitly requires `dow='*'` to
avoid ambiguity.

## Patterns covered

| Expr         | Result |
|---|---|
| `0 2 * * *`  | `At 02:00, every day` |
| `0 3 * * *`  | `At 03:00, every day` |
| `0 9 * * 1`  | `At 09:00 on Monday` |
| `0 9 1 * *`  | `At 09:00 on the 1st of each month` (was: raw expr, no hint) |
