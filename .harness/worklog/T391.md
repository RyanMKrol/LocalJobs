# T391 — un-ignore for the 4 audit-style ignore lists

## Built
- `src/db/store.ts`: added `unignoreSurfacedItem(jobName, itemKey)` — deletes an
  `ignored` ledger row (delete-and-refresh, mirrors `unstickWorkItem`), no-op
  (returns 0) on a non-ignored/absent key. No bulk variant, per spec.
- `src/api/server.ts`: added 4 endpoints mirroring their `ignore` counterparts
  exactly (validation + guard):
  - `POST /api/movie-gaps/:tmdbId/unignore`
  - `POST /api/movie-recs/:tmdbId/unignore`
  - `POST /api/tv-recs/:tmdbId/unignore`
  - `POST /api/missing-seasons/:tmdbId/:season/unignore`
  Each returns `{ ok: true, unignored: <rows> }`.
- `dashboard/app/lib/api.ts`: matching client functions `unignoreMovieGap`,
  `unignoreMovieRec`, `unignoreTvRec`, `unignoreMissingSeason`.
- `dashboard/app/workflows/[name]/page.tsx`: each of the 4 managers
  (`TvRecsManager`, `MovieRecsManager`, `MovieGapsManager`,
  `MissingSeasonsManager`) gets an `unignore(...)` handler (confirm-gated,
  reusing existing `busy`/`err` state) and a new action column (`<th></th>` +
  per-row "↺ Un-ignore" button) in its `IgnoredSection` table.
- `dashboard/app/globals.css`: added `.ignored-section table th:last-child` /
  `td:last-child { width: 1%; white-space: nowrap; }` (mirrors the existing
  `.movie-gaps-scroll` rule) since MovieGapsManager/MissingSeasonsManager's
  Ignored tables aren't nested inside `.movie-gaps-scroll`.
- `CLAUDE.md`: extended the "Ignore-to-suppress a SURFACED item (T145)" section
  with a new bullet documenting `unignoreSurfacedItem` + the 4 endpoints + the
  UI control + the delete-and-refresh design decision and its re-notify
  consequence.
