# T443 — plex-language-fix apply stage

## What was built
- `src/core/plex-client.ts`: added `plexPutStreams(partId, audioStreamId, subtitleStreamId?)`
  (PUT /library/parts/<id>?audioStreamID=&subtitleStreamID=&allParts=1) and
  `triggerButlerBackup()` (POST /butler/BackupDatabase), both reusing `resolvePlexHost()` and the
  existing scoped insecure-TLS agent.
- `src/workflows/plex-language-fix/stages/apply.job.ts` + `apply.ts` + `apply.test.ts`: reads
  `data/out/language-scan.json`, filters to `status: 'change'` entries with a usable
  `proposedAudio.streamId` (skipping/warning on malformed ones without marking them
  success/failed), triggers a Butler backup once before the first real PUT (WARN-only on
  failure), applies each qualifying file via `plexPutStreams`, records every processed file on
  the `work_items` ledger (`success`/`failed`, keyed by partId), writes a self-contained per-run
  `applied-log-<timestamp>.json` (before + after audio/subtitle per file), and throws if any file
  failed to apply this run (per the repo's item-loop convention).
- `contracts.ts`/`scan.job.ts`: wired the `plex-language-scan → plex-language-apply` gate, reusing
  the same `plexLanguageScanContract()` factory on both sides (producer + consumer).
- `plex-language-fix.workflow.ts`: added `{ job: 'plex-language-apply', dependsOn:
  ['plex-language-scan'] }`.
- `scripts/plex-language-undo.ts` + `plex-language-undo.test.ts`: manual, never-scheduled revert
  tool. Reads the most recent (or given) applied-log, computes the before/after-swapped revert per
  entry, defaults to dry-run, `--apply` to actually call `plexPutStreams`. Writes its own
  `*-undo-results-*.json` log.
- Docs: root `CLAUDE.md` (workflow table row, `plex-client.ts` file-map row, `scripts/*` file-map
  row) and `src/workflows/plex-language-fix/CLAUDE.md` updated to describe the apply stage, the
  Butler-backup + per-file-undo safety net, and the full-automation (no ambiguous carve-out, no
  per-run sign-off) decision.
