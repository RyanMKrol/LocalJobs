# T197 worklog

## What was done

Modified `src/core/notifier.ts` `_sendNtfyToUrl`:
1. **Better diagnostics**: catch block now extracts `e.cause` (Node network errors carry the underlying reason like ENOTFOUND/ECONNREFUSED on `e.cause.code`) and appends it to the error string, e.g. "fetch failed (ECONNREFUSED: connect ECONNREFUSED 127.0.0.1:80)".
2. **Bounded retry**: wrapped the `_fetch` call in a `for` loop (up to `NETWORK_RETRIES = 2` extra attempts, `NETWORK_RETRY_DELAY_MS = 500ms` between). HTTP errors (4xx/5xx) return immediately without retrying; only thrown network errors are retried. The 429 backoff path is untouched. The function still never throws.
