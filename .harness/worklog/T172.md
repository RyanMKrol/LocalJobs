# T172 worklog

## What was done

Fixed two styling issues in the Input → Output mapping panel on the workflow-run detail page.

### Issue 1: Inconsistent corner radii (table clipping)

The `.panel` wrapper around the IO table had `border-radius: 10px` but no `overflow: hidden`,
causing the table's square corners to poke out past the rounded border at the bottom. Fixed by
adding `style={{ overflow: 'hidden' }}` inline on that specific panel div in
`dashboard/app/workflow-runs/[id]/page.tsx`. This clips the inner table to the rounded boundary
without affecting other panels.

### Issue 2: Footnote hugging the sides

The `.io-footnote` paragraph had `padding-top: 8px` but no horizontal padding, so it sat flush
against the panel edges. Fixed by changing `padding-top: 8px` to `padding: 8px 14px 0` in
`dashboard/app/globals.css`, matching the `14px` horizontal padding used by table `td`/`th` cells.
