# T387 — Verify StageIoPanel against parallel fan-out and outputJob-override

## What I did

Added `/stage-io` fixtures + routing in `dashboard/scripts/_dashboard-harness.mjs` for the two
DAG shapes T387 asks about:

1. **Parallel fan-out** — `movie-recommendations` (`movie-recs-run`): `movie-snapshot` fans out
   to 8 sibling `rec-*` branches in the same wave, each getting a DISTINCT single output item
   (`movieRecBranchOutput`), plus `franchise-gaps` / `rec-merge` / `movie-gaps-notify` fixtures
   and an `overall` fixture (root = `movie-snapshot`, terminal = `movie-gaps-notify`). Wired into
   `fixtureFor` at `/api/workflow-runs/movie-recs-run/stage-io`.
2. **`outputJob` override** — `stocks-sync` (`stocks` run): added `stocksStageIo` (per-stage
   inputs/outputs for `stocks-snapshot`/`stocks-watch`/`stocks-notify`) and `stocksStageIoOverall`,
   which sets BOTH `inputs` and `outputs` to `stocks-snapshot`'s rows (mirroring the real
   `outputJob: 'stocks-snapshot'` manifest override — `stocks-notify` never records `work_items`).
   Wired into `fixtureFor` at `/api/workflow-runs/stocks/stage-io`.

Added 3 new `FLOWS` entries:
- `stage-io-fan-out-branch-auteur` / `stage-io-fan-out-branch-canon` — click the `rec-auteur` and
  `rec-canon` chips on `/workflow-runs/movie-recs-run` to prove each branch shows ONLY its own
  input/output, not siblings' rows.
- `stage-io-outputjob-override-overall` — the default "Overall" tab on `/workflow-runs/stocks`
  (also the existing `workflow-run-stocks-io` PAGES baseline, captured again explicitly here).

No fixture existed for `movie-recs-run/stage-io` or `stocks/stage-io` before this task — every
member job's tab and the Overall tab previously fell through to the generic catch-all
(`/api/workflow-runs/*` → `{ run, jobs, logs, gates }`), which is the wrong response shape for
`StageIoPanel` (expects `{ inputs, outputs, ... }`). This wasn't screenshotted as a rendering bug
because `StageIoBlock`/`StageIoOverallBlock` bail out with `if (!data) return null` when `usePoll`
hasn't resolved a matching shape yet, so no crash — but the panel would have rendered nothing
useful for these two workflows before this task's fixtures existed.

## Bug found?

No bug in `StageIoLists.tsx` — the component correctly scopes each stage's own inputs/outputs and
correctly renders whatever `outputJobs` the API's `overall` response names. This task was purely
about adding the missing fixtures to exercise those two DAG shapes; no code fix was needed.
