# T592 — gate padlock collides with intervening node on a skip-column edge

2026-07-16

## What I did

Fixed `dashboard/app/components/DagFlow.tsx`'s `GateEdge`/`buildLayout` so a same-row
"skip-column" gate edge (a source/target pair whose Y is equal but whose wave-column
gap is 2+, e.g. `plex-language-discover -> plex-language-evaluate` which coexists
with the `discover -> resolve -> evaluate` chain) never paints its padlock label on
top of an unrelated intervening node.

`buildLayout` now computes each node's real footprint (`nodeBox`, from the already-
computed `colX`/`columnY`) and, for every dependency edge, checks whether the edge is
a same-row skip-column edge (`jobWave - depWave >= 2` and `sourceY === targetY`). If
so, it computes the geometric label midpoint the same way the reported bug does
(`(sourceRight + targetLeft) / 2`, at the shared row Y) and tests it against every
OTHER node's footprint. On a genuine collision it sets `labelOffsetY` on the edge's
data — `+(NODE_H/2 + 16)` if the row is topmost (nothing above to move into),
`-(NODE_H/2 + 16)` otherwise — bowing just the *label* off the row while leaving the
edge path itself untouched (approach 2 from the spec: simpler and still correct,
since only the label needs to clear the collision).

`GateEdge` applies `labelOffsetY` (default 0) on top of `getBezierPath`'s real
`labelY` at render time. The check is fully general — driven off actual node
X/Y boxes, not any hardcoded stage name — so it applies to any workflow with the
same same-row multi-hop shape, not just `plex-language-fix`.

Also updated `dashboard/scripts/_dashboard-harness.mjs`: the
`plex-language-fix-run`/`plex-language-fix` fixtures previously hardcoded
`gates: []` on both handlers, so the existing `workflow-run-plex-language-fix`
PAGES screenshot never painted any padlocks and couldn't exercise this bug at
all. Added a `plexLanguageFixGates` array (mirroring `movieRecsGates`'s shape)
covering `discover -> resolve`, `discover -> evaluate` (the skip edge),
`resolve -> evaluate`, and `evaluate -> apply`, wired into both the
`/api/workflow-runs/plex-language-fix-run` and `/api/workflows/plex-language-fix`
handlers.
