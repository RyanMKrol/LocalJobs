# T345 — Boxed empty-state panel for TV Recommendations

## What changed
- `dashboard/app/globals.css`: added `.empty-state-panel` — a reusable "boxed empty-state"
  convention (flex-centred, min-height, padding) for a panel whose only content is one
  centred muted sentence. Documented as general-purpose (not IO-specific), sitting next to
  the existing `.io-empty-state` rule (left untouched).
- `dashboard/app/workflows/[name]/page.tsx`: `TvRecsManager`'s `data.generatedAt == null`
  branch now renders `<div className="panel"><p className="empty-state-panel">…</p></div>`
  instead of a bare `<p className="muted">`. Exact sentence text unchanged. No other
  empty-state text in this file was touched (MovieRecsManager, Franchise Gaps, Missing
  Seasons empty states are explicit out-of-scope follow-ups per the spec).
- `dashboard/scripts/_dashboard-harness.mjs`: added a `tv-recs-empty` FLOWS entry that
  overrides `/api/tv-recs` (page-level `page.route`, which takes precedence over the
  context-level `routeApi`) to return `generatedAt: null, recommendations: []`, then
  reloads — so visual-check now captures a screenshot of the actual empty state (the
  default fixture always has recommendations, so no existing screenshot exercised this).
- `CLAUDE.md`: added a Conventions bullet documenting `.empty-state-panel` next to the
  existing "Shared dashboard UI components (T268)" bullet.
