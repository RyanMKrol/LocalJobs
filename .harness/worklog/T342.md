# T342 — Extract a shared table/category-panel component and migrate Services + Workflows onto it

## What was built (within scope)

- Added `dashboard/app/components/CategoryTable.tsx` — a small, focused component (following the
  `RunButton.tsx`/`Pill.tsx` convention: header comment documenting props, named export). It owns
  the `.panel` wrapper, the `<h2>{label}</h2>` category heading, and the `<table>`/`<colgroup>`/
  `<thead>` scaffold. Spacing between stacked panels comes for free from the existing `.panel +
  .panel` CSS rule (globals.css:88, landed by T340) — no new CSS was added. Chose the simpler
  "columns config + children for tbody" shape over a fully generic table component, since neither
  call site needs row-level genericism (both already build their own `<tr>`/`<td>` markup with
  page-specific interactivity — click handlers, inline edit state, `Pill`/`RunButton` usage, the
  stuck-item popover trigger).
- Migrated `dashboard/app/services/page.tsx`: each category's `<div className="panel"><h2>…
  <table className="services-table"><colgroup>…</colgroup><thead>…</thead><tbody>…` block now
  renders through `<CategoryTable label=… tableClassName="services-table" columns={[…]}>` with the
  existing `<tbody>` row-mapping passed as children, unchanged.
- Migrated `dashboard/app/workflows/page.tsx`: same pattern — the per-category `<div
  className="panel"><h2>…<table><thead>…` block now renders through `<CategoryTable label=…
  columns={[…]}>`, preserving the `RunButton`, `Pill`, stuck-item popover trigger, and all cell
  content unchanged.
- Both migrations preserve column widths/alignment (`width`/`align` per column, matching the
  pre-existing colgroup on the services table and the `textAlign: 'center'` headers on the
  workflows table) and all existing behaviour — no page-visible change, this is a pure structural
  refactor.
