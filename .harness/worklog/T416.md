# T416 — Formalize item-loop-job fail-on-failed-item convention

Documentation-only task. Added a new "Job conventions" bullet to root `CLAUDE.md`:
an item-loop job must `throw` a summarizing error at the end of its run if its own
tally shows `failed > 0` (excluding intentional `'skipped'` soft-stops like a quota
pause), so the run is recorded `'failed'`. Explained why no new framework mechanism
is needed — `executeDag` already blocks downstream deps on any non-`'success'`
status, `runAttempts`/`maxRetries` already retries, and idempotent `!isWorkItemDone`
filtering already resumes correctly on retry/rescan.

Added a short pointer/reinforcement to `src/workflows/CLAUDE.md`, mirroring the
existing gate-coverage and detail-convention reinforcement pattern.

No production job code was changed (per spec — retrofitting actual jobs is
separate follow-up work).
