# T071 — Show validation gates on the workflow definition graph

## Attempt 1 — 2026-06-22

### What was done

Added structural gate markers to the workflow definition page (`/workflows/[name]`).

**Changes:**

1. `src/api/server.ts` — `GET /api/workflows/:name` now includes `gates: gatesForWorkflow(name)` in the response. `gatesForWorkflow` was already defined; this just exposes it on the definition endpoint. Updated the comment on the run endpoint that said "structure-only view never gets these".

2. `dashboard/app/lib/api.ts` — Added `StructuralGate` interface (`{key, producer, consumer, description?}`). Added `gates?: StructuralGate[]` field to the `Workflow` interface.

3. `dashboard/app/components/Dag.tsx` — Added `StructuralGateChips` component for rendering gate chips without run state. Added `structuralGates` and `lastRunId` props to the `Dag` component. When no `workflowRunId` is set, structural chips are rendered per consumer node. If `lastRunId` is provided, chips link to that run's gate detail page; otherwise they render as non-interactive `<span>` elements.

4. `dashboard/app/globals.css` — Added `.dag-gate.structural { border-style: dashed; }` to distinguish structural (no run state) gates from run-state-coloured ones.

5. `dashboard/app/workflows/[name]/page.tsx` — Passes `structuralGates={p.gates}` and `lastRunId={p.last_run?.id}` to `<Dag>`.
