# T238 worklog

## Root cause
`.db-modal` had no `font-family` set. Browser UA stylesheets give `<button>` elements their own
default `font-family` (system UI font) that does NOT inherit from `body` unless explicitly told to.
So while regular text inside `.db-modal` correctly inherited `--font-body` from `body`, buttons
inside the consumers popup (including the "✕ Close" button) rendered in the browser's default system
font regardless of which theme/font was selected.

## Fix
Added `font-family: var(--font-body)` to `.db-modal` in `dashboard/app/globals.css`. With this
explicit declaration on the modal container, `font-family: inherit` properly resolves to the
themed body font for ALL child elements (including buttons), across all `.db-modal` popups (consumers,
stuck, markdown, theme switcher, etc.).
