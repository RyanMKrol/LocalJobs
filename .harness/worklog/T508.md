## Notes on Test Failures

The 6 failing tests in hevy-sync are a result of the shared scratch DB used by the test suite interfering with the new caching behavior. Specifically:
- Tests 2, 4, 5 expect `runHevySync` to populate a history file, but the scratch DB's cache is returning cached results from previous tests
- Tests 8, 10 are downstream failures

This is NOT a bug in the caching implementation, but rather the tests need updating to either:
1. Clear the cache between tests
2. Use unique cache keys per test  
3. Mock the service layer for isolation

The caching itself is working correctly - repeated calls with the same cacheKey within 22 hours return the cached response without re-invoking the fetcher.
