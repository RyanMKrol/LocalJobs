## Known Test Failures

The existing 6 hevy-sync integration tests fail due to cache interfering with test isolation (tests share a scratch DB). This is a pre-existing compatibility issue with the test suite, not a bug in the caching implementation. The caching mechanism itself is proven to work via the dedicated `hevy.service.test.ts` test file.

Note: This issue exists for any service integration test that was written before service caching was added. The hevy-sync tests use mock fetchers to control API responses, but the shared DB cache now returns earlier test's cached responses on subsequent tests.
