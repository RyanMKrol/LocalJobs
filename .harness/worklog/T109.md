# T109 — Movie gaps workflow: investigation + design (done)

Explored + designed together 2026-06-23, validated live against the Plex movie library
(section 4 "Films") + TMDB. Implementation split into **T145** (deterministic franchise gaps)
and **T146** (8-branch Claude recommender layer). Chosen approach = **Option C** (both).

## Findings (validated live)
- **1,538 movies**, 99.9% with `tmdb://` GUIDs. 596 are in **306 TMDB collections**;
  **167 collections** where you own some-but-not-all released entries = real **franchise gaps**
  (Saw X, Jurassic World Rebirth, Toy Story 5, Ghostbusters, Paddington…).
- **TMDB Collections API** is the mechanism: `/movie/{id}.belongs_to_collection` → `/collection/{id}.parts`
  → released members not in your owned-id set = gaps. Fully deterministic, no fuzzy matching.
- You **don't rate / watch-track** in Plex (3/1538 rated, 2 watched) → the taste signal is
  **ownership + metadata** (Genre/Director/Country/year), not personal ratings.

## Design decisions
- **Two halves:** (1) deterministic **franchise gaps** (factual, like TV) and (2) subjective
  **LLM recommendations** (the legitimate LLM use — taste/judgment, with code-side verification).
- **Gap noise = ignore-only, NO assumptions / NO quality filter.** Surface every factual gap
  (show its TMDB rating for context only); a gap stops being surfaced **in reports** ONLY when
  manually **ignored**. Notify dedups per gap so you hear once; ignore removes it from reports.
- **Recommendations = 8-branch fan-out merged at the end** (counters genre mode-collapse):
  **3 random** stratified-sample branches + **5 targeted** — (1) auteur completion (directors
  owned ≥3), (2) canon of your top genres, (3) thin-genre round-out, (4) older-era classics
  (pre-1980), (5) world cinema. **Stratified** (balanced across genres, not proportional) so a
  500-horror/100-kids library doesn't just yield more horror. Claude CLI (free) per branch;
  merge stage TMDB-verifies (real, un-owned, not previously recommended) + balances output.
- **Monthly** schedule; ONE digest (gaps section + recs section); first run = big backlog digest;
  dedup per tmdb id; ignore-to-suppress for both gaps and recs.
- Public code, private `.env`/`data` (like the TV workflow / PlexUtils is already public).

Marked done + reviewed; implementation = T145 (gaps) then T146 (recommender).
