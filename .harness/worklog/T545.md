# T545 — memoize MovieGapsManager's groupByCollection

2026-07-15

## What I did
`MovieGapsManager` (`dashboard/app/components/MovieGapsManager.tsx`) computed
`groupByCollection(active)` three times per render (the summary counts and the
active table's `.map`) and `groupByCollection(ignored)` once more for the
ignored table. Added two `useMemo` hooks — `activeGroups = useMemo(() =>
groupByCollection(active), [active])` and `ignoredGroups = useMemo(() =>
groupByCollection(ignored), [ignored])` — and replaced every call site with the
memoized value. No behaviour/markup change, just fewer duplicate computations
per render.

(Only `MovieGapsManager` was in scope per the spec — `MissingSeasonsManager`
has an analogous `groupByShow` duplication but the task's `## Done when` only
covers "the active (and ignored) collection grouping", i.e. movie franchise
gaps, so I left it untouched.)
