# T078 — Backlog section headers: add an expand/collapse caret affordance

## Outcome: done (feature already implemented in T067)

## Investigation

T078 was authored before T067 ran. T067 ("make every backlog section collapsible",
commit `0615e57`) added the full caret affordance to `.section-heading-summary`:

```css
.section-heading-summary::before { content: '▾ '; font-size: 12px; opacity: .7; }
details:not([open]) .section-heading-summary::before { content: '▸ '; }
```

- `▾` (down) when section is open
- `▸` (right) when section is collapsed

This satisfies every doneWhen criterion:
- Caret points one way collapsed (▸), clearly different when expanded (▾) ✓
- header is obviously clickable (`cursor: pointer` on the class) ✓
- consistent across all three sections (same class applied to all) ✓
