# T337 — stock-digest workflow

Added `src/jobs/stock-digest/` as a brand-new, standalone example workflow (own
folder, own `stock-digest.workflow.ts`, own weekly schedule `'0 8 * * 1'`,
`category: 'regular-maintenance'`) — deliberately NOT a stage on `stocks-sync`,
per the owner's explicit direction in the spec.

## What was built

- `config.ts` — `stockDigestConfig` (`outDir`, `moversCount`) + `reportPathFor(weekKey)`.
- `stages/stock-digest-build.ts` — the single stage:
  - Pure helpers: `weekKey`/`weekLabel` (ISO-8601 week), `gainPct` (ported from
    `stocks-watch`'s formula), `portfolioSharePct`, `pickMovers` (top-N winners/losers),
    `buildFacts` (structured facts object), `buildDigestPrompt`, `readPortfolio`
    (tolerant of missing/malformed file, mirrors `stocks-watch`'s `readPortfolio`).
  - `runStockDigestBuild`: reads `stocksSyncConfig.portfolioJsonPath` via a plain
    relative import of `../stocks-sync/config.js` and the `NormalizedPosition` type
    from `../stocks-sync/stages/stocks-snapshot.js` (no duplicated shape, no absolute
    path — the new cross-workflow-read pattern called out in the spec). Missing/empty
    portfolio → WARN log + clean skip (no throw). Otherwise computes facts in code
    (never lets Claude invent numbers), calls `runClaude` with a structured-facts
    prompt, writes `data/out/stock-digest-<weekKey>.md`, and records the artifact via
    `markWorkItem(ctx, weekKey, 'success', { detail: { markdown } })` (T110 convention).
  - Idempotent per ISO week via the `work_items` ledger keyed by `weekKey` — a
    same-week re-run overwrites the file rather than duplicating it (verified in tests).
- `stages/stock-digest-build.job.ts` — job definition, `timeoutMs: 300_000` (Claude call).
- `stock-digest.workflow.ts` — one-stage `WorkflowDefinition`, doc comment explains the
  weekly-after-stocks-sync timing and the cross-workflow read.
- `stages/stock-digest-build.test.ts` — hermetic unit tests (no live Trading212/Claude
  calls; stub `claudeRunner` + scratch DB + tmp files) covering: `weekKey`/`weekLabel`
  (including a year-boundary case), `gainPct` (incl. divide-by-zero guard), `portfolioSharePct`,
  `pickMovers`, `buildFacts`, `readPortfolio` (missing/malformed/valid), and end-to-end
  `runStockDigestBuild` (missing file skip, empty array skip, successful write + ledger,
  Claude failure throws, idempotent same-week re-run overwrites).
