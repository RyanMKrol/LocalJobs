## What I did

Re-attempt of T565 (blocked twice at both model tiers because the original task's `scope` never
included the CLAUDE.md files the spec's own "## Done when" required — an unfixable trap under the

actually in scope.

1. Added 3 new shared exports to `src/core/plex-client.ts`:
   - `PlexAllResponse<T>` — the `{ MediaContainer?: { Metadata?: T[] } }` response shape every
     workflow's `interface PlexAllResponse<T>` re-declared identically.
   - `extractTmdbId(guids)` — pulls the numeric id off a title's `tmdb://` GUID, `null` when
     absent (never guessed) — same logic `missing-tv-seasons/plex.ts` and
     `plex-language-fix/lib.ts` each defined independently.
   - `fetchSectionMetadata<T>(section, opts)` — a thin wrapper over `plexGet` for the
     `/library/sections/<n>/all[?query]` pattern, routed through `callService('plex', ...)`
     (T578's existing migration), returning `MediaContainer.Metadata ?? []` directly. Preserves
     the exact `cacheKey` semantics each call site relied on: default `plex:<path>` for a cached
     read, `cacheKey: null` to explicitly opt an uncached probe out of the 3-hour response cache
     (used by `plex-profiles`' `resolveInputKeys()` and `missing-movies`'s snapshot, which was
     ALREADY uncached before this change — preserved, not newly cached).

2. `missing-tv-seasons/plex.ts` and `plex-language-fix/lib.ts` no longer define their own
   `extractTmdbId` — both now `import` the core implementation and `export { extractTmdbId }` as a
   thin re-export, so every existing importer (`movies.ts`, `tv-recs/tv-shows.ts`, `discover.ts`)
   keeps working via the transitive chain with zero further changes needed to those files (they
   weren't in this task's scope).

3. Replaced the local `interface PlexAllResponse<T>` + inline `callService(plexGet(...))` fetch
   pattern with `fetchSectionMetadata` calls at all 6 in-scope call sites: `movies/stages/
   snapshot.ts`, `tv-recs/stages/tv-snapshot.ts`, `missing-tv-seasons/stages/snapshot.ts` (2
   calls: shows + episodes), `missing-movies/stages/snapshot.ts`, `plex-space-saver/stages/
   scan.ts` (3 calls: movies + shows + episodes), `plex-profiles/stages/build.ts` (3 section
   listings in `runBuild` + 2 in `resolveInputKeys`, both uncached). `plex-profiles/stages/
   build.ts`'s per-item detail fetch (`/library/metadata/<ratingKey>`, not a section listing)
   keeps its direct `plexGet`+`callService` call but now imports `PlexAllResponse<T>` from core
   instead of its own duplicate `PlexMetadataResponse<T>` interface (identical shape).

4. No change needed to `movies/movies.ts` or `tv-recs/tv-shows.ts` — both already imported
   `extractTmdbId` from `missing-tv-seasons/plex.js` and re-exported it; since that file now
