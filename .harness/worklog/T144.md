## Shape (why it inverts the usual pattern)
- No member declares `inputKeys()` → the workflow is **not limitable**; scheduled-only,
  always unlimited. Inputs are discovered (the whole Plex library), not a static file.
- snapshot + season-check **re-compute fresh every run** (no skip-if-done).
- Only `plex-seasons-notify` touches the `work_items` ledger — as a **"already
  notified?" log**, keyed `(plex-seasons-notify, "<tmdbId>::S<season>")`. Rows are
  recorded only for ACTIONABLE shows (so the IO panel isn't 600+ "up to date" rows).
  First run = one big digest of the whole backlog; nothing new = no push.
- All stages key by `tmdbId` → `root_key = item_key` naturally (no lineage args).
- Match is ALWAYS by `tmdb://` GUID; a GUID-less show is flagged "unverifiable", never guessed.
- Ended/canceled shows are NOT skipped (`evaluateShow` never inspects `status`) — revivals.
- Plex self-signed cert handled by a scoped `https.Agent({ rejectUnauthorized:false })`
  in `client.ts` for Plex requests ONLY (global TLS verification untouched). JSON is
  requested (`Accept: application/json`) so items arrive under `MediaContainer.Metadata`.
- TMDB routed through the free, rate-limited `tmdb` service (`paid:false`, ~10/s default).
