# T454 — bulk unignore backend

2026-07-07

## What was built
- `unignoreSurfacedItems(jobName, itemKeys)` in `src/db/store.ts`, directly below
  `unignoreSurfacedItem`, mirroring `ignoreSurfacedItems`'s transaction-wrapped
  batch shape: loops the keys inside a single `db.transaction`, deleting each
  `ignored` ledger row, and returns the total rows removed. Empty array short-circuits
  to 0 without opening a transaction (matches `ignoreSurfacedItems`).
- Four new mutating routes in `src/api/server.ts`, each placed next to its
  ignore-bulk (or single-item unignore) twin, same body validation (400 on
  malformed input) and behind the existing loopback/token mutation guard:
  - `POST /api/movie-gaps/unignore-bulk { tmdbIds }`
  - `POST /api/movie-recs/unignore-bulk { tmdbIds }`
  - `POST /api/tv-recs/unignore-bulk { tmdbIds }`
  - `POST /api/missing-seasons/unignore-bulk { items: { tmdbId, season }[] }`
  Each returns `{ ok: true, unignored: <count> }`.
- Added `unignoreSurfacedItems` to the `src/db/store.ts` import list in `server.ts`.

## Tests
- Added a unit test block in `src/db/store.test.ts` (T454): partial-batch removes
  exactly the supplied keys and leaves others ignored; a never-ignored key is a
  no-op contributing 0; empty array returns 0.
