# T065 — Validation gate: visual representation of what is being checked

## Goal
Make the validation-gate page show, left-to-right, **Output → Gate(what it
checks) → Input**, so a non-expert reader understands what the gate validates and
what actually flowed — expected shape + a sample/summary + pass/fail per
expectation. Contracts must be able to declare an expected shape for display.
Builds on T063 (dedicated gate page) + T064 (input/output framing).

## What I changed
- **`src/core/types.ts`** — new display contracts:
  - `ShapeExpectation { label, detail? }`, `ArtifactShape { summary, format?, expectations[] }`,
    `ExpectationResult { label, ok, actual? }`.
  - `ArtifactContract` gains optional `shape?: ArtifactShape`.
  - `GateResult` gains optional `checks?: ExpectationResult[]` (per-expectation,
    aligned to the shape by label) and `sample?: string` (a small summary of the
    actual artifact). `ok`/`violations` semantics unchanged.
- **`src/jobs/places/contracts.ts`** and **`src/jobs/perfumes/contracts.ts`** —
  each contract now declares a `shape` and its `check()` builds `checks[]` + a
  `sample`. A shared local `fromChecks(checks, sample)` derives `ok`/`violations`
  from the failed checks, so the executor's gate enforcement is byte-for-byte
  equivalent (still fails on `ok:false`). Labels in `shape.expectations` are kept
  identical to the labels the `check` emits (asserted by a new test).
- **`src/api/server.ts`** — new `GET /api/workflow-runs/:id/gates/:producer/:key`.
  Classifies the gate state for the run, then runs each side's contract `check()`
  LIVE (output = producer's `produces[key]`, input = consumer's `consumes[key]`),
  returning `{ gate, output:{shape,result}, input:{shape,result} }`. Reads `data/`
  files only — never a paid/remote call.
- **`dashboard/app/lib/api.ts`** — mirror types (`ArtifactShape`,
  `ExpectationResult`, `GateResult`, `GateSide`, `GateInspection`) + `api.gateInspection`.
- **`dashboard/app/workflow-runs/[id]/gates/[producer]/[key]/page.tsx`** —
  rebuilt as a left-to-right flow: `SideCard` (Output) → arrow → centre Gate card
  (key + state + what it validates) → arrow → `SideCard` (Input). Each SideCard
  shows the declared shape (summary + format + expectations) paired with the live
  per-expectation ✓/✗ and the `sample` of what flowed; links to the relevant run
  logs. Polls `gateInspection` (2s) alongside the existing workflow-run poll.
- **`dashboard/app/globals.css`** — `.gate-flow` / `.gate-card` / `.gate-arrow` /
  `.gate-expects` styling; responsive block stacks the three cards vertically and
  rotates the arrows at ≤640px.
- **Docs** — README typed-artifact-contracts section + CLAUDE.md validation-gates
  convention updated to describe `shape` / `checks` / `sample` and the new
  endpoint; `.harness/LIMITATIONS.md` notes that the page's "actual" reflects the
  artifact on disk NOW (live re-check), not a per-run snapshot.
- **Tests** — `src/core/contracts.test.ts`: every contract declares a shape with a
  summary + non-empty expectations; a passing check reports per-expectation
  results aligned to the shape (by label) + a sample containing real data; a
  failing check marks the offending expectation `ok:false` with the actual value
  surfaced.
