# T609: Add listServiceCacheRows() store function for browsing service_cache rows

## Summary
Implemented `listServiceCacheRows()` function in `src/db/store/services.ts` to list actual rows from the service_cache table instead of just per-service counts.

## Implementation Details

### Added to `src/db/store/services.ts`:
1. **`ServiceCacheRow` interface**: Exports the row shape with fields:
   - `service_name: string`
   - `cache_key: string`
   - `response_json: string` (raw JSON string, not parsed)
   - `cached_at: string`

2. **`listServiceCacheRows(serviceName?: string)` function**:
   - Returns all rows from service_cache ordered by `cached_at DESC` (most recently cached first)
   - When `serviceName` is provided, filters to only that service's rows
   - When `serviceName` is undefined, returns all rows across all services
   - response_json is returned as a raw stored string (no server-side parsing)
   - No pagination/limiting logic (deferred to API layer)

### Tests Added to `src/db/store.test.ts`:
1. Imported `listServiceCacheRows` from the store barrel
2. Added comprehensive test coverage:
   - Inserts ≥2 rows across ≥2 different services with explicit timestamps for deterministic ordering
   - Verifies `listServiceCacheRows()` with no args returns all rows in cached_at DESC order
   - Verifies `listServiceCacheRows(serviceName)` filters correctly to only matching rows
   - Confirms response_json is returned as raw string (not parsed JSON)
   - Tests unknown service returns empty array
