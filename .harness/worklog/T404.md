# T404 — Rec top-up: also exclude owner-ignored recs from re-prompted branches

## What was built

- `src/db/store.ts`: `ignoreSurfacedItem(jobName, itemKey, detail?)` now accepts an
  optional `detail`. On a fresh row it's stored as-is; on an existing row, an
  omitted `detail` arg PRESERVES whatever was already there (`COALESCE(excluded.detail,
  work_items.detail)`) instead of clobbering it to `NULL`. Added
  `ignoredWorkItemDetails(jobName)` — returns `{ itemKey, detail }[]` for one job's
  `ignored` rows (mirrors `ignoredItemKeys` but with parsed detail).
- `src/api/server.ts`: `POST /api/movie-recs/:tmdbId/ignore` and the tv-recs
  equivalent now look up the tmdbId in the currently-loaded `recommendations.json`
  and pass `{ title, year }` as `detail` when found — so a pre-notify ignore
  captures a recoverable title. Falls back to no detail if the id isn't in the
  current file (documented limitation, not a regression).
- `src/jobs/movies/stages/notify.ts` / `src/jobs/tv-recs/stages/tv-recs-notify.ts`:
  the `markWorkItem(..., 'success', { detail })` call for a newly-notified rec now
  additionally carries `title`/`year` fields alongside the existing `name`/`markdown`
  — additive only, `name`'s format unchanged.
- `src/jobs/movies/stages/recommend.ts` / `src/jobs/tv-recs/stages/recommend.ts`:
  added `ignoredSuggestionTitles(jobName)` — reads the ledger's `ignored` rows via
  the new store helper, extracts usable `title`/`year`, formats identically to
  `allHistoryTitles`. Legacy rows with no usable title (e.g. only the old
  `"Title (lens)"` `name` string) are skipped, not parsed.
- `runBranch` in both `recommend.ts` files now builds `alreadySuggested` from the
  union (deduped via `Set`) of `allHistoryTitles(...)` and `ignoredSuggestionTitles()`
  — so the FIRST branch pass, not just top-up, avoids ignored titles too.
- `src/jobs/movies/stages/merge.ts` / `src/jobs/tv-recs/stages/tv-rec-merge.ts`:
  same union applied to `buildDefaultTopUp`'s `alreadySuggested`, PLUS the
  `considered` list (the `exclude` arg handed to the injected `topUp` function) is
  seeded with `ignoredSuggestionTitles()` up front, so even round 1 of a top-up
  excludes a pre-notify-ignored title that was never in the history file.

## Tests added

- `src/db/store.test.ts`: fresh ignore with detail → detail stored; ignoring an
  existing `success` row with a detail, calling with no detail arg → prior detail
  preserved.
- `src/jobs/movies/stages/recommend.test.ts`: `ignoredSuggestionTitles` — usable
  title+year formatted/included, title-only included, legacy name-only/no-detail
  rows skipped without crashing.
- `src/jobs/movies/stages/merge-quality.test.ts` and
  `src/jobs/tv-recs/stages/tv-rec-merge.test.ts`: regression test for the exact
  gap — mark a tmdbId `ignored` with a title-carrying detail WITHOUT ever
  appending it to history, run the merge top-up path, assert the title appears in
  the `exclude` list passed to the injected `topUp` function.
