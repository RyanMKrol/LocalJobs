# T309 — Delete dead pixel-picnic/default/spacemono/reduce-motion/emoji-minimise theme code

## What changed

- `dashboard/app/globals.css`:
  - Removed the `pixel-picnic` family CSS block (dark-base + light-override).
  - Removed the `default` family's `[data-theme="default"][data-mode="light"]` override rule
    (unreachable — `data-theme` is never written).
  - Removed the Space Mono font-variable rule (`html[data-font="spacemono"]`); folded the
    Baloo rule's declarations directly onto `:root` (already done in T308's `:root` block —
    the separate `html[data-font="baloo"]` selector was removed as dead).
  - Removed the `html[data-motion="reduced"]` kill-switch block.
  - Unconditionalized the "joyful accent" rules that were gated behind
    `html:not([data-motion="reduced"])` / `html[data-theme]:not([data-theme="default"])`:
    emoji badges/sparkles, card radius, hover-lift, table row accent, success-badge spark
    flash, progress-bar sheen, stat-number pop — all now apply unconditionally since
    sunny-8bit + Baloo is the dashboard's one and only look (T308).
  - Rewrote the top-of-section banner comment to describe the single-theme/single-mode
    reality instead of the old 4-attribute switcher.
- `dashboard/app/layout.tsx`: already fully clean from T308 (no leftover Space Mono / data-theme
  / data-font / data-motion references) — no change needed.
- `dashboard/app/ui.tsx`: deleted `useTheme`, `useFont`, `useMotion`, `useHtmlPref`,
  `prefersReducedMotion`, the `THEMES`/`FONTS` arrays, and the `ThemeId`/`FontId` types. Kept
  `useMode`/`resolveMode`/`ModeId` (still used by the sun/moon toggle) and `ThemeControls`.
- `dashboard/scripts/_dashboard-harness.mjs`: trimmed `seedTheme`'s params to just `{ mode }`
  (dropped `theme`/`font`/`motion`, now unused) and updated its doc comment.
- `dashboard/scripts/visual-check.mjs` (living-artifact script, always in-scope per the harness
  rule): removed the `VISUAL_THEMES`/`THEMES` per-theme capture loop (no more theme families to
  iterate) and the `theme` param threading through `capture()`/`seedTheme()`; now takes exactly
  one screenshot per PAGES/FLOWS entry. This was necessary to satisfy the Done-when grep across
  `dashboard/scripts` (the old `THEMES`/`pixel-picnic`/`data-theme` references lived here in
  doc comments + a live loop variable).
