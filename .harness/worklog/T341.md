# T341 — align Services page table columns across categories

## Change
- `dashboard/app/services/page.tsx`: each per-category `<table>` now has a
  `services-table` class and an identical `<colgroup>` (40% / 16% / 16% / 16% / 12%)
  right after the opening tag, so all 4 category tables (CLI tools, Website scrapes,
  APIs, Uncategorized) size their Service / Rate-min / Rate-day / Rate-month /
  actions columns identically regardless of per-table content width.
- `dashboard/app/globals.css`: added `.services-table { table-layout: fixed; }` so
  the `<colgroup>` widths are enforced rather than just hinted (default
  `table-layout: auto` lets content re-negotiate column widths per table, which was
  the original misalignment cause). Reverted to `table-layout: auto` inside the
  existing `@media (max-width: 640px)` block, matching the mobile-first behaviour
  of every other table on the page (fixed-width columns caused genuine overflow at
  phone width even though the panel itself scrolls — a per-element boundary-crossing
  check flags this since `overflow-x` on `<table>` defaults to `visible`).

## Verification
- `npx tsc --noEmit` — pass.
- `npm test` — 127/127 pass.
- `npm --prefix dashboard run build` — succeeds.
- `node dashboard/scripts/visual-check.mjs` — all 27 pages captured; opened
  `dashboard/scripts/visual-out/services.png` by eye: Rate/min, Rate/day,
  Rate/month, and the actions column now start at identical x-positions across
  the CLI tools / Website scrapes / APIs / Uncategorized tables (confirmed both
  before and after the mobile-layout media-query fix — desktop rendering
  unchanged by that addition).
- `node dashboard/scripts/mobile-check.mjs` — first run (before the media-query
  fix) FAILED: fixed 640px-independent column widths forced `table.services-table`
  (and descendants) past their own box at 402px viewport width (`overflow-x:
  visible` default on `<table>`), 12 boundary-crossing offenders on `/services`.
  Added `.services-table { table-layout: auto }` inside the existing
  `@media (max-width: 640px)` block to revert to auto column sizing at phone
  width (same behaviour every other table on the page already has) — re-ran and
  all 16 pages pass, 0 offenders.

No other files touched; `dashboard/scripts/_dashboard-harness.mjs` already had
`/services` fixtures spanning all 4 categories (including a mix of `no limit` and
bar-rendered numeric cells), so no harness update was needed for this task.
