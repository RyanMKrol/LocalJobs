# T273 — Migrate Job page to shared component library

## Finding: nothing applicable to migrate

Reviewed `dashboard/app/jobs/[name]/page.tsx` against the shared components from T268
(`dashboard/app/components/`: `RunButton`, `Pill`, `DagFlow`, `WorkflowOutputSection`):

- **`RunButton`** wraps `.btn.btn-run` for workflow run-trigger actions (with "already
  running" disabled state). The Job page has NO run button — per T070 a job is a
  read-only workflow member (no run-now, no schedule, no enable toggle; you run the
  *workflow*, never the job). The page's only buttons are plain `.btn` "Unstick"/
  "Unstick all" actions, which are a different semantic (no "already running" concept)
  and would misrepresent the action if wrapped in `RunButton`.
- **`Pill`** wraps `.pill` chips (e.g. on/off, reviewed/unreviewed). The Job page has
  no pill/chip UI at all — the only status rendering is `StatusBadge` (from
  `../../ui`), which the page already uses (imported and applied to run-history rows).
- **`DagFlow`** / **`WorkflowOutputSection`** are workflow-run-graph / output-panel
  components; the Job page has neither a DAG view nor an output section (those live on
  the workflow detail page).
- Considered swapping the inline stuck-items table for the existing `StuckPopover`
  (`ui.tsx`) reuse pattern, but that would NOT be like-for-like: the Job page's inline
  table renders extra per-item debug fields (`snippet`, `debugFile`, `finalUrl`,
  `pageTitle`, `httpStatus`, `textLength`) that `StuckPopover` doesn't render, so
  swapping would change the rendered output — out of scope for a like-for-like swap,
  and `StuckPopover` isn't one of the T268 `components/` additions this task targets.

Conclusion: the Job page already uses the one applicable shared piece (`StatusBadge`)
and has no other UI surface matching `RunButton`/`Pill`/`DagFlow`/`WorkflowOutputSection`.
No code change made — migrating would mean either misusing a component for a mismatched
semantic or changing behaviour, both disallowed by the "like-for-like, no appearance/
behaviour change" requirement.
