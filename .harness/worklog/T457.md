# T457 — detail-hint pills on StageIoItemRow

2026-07-08

## What I did

- `dashboard/app/components/StageIoLists.tsx`: added `detailHints(detail)` (exported) which
  extracts up to 4 extra scalar (`string`/`number`/`boolean`) fields from a work item's
  `detail` blob, skipping `name`/`markdown`/`path`/`format`/`attempts` and any
  object/array/null/undefined value. Each key is humanized via a new exported
  `humanizeDetailKey` (camelCase/snake_case → Title Case). String values > 80 chars are
  truncated with an ellipsis. `StageIoItemRow` now renders these as a `.stage-io-item-hints`
  row of `.stage-io-item-hint` pills beneath the name/key line, on both input and output rows,
  unconditionally (not gated behind the artifact-preview click).
- `dashboard/app/globals.css`: added `.stage-io-item-hints` (flex-wrap row) and
  `.stage-io-item-hint` (small muted pill, reusing the existing `.pill` visual language —
  `--panel-2`/`--border`/`--muted`).
- `dashboard/app/components/StageIoLists.test.ts` (new): 9 `node:test` cases against
  `detailHints`/`humanizeDetailKey` — name-only, extra scalars, snake_case keys, long-string
  truncation, nested object/array skipped, cap at 4, excluded bookkeeping keys, null/undefined
  detail. Wrapped in an async `main()` (not top-level `await test(...)`) since `dashboard/`
  has no `"type": "module"` in its `package.json`, which makes `tsx --test` emit cjs and
  reject top-level await — mirrored `dashboard/scripts/nav-check.test.ts`'s existing pattern
  for the same reason.
- `dashboard/scripts/_dashboard-harness.mjs` (Living Artifact Rule): the places-shaped
  `placesResolveOutput`/`placesEnrichOutput` fixtures previously had ONLY `detail.name` — added
  `detail.placeId` (resolve) and `detail.rating`/`detail.type`/`detail.address` (enrich),
  mirroring the real `places/stages/resolve.ts`/`enrich.ts` `markWorkItem` calls, so
  `visual-check.mjs`/`mobile-check.mjs` actually exercise + screenshot the new hint pills. (The
  pre-existing `stock-digest` fixtures — `stock-portfolio-snapshot`, `stock-sector-lookup` —
  already carried extra scalar detail fields, so they exercised the new rendering unchanged.)
