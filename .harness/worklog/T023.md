# T023 — Security hardening of dashboard + API

## Goal
Stop remote actors abusing the run/toggle endpoints: tighten CORS off `*`,
confirm loopback bind (configurable), guard mutating endpoints with a
loopback/token check — while keeping the read-only local dashboard working.

## What was done (reconciled from interrupted attempt — all outstanding work confirmed present)

`src/config.ts`
- `apiHost` (`LOCALJOBS_HOST`, default `127.0.0.1`) — bind address, configurable.
- `allowedOrigins` (`LOCALJOBS_ALLOWED_ORIGINS`, default local dashboard origins)
  — CORS allowlist, comma-split. Never `*`.
- `authToken` (`LOCALJOBS_TOKEN`, default empty) — shared secret for non-loopback mutations.

`src/api/server.ts`
- `json()` no longer emits `Access-Control-Allow-Origin: *`.
- `applyCors()` reflects the request Origin ONLY when allowlisted (+ `Vary: Origin`);
  disallowed origin gets no ACAO header so the browser blocks the response.
- `authoriseMutation()` — loopback always allowed; non-loopback caller must present
  the configured token (`X-LocalJobs-Token` or `Authorization: Bearer`); no token
  configured ⇒ remote mutations impossible.
- Every POST routes through the guard → 401 `unauthorised` when it fails. GETs stay open.
- `createApiServer(opts)` split out (injectable `isLoopback`) so tests drive it on an
  ephemeral port; `startApi()` binds `config.apiHost`.

`src/api/server.test.ts` (new, auto-discovered by run-tests.ts)
- Pure helpers: `originAllowed` (rejects `*`/unknown/undefined), `isLoopbackAddress`
  (127.0.0.1/::1/::ffff:127.0.0.1 vs Tailscale CGNAT), `authoriseMutation`
  (loopback allowed; remote rejected w/o token; remote allowed w/ header or bearer).
- Integration on ephemeral server: allowed Origin reflected (not `*`); disallowed
  Origin gets NO ACAO; remote POST no-token → 401; remote POST right-token → 404
  (guard passed to routing); loopback POST → 404 (guard passed).
