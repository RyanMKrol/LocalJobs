# T167 worklog — Progress bar: stop scrolling barber-pole animation once run is finished

## What was done

The joyful-theme animated progress bar (`joy-progress-sheen`) was applying to ALL `.progress > span`
elements in joyful themes, including bars for completed/terminal runs.

**Fix:**
- `dashboard/app/ui.tsx`: Added optional `done?: boolean` prop to `ProgressBar`. When `done` is
  true (or `pct >= 100` by default), adds class `done` to the `.progress` div.
- `dashboard/app/runs/[id]/page.tsx`: Passes `done={run.status !== 'running'}` so terminal runs
  get the `done` class immediately.
- `dashboard/app/globals.css`: Changed the animation selector from `.progress > span` to
  `.progress:not(.done) > span` — animation only runs when the bar does NOT have the `done` class.
  The static stripe texture remains on done bars; only the scrolling motion stops.

**Workflow-run page check:** `/workflow-runs/[id]/page.tsx` has no visual `.progress` bar —
it shows text only (`"N of M stages · X%"`), so there was nothing to fix there.
