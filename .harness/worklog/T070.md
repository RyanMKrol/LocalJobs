# T070 — Strip workflow-level properties from individual jobs

## Goal
A job is only ever a workflow member (T037), so workflow-level concerns must not
live on individual jobs. Remove `schedule`, the `enabled` toggle, `next-run`, the
`instructions` panel, and the per-job Run-now button wherever integrated. The job
page becomes a read-only member view (status · run history · logs). Workflows keep
schedule/enabled/run/next-run.

## What I changed
- **`src/core/types.ts`** — removed `instructions?` and `schedule?` from
  `JobDefinition`; rewrote the doc comment to state a job carries no workflow-level
  properties. Kept `timeoutMs`/`maxRetries`/`produces`/`consumes`/`run`.
- **`src/db/schema.sql`** — `jobs` table no longer has `schedule` or `enabled`
  columns (fresh DBs get the trimmed table).
- **`src/db/index.ts`** — added exported, idempotent `migrateDropJobColumns(db)`
  (called in `openDb`) that drops the legacy `schedule`/`enabled` columns from an
  existing `jobs` table without row loss (neither is indexed → safe DROP COLUMN).
- **`src/db/store.ts`** — `syncJob`/`upsertJobStmt` no longer write schedule/enabled;
  `JobRow` trimmed; removed `setJobEnabled`.
- **`src/core/scheduler.ts`** — removed the always-null per-job `nextRun()`.
- **`src/api/server.ts`** — `jobView` no longer returns `next_run`/`instructions`;
  removed `POST /api/jobs/:name/run` and `POST /api/jobs/:name/toggle`; dropped now-
  unused `runJob`, `nextRun`, `setJobEnabled` imports. Left explanatory NOTEs.
- **`dashboard/app/lib/api.ts`** — `Job` type trimmed (no schedule/enabled/next_run/
  instructions); removed `api.runNow`/`api.toggle`.
- **`dashboard/app/jobs/[name]/page.tsx`** — removed Run-now button, enable toggle,
  instructions panel, and the Schedule/Enabled/Next-run rows. Now a read-only member
  view: links to its workflow, Timeout/Max-retries, stuck items, run history.
- **Example jobs** — removed `instructions`/`schedule` fields from all 8
  (`places/*.job.ts`, `perfumes/*.job.ts`); the workflow manifests already carry the
  real schedule. Updated two stale "see job instructions" error strings → "see README".
- **`src/daemon.ts`** — clarified the enabled-toggle comment (workflows + services, not jobs).
- **Docs** — CLAUDE.md (add-a-job example + a new T070 convention bullet), README
  (Triggering jobs section + Job-detail dashboard description).
