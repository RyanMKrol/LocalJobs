# T061 — Backlog view: DAG / dependency graph

## Approach

Added a DAG view to the `/backlog` page alongside the existing list view, using the same wave-layout approach as the existing `Dag.tsx` component.

### Files changed

- `dashboard/app/components/Dag.tsx` — added `computeBacklogWaves` (wave algorithm adapted for `BacklogTask.dependsOn`) and exported `BacklogDag` component
- `dashboard/app/backlog/page.tsx` — added `view` state toggle (List / DAG), `selectedId` state, rendered `BacklogDag` in DAG mode with `DagDetailPanel` for selected task
- `dashboard/app/globals.css` — added `.dag-node.done`, `.dag-node.needs-human`, `.dag-node.selected`, `button.dag-node:hover`, `.backlog-dag-scroll`, `.dag-legend.*`, `.view-toggle` styles

### Design decisions

- Node colouring: `done` → green (reuses success style pattern), `running` (blue) for the next-buildable task, `needs-human` → accent, default for other pending
- Click to select a node → detail panel appears below the DAG showing full task info (title, model ladder, do/doneWhen)
- DAG is horizontally scrollable (`.backlog-dag-scroll` with `overflow-x: auto; flex-wrap: nowrap`)
- `title` attribute on each node shows task title on hover for quick inspection without selecting
