# T249 — Sortable column headers for recommendation tables

## What was built

Added client-side sortable column headers to both the movie (`MovieRecsManager`) and TV
(`TvRecsManager`) recommendation tables on the workflow detail page.

### Changes

**`dashboard/app/workflows/[name]/page.tsx`**
- Added `RecSortCol` + `SortDir` types and a generic `sortRecs<T>` comparator.
- Added `SortTh` component: renders a `<th>` with `.sort-th` / `.sort-th-active` classes,
  click toggles asc/desc (same column) or resets to desc (new column).
- Both `MovieRecsManager` and `TvRecsManager` gain `sortCol` / `sortDir` state (default: `tmdb` / `desc`),
  a `handleSort` callback via `useCallback`, and a `sortedActive` derived list.
- Active rows are sorted through `sortedActive`; the `SortTh` headers replace the old static `<th>` cells.

**`dashboard/app/globals.css`**
- Added `.sort-th`, `.sort-th:hover`, `.sort-th-active` — cursor pointer, theme-variable accent colour for
  the active header. No new phone-width rules needed (tables already scroll in `.panel`).

**`dashboard/scripts/_dashboard-harness.mjs`**
- Added `rec()` factory + `movieRecs` / `tvRecs` fixtures with 4 active + 1 ignored item each.
- Added `/api/movie-recs` and `/api/tv-recs` + `/api/movie-gaps` routes to `fixtureFor`.
- Added `workflow-movie-recs` and `workflow-tv-recs` to `PAGES`.
- Added `movie-recs-sorted-by-lens` flow (clicks the Lens header to show lens-grouped rows) and
  `tv-recs-default-sort` flow (screenshots the default TMDB-desc state).
