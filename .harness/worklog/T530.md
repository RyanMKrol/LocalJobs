# T530 — 2026-07-13

Replaced `src/api/server.ts`'s ~55-block sequential if/else positional-path
dispatch (`parts[1] === 'workflows' && parts[3] === 'run'`) with a declarative
`routes: Route[]` table + a small exact-segment-length matcher (`matchRoute`).

## What changed

- Added `type Route = { method, pattern, handler }` and a `RouteCtx` type
  (`req, res, url, parts, params`) passed to every handler.
- `matchRoute(method, pathParts, routes)`: requires the pattern and path to
  have the SAME segment count (the actual fix — the old chain had no length
  check on several routes, e.g. `parts[3] === 'runs'` matched ANY path with
  `'runs'` at that position regardless of what followed it), matches literal
  segments exactly, and captures `:name`-style segments into `params`.
- Converted every existing if-block into a `{ method, pattern, handler }`
  entry in a single `routes` array — same order as the original file, though
  order is no longer semantically load-bearing (no two routes at the same
  method + segment count can overlap; a literal-vs-literal mismatch or a
  differing segment count always disambiguates).
- Kept everything in `src/api/server.ts` (single file) rather than splitting
  into `src/api/routes/` — the spec called the split "optional but preferred";
  given the size and the need for a careful, zero-behavior-change conversion
  I kept it in one file to minimize risk. All existing helper functions
  (`jobView`, `workflowView`, `gatesForWorkflow`, `resolveOutputForm`,
  `resolveBulkScope`, `safeOutputMarkdown`/`safeOutputFile`, etc.) are
  unchanged, still module-level in `server.ts`.
- `createApiServer`'s request handler is now: CORS → OPTIONS short-circuit →
  POST mutation-auth guard → `matchRoute` → handler → same try/catch → 500.
  Byte-identical CORS/auth/500 behavior preserved.

## Behavior-change note (intentional, per spec)

Several routes previously had NO explicit `parts.length` check at all (only a
positional literal check), so they loose-matched: `GET /api/jobs/:name/runs`,
`GET /api/workflows/:name/runs`, `POST /api/workflows/:name/run` (and
`/toggle`, `/schedule`, `/concurrency`, `/notify`, `/certify`,
`/reset-output`), `POST /api/jobs/:name/prune`, `/timeout`, and the
movie-gaps/movie-recs/tv-recs/missing-seasons `:id/ignore` and `:id/unignore`
routes. With the new exact-length matcher, a path with extra trailing segments
on any of these now correctly 404s instead of loose-matching. This is exactly
the bug class T530 exists to kill, per the spec.

in this commit is my own single-file implementation, verified independently.
