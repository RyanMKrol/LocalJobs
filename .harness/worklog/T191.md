# T191 worklog

## What was built

Replaced the per-task "Mark as reviewed" toggle in the Backlog Done section with a
bulk checkbox-selection flow that reviews multiple tasks in a SINGLE git commit.

### API changes (`src/api/server.ts`)

- Added `setReviewEntries(raw, ids[], reviewed, at)` pure helper — same as
  `setReviewEntry` but sets multiple ids in one pass. Exported for unit testing.
- Added `writeReviewEntries(path, ids, reviewed, at)` — atomic temp+rename write for
  the bulk case.
- Extended `commitReviewsFile` with an optional `commitMessage` param so the bulk path
  can supply its own message (`reviews: bulk N reviewed [skip ci]`).
- Added `defaultCommitReviewsBulk(reviewsPath, ids)` — resolves repo root and calls
  `commitReviewsFile` with the bulk message.
- Added `commitReviewsBulk` injectable to `createApiServer` opts (parallel to
  `commitReviews`).
- Added `POST /api/backlog/reviewed-bulk { ids: string[] }` endpoint: validates non-
  empty array of `T\d+` ids (400 otherwise), writes reviews.json for all ids in one
  atomic disk write, commits once under the same `serializeReview` mutex + repo lock.

### Tests (`src/api/server.test.ts`)

- Imported `setReviewEntries`.
- Added unit test for `setReviewEntries`: empty raw, multiple ids set, sibling
  preserved, overwrite.
- Added integration test for `POST /api/backlog/reviewed-bulk`: verifies ids written,
  pre-existing entry preserved, response fields.
- Added test for 400 on empty ids and invalid id format.

### Dashboard (`dashboard/app/lib/api.ts`)

- Added `markReviewedBulk(ids: string[])` calling `POST /api/backlog/reviewed-bulk`.

### Dashboard (`dashboard/app/backlog/page.tsx`)

- Removed per-task review toggle button entirely from the Done list.
- Added per-row checkbox on unreviewed done rows (reviewed rows get a spacer for alignment).
- Added select-all header checkbox covering all visible unreviewed done tasks.
- Added "Mark N as reviewed" action button that POSTs to the bulk endpoint and refreshes.
- Filter-change clears the selection set.

### Docs (`CLAUDE.md`)

- Updated the architecture "ONE deliberate write exception" bullet to describe both
  the per-task and bulk endpoint and their single-commit guarantee.
- Updated the harness `reviewed`-flag section to describe both endpoints.
