# T227 worklog

## What was done

Unified the styling of the grouped output/manager sections on the workflow detail page.

### Problems fixed

1. **Inverted background**: The `output-section` wrapper had `background: var(--panel-2)` (the darker inset
   surface) wrapping inner `.panel` items (lighter) — the opposite of every other page section. Fixed by
   removing the outer wrapper entirely (replaced `<div className="output-section">` with `<>` fragments),
   so the sections now follow the same `<h2>...</h2><content>` pattern as Graph, Runs, and Danger zone.

2. **Inconsistent containment**: The managers had an extra beige outer container that Runs did not. Removed.

3. **"Ignore all" button jammed in corner**: Panels inside `.movie-gaps-scroll` had no padding, so the
   group header flex-row (title + button) was flush against the border-radius. Added
   `.movie-gaps-scroll .panel { padding: 12px 16px; }` to give proper inset.

4. **Per-row Ignore button in wide empty column**: Removed `style={{ textAlign: 'right' }}` from action
   `<td>` elements and added `.movie-gaps-scroll table th:last-child, td:last-child { width: 1%; white-space: nowrap; }`
   so the action column is compact and just wide enough for the button.

5. **Group titles flush-left against border**: Fixed by the same panel padding (point 3).

### Files changed

- `dashboard/app/workflows/[name]/page.tsx`: Removed outer `output-section` div from all 4 managers
  (TvRecsManager, MovieRecsManager, MovieGapsManager, MissingSeasonsManager); removed `textAlign: right`
  from 3 action `<td>` elements.
- `dashboard/app/globals.css`: Removed `.output-section` container rules (border/background/padding/radius);
  removed `.output-section h2`, `.output-section table`, `.output-section .panel` rules; added
  `.movie-gaps-scroll .panel { padding: 12px 16px; }` and compact last-column rule.

### Verification

- `npx tsc --noEmit`: clean
- `npm test`: all tests passed
- `npm --prefix dashboard run build`: clean build
- `node dashboard/scripts/mobile-check.mjs`: all 9 pages pass (0px overflow, 0 boundary crossings)
- Theme variable-driven: the sections now use `var(--panel)` as their surface (via `.panel`), which is
  correctly overridden by all 3 theme families in both light and dark modes.
