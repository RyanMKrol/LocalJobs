# T002 — Self-contain the perfume profile template

## What was done
The perfumes job config leaked the machine's topology via a hardcoded absolute
path: `templatePath: '/Users/ryankrol/Development/perfume-markdown/research/_TEMPLATE.md'`.
Made the repo self-contained:

1. **Copied the template content** from the external repo's `_TEMPLATE.md` into a
   new in-project file `src/jobs/perfumes/profile.template.md` (verbatim — same
   frontmatter keys, section headings, and example values).
2. **`config.ts`** — `templatePath` now resolves RELATIVE to the job dir:
   `process.env.PERFUMES_TEMPLATE_PATH ?? resolve(here, 'profile.template.md')`.
   No absolute path, no external repo. Optional env override added.
3. **`build.ts`** — unchanged; its `existsSync(templatePath) ? read : FALLBACK_TEMPLATE`
   path still works (FALLBACK_TEMPLATE kept as required).
4. **`.env.example`** — documented `PERFUMES_TEMPLATE_PATH` (commented, with the
   in-project default noted).
5. **`CLAUDE.md`** — added a "repo is self-contained — no absolute paths" convention.
6. **Test** — added `src/jobs/perfumes/config.test.ts` asserting the default
   resolves to the in-project file, is absolute, doesn't reference `perfume-markdown`,
   and that the file exists + is template-shaped (starts with `---`, has `## Sources`).

## Privacy note
