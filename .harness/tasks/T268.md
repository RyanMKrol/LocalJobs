## Do

Create a shared **UI component library** for the dashboard, so the same element isn't re-implemented
ad hoc across pages (the Run button is styled/behaves differently on Overview vs Workflows vs the
workflow detail page — the cited drift). This is the FIRST task of a phased effort: **create the
library here**; migrating each page to use it is a separate per-page task (T269–T277, which depend on
this).

Do:
1. **Scan every dashboard page** (`dashboard/app/**/page.tsx` — overview, workflows, workflow detail,
   workflow-run, job, run, services, backlog, the two gate pages) and inventory the recurring/
   duplicated UI: the **Run button** (Overview cards / Workflows list / workflow detail — each with its
   own markup + disabled/"Running…" handling), the various **`.pill`** chips (status / reviewed /
   done / failed / human / buildable — i.e. a `<Pill kind=…>` covering the "pill for any enumeration"
   idea), the **action buttons** (`.review-toggle` etc.), **panels/cards**, **tables**, and the
   **modal/popover chrome** (`.db-modal*`, the bulk `StuckPopover`). Record the inventory (a short
   note — what's duplicated, where, and which become components).
2. **Extract the genuinely-shared ones into reusable components** under `dashboard/app/components/`
   (consolidating with the partial existing shared layer in `dashboard/app/ui.tsx` — `StatusBadge`,
   `ThemeControls`, `StuckPopover`, etc.). Each component WRAPS the existing CSS-variable class idiom
   in `globals.css` (`.pill*`, `.review-toggle`, `.btn*`, `.panel`, `.db-modal*`) — do NOT introduce a
   new styling system or change the rendered look. Start with the highest-value: `<RunButton>` and
   `<Pill>`.
3. **Codify the convention** in `CLAUDE.md` (the dashboard section, alongside the mobile-responsive /
   theme-variable / visual-check rules): new dashboard UI MUST reuse these shared components rather
   than re-implementing one-offs; new shared elements go in `dashboard/app/components/`.

This task CREATES the library + convention; it does NOT migrate the pages (that's T269–T277). It may
adopt a component in one reference spot only if needed to prove it renders.

## Done when

- `dashboard/app/components/` contains the extracted shared components (at minimum `<RunButton>` and
  `<Pill>`), each wrapping the existing class idiom (no visual change, theme-variable-driven), with a
  recorded inventory of what was found duplicated and what became a component.
- The reuse convention is documented in `CLAUDE.md` (reuse shared components; where they live).
- `npm --prefix dashboard run build`, `npx tsc --noEmit`, and `node dashboard/scripts/mobile-check.mjs`
  are green; `node dashboard/scripts/visual-check.mjs` is green (no page should look different yet —
  this task doesn't migrate pages). The harness (`PAGES`/fixtures/`FLOWS`) is updated only if needed.
