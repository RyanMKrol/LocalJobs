## Do

Fix the invisible gate **padlock** marks on the workflow DAG (regression from T223). In
`dashboard/app/components/DagFlow.tsx` the padlock is built as an `<a><svg>…</svg></a>` React node and
assigned to a React Flow built-in (`type: 'default'`) edge's `label`. React Flow v12 renders a
built-in edge's label inside an SVG `<text>` element, where an `<a>`/`<svg>` can't paint AND the
zero-width `getBBox()` forces the wrapping group to `visibility: hidden` — so NO padlock ever shows,
on any gated edge, in either the run view (`/workflow-runs/<id>`) or the definition view
(`/workflows/<name>`).

Render the gate marks via React Flow's **`EdgeLabelRenderer`** using a small **custom edge type**
(an HTML portal overlay positioned at the edge midpoint) instead of the default edge's `EdgeText`, so
the HTML `<a>`/`<svg>` padlock has a real DOM home and actually paints. Preserve everything T223
already built: state colour-coding (failed = red/prominent, passed = green/subtle, pending/structural
= muted-grey/small), the hover tooltip (`key — description`), and the click-through to the gate detail
page (`gateHref`). Keep it theme-variable-driven and mobile-safe.
