## Overview

On the dashboard's Logs page (`/logs`), each log line currently renders as `<timestamp> [source]
[LEVEL] message`. The owner wants the level tag to come immediately after the timestamp, before the
job/workflow source label: `<timestamp> [LEVEL] [source] message`. This is a purely mechanical JSX
reorder of two existing elements — no new rendering logic, no CSS redesign.

## Do

In `dashboard/app/logs/page.tsx`, inside the `lines.map((l) => ...)` block (currently around lines
161-174), swap the JSX order of the source-label block and the level-tag block so the level tag
renders BEFORE the source label. The final per-line order must be:

1. timestamp (`<span className="ts">{fmtTime(l.ts)}</span>`)
2. level tag (`<span className={`log-lvl-tag log-lvl-tag--${l.level}`}>[{l.level.toUpperCase()}]</span>`)
3. source label (the existing `sourceLabel && (...)` block — link when `href` is available, plain
   `<span className="mono">` otherwise)
4. message (`{l.message}`)

Preserve the existing spacing mechanism exactly as-is, just relocated with the two blocks: keep the
timestamp's own `margin-right` (`.ts` in `globals.css`) as the separator before the level tag (no
explicit space text node needed there, matching how it already separates the timestamp from
whatever follows it today); keep the level tag's own `margin-right` (`.log-lvl-tag` in
`globals.css`) plus an explicit `{' '}` text node as the separator before the source label (mirroring
the existing double-spacing pattern that currently sits between the level tag and the message); and
keep the single explicit `{' '}` text node as the separator before the message (mirroring the
existing space that currently sits between the source label and the level tag).

Do NOT change any CSS. `.log-lvl-tag`'s `margin-right: 6px` already separates it from whatever
immediately follows it — it works identically whether that's the message (today) or the source label
(after this change) — no adjustment needed. `.ts`'s `margin-right: 10px` is unaffected since the
timestamp still leads the line. This is confirmed by reading the current CSS in `globals.css` before
making the change.

Judgment call (not asked of the owner — a confident, low-risk inference): the timestamp stays FIRST,
ahead of the level tag. The owner's wording was "the level should be the very first thing before the
job" — about level vs. source specifically, not level vs. timestamp — and a leading timestamp is the
universal log-line convention. Final order: timestamp → `[LEVEL]` → `[source]` → message.
