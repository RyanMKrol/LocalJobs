## Overview
The '✕ Ignore all' button in a GroupedManager group-header row sits 4px further right than the per-item '✕ Ignore' buttons below it, because the group-header <td> uses a narrower horizontal padding than a normal <td>. Fix the padding mismatch so both buttons' right edges line up.

## Do
In dashboard/app/globals.css, the default cell rule at (currently) line 107 reads `th, td { text-align: left; padding: 8px 18px; border-bottom: 1px solid var(--border); }` — 18px horizontal padding. The group-header override at (currently) line 232 reads `.table-group-header td { background: color-mix(in srgb, var(--accent) 16%, var(--panel-2)); font-weight: 600; padding: 8px 14px; border-top: 1px solid var(--border); border-bottom: 1px solid color-mix(in srgb, var(--accent) 40%, var(--border)); }` — 14px horizontal padding, 4px narrower on each side. Both classes are used by dashboard/app/components/GroupedManager.tsx (the '✕ Ignore all' button lives inside a flex div inside `.table-group-header td`, right-aligned via justifyContent: 'space-between'; the per-item '✕ Ignore' button lives in a plain trailing `<td>`). Change `.table-group-header td`'s padding from `8px 14px` to `8px 18px` so the horizontal padding matches the default td exactly — keep the 8px vertical padding and every other property (background, font-weight, border-top, border-bottom) unchanged. This is a single-value CSS edit, not a rule restructure.

## Done when
`.table-group-header td` in dashboard/app/globals.css has `padding: 8px 18px` (horizontal padding now matches the default `th, td` rule's 18px). No other property of the rule changed. `npm run build` in dashboard/ succeeds. Run `node dashboard/scripts/visual-check.mjs` after building and visually confirm (via the screenshots it writes) that on a page using GroupedManager (missing-movies or missing-tv-seasons workflow detail page) the 'Ignore all' button's right edge now aligns with the per-item 'Ignore' buttons' right edge directly below it, with no new horizontal overflow or layout regression in the group-header row.
