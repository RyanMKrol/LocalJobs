## Overview
src/api/server.ts is one ~1,150-line request callback of ~55 sequential `if` blocks doing positional path matching (`parts[1] === 'workflows' && parts[3] === 'run'`). This has ALREADY produced loose-match bugs — e.g. `GET /api/workflows/:name/runs` at line ~962 matches ANY path where `parts[3] === 'runs'` regardless of length. Replace the if/else chain with a small no-dependency route table + a ~20-line matcher that does EXACT segment-length matching and `:param` extraction, killing the loose-path bug class and making route ordering irrelevant.

## Do
In src/api/server.ts (staying framework-free — no express/router dep, per the deliberate no-dependency choice) define `type Route = { method: string; pattern: string; handler: (ctx) => Promise<void>|void }` and a `const routes: Route[]` array. Write a ~20-line matcher: split the request path and each `pattern` on '/', require EQUAL segment count (this is the fix — `parts.length` exact match), match literal segments, and collect `:name`-style segments into a `params` object passed to the handler. Convert each existing `if (method === … && parts[…] === …)` block into a `{ method, pattern: '/api/workflows/:name/runs', handler }` entry, moving the handler bodies out either into per-domain files under src/api/routes/ (mirroring the store split — optional but preferred) or into named functions in server.ts. A no-match falls through to the existing 404. CRITICAL: this must be a zero-behavior-change move EXCEPT that paths with extra trailing segments now correctly 404 instead of loose-matching. Add route-matcher unit tests (exact-length match, `:param` extraction, 404 fallthrough, and a regression test proving `GET /api/workflows/foo/runs/extra` does NOT match the `/api/workflows/:name/runs` route). Do this AFTER the store split lands (separate commit) and land the route table as its OWN commit separate from any behavioral API fix, per R03's sequencing note. Update the CLAUDE.md architecture/api notes to describe the route table.

## Done when
server.ts routing is driven by a `routes: Route[]` table + a length-exact matcher (no positional `parts[n]===` chain remains for dispatch); the full ~100-case server.test.ts passes unchanged (response shapes/status codes byte-identical); new route-matcher unit tests pass including a regression asserting no route matches a path with extra segments; `npx tsc --noEmit` + `npm test` green; CLAUDE.md updated.
