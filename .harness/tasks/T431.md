## Overview

Part of a repo-wide sweep giving every job a real paragraph-length `description` (surfaced on
`/jobs/[name]` and `/runs/[id]`) instead of a one-liner. This task covers the 3 jobs in the
`missing-tv-seasons` workflow (folder `src/workflows/missing-tv-seasons/`). The mechanism was already
built by the foundational task this depends on — this is pure content authoring.

## Do

Rewrite the `description` string of each of the following 3 job files to a real 3-6 sentence
paragraph — what the stage does step by step, what it reads/writes, and any notable behavior
(idempotency via the "have I already notified this?" ledger pattern, TMDB/Plex specifics). Ground each
in `src/workflows/missing-tv-seasons/CLAUDE.md` (if present) and the stage's own code. Plain prose
only, no markdown syntax:

- `src/workflows/missing-tv-seasons/stages/snapshot.job.ts` (`plex-tv-snapshot`) — currently: "Stage
  1: snapshot the Plex TV library by GUID — each show + its highest owned regular season."
- `src/workflows/missing-tv-seasons/stages/season-check.job.ts` (`tmdb-season-check`) — currently:
  "Stage 2: check TMDB by GUID for complete seasons the owner is missing (ended shows included)."
- `src/workflows/missing-tv-seasons/stages/notify.job.ts` (`plex-seasons-notify`) — currently: "Stage
  3: digest-push the newly-detected complete seasons you don't own (deduped per show+season)."

Also read root `CLAUDE.md`'s description of this workflow (it re-scans fresh every run and is
idempotent only at the final notify stage via the work_items ledger as a notification log, not a
work-done log) so each paragraph reflects that this is an audit/alert workflow, not a build pipeline.

## Done when

- All 3 `missing-tv-seasons` jobs have a real, accurate paragraph description grounded in their actual
  behavior — not a one-liner.
