## Do

On a run's progress bar, the small **hint text** appended after the percentage (e.g. *"20% ·
verifying 61"*) is often unclear — "verifying 61" doesn't tell the owner anything useful out of
context. **Hide the progress-message hint and show only the percentage** (e.g. just "20%").

- On the run detail page (`dashboard/app/runs/[id]/page.tsx`, and any shared progress-bar component
  in `dashboard/app/ui.tsx`), stop rendering the `ctx.progress` MESSAGE next to the percentage —
  display only the `NN%`. The bar fill + the percentage stay; only the trailing "· <message>" hint
  is removed.
- The detailed narration isn't lost — it still lives in the **logs** (where per-item detail belongs,
  and where T174/T176 are making it more verbose). This change only declutters the progress bar.
- Apply consistently wherever a progress bar currently renders the message hint (the run detail page
  primarily; check workflow-run / list progress bars and keep them percentage-only too for
  consistency).
- Keep the percentage + bar + the completed/running colouring unchanged. Mobile-responsive.

(Note: this is display-only — jobs can keep calling `ctx.progress(pct, message)`; the message simply
isn't shown on the bar anymore. No job code needs to change.)

## Done when

A run's progress bar shows only the percentage (e.g. "20%") with no trailing message hint, on the run
detail page and any other progress-bar display, while the bar fill, percentage, and status colouring
are unchanged and the detailed narration remains in the logs; `npm --prefix dashboard run build`
passes and `node dashboard/scripts/mobile-check.mjs` stays green.
