## Overview
The launchd installers bake a frozen nvm node path into the plist at install time, so a later nvm upgrade/uninstall silently runs old Node or kills the daemon at boot. Resolve node at run time in the wrapper instead, drop the baked path from both plists, and fix the stale 'http://localhost:3001' comment in the dashboard installer.

## Do
1) In start-daemon.sh, resolve the node bin dir at RUN time (source nvm / read the .nvmrc from CI-TEST-03 / probe common node locations, fall back to PATH) instead of relying on a plist-injected NODE_BIN_DIR. 2) Remove the baked NODE_BIN_DIR from install-launchd.sh's generated plist (plist keeps only the wrapper path). 3) Apply the same run-time-resolution shape to the dashboard agent (route its `$NODE_BIN_DIR/npm` invocation through a wrapper or otherwise resolve node at run time so no frozen path is baked in). 4) Fix the stale `http://localhost:3001` header comment in install-dashboard-launchd.sh to 4788. 5) Ensure both installers re-run cleanly over an existing install. 6) Optionally (only if the owner confirms in scope) modernize `launchctl load/unload` to `bootstrap/bootout gui/$(id -u)`; otherwise leave load/unload as-is.

## Done when
Manual on the Mini (installer scripts have no test harness): `nvm install <new>` + `launchctl kickstart -k` picks up the new Node without re-running the installer; removing the old nvm version does not kill the daemon at boot; both installers re-run cleanly over an existing install; the generated plists no longer bake a frozen nvm node path; the install-dashboard-launchd.sh header comment reads 4788.
