## Overview
Adopt response caching for the `tmdb` service (the largest adopter — ~8 read sites across 4
workflows) so repeated TMDB reads within a 22-hour window are served from `service_cache` instead of
re-fetched. Outcome of the T479 caching-adoption decision. The caching mechanism (the `service_cache`
table, `callService`'s `cacheKey` option, per-service `cacheTtlMs`) already exists (T451/T476).

## Do
1. Set `cacheTtlMs: 79_200_000` (22 hours) on the `tmdb` `ServiceDefinition` in
   `src/services/tmdb.service.ts`.
2. Add a stable `cacheKey` to every PRODUCTION `callService('tmdb', ...)` READ site:
   - `src/workflows/plex-language-fix/lib.ts`
   - `src/workflows/tv-recs/stages/tv-rec-merge.ts`
   - `src/workflows/missing-tv-seasons/stages/season-check.ts` (2 calls)
   - `src/workflows/movies/stages/franchise-gaps.ts` (2 calls)
   - `src/workflows/movies/stages/merge.ts`
3. For the `() => tmdbGet(path)` sites, key by the request path: `` cacheKey: `tmdb:${path}` `` (e.g.
   `tmdb:/tv/123/season/2`). For the search-query sites (`tv-rec-merge.ts`, `merge.ts`) derive a
   stable key from the search query + media type (e.g. `tmdb:search:movie:<query>`).
4. Do NOT modify `src/workflows/plex-language-fix/stages/resolve.ts` — it already passes its own
   `tmdb-language:<type>:<tmdbId>` cacheKey; leave it as-is (the new 22h TTL applies to it too, which
   is fine). Do NOT refactor `tmdbGet` itself to route through `callService` — the existing sites
   already wrap `callService('tmdb', () => tmdbGet(...))`, so that would double-wrap/double-meter.
5. Cache read/GET calls only.

## Done when
- `cacheTtlMs: 79_200_000` is set on the `tmdb` service def.
- Every production `callService('tmdb', ...)` site listed above passes a stable `cacheKey` derived
  from its path or search query; `resolve.ts`'s existing cacheKey is untouched.
- `npx tsc --noEmit` and `npm test` are green.
- Verified a repeated `tmdbGet` for the same path within 22h is served from `service_cache` (a
  scratch-DB check via `getCachedServiceResponse`, or an equivalent hermetic assertion) rather than a
  second live call — no live paid/quota'd TMDB call is required to prove this.
