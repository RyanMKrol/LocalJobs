## Overview
Add a read-only store function that lists the actual rows of the service_cache table (service_name, cache_key, response_json, cached_at) instead of just the existing per-service counts, so a later API/UI layer can browse cache contents.

## Do
In src/db/store/services.ts, alongside the existing serviceCacheCounts() (~line 247-256) and clearServiceCache() (~line 259-268), add a new exported function `listServiceCacheRows(serviceName?: string)` that queries `SELECT service_name, cache_key, response_json, cached_at FROM service_cache` — filtered to `WHERE service_name = ?` when `serviceName` is passed, else all rows — ordered by `cached_at DESC` (most recently cached first). Define and export a `ServiceCacheRow` interface: `{ service_name: string; cache_key: string; response_json: string; cached_at: string }` (mirror the style of the existing `ServiceCacheCount` interface just above `serviceCacheCounts`). Do NOT parse `response_json` server-side in the store layer — return it as the raw stored string; the API/UI layer decides how to render it. Do NOT add pagination/limiting logic in this task — that is a separate concern for the API layer (task service-cache-browse-b) if the owner confirms it's wanted; this store function just returns all matching rows for the given filter.

## Done when
`listServiceCacheRows()` (no args) returns every service_cache row ordered by cached_at DESC; `listServiceCacheRows('some-service')` returns only that service's rows in the same order; the `ServiceCacheRow` type is exported from src/db/store/services.ts. A new test in src/db/store.test.ts (the existing store-level test file that covers src/db/store/*.ts via the store.ts barrel — there is no separate per-domain test file for services.ts) inserts ≥2 rows across ≥2 different service_name values directly via the DB (or via setCachedServiceResponse), then asserts listServiceCacheRows() returns all of them in cached_at-descending order, and that the serviceName-filtered call returns only the matching subset. `npx tsc --noEmit` and `npm test` are green.
