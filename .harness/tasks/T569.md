## Overview
All 16 service definitions repeat the same name/category/description + three unvalidated Number(process.env...) lines + paid + rateLimitSource shape. This unit migrates each onto the defineService() factory from R04a (each shrinks to ~10 declarative lines, still one file per service), fixes vercel's missing rateLimitSource, and proves via a before/after effectiveLimits snapshot test that nothing changed. It completes the bundled B19 env-validation fix by routing every numeric env read through the validating envInt.
