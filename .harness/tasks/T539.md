## Overview
Node is pinned only in CI; local and the launchd plists can silently diverge. Add an `engines.node` range to both package.jsons and a repo-root .nvmrc so all three environments (CI, local, launchd) agree on the Node major. This is the shared pin referenced by both the CI-hardening and launchd-installer tasks.

## Do
1) Add `"engines": { "node": ">=22 <23" }` to package.json. 2) Add the same to dashboard/package.json. 3) Create a repo-root `.nvmrc` containing `22`. 4) Keep the value consistent with CI's setup-node node-version (22). Do NOT enable engine-strict enforcement (advisory-warn is the intended behavior) unless the owner asks.

## Done when
`.nvmrc` exists at repo root pinning 22; both package.json and dashboard/package.json declare `engines.node` matching the CI Node major; `npm install` under an out-of-range Node major warns per engines; `npm test`, `npx tsc --noEmit`, and `npm --prefix dashboard run build` still pass. Confirm the exact range/value with the owner if unsure (pending-questions #18).
