## Do

The workflow graph (`dashboard/app/components/Dag.tsx`) is **left-aligned** in its container, so a
graph narrower than the container — e.g. the linear 3-node TV-seasons graph (`plex-tv-snapshot →
tmdb-season-check → plex-seasons-notify`) — sits flush against the left edge with a large empty gap
on the right (see screenshot). **Center the graph horizontally** in its container when it's narrower
than the available width.

- Center the wave/column layout horizontally (e.g. `justify-content: center` on the flex row of
  waves, or `margin: 0 auto` on the graph content) so a narrow graph sits in the middle, not
  flush-left.
- A graph that **fills or exceeds** the container width must be unaffected (it already uses the full
  width / scrolls or wraps); only narrow graphs get centered.
- Keep the phone (stacked) layout sensible — centered there too if appropriate.
- Don't disturb the edge/gate rendering. Mobile-responsive.

## Done when

A workflow graph narrower than its container is horizontally centered (no longer flush-left with a
big empty gap on the right); full-width / wrapping / scrolling graphs are unaffected; edge + gate
rendering is intact; `npm --prefix dashboard run build` passes and
`node dashboard/scripts/mobile-check.mjs` stays green.
