The `workouts-sync` workflow (`src/jobs/workouts-sync/`) is a 2-stage DAG: `hevy-sync` fully re-paginates the Hevy workout API (https://api.hevyapp.com/v1/workouts) every run and idempotently appends any newly-discovered workouts to a local full-history file, `src/jobs/workouts-sync/data/out/workouts-history.json` (no DynamoDB — a local JSON accumulator, per T298). `workouts-progress` (depends on `hevy-sync`) then reads that history file and computes a per-exercise 6-month progress comparison — best single set, total volume, and estimated 1-rep-max (Epley formula) — between a 'current period' (the most recently completed calendar month) and a 'baseline period' (the calendar month exactly 6 months before that), and asks Claude to narrate the comparison into a markdown report at `src/jobs/workouts-sync/data/out/workouts-progress.md` (raw numbers also written to `progress-data.json`).
