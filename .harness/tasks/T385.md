## Overview

Add a new "Overall" tab to `StageIoPanel` (`dashboard/app/components/StageIoLists.tsx`) that shows the whole workflow's own potency/ledger key(s) as Inputs and its terminal output(s) as Outputs, using the `overall=true` endpoint mode from T384. Make it the DEFAULT selected tab (shown before the per-stage tabs) — the owner specifically asked for this to be the first thing a person sees on a run page, since it answers "did this workflow's periodic run produce its result" at a glance.

## Do

In `dashboard/app/lib/api.ts`, add a client function (e.g. `workflowRunStageIoOverall(id: string)`) hitting `GET /api/workflow-runs/:id/stage-io?overall=true`, and extend the `StageIo` interface (or add a sibling type) to include the `outputJobs`/`predecessorJobs` fields T384's response carries.

In `dashboard/app/components/StageIoLists.tsx`, add an "Overall" entry to the tab bar `StageIoPanel` already renders (the `.io-job-filter-chip` bar added for the per-stage tabs) — it must be a DISTINCT tab from "All stages" (which stacks every per-stage block) and from any individual stage tab. Selecting "Overall" renders the SAME two-column Inputs/Outputs layout (`StageIoColumn`/`.stage-io-columns`) the per-stage blocks already use, fed by the new `overall=true` data instead of a single stage's data. Change `StageIoPanel`'s default `selectedJob` state so "Overall" is selected on mount, not the first per-stage job as it is today.

Update `dashboard/scripts/_dashboard-harness.mjs`: add a `stock-digest-run/stage-io?overall=true` fixture entry (mirroring the existing `stockDigestStageIo` per-job fixtures already in that file) so `mobile-check.mjs`/`visual-check.mjs` actually exercise the new tab — the existing `workflow-run-stock-digest` PAGES entry will now capture the Overall tab by default since it becomes the default selection; add or update a FLOW if a screenshot of a NON-default state (e.g. clicking back to a per-stage tab) is still needed for visual confirmation coverage.
