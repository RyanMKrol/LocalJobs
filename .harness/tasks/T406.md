## Overview

`movie-recommendations`'s notify stage (`src/jobs/movies/stages/notify.ts`) is the only stage in
this workflow that calls `markWorkItem`, using the ledger as a "have I already notified this
franchise-gap / recommendation?" log rather than a per-stage work-done log (this is expected — see
root `CLAUDE.md`'s T145 audit-style-workflow pattern). Both ledger entries already reference the
shared monthly report (`detail.markdown`), which satisfies the "stage writes a file → reference it"
half of the `detail` convention. But the `detail` blob for each entry still only restates the
item's identity (film title / collection name / lens) — it does not carry the actual FINDING that
was computed for that item (why it's a gap, why it was recommended, its TMDB rating). Someone
looking at the dashboard's Inputs & Outputs panel for a single ledger row has no way to see the
reasoning without opening the full markdown report and finding the row themselves. Enrich `detail`
on both `markWorkItem` calls so each ledger row is self-describing.
