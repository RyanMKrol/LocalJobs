## Do

The movies workflow logs report COUNTS without identifying WHICH items, so they aren't actionable —
contrary to this repo's "be verbose by default / enumerate notable items" logging convention.
Concrete example: the snapshot stage (`src/jobs/movies/stages/snapshot.ts:37`) logs *"N movie(s)
have no tmdb:// GUID — they can't be checked for franchise gaps (never guessed)."* but never names
them, so when a film is silently excluded from gap-checking you can't tell which one (the owner's
complaint: "which item has no tmdb? who knows…").

- **Snapshot stage:** when movies have no `tmdb://` GUID, **enumerate them by title** (plus year and
  the Plex `ratingKey` where available) — one per line or a compact bulleted list — not just a
  count. So the owner can see exactly which films can't be gap-checked and go fix the Plex match.
  (The snapshot already has the full movie list in memory, so the titles are right there.)
- **Audit the other movies stages** (`franchise-gaps`, the `rec-*` recommender branches, `rec-merge`,
  `movie-gaps-notify`) for the same "count only, no identity" pattern, and add item-level detail
  wherever a decision **excludes / skips / dedupes / drops** something — name the title(s) /
  tmdbId(s) involved, not just totals. Keep the existing summary counts; ADD the enumerations
  alongside them.
- Be sensible about volume: if a list could be very large, log the first ~20 and note "… and M
  more" (the no-GUID list is typically small, so list it in full). Use the right `ctx.log` levels
  (`warn` for exclusions like the no-GUID case).
- **Logging only — no behaviour change.** Don't alter what's snapshotted/checked/recommended, just
  how verbosely it's narrated.

## Done when

The movies snapshot stage names the specific movies that have no `tmdb://` GUID (title + year /
ratingKey) instead of only a count, and the other movies stages that previously logged count-only
exclusion/skip/dedupe decisions now identify the items involved (titles / tmdbIds), with the summary
counts kept alongside; no behavioural change; `npx tsc --noEmit` and `npm test` are green.
