## Do

Add the **LLM recommendation layer** (the subjective half of the movie audit, designed in T109) to
the `movies` workflow built in T145 (this task `dependsOn` T145). This is the genuine LLM use case
— *taste-based discovery* — with **code-side verification** so the model never invents films or
recommends ones already owned. The design (T109) is **8 recommender branches merged at the end**,
specifically chosen to counter genre **mode-collapse** (a 500-horror / 100-kids library must not
just yield more horror). Uses **Claude CLI** (free, the perfumes `claude.ts` pattern: pipe a prompt
to `claude -p --output-format json`, parse `.result`; reuse/extract that helper, don't duplicate).
Runs monthly, merged into the same digest as the franchise gaps.

### The 8 recommender branches — a fan-out DAG (snapshot → 8 branches → merge)
Each branch builds a prompt from T145's **taste profile + a sampled slice** of the owned library,
asks Claude for ~5 diverse films **not owned** (each with a one-line reason), and returns raw
`{ title, year, reason, lens }`. **Use STRATIFIED sampling everywhere** — sample roughly EQUAL
counts across genres/eras, NOT proportional — so the dominant genre doesn't drown out the rest.

**3 random branches** — each gets a DIFFERENT stratified-random slice (per-branch seed/noise so
they diverge) of ~40–60 owned titles; prompt ≈ "given these films I own, recommend diverse films
I'd enjoy that aren't in this list." Pure serendipity.

**5 targeted branches** (2 depth + 3 breadth — the split chosen in T109):
1. **Auteur completion** (depth) — directors owned ≥3 times + their owned titles → acclaimed films
   by *those directors* the owner doesn't own.
2. **Canon of top genres** (depth) — the owner's strongest genres + sample titles → canonical /
   acclaimed films IN those genres they somehow missed (blind spots in their strengths).
3. **Thin-genre round-out** (breadth) — genres owned FEW of → acclaimed films there (broadens
   without amplifying the dominant genre).
4. **Older-era classics** (breadth) — the decade histogram (skews 2000s–2020s, sparse pre-1980) →
   acclaimed **pre-1980** classics from the under-owned eras.
5. **World cinema** (breadth) — the country mix (Anglophone-heavy) → acclaimed **non-English**
   films the owner under-owns.

Every branch's prompt must instruct: recommend films NOT already owned, NOT sequels to franchises
being completed (that's T145's job), be diverse, and give a reason. Model the branches as
**parallel branch jobs** (true to the design + a clean framework fan-out) — or one job running the
8 prompts; prefer parallel jobs for DAG legibility. All Claude calls go through the `claude-cli`
service. Handle an LLM returning junk / no parseable JSON gracefully (skip that branch, don't fail
the run).

### Merge stage — code enforces correctness
Pool all 8 branches' raw suggestions, then for each suggestion: **TMDB search** (title + year) →
must resolve to a real tmdb id → must NOT be in the owned set → must NOT be in the
**already-recommended ledger** (keyed by recommended tmdb id, so a film is never recommended twice
across months). Dedupe across branches. **Balance the final output** (cap per genre so the digest
isn't one flavor); keep ~10–15. **Feed recent recommendations back into each branch's prompt** so
successive months vary. TMDB calls via the `tmdb` service.

### Output · dedup · ignore
- Add a **Recommendations** section to the monthly markdown report + the digest notification,
  SEPARATE from the franchise-gaps section, each rec showing its lens + reason + TMDB link.
- Dedup per recommended tmdb id (never re-recommend). **Ignore-to-suppress** for recs too — reuse
  the surfaced-item ignore mechanism from T145: an ignored ("not interested") rec never resurfaces
  in future reports/notifications. Owner action only.

### Framework shape · tests · privacy
- Same inversion as T145: no `inputKeys()`; recompute each run; only the dedup/ignore ledgers carry
  state. Reuse the `claude-cli` service + `claude.ts` pattern + the `tmdb` service.
- **Hermetic tests** (NO live Claude/TMDB): mock a branch's Claude output and assert the merge
  TMDB-verifies against synthetic fixtures — **drops** a hallucinated title (no TMDB match), an
  already-owned title, and a previously-recommended title; **keeps** a valid novel one. Assert
  cross-branch dedup, output balancing (cap per genre), and that **ignore suppresses** a rec.
  Assert the **stratified sampler is balanced, not proportional** (e.g. a 500/100 split yields a
  ~balanced sample). Cover a branch returning no-JSON/garbage without failing the run.
- Public code; tokens/library data stay private. Live end-to-end run is the OWNER's.
