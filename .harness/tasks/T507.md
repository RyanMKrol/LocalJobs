## Overview
Cache `lastfm` top-albums / top-tracks reads for 22 hours. Outcome of the T479 caching-adoption
decision; the caching mechanism already exists (T451/T476).

## Do
1. Set `cacheTtlMs: 79_200_000` (22 hours) on the `lastfm` `ServiceDefinition` in
   `src/services/lastfm.service.ts`.
2. Add a stable `cacheKey` to all 4 `callService('lastfm', ...)` read sites in
   `src/workflows/listening-digest/stages/listening-digest.ts` (top-albums + top-tracks, for both the
   current `period` and the `trailingPeriod`):
   - top-albums calls → `` `lastfm:top-albums:${period}` `` (using the actual period passed)
   - top-tracks calls → `` `lastfm:top-tracks:${period}` ``
3. Cache reads only.

## Done when
- `cacheTtlMs: 79_200_000` is set on the `lastfm` service def.
- All 4 call sites pass a stable `lastfm:<method>:<period>` cacheKey reflecting the exact period used.
- `npx tsc --noEmit` and `npm test` are green.
- Verified a repeated top-albums/top-tracks fetch for the same period within 22h is served from
