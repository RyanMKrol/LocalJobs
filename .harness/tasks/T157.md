## Do

Optimise the **Input → Output mapping** output column so it renders smoothly instead of janking.
Today each `OutputCell` (`dashboard/app/workflow-runs/[id]/page.tsx`) does a mount-time `useEffect`
that calls `api.workflowRunOutput(runId, outputJob, outputKey)` — which fetches the **full
multi-KB markdown content** for that row — and until it resolves the cell shows a *fallback* (the
raw `outputKey` in mono), then **re-renders into the filename/meta layout** once the fetch lands.
Doing that per row means N upfront heavy fetches AND a layout shift (the column visibly grows /
shifts left-right just after first paint).

Make it two-phase so the cell renders its **final shape immediately from data already on the row**,
and the heavy content loads lazily:

1. **Phase 1 — instant, no fetch.** The filename and name are already available without fetching:
   `row.outputDetail.markdown` holds the recorded markdown path (T110) → its basename is the
   filename, and `row.outputDetail.name` is the label. Render the FINAL meta cell (filename link +
   "click to preview") straight from `row.outputDetail` on first paint — same structure/width as
   the loaded state, so there's no reflow. (Keep the existing `—` for `!row.outputKey`, and for a
   row with an output key but no `markdown` path, show the key/name with no preview — still no
   fetch.)
2. **Phase 2 — lazy.** Fetch `api.workflowRunOutput(...)` only **when the user opens the preview**
   (the `onOpen`/click handler), not on mount — so the multi-KB markdown is loaded on demand for the
   one row being previewed, never for all rows upfront. (If you'd rather have the content warmed,
   fetch it in the background AFTER first paint via an idle/deferred pass rather than synchronously
   on mount — but on-click is preferred and avoids loading files the user never opens.) Show a tiny
   "loading…" state inside the popover while it fetches; keep the existing markdown popover + the
   no-artifact fallback.

The file **size** (`out.bytes`) currently shown in the cell requires the fetch — since it's no
longer fetched on mount, either drop the size from the cell (the "click to preview" affordance is
enough) or load it lazily; don't reintroduce a mount-time fetch just for the size. The
`workflowRunOutput` endpoint is unchanged (still the on-demand content source); this is a
client-side change to WHEN it's called.

The net effect: the output column paints its filenames immediately with a stable layout (no jank),
and heavy markdown is fetched only on preview. Stay mobile-responsive.

## Done when

The Input → Output output column renders each row's filename immediately from `row.outputDetail`
(no per-row mount fetch), so the column no longer shifts/grows after first paint; the full markdown
is fetched only when a row's preview is opened (or via a deferred background pass), with a loading
state in the popover and the no-artifact fallback intact; no mount-time content fetch remains.
`npm --prefix dashboard run build` passes and `node dashboard/scripts/mobile-check.mjs` stays green.
