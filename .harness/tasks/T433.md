## Overview

Part of a repo-wide sweep giving every job a real paragraph-length `description` (surfaced on
`/jobs/[name]` and `/runs/[id]`) instead of a one-liner. This task covers the 11 jobs in the `tv-recs`
workflow (`tv-recommendations`). The mechanism was already built by the foundational task this depends
on — this is pure content authoring.

## Context

Mirrors the `movies` workflow's shape exactly. The 8 recommender branch jobs (`tv-rec-random-1/2/3`,
`tv-rec-creator`, `tv-rec-canon`, `tv-rec-thin-genre`, `tv-rec-older-era`, `tv-rec-world`) are thin
wrappers; each branch's OWN `description` lives in a shared spec object in
`src/workflows/tv-recs/stages/branches.ts` (currently short one-liners like "Stage: TV serendipity
branch ${n} — recommends diverse shows from a stratified-random slice of the owned library."). Edit
the descriptions THERE, not in the individual `tv-rec-*.job.ts` wrapper files.

## Do

Rewrite each branch's `description` in `src/workflows/tv-recs/stages/branches.ts` (8 branches) to a
real 3-6 sentence paragraph — what makes that branch's recommendation lens distinct, what it reads (the
TV snapshot + taste profile) and writes (its raw suggestion list for `tv-rec-merge` to pool). Then do
the same for the 3 non-branch job files:

- `src/workflows/tv-recs/stages/tv-snapshot.job.ts` (`tv-snapshot`) — currently: "Stage 1: snapshot the
  Plex TV library by GUID — each show, its TMDB id, and taste metadata (genres/roles/decades/
  countries)."
- `src/workflows/tv-recs/stages/tv-rec-merge.job.ts` (`tv-rec-merge`) — currently: "Merge stage: pool
  the 8 TV recommender branches' raw suggestions, TMDB-verify each (real TV show, un-owned, not
  previously recommended), dedupe across branches, and balance per genre into the final recommendation
  list."
- `src/workflows/tv-recs/stages/tv-recs-notify.job.ts` (`tv-recs-notify`) — currently: "Final stage:
  ONE monthly digest of newly-verified TV show recommendations (owner-ignored + already-notified
  excluded; each pick announced exactly once)."

Ground every paragraph in root `CLAUDE.md`'s detailed `tv-recommendations` section (it documents the
full DAG, taste-profile shape, and merge/notify behavior) and each file's own code. Plain prose only,
no markdown syntax.

## Done when

- All 11 `tv-recs` jobs (8 branches in `branches.ts` + `tv-snapshot`, `tv-rec-merge`,
  `tv-recs-notify`) have a real, accurate paragraph description grounded in their actual behavior —
  not a one-liner.
- `npx tsc --noEmit` and `npm test` are green.
