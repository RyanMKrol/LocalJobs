## Overview
Human sign-off step: the owner looks at the five 'Top of mind' placement/treatment variants built on the Overview page (in the running dashboard, via the `?tom=1`..`?tom=5` query param) and picks a favorite, so the follow-up task knows which one to hardcode.

## Do
Run the dashboard locally, visit the Overview page with each of `?tom=1` through `?tom=5` (built by the prior task), compare the five 'Top of mind' placements/treatments, and decide which one to keep.

## Done when
The owner has viewed all five variants in the running dashboard and recorded which variant number (1-5) they want kept, e.g. as a note in the worklog or communicated directly for the next task to consume. Mark this task done via `.harness/scripts/mark-done.sh` once the decision is made.
