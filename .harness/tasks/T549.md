## Overview
Teach fmtDuration to show hours and make fmtRelative robust to already-ISO inputs and clock skew, matching fmtTime's guards.

## Do
In dashboard/app/ui.tsx: (1) fmtDuration (:64-71) tops out at minutes — a 2h5m run renders '125m 12s'. Add an hours tier so durations past 60 minutes render as 'Xh Ym'. (2) fmtRelative (:80-91) unconditionally does new Date(t.replace(' ','T')+'Z'), so an already-ISO input (ending in Z, as fmtTime at :73-78 guards for) yields NaN -> 'NaNs ago', and a future timestamp under clock skew yields '-2s ago'. Reuse fmtTime's includes('Z')||includes('T') guard so it parses an ISO string correctly, and clamp a negative diff to 'just now'. Add unit tests for both.

## Done when
fmtDuration renders 'Xh Ym' for durations over an hour; fmtRelative parses an already-ISO string without NaN and clamps negatives to 'just now'; unit tests cover both; npx tsc --noEmit, npm test, and npm --prefix dashboard run build are green.
