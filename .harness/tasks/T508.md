## Overview
Cache `hevy` workout-ingestion reads for 22 hours. Outcome of the T479 caching-adoption decision; the
caching mechanism already exists (T451/T476).

## Do
1. Set `cacheTtlMs: 79_200_000` (22 hours) on the `hevy` `ServiceDefinition` in
   `src/services/hevy.service.ts`.
2. Add a stable `cacheKey` to the `callService('hevy', ...)` read site(s) in
   `src/workflows/workouts-sync/stages/hevy-sync.ts`, keyed by the endpoint + page (or workout id)
   being fetched: `` `hevy:${endpoint}:${pageOrId}` `` (use whatever stable identifier the call
   actually varies over — e.g. the page number for a paginated workout list).
3. Cache reads (GET) only.

## Done when
- `cacheTtlMs: 79_200_000` is set on the `hevy` service def.
- Each `callService('hevy', ...)` read site passes a stable cacheKey derived from its endpoint + page/id.
- `npx tsc --noEmit` and `npm test` are green.
