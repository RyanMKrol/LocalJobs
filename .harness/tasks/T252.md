## Do

Build the shared **`dynamodb` service** — the single cross-repo DynamoDB client that every ingestion
pipeline (workouts / listens / projects) reuses. Create `src/services/dynamodb.service.ts`
default-exporting a self-contained `ServiceDefinition` (the registry discovers it from
`src/services/`), wrapping an `@aws-sdk/lib-dynamodb` `DynamoDBDocumentClient` over
`@aws-sdk/client-dynamodb`. Add both SDKs to `package.json` (mirror the website's `src/lib/dynamo.js`).
Read AWS creds + region + table names from `process.env` (never hardcoded). Expose helpers the
pipelines call through `callService('dynamodb', …)` so reads/writes are metered + rate-limited via the
SQLite `service_usage` meter, exactly like `gemini`/`google-places`.

**Spend governance — respect the DynamoDB free tier; be conservative.** AWS Always-Free DynamoDB is
≈ 25 provisioned RCU + 25 WCU + 25 GB (~200M requests/month). Nothing here is time-sensitive, so set
**conservative, env-overridable** limits well under that (a modest `ratePerMinute` — these syncs are
infrequent, small batches — and a monthly cap far below the free-tier request budget), so the service
soft-fails (`QuotaExceededError`) long before any cost. Research/refine the exact numbers when
building; default cautious. Put the daily-from-monthly math through `src/services/lib.ts` if a daily
cap applies.

Unit-test with the AWS client **mocked** — no live AWS calls in tests, never hit real DynamoDB in CI.
