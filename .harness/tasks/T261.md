## Do

Close the human-done handoff loop: make the dashboard "Mark done" on a `needs-human` task actually
**unblock its dependents** in the build loop. Today `.harness/human-done.json` is a display-only
overlay — the loop's `task_done()` (`.harness/loop.sh`) reads ONLY `TASKS.json` `.status=="done"` and
never consults the overlay, so a `needs-human` blocker marked done in the dashboard stays `pending` in
the loop's eyes and its buildable dependents are blocked forever (the T251 → pipelines gotcha).

Add a **pre-flight reconcile in `.harness/loop.sh`** (runs once per iteration, BEFORE task selection):
read `.harness/human-done.json`; for each id marked `done: true` **that is a `needs-human` task (its
`gate == "needs-human"`) and is not already `status=="done"` in `TASKS.json`**, set that task's
`TASKS.json` `status` to `done` and commit it (`[skip ci]`) + push (reuse the existing commit/push
pattern; this is the same kind of status write `mark_done` does). Then selection proceeds and the
task's dependents become eligible.

Hard constraints:
- **Only `needs-human` tasks** are promoted. NEVER flip a buildable (`gate == null`) task done from
  the overlay — buildable status stays the loop's own (set by `mark_done` after a real build).
- **Do NOT call `record_outcome`** / append to `outcomes.jsonl` — a `needs-human` task is not
  loop-built and is not a calibration input; this is a plain status write + commit only.
- Preserve the decoupling: the dashboard still writes ONLY the overlay; the loop remains the SOLE
  `TASKS.json` writer. Idempotent (re-running promotes nothing already done).
- Update the docs that currently say human-done "never modifies TASKS.json status" so they describe
  the reconcile (root `CLAUDE.md` human-done bullet, `.harness/CLAUDE.md`, `.harness/HARNESS.md`).

⚠️ This edits `loop.sh` — the loop's own driver — and loop.sh is NOT covered by CI or `LOCAL_DOD`
(tsc/test/build don't touch bash). So verification CANNOT rely on CI; you MUST verify behaviourally
(below) and keep the change minimal and `bash -n`-clean, or you can silently break the loop.
