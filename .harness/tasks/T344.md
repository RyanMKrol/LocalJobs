## Do

Reintroduce a selectable 'System' mode to the dashboard's light/dark control (`ThemeControls`, `dashboard/app/ui.tsx`), building on top of the prerequisite flicker-fix task's lazy-init `useMode()`. This is a DELIBERATE UX reversal of T308 (which collapsed a prior Dark/Light/System tri-state, T190, down to a binary click-to-toggle button) — the owner explicitly asked for System back, with a computer/robot-style icon.

**Decided interaction shape — implement exactly this, do not redesign:** cycle-on-click through THREE states in a fixed order using the SAME single, compact header button used today — Dark → Light → System → Dark → … — rather than a dropdown/popover/menu. Reasoning (for context, not to be re-litigated): T308's rationale for a compact, unobtrusive single-button control still holds, and the popover CSS the old T190 tri-state picker used (`.theme-popup`/`.theme-grid`/`.theme-opt`/`.theme-section`, visible via `git show 952829b -- dashboard/app/ui.tsx`) was deleted by T308's hardcode commit — reinstating a full picker menu would mean inventing new popover UI, which is out of scope for reintroducing just the mode choice. Each state gets its own icon and accurate accessible label:
  - Dark: 🌙 — clicking moves to Light.
  - Light: ☀️ — clicking moves to System.
  - System: 🖥️ (a computer icon, per the owner's 'computer/robot-style' request) — clicking moves to Dark. In System state the rendered appearance (`data-mode`) follows the OS `prefers-color-scheme` live, exactly like the original T190 System option.

Implementation steps:
1. Change `ModeId` back to a tri-state type: `export type ModeId = 'dark' | 'light' | 'system';`
2. Update `resolveMode(stored, osPrefersDark)` to explicitly handle `stored === 'system'` the same way it already handles `null` (follow `osPrefersDark`), for clarity.
3. Update `useMode()`:
   - Its lazy `useState` initializer (from the prerequisite task) must read the STORED mode choice from `localStorage.getItem('localjobs.mode')` synchronously when present, falling back to `'system'` when absent — NOT infer the icon from `data-mode` alone, since `data-mode` only ever holds the resolved `light`/`dark` value and never `'system'`, so System's correct icon can't be recovered from it.
   - Rename/replace `toggle()` with a `cycle()` that advances `dark → light → system → dark`. On landing on `'system'`, WRITE the literal string `'system'` to `localStorage` (do not remove the key) and set `data-mode` to the OS-resolved effective value.
   - Keep a `matchMedia('(prefers-color-scheme: dark)')` change listener attached ONLY while `mode === 'system'` (mirroring the old T190/pre-T142 `useTimeMode` pattern) so the page reacts live to OS changes while in System mode, and detach it otherwise.
4. Re-check the pre-paint script (`PREPAINT`) in `dashboard/app/layout.tsx`: it currently treats any stored value that isn't exactly `'light'` as dark-forcing. Verify it still resolves correctly now that `'system'` is a valid explicit stored value (i.e. `mo==='dark'` → force dark, `mo==='light'` → force light, anything else including `'system'` or absent → follow OS `prefers-color-scheme`) — adjust only if you find an actual mismatch.
5. Update `ThemeControls`: wire the button's `onClick` to the new cycle function; update `aria-label`/`title` to state the CURRENT mode and what clicking does (e.g. `'System mode (follows OS) — click for dark mode'`); render 🌙 / ☀️ / 🖥️ per the current `mode`.
6. Update the doc comment block above `useMode`/`ThemeControls` (currently states 'strictly binary, no System option' per T308) to describe the new tri-state cycling behaviour, referencing this task.
7. Update root `CLAUDE.md`'s theme/mode section (the `data-mode` bullet describing the T308 binary toggle) to describe the reintroduced System option and the Dark→Light→System cycle interaction — required by this repo's Definition of Done ('documentation is part of every change').

Out of scope: the theme-family (`sunny-8bit`/`pixel-picnic`/`default`) and font system stay hardcoded exactly as T308 left them — this task concerns ONLY the light/dark/system mode control.
