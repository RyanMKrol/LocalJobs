## Do

Make `dependsOn` task ids **clickable** on the Backlog view so you can jump to a dependency and read
it. In `dashboard/app/backlog/page.tsx`, the dependency ids currently render as plain text in two
places: the waiting-section "needs: T001, T002" pill, and each row's expanded body ("depends on:
<ids>"). Turn each id into a clickable control that, on click, **scrolls the referenced task's row
into view AND auto-expands it** (opens its `CollapsibleRow` body showing its spec/details). If the
target lives in a section that's collapsed by default (e.g. Done), **expand that section too** so the
row is visible. A brief highlight on the landed row is a nice touch (optional).

Implementation notes / anchors: `CollapsibleRow` owns its `expanded` state locally — to let one row
open another, lift the expanded state to the parent `BacklogPage` (a controlled `expandedId` /
`openTask(id)` handler passed down), or give each row a stable DOM id (e.g. `id="task-T123"`) and use
`scrollIntoView` + a shared "open this id" signal. Tasks are split across the Ready / Waiting /
Needs-human / Done sections, and a dependency may live in any of them. Keep it mobile-responsive (the
`@media (max-width: 640px)` block) and theme-variable-driven.
