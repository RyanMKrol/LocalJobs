in T413: remove the now-redundant ISIN/OpenFIGI resolution logic from `stocks-snapshot`, so it goes

resolution work `stocks-snapshot` was still doing on top of that.

## Context (read before making any change)

After T413, `stocks-sync` is a 5-job DAG: `stocks-fetch` → `stocks-resolve-names` →
`stocks-snapshot` → `stocks-watch` → `stocks-notify`. `stocks-resolve-names` now resolves each
position's company `name` via Trading212's metadata endpoint and writes `data/out/
named-positions.json`. `stocks-snapshot` reads that file — but as of T413 it ALSO still
independently calls `fetchInstrumentsMetadata`/`resolveTickers`/`resolveOpenFigiTickersBatched`
itself (the OLD ISIN/OpenFIGI resolution, left in place deliberately by T413 to keep that task
small) — meaning the Trading212 metadata endpoint currently gets called TWICE per workflow run
(once in `stocks-resolve-names`, once in `stocks-snapshot`), which is wasteful. This task removes
`stocks-snapshot`'s copy entirely.

`src/services/trading212.service.ts` is a shared service file also used independently by the
`stock-digest` workflow for its own ISIN/OpenFIGI resolution (Finnhub industry lookup) — **this
task must not touch `stock-digest` or any of the shared service's ISIN/OpenFIGI exports**
(`resolveTickers`, `resolveOpenFigiTickersBatched`, `fetchInstrumentsMetadata`,
`Trading212Instrument`, `NormalizedPosition`'s `isin?`/`resolvedTicker?` fields) — only
`stocks-snapshot.ts`'s own USE of them is being removed, not the shared functions themselves.
