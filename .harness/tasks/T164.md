## Do

The cron-schedule **hint** (the "?" help that shows a plain-English description of the schedule) is
missing from the **movies** workflow row on the Workflows page (`/workflows`), even though movies
has a schedule (`0 9 1 * *` — monthly, the 1st at 09:00).

Root cause (NOT a duplicated-component problem — the owner's hunch was reasonable but it's not
that): the schedule IS rendered by the single shared `CronBadge` component (`dashboard/app/ui.tsx`),
which calls `cronToEnglish(expr)`; when `cronToEnglish` **can't translate** an expression it returns
the expr unchanged, and `CronBadge` then renders just the raw expr with **no "?" hint** (`if
(english === expr) return <span className="mono">{expr}</span>`). `cronToEnglish` doesn't handle the
**monthly / day-of-month** pattern, so movies gets no hint.

Fix:
1. **Extend `cronToEnglish`** to describe the patterns the workflows actually use — especially the
   **monthly day-of-month** form (`0 9 1 * *` → e.g. "at 09:00 on the 1st of each month"). Audit the
   others in use and make sure each returns a sensible English string so the "?" hint appears for
   all of them: `0 2 * * *` / `0 3 * * *` (daily), `0 9 * * 1` (weekly, Mondays), `0 9 1 * *`
   (monthly). For genuinely-unusual expressions it's fine to keep falling back to the raw expr with
   no hint — but the common workflow patterns must all be covered.
2. **Framework-wide consistency scan (the owner's good instinct):** confirm EVERY place a
   workflow/cron schedule is displayed routes through the single shared `CronBadge`, not a bespoke
   renderer. A quick look shows the Workflows list (`workflows/page.tsx`) and the workflow detail
   (`workflows/[name]/page.tsx`) both use `CronBadge`; verify nothing else (overview, etc.) renders
   a schedule directly, and route any such case through `CronBadge` so the hint is consistent.
3. `cronToEnglish` is a pure function — add a small unit test (it lives in the dashboard, so verify
   via the build + a manual check, or factor it so it's testable) asserting the daily/weekly/monthly
   patterns each translate. Mobile-responsive.
