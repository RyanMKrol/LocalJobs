## Overview

The dashboard's nav tab and page heading were already renamed from "Services" to "Integrations"
(`dashboard/app/layout.tsx:51`, `dashboard/app/services/page.tsx:137`), but the URL is still
`/services` because the Next.js App Router route folder (`dashboard/app/services/`) was never
renamed. Rename the route folder to `dashboard/app/integrations/`, update the nav link, and add a
redirect from the old `/services` path so it isn't a dead link if bookmarked/muscle-memoried.

## Do

1. Rename the route folder `dashboard/app/services/` → `dashboard/app/integrations/` (the folder
   contains a single `page.tsx`; App Router routes are named by folder path, so this alone moves the
   page from `/services` to `/integrations`). Keep the file's contents unchanged except for anything
   below.
2. Update the nav link in `dashboard/app/layout.tsx` (currently line 51:
   `<a href="/services">Integrations</a>`) to `<a href="/integrations">Integrations</a>`.
3. Add a redirect from the old path so `/services` keeps working (bookmarks / muscle memory) instead
   of 404ing. Use Next.js's `redirects()` in `dashboard/next.config.js`, alongside the existing
   `rewrites()` block (both can coexist on the same `nextConfig` object):
   ```js
   async redirects() {
     return [
       { source: '/services', destination: '/integrations', permanent: false },
     ];
   },
   ```
   Use `permanent: false` (a 307) rather than `permanent: true` (308) — this is an internal
   single-user dashboard, not a public site needing SEO-grade permanence, and a temporary redirect
   avoids browsers/proxies aggressively caching the old→new mapping in a way that's annoying to undo
   if the route is ever renamed again later.
   Do **not** touch anything under `/api/services/**` (`dashboard/app/lib/api.ts`,
   `src/api/server.ts`) — that's the daemon's HTTP API for the `services` DB table/concept and is
   unrelated to this dashboard-route rename; only the dashboard PAGE path changes.
4. Update the hermetic dashboard test harness `dashboard/scripts/_dashboard-harness.mjs` (the single
   living artifact for page list + fixtures + interaction flows — required by this repo's
   CLAUDE.md/`.harness/CLAUDE.md` UI convention, not optional):
   - The `PAGES` entry `{ name: 'services', path: '/services' }` → change `path` to `/integrations`
     (keep the `name` as `'services'` or rename to `'integrations'`, your call, just keep it
     consistent with the `FLOWS` entry below).
   - The `FLOWS` entry for the "Consumers of …" / `service-consumers-modal` flow currently has
     `path: '/services'` — update it to `/integrations` too, so the flow still opens the right page
     before clicking into the modal.
   - Leave the `/api/services` fixture interception (`pathname === '/api/services'`,
     `pathname.startsWith('/api/services/')`) unchanged — that's the API path, not the dashboard
     route, and stays `/api/services` per point 3 above.
5. Note for context only (no action needed): T342 ("Extract a shared table/category-panel component
