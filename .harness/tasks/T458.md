## Overview
A non-markdown output artifact (e.g. a JSON file) is unreadable when previewed today: the workflow-run page's Stage I/O popover (StageIoLists.tsx) ALWAYS renders content through the markdown viewer regardless of what format the item declared, which collapses a pretty-printed JSON file's indentation into one unbroken paragraph. The OTHER Input/Output surface (WorkflowOutputSection.tsx, used on a workflow's own definition page) already has a working format-dispatch mechanism (T282) but only a `markdown` renderer was ever registered — everything else silently falls back to a raw, unstyled `<pre>` block. Every workflow's stages already correctly declare `detail.format` at the data layer (confirmed for places/perfumes/stocks-sync/plex-language-fix/listening-digest/stock-digest) — the gap is purely on the rendering side, in two different places that don't share code. Add real `json` and `text` renderers to one shared dispatch mechanism, and wire BOTH surfaces through it.
