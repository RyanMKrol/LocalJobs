## Overview

Part of a repo-wide sweep giving every job a real paragraph-length `description` (surfaced on
`/jobs/[name]` and `/runs/[id]`) instead of a one-liner. This task covers the 4 jobs in the
`stocks-sync` workflow. The mechanism was already built by the foundational task this depends on —
this is pure content authoring.

## Do

Rewrite the `description` string of each of the following 4 job files to a real 3-6 sentence
paragraph — what the stage does step by step, what it reads/writes, and any notable behavior (the
read-only Trading212 integration, ISIN/ticker resolution, the fresh-breach vs. per-run ledger key
split). Ground each in root `CLAUDE.md`'s detailed `stocks-sync` section and the stage's own code.
Plain prose only, no markdown syntax:

- `src/workflows/stocks-sync/stages/stocks-fetch.job.ts` (`stocks-fetch`) — currently: "Fetch the
  owner's current open equity positions from Trading212 Invest (and ISA, if configured), normalize +
  tag each by account, and write data/out/raw-positions.json for stocks-snapshot to resolve. No ticker
  resolution here."
- `src/workflows/stocks-sync/stages/stocks-snapshot.job.ts` (`stocks-snapshot`) — currently: "Resolve
  each fetched position's ISIN + real-world ticker (Trading212 instruments-metadata + OpenFIGI), then
  write data/out/portfolio.json + data/out/portfolio.md. Idempotent per calendar day."
- `src/workflows/stocks-sync/stages/stocks-watch.job.ts` (`stocks-watch`) — currently: "Check every
  held position's gain since average buy price and record whether it freshly breached 30%+, writing
  this run's fresh breaches for stocks-notify to send."
- `src/workflows/stocks-sync/stages/stocks-notify.job.ts` (`stocks-notify`) — currently: "Send ONE
  push naming every position that freshly breached 30%+ above its average buy price this run (per
  stocks-watch's fresh-breaches.json); a no-op when nothing breached."

Note the strictly READ-ONLY nature of the Trading212 integration (GET-only, per root `CLAUDE.md`'s
non-negotiable broker rule) is worth stating plainly in `stocks-fetch`'s paragraph.
