## Overview
The admin page's "Delete all workflow output" bulk action currently resets every non-running workflow unconditionally, with no awareness of the owner's per-workflow `certified` flag (T497, the plain user-set "reviewed & settled" marker). This adds a certified-check so a certified workflow is skipped by the bulk reset the same way an actively-running workflow already is, while leaving the per-workflow 'Danger zone → Clear output data' button (POST /api/workflows/:name/reset-output) fully unaffected — that endpoint still resets a certified workflow if the owner explicitly clicks it on that workflow's own page.
