## Overview
Three Claude-narrating jobs treat a rate/usage limit as a hard failure, so they burn their whole retry budget against a limit that won't clear in minutes (and project-summarize keeps hammering every remaining repo). Make a rate-limit a soft pause/break that defers to the next run, matching how perfumes/movies/tv-recs already handle it.

## Do
In each of the three jobs, before the existing generic `if (!result.ok) throw` handler, branch on result.rateLimited (already provided by src/services/claude.ts). (1) workouts-progress.ts (~247-249) and (2) stock-digest-build.ts (~271-273) are single-artifact stages: on rateLimited, log a clear warn (e.g. 'Claude rate/usage limit hit — pausing this stage, will retry next run', level 'warn') and RETURN early without writing output, without marking any ledger row, and without throwing — so the item stays un-done and a later scheduled/manual run resumes it. (3) project-summarize.ts (~244-246) is a per-repo loop: on rateLimited, log the same kind of warn and BREAK the loop so the remaining repos are left for the next run — do NOT mark the current repo 'failed', do NOT continue hammering, and do NOT let the rate-limit break trip the T416 'throw if failed>0' end-of-loop rule (a rate-limit pause is a soft-stop like a quota skip, not a per-item failure). In all three, a NON-rate-limit failure (result.ok false, rateLimited false) keeps its existing throw/failed path unchanged. Mirror the wording/shape already used in movies/stages/recommend.ts (~155).

## Done when
workouts-progress and stock-digest-build soft-return on a Claude rateLimited outcome (warn logged, no output written, no ledger mark, no throw, item left un-done); project-summarize breaks its repo loop on rateLimited without failing the run or marking the current repo failed. A non-rate-limit Claude failure still throws/fails as before in all three. A new unit test per job injects a rateLimited runner and asserts the soft outcome, plus a companion assertion that a plain failure still fails. Existing tests green; npx tsc --noEmit and npm test pass.
