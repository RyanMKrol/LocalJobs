## Overview

The "Ignored (N)" sub-sections on the movie-recommendations / tv-recommendations / missing-tv-seasons
workflow detail pages have their heading and subtext hugging the panel's top-left edge with no
breathing room. Fix the styling and, while doing so, de-duplicate the byte-identical markup that's
currently copy-pasted 4 times into one shared `IgnoredSection` component.

## Do

**Root cause (already confirmed, high confidence).** `dashboard/app/workflows/[name]/page.tsx` has
the IDENTICAL "Ignored" block duplicated 4 times — `TvRecsManager` (~line 165), `MovieRecsManager`
(~line 303), `MovieGapsManager` (~line 469), `MissingSeasonsManager` (~line 643):

```tsx
<div className="panel">
  <h3 style={{ fontSize: 15, marginTop: 0 }}>Ignored ({ignored.length})</h3>
  <p className="muted" style={{ fontSize: 13 }}>...</p>
  <table>...</table>
</div>
```

Two compounding causes:
1. The inline `marginTop: 0` on the `<h3>` explicitly zeroes its top margin, removing any vertical
   gap between the panel's top border and the heading.
2. Neither the `<h3>` nor the `<p className="muted">` has any left margin/padding, and `.panel` itself
   has **zero padding** (`dashboard/app/globals.css:88`, `.panel { ... }` — no padding declared) — so
   both lines sit flush against the panel's left border too.

There's a second wrinkle worth understanding before you fix this: `TvRecsManager` and
`MovieRecsManager` wrap their tables (including the Ignored panel) in a `<div
className="movie-gaps-scroll">`, and `.movie-gaps-scroll .panel { padding: 12px 16px; ... }`
(`globals.css:221`) DOES give those two some padding already — but `MovieGapsManager` and
`MissingSeasonsManager` do NOT use that wrapper for their Ignored panel, so theirs get literally zero
padding from any source. This is why the bug is more visually severe in some instances than others
(the reported screenshot — subtext "Suppressed by you — never reported or notified, even though you
don't own them" — matches `MovieGapsManager`, one of the fully-unpadded ones). **Don't rely on the
`.movie-gaps-scroll` wrapper for the fix** — the new shared component must own its own spacing so it
looks correct wherever it's used, wrapped or not.

**Fix: extract a shared `IgnoredSection` component AND fix the spacing in the same change.** Create
`dashboard/app/components/IgnoredSection.tsx`, following the existing shared-component convention
(T268 — see `RunButton.tsx` / `Pill.tsx` for the pattern) exporting something like:

```tsx
export function IgnoredSection({
  count,
  subtitle,
  children,
}: {
  count: number;
  subtitle: string;
  children: ReactNode;
}) {
  return (
    <div className="panel ignored-section">
      <h3 className="ignored-section-heading">Ignored ({count})</h3>
      <p className="muted ignored-section-subtitle">{subtitle}</p>
      {children}
    </div>
  );
}
```

Deliberately keep the table content as `children` (NOT baked into the component) — each of the 4
call sites has a DIFFERENT table shape (`Show/Year/Genre`, `Film/Year/Genre`, `Film/Collection/Year`,
`Show/Season`), so the component should only own the panel chrome (padding + heading + subtitle
spacing), not the table. This also means a later task adding a per-row "un-ignore" action column can
just add a `<th>`/`<td>` inside each manager's own table — no fight over this component's shape.

Add the CSS in `dashboard/app/globals.css`:
```css
.ignored-section { padding: 16px 18px; }
.ignored-section-heading { margin: 0 0 4px; font-size: 15px; }
.ignored-section-subtitle { margin: 0 0 10px; font-size: 13px; }
```
**Specificity gotcha to get right:** `TvRecsManager`/`MovieRecsManager` nest this panel inside
`.movie-gaps-scroll`, and `.movie-gaps-scroll .panel { padding: 12px 16px; ... }` has the SAME
specificity (two classes) as a bare `.ignored-section` rule — so whichever rule is declared LATER in
the cascade wins. Place the new `.ignored-section` rule AFTER `.movie-gaps-scroll .panel` in
`globals.css` (or give it a selector like `.panel.ignored-section` to be unambiguous), and confirm
visually (see Done when) that the padding actually applies in ALL 4 locations, not just the two that
already had some padding from the wrapper.

Replace all 4 duplicated blocks in `dashboard/app/workflows/[name]/page.tsx` with
`<IgnoredSection count={ignored.length} subtitle="...">` wrapping each manager's existing `<table>`
(keep each manager's own subtitle wording — they currently differ slightly, e.g. "Dismissed by you —
never recommended or notified again." vs "Suppressed by you — never reported or notified, even though
you don't own them." — preserve that per-manager wording, just pass it as the `subtitle` prop).

Keep mobile-responsive (the existing `@media (max-width: 640px)` block in `globals.css` — no new rule
should need to be added there unless the new padding causes overflow at 402px, which it shouldn't).
