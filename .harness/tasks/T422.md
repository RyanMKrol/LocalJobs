## Overview

`project-summarize` (`src/workflows/projects-sync/stages/project-summarize.ts`) is the exact job whose
real incident triggered the T394 convention ("an item-loop job must fail its own run if it failed
any item this run"). A real run reported the job/run status `succeeded` while its own logs showed
`error: failed to summarize RyanMKrol/LocalJobs: ...` and a final summary line of
`project-summarize complete ā€” 0 summarized, 0 skipped` ā€” i.e. every repo in that run failed, yet
the run was recorded as a success because the per-repo `catch` block swallows the error. Fix
`runProjectSummarize` to throw when any repo failed this run, so the RUN itself is correctly
marked failed, downstream DAG dependents are blocked, and the job's own retry budget kicks in
(a retry will naturally skip already-`success` items via the existing marker check and resume at
the first not-yet-done one).
