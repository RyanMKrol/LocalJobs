## Do

Replace the per-task "Mark as reviewed" toggle in the Backlog **Done** section with a bulk
checkbox-selection flow that reviews multiple tasks in a SINGLE git commit.

Today each done backlog task has its own review button → `POST /api/backlog/:id/reviewed`, which the
daemon handles in `src/api/server.ts` by writing `.harness/reviews.json` (via the pure
`setReviewEntry(raw, id, reviewed, at)` helper + temp-file/rename) and then committing+pushing that one
file `[skip ci]` under the `src/core/repo-lock.ts` mkdir lock (`commitReviewsFile` /
`defaultCommitReviews`). That's one commit per task.

API (`src/api/server.ts`):
- Add a pure helper `setReviewEntries(raw, ids[], reviewed, at)` that sets every id in `ids` to
  `{ reviewed, at }` in the reviews map and returns the serialized JSON (mirroring `setReviewEntry` but
  for many ids), so it is unit-testable with no git.
- Add a new bulk endpoint `POST /api/backlog/reviewed-bulk` accepting `{ ids: string[] }` (and
  `reviewed` defaulting to `true` — this feature only ever marks reviewed). It writes reviews.json for
  ALL ids at once (temp+rename) and commits+pushes ONCE under the same repo lock, with a single commit
  message covering the batch (e.g. `reviews: bulk N reviewed [skip ci]`). Reuse the existing
  serialization mutex so concurrent review POSTs can't interleave. It is a mutating endpoint and MUST
  sit behind the same loopback/token guard as the other POST routes. Validate the body (non-empty array
  of string ids) → 400 on bad input.

Dashboard (`dashboard/app/backlog/page.tsx`, `dashboard/app/lib/api.ts`):
- Remove the per-task review toggle entirely from the Done list. Add a checkbox on each unreviewed done
  row, a select-all header checkbox (selects/deselects all currently-unreviewed done tasks), and a
  single "Mark as reviewed" action that POSTs the selected ids to the bulk endpoint and refreshes.
- This is mark-reviewed-only — there is no bulk un-review, and removing the per-task toggle means the
  single-task un-review affordance goes away (an accepted consequence). Already-reviewed done tasks
  render as reviewed (no checkbox).

Update `CLAUDE.md`'s reviews.json note (the architecture "ONE deliberate write exception" bullet + the
