## Overview

Part of a repo-wide sweep giving every job a real paragraph-length `description` (surfaced on
`/jobs/[name]` and `/runs/[id]`) instead of a one-liner. This task covers the 11 jobs in the `movies`
workflow (`movie-recommendations`). The mechanism was already built by the foundational task this
depends on — this is pure content authoring.

## Context

The 8 recommender branch jobs (`rec-random-1/2/3`, `rec-auteur`, `rec-canon`, `rec-thin-genre`,
`rec-older-era`, `rec-world-cinema`) are thin wrappers generated by a shared factory
(`makeBranchJob(id)` in `src/workflows/movies/stages/recommend.ts`); each branch's OWN `description`
actually lives in a shared spec object in `src/workflows/movies/stages/branches.ts` (one `description`
field per branch, currently short one-liners like "Stage: serendipity branch ${n} — recommends diverse
films from a stratified-random slice of the owned library."). Edit the descriptions THERE, not in the
individual `rec-*.job.ts` wrapper files (those files have no `description` field of their own to
change).

## Do

Rewrite each branch's `description` in `src/workflows/movies/stages/branches.ts` (8 branches) to a
real 3-6 sentence paragraph — what makes that branch's recommendation lens distinct (its prompt
strategy / library slice), what it reads (the movie snapshot + taste profile) and writes (its raw
suggestion list for `rec-merge` to pool). Then do the same for the 3 non-branch job files, each in its
own file:

- `src/workflows/movies/stages/snapshot.job.ts` (`movie-snapshot`) — currently: "Stage 1: snapshot the
  Plex movie library by GUID — each movie, its TMDB id, and taste metadata (genres/directors/decades/
  countries)."
- `src/workflows/movies/stages/franchise-gaps.job.ts` (`franchise-gaps`) — currently: "Stage 2: detect
  franchise gaps via the TMDB Collections API — released franchise films you don't own (no quality
  filter)."
- `src/workflows/movies/stages/merge.job.ts` (`rec-merge`) — currently: "Merge stage: pool the 8
  branches' raw film suggestions, TMDB-verify each (real, un-owned, not previously recommended), dedupe
  across branches, and balance per genre into the final recommendation list."
- `src/workflows/movies/stages/notify.job.ts` (`movie-gaps-notify`) — currently: "Final stage: ONE
  combined monthly digest + report — the newly-detected franchise gaps you don't own AND the
  TMDB-verified film recommendations (separate sections; deduped per film; owner-ignored items
  excluded)."

Ground every paragraph in root `CLAUDE.md`'s detailed `movie-recommendations` section (it documents the
DAG shape, the quality bar, the re-prompt top-up loop, and the merge/notify behavior in depth) and each
file's own code. Plain prose only, no markdown syntax.

## Done when

- All 11 `movies` jobs (8 branches in `branches.ts` + `movie-snapshot`, `franchise-gaps`, `rec-merge`,
  `movie-gaps-notify`) have a real, accurate paragraph description grounded in their actual behavior —
  not a one-liner.
- `npx tsc --noEmit` and `npm test` are green.
