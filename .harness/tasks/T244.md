## Do

Make `dashboard/scripts/visual-check.mjs` capture modal/overlay interaction flows faithfully by
adding an opt-in **per-flow viewport-capture mode**. Today every capture uses
`page.screenshot({ fullPage: true })`, but a modal's dimmed backdrop (`.db-modal-overlay`) only
covers the viewport — so on a modal flow (e.g. `overview-theme-picker`) only the top viewport-band is
dimmed and the page below the fold shows through at full brightness, misrepresenting how the modal
actually looks to a user (who only ever sees it over the viewport).

Add a per-flow flag in the `FLOWS` list in `dashboard/scripts/_dashboard-harness.mjs` (e.g.
`viewport: true` / `fullPage: false`) and honour it in `capture()` in `visual-check.mjs`: a flow so
marked is screenshotted at viewport size (`fullPage: false`) instead of full-page. Mark the existing
`overview-theme-picker` flow as a viewport/modal flow. This is **tooling-only** — do NOT change any
product/dashboard CSS (the overlay is not broken for real users; this only affects screenshot
fidelity).

## Done when

- A `FLOWS` entry can declare it should be captured at viewport size, and `capture()` in
  `visual-check.mjs` honours it (`fullPage: false` for those; `fullPage: true` for everything else).
- The `overview-theme-picker` flow is marked as a viewport/modal flow.
- Running `node dashboard/scripts/visual-check.mjs` exits 0 with all captures passing, and visually
  inspecting `dashboard/scripts/visual-out/overview-theme-picker.png` shows the dimmed backdrop
  covering the **full frame** — no un-dimmed page content below the fold.
- All other (non-modal) flows and all `PAGES` still capture `fullPage` exactly as before (their
  screenshots are unchanged in extent).
- NOTE: this task touches ONLY the visual-check harness scripts (`visual-check.mjs` +
  `_dashboard-harness.mjs`), which are the scope-exempt living-artifact files — NOT rendered product
  UI. So its verification is the **visual-check run + screenshot inspection above**, and it does NOT
  require `npm --prefix dashboard run build` or `node dashboard/scripts/mobile-check.mjs`.
