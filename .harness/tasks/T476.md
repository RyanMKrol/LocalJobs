## Overview
The repo's generic response cache (service_cache / T451, driven by callService(name, fn, { cacheKey })) currently uses ONE hardcoded global TTL constant (SERVICE_CACHE_TTL_MS in src/core/services.ts, 5 minutes) for every 'api'-category service. This task makes the TTL configurable per service so a slow, expensive-to-crawl dependency like Plex can use a longer window than a cheap lookup like TMDB.

## Do
Add an optional `cacheTtlMs?: number` field to `ServiceDefinition` in src/core/types.ts (a code-level field, not dashboard-editable/persisted — same treatment as the existing code-only `minIntervalMs`/`maxJitterMs` fields, NOT the same treatment as the dashboard-editable rate/daily/monthly caps). In src/core/services.ts, change callService's cache-check/cache-write logic to use `service.cacheTtlMs ?? SERVICE_CACHE_TTL_MS` (the existing global constant becomes the fallback default, not the only value) wherever the TTL is read to decide whether a cached row is still fresh. Set `cacheTtlMs: 3 * 60 * 60 * 1000` (3 hours) on the `plex` ServiceDefinition in src/services/plex.service.ts, with a comment explaining why Plex specifically gets a longer window (a full-library crawl is expensive; Plex library contents change slowly enough that a multi-hour-stale cache is acceptable, unlike TMDB's cheaper per-lookup calls which keep the existing 5-minute default). Do not change any other service's TTL.

## Done when
`ServiceDefinition.cacheTtlMs` exists as an optional field in src/core/types.ts; callService's cache freshness check in src/core/services.ts uses the service's own `cacheTtlMs` when set, falling back to the existing `SERVICE_CACHE_TTL_MS` constant when unset, with no behavior change for any service that doesn't set the field; `plex.service.ts` sets `cacheTtlMs` to exactly 3 hours (10800000 ms); a unit test in src/core/services.test.ts proves a service with a custom `cacheTtlMs` uses that value instead of the global default (e.g. a cached entry older than the global default but younger than the custom TTL is still served as a cache hit); `npx tsc --noEmit` and `npm test` are green.
