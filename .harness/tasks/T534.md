## Overview
Once R03's route table exists (this unit depends on conv-api-refactor-route-table), a `GET /api` endpoint that returns the table as `[{ method, pattern }]` is ~5 lines — a poor-man's OpenAPI making the API self-describing for future sessions/agents at near-zero cost. This CANNOT land standalone; it needs the route table.

## Do
Add a `GET /api` route (as an entry in the `routes` table itself) whose handler returns `json(res, 200, { routes: routes.map(r => ({ method: r.method, pattern: r.pattern })) })` — deriving directly from the same `routes` array the matcher uses, so it stays automatically in sync as routes are added/removed. It's a read (GET) so it needs no mutation guard. Add a server.test.ts case asserting `GET /api` returns a non-empty array of `{ method, pattern }` including at least one known route (e.g. the `/api/workflows/:name` pattern).

## Done when
`GET /api` returns `{ routes: [{ method, pattern }, …] }` derived from the live route table; a server.test.ts case asserts the shape and a known pattern is present; `npx tsc --noEmit` + `npm test` green.
