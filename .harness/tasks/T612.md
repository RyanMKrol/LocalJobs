## Overview
The shared status pill (StatusBadge / .badge, dashboard/app/globals.css) can wrap its emoji onto its own line above the text label instead of staying on one line, because .badge has no white-space: nowrap or flex-shrink: 0. This has been observed in two structurally different containers — a narrow column-flex cell (.last-run-cell on the Workflows list page) and a shrinking row-flex list item (.stage-io-item in the Stage I/O / Outputs panel) — confirming the fix belongs on the shared .badge class itself, not a per-page patch.

## Do
In dashboard/app/globals.css, add `white-space: nowrap;` and `flex-shrink: 0;` to the `.badge` rule (currently lines 116-119: `display: inline-block; padding: 2px 9px; border-radius: 999px; font-size: 12px; font-weight: 600; border: 1px solid transparent;`). Do not touch StatusBadge's markup in dashboard/app/ui.tsx (lines 42-50) or .badge-emoji (globals.css line 621) — the fix is scoped to the .badge rule's own CSS properties. After the change, rebuild the dashboard and restart it per the root CLAUDE.md 'Always restart what you changed' rule (`npm --prefix dashboard run build && launchctl kickstart -k gui/$(id -u)/com.ryankrol.localjobs-dashboard`).

## Done when
dashboard/app/globals.css's `.badge` rule includes both `white-space: nowrap` and `flex-shrink: 0`. `npm --prefix dashboard run build` succeeds. Run `node dashboard/scripts/visual-check.mjs` and visually confirm (look at the screenshots) that: (1) on the Workflows list page, the 'Last Run' column's status pill renders its emoji and label on ONE line (not emoji-above-label-below) inside the pill shape; (2) on a workflow-run detail page's Outputs / Stage I/O panel, each item's status pill also renders on one line, not compressed/wrapped internally. Additionally run `node dashboard/scripts/mobile-check.mjs` and confirm it stays green (no new horizontal overflow at ~402px) — this is explicitly called out because `flex-shrink: 0` on a globally-used component is exactly the kind of change that could force overflow in a narrow flex row that used to rely on the badge being compressible, so mobile-check must be checked, not assumed safe, on top of the normal layer=='ui' auto-covered visual verification.
