## Do

Add a **"Copy logs" button** to the log view so the owner can grab ALL the logs for a run with one
click. Today the logs stream, scroll, and get long, so there's no easy way to copy the full text.

- On the run detail page's log panel (`dashboard/app/runs/[id]/page.tsx`) add a small **Copy** button
  (e.g. in the log panel header) that copies the **full** log content — every line, in order, with
  its timestamp + level + message — to the clipboard via `navigator.clipboard.writeText`. Copy the
  COMPLETE log, not just the lines currently scrolled into view.
- Give brief click feedback (e.g. the button flips to "Copied!" for ~1–2s, then back).
- Apply it to every log view that shows run/framework logs — the **job run detail** page AND the
  **workflow-run** framework-log view (if it has one) — ideally via a shared treatment/component so
  it's consistent.
- Style it as a standard themed button (consistent across the default + joyful themes). Handle the
  clipboard-unavailable case gracefully (e.g. fall back to selecting the text / a non-fatal
  message). Mobile-responsive.

## Done when

Each log view has a "Copy" button that copies the full ordered log text (timestamp + level +
message) to the clipboard with click feedback ("Copied!"); it works on the run detail (and the
workflow-run framework) log views via a shared, consistent treatment; it's a themed, mobile-safe
button with a graceful fallback when the clipboard API is unavailable; `npm --prefix dashboard run
build` passes and `node dashboard/scripts/mobile-check.mjs` stays green.
