## Overview
Adds a mechanically-enforced test — mirroring the EXISTING `src/core/gate-coverage.test.ts` pattern exactly — that walks every job the registry loads and fails `npm test` if any job declares `inputKeys()` without also declaring a valid `inputKeysService` (T487's new field) naming a real, registered service. This is the actual teeth behind the `inputKeysService` contract; without this test the field is just documentation. Depends on T487 (the field must exist and be correctly retrofitted onto all 5 current `inputKeys()`-declaring jobs before this test can pass).

## Do

1. Load every workflow/job the registry discovers (same mechanism `gate-coverage.test.ts` already uses — the registry scans `src/workflows/**/*.job.ts` and `src/services/*.service.ts` + `src/workflows/**/*.service.ts`).
