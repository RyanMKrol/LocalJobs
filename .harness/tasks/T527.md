## Overview
tv-recs-notify marks recommendations notified even when the monthly digest push FAILS, silently losing the month's TV recs forever. On res.ok===false it still records every rec 'success' in the announce-once ledger AND appends them to recs-history.json, so if ntfy is down the digest never reaches the owner yet every rec is permanently marked notified and next month's recs are different. Fix it to throw on a failed push before marking, matching the movies/missing-tv-seasons sibling notify stages.

## Do
In src/workflows/tv-recs/stages/tv-recs-notify.ts, after the digest push at line 92 and the result log at line 93, add a guard `if (!res.ok) throw new Error(`Digest push failed — ${res.error}`);` BEFORE the ledger-mark loop (lines 96-110, markWorkItem RECS_JOB/recKey 'success') and BEFORE the appendHistory call (line 113). This mirrors the established pattern in src/workflows/movies/stages/notify.ts (lines 148-152: log the push result, then `if (!res.ok) throw` before the mark loops) and src/workflows/missing-tv-seasons/stages/notify.ts — a copy-paste omission when this stage was cloned from movies. Do NOT change any behaviour on a successful push (res.ok true): the mark loop, appendHistory, report write, and progress/log lines stay exactly as they are. The report write (writeReport, line 80) already happens BEFORE the push and should stay there — it's the full active-backlog report, independent of whether the digest was delivered; only the ledger mark + history append (the 'announce-once' state) must be gated behind a delivered push. On a thrown push failure the run fails honestly, maxRetries engages, and the next run re-notifies because no rec was recorded done and recs-history.json was never written.

## Done when
On a failed push the stage throws BEFORE marking the ledger or appending history, so the run is recorded failed (engaging maxRetries) and a later run re-sends. A successful push behaves exactly as before (recs marked 'success', history appended, report written). A new/extended test in src/workflows/tv-recs/stages/tv-recs-notify.test.ts injects a failing pushFn (returns { ok: false, error }) and asserts: the run throws; isWorkItemDone(RECS_JOB, recKey(id), 1) is false for every rec that would have been notified; and recs-history.json is unchanged (either absent or byte-identical to before the run). An existing success-path test still passes (recs marked done, history appended). Definition of Done: `npx tsc --noEmit` and `npm test` both green.
