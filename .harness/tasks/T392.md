## Overview

The owner hit a Plex request that timed out after 30 seconds even though the Plex server was reachable — it just needed more time to respond. Bump the shared Plex client's default per-request timeout from 30 seconds to 5 minutes so slow-but-healthy responses don't get killed early.

## Do

`src/jobs/plex/config.ts:34` currently reads:

```ts
requestTimeoutMs: Number(process.env.PLEX_REQUEST_TIMEOUT_MS ?? 30_000),
```

Change the fallback default from `30_000` to `300_000` (5 minutes — matches the owner's own suggestion of "about five minutes"):

```ts
requestTimeoutMs: Number(process.env.PLEX_REQUEST_TIMEOUT_MS ?? 300_000),
```

This value is `plexConfig.requestTimeoutMs`, consumed by `src/jobs/plex/client.ts`'s `plexGet` as the actual per-request HTTP timeout (around line 270, `timeout: plexConfig.requestTimeoutMs`, and line 291's `req.on('timeout', ...)` handler that throws `Plex request timed out for ${path} (${plexConfig.requestTimeoutMs}ms)`). It is still fully env-overridable via `PLEX_REQUEST_TIMEOUT_MS` — this task only changes the fallback default, not the mechanism.

**Do NOT touch** the separate `PROBE_TIMEOUT_MS` constant near the top of `client.ts` (~line 24, `Number(process.env.PLEX_PROBE_TIMEOUT_MS ?? 1_500)`) — that is the DHCP-host-discovery LAN-scan per-probe timeout, a deliberately short, unrelated value ("short so a dead host fails fast during a scan", the T149 self-healing IP resolution feature). This task is scoped strictly to the ongoing-request timeout (`requestTimeoutMs`), once the correct host has already been found. A sibling backlog task covers a DHCP-scan-budget issue in the same file — keep this task's diff limited to the `requestTimeoutMs` default only.

Also update `.env.example` at the `PLEX_REQUEST_TIMEOUT_MS` entry (around line 64-65):

```
# Optional Plex request timeout (ms). Default 30000.
# PLEX_REQUEST_TIMEOUT_MS=30000
```

to reflect the new default:

```
# Optional Plex request timeout (ms). Default 300000 (5 minutes).
# PLEX_REQUEST_TIMEOUT_MS=300000
```

This client (`src/jobs/plex/client.ts`) is shared by every Plex-touching workflow (`missing-tv-seasons`, `tv-recommendations`, `movie-recommendations`), so this one config change benefits all three.

Root `CLAUDE.md` and `README.md` do not currently mention this specific timeout value (checked — no `PLEX_REQUEST_TIMEOUT_MS`/`30000`/`30_000` references outside `config.ts`, `client.ts`, and `.env.example`), so no further doc updates are required beyond the `.env.example` line above.
