## Do

**BACKEND half** of decoupling the workflow Output section from markdown-only (the dashboard renderer
dispatch is the dependent task **T282** — do NOT touch dashboard files here).

Today every workflow output is treated as a markdown artifact: the output endpoint
(`GET /api/workflows/:name/output?job=&key=` and the per-run variant, `safeOutputMarkdown` in
`src/api/server.ts`) reads a `.md` file from the job's `data/out/` tree and the dashboard renders it
as markdown.

Make an output work item able to **declare its render form** — a `format`/`kind` recorded on the
item's `detail` via `markWorkItem` (`detail` is freeform JSON, so no schema/store change is needed) —
and **extend the read-only output API to serve a declared NON-markdown form's data**, keeping the
SAME guards (confined to each job's `data/out/` tree, realpath/prefix checks, files only, no
traversal, no paid/remote calls). `markdown` stays a form with its current behaviour. Document the
convention + how to add a new form in `CLAUDE.md`'s Output-section section, so T282 (renderer) and
T263 (a structured size table) can add a form without re-plumbing the endpoint.

Scope: `src/api/server.ts`, `CLAUDE.md`. This is the API + convention half only.
