## Do

For workflow graphs **wider than their container** (`dashboard/app/components/Dag.tsx`), the graph
currently wraps onto new rows that each run **left-to-right**, so the eye has to jump all the way
back to the far left at the start of every row. Lay wide graphs out in a **boustrophedon ("snake")
pattern** instead: row 1 left→right, then down, row 2 **right→left**, then down, row 3 left→right,
etc. — so the flow is continuous and easier to parse (the end of one row sits directly above the
start of the next).

**What triggers this — be precise.** The trigger is the graph's **horizontal extent**: a workflow
with **many sequential waves** (a long pipeline LENGTH) whose columns overflow the container's
width. It is **NOT** triggered by a **tall** wave — a single wave with many parallel stages stacked
vertically (e.g. the movies workflow's 8 `rec-*` branches in one column) is only ~4 waves wide, fits
horizontally, and must **NOT** snake. Tall is fine; only horizontal overflow from a long sequence of
waves snakes.

- When the wave columns exceed the container width and wrap, **reverse the visual direction of
  alternate rows** (odd rows L→R, even rows R→L) so consecutive waves stay adjacent across each row
  break rather than restarting at the left.
- **Route the connectors to follow the snake:** the edge from the last node of an L→R row to the
  first node of the next (R→L) row should drop essentially straight **down** (they're vertically
  adjacent), and arrows within a reversed row point **leftward**. Gate marks stay on their edges,
  consistent with the connector rendering from **T175** (this builds on that — hence the dependency).
- Only applies when the graph **wraps** (exceeds container width). A narrow single-row graph is
  unaffected (and is centered per T180). The **phone** layout already stacks vertically into a
  single column, so snaking is a desktop/wide concern — keep the phone layout unchanged/sensible.
- Verify with a graph that genuinely has **many sequential waves** (a synthetic long-pipeline DAG —
  enough waves to overflow the container width). **Do NOT use the movies workflow as the example:**
  it's only ~4 waves wide (just tall), so it fits horizontally and must keep rendering as a single
  non-snaked row. Confirm the snake reads as one continuous flow with connectors/arrows following it,
  and that short/narrow + tall-but-not-wide graphs (like movies) still render as before.

## Done when

A workflow graph wider than its container wraps in a snaking / boustrophedon layout (alternate rows
reverse direction) with connectors routed to follow the snake (straight down between rows, leftward
within reversed rows) and gate marks on their edges; narrow graphs are unaffected; it's readable and
correct at desktop width and sensible on phone; `npm --prefix dashboard run build` passes and
`node dashboard/scripts/mobile-check.mjs` stays green.
