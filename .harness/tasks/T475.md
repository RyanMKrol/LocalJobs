## Overview
The dashboard has 5 `*_overridden` flags across services/workflows/jobs (services.limits_overridden, workflows.schedule_overridden, workflows.max_concurrency_overridden, workflows.notify_enabled_overridden, jobs.timeout_ms_overridden) that persist a dashboard edit forever in SQLite but never feed back into the committed manifest/service-definition code default. This task builds a small, single-stage, report-only audit workflow (mirroring src/workflows/plex-space-saver/'s shape — no LLM, plain SQLite reads, weekly cron) that surfaces any override that has been live and unchanged for 2+ weeks as a candidate to fold into code, plus documents the convention that overrides are provisional, not permanent.
