## Overview
Add a one-to-two-sentence `idempotencyNote` to each of the 17 workflow manifests (the field added by the dependent wiring task), explaining in plain language HOW that specific workflow is idempotent — what it tracks and whether/how items get re-evaluated — so the owner can understand a workflow's idempotency model at a glance on its dashboard detail page. This is real per-workflow authoring, not boilerplate: each workflow's idempotency model genuinely differs, and a wrong summary is worse than none, so every sentence below MUST be verified against that workflow's own current `CLAUDE.md` (and code, if the CLAUDE.md is ambiguous) before being added — do not copy the drafts below blindly if the workflow's actual current behavior has since diverged from what's written here.
