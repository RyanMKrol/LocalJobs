## Overview
Cache `finnhub` company-profile / industry lookups (industry classification rarely changes) for 22
hours. Outcome of the T479 caching-adoption decision; the caching mechanism already exists
(T451/T476).

## Do
1. Set `cacheTtlMs: 79_200_000` (22 hours) on the `finnhub` `ServiceDefinition` in
   `src/services/finnhub.service.ts`.
2. Add a stable `cacheKey` to the `callService('finnhub', ...)` read site in
   `src/workflows/stock-digest/stages/stock-sector-lookup.ts`, keyed by the Finnhub query symbol
   (the resolved ticker used for the lookup): `` `finnhub:profile:${symbol}` ``.
3. Cache the read (company-profile GET) only.

## Done when
- `cacheTtlMs: 79_200_000` is set on the `finnhub` service def.
- The call site passes `finnhub:profile:<ticker>` as its cacheKey.
- `npx tsc --noEmit` and `npm test` are green.
- Verified a repeated profile lookup of the same ticker within 22h is served from `service_cache`
  (hermetic scratch-DB check) rather than a second live call.
