## Overview
Every workflow detail page should show a short, plain-language sentence explaining HOW that specific workflow is idempotent (what it tracks, whether/how items get re-evaluated), so the owner can understand a workflow's idempotency model at a glance without reading its CLAUDE.md or source. This unit adds the field/schema/API/UI wiring only — mirroring the existing `category` field's mechanism exactly (manifest-owned, no dashboard edit UI, no `_overridden` column, always refreshed from code on sync). A separate follow-up task authors the actual 17 per-workflow sentences.
