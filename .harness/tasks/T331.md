**Evaluate each file against 4 criteria:**
1. **Inconsistencies** — does the file contradict another in-scope file, or contradict the actual current code/behaviour it describes?
2. **Excessive verbosity** — sections that restate the same point multiple times, or over-explain something a shorter rule + example would cover.
3. **Cuttable content** — narrow one-off incident narratives, superseded conventions, or detail that no longer earns its space (especially relevant to `CLAUDE.md`'s 114KB size and `LIMITATIONS.md`'s 37KB size).
4. **Staleness** — describes behaviour, file paths, ports, or conventions that no longer match the current code.

## Done when

Each of the 16 in-scope files listed above has been read and reviewed against all 4 criteria. Findings are written down (e.g. as a short review note per file, or inline as a list) covering: any contradictions found (with the two conflicting statements/files named), any sections flagged as excessive verbosity or cuttable (with a proposed cut or compression), and any stale content found (with what it claims vs. what's actually true now). For findings the owner agrees should be acted on, the corresponding edits are made to the file(s) in the same task (trimming/fixing/updating) and the change is committed. It is acceptable for the outcome to be "reviewed, no changes needed" for a given file as long as that file was genuinely read and considered against all 4 criteria, not skipped.
