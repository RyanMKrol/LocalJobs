## Overview

Part of a repo-wide sweep giving every job a real paragraph-length `description` (surfaced on
`/jobs/[name]` and `/runs/[id]`) instead of a one-liner. This task covers the 2 jobs in the
`workouts-sync` workflow. The mechanism was already built by the foundational task this depends on —
this is pure content authoring.

## Do

Rewrite the `description` string of each of the following 2 job files to a real 3-6 sentence
paragraph — what the stage does step by step, what it reads/writes, and any notable behavior
(idempotency, the 6-month baseline/current comparison, which sets get skipped). Ground each in
root `CLAUDE.md`'s `workouts-sync` section and the stage's own code. Plain prose only, no markdown
syntax:

- `src/workflows/workouts-sync/stages/hevy-sync.job.ts` (`hevy-sync`) — currently already a two-
  sentence description ("Paginate the Hevy API and append new workouts to a local full-history JSON
  file. Idempotent via the work_items ledger (keyed by Hevy workout id).") — expand it into a fuller
  paragraph covering pagination, rate-limiting via `src/services/hevy.service.ts`, and exactly what
  "full data" means (title, exercises, sets).
- `src/workflows/workouts-sync/stages/workouts-progress.job.ts` (`workouts-progress`) — currently:
  "Compute a per-exercise 6-month progress comparison (best set / volume / est. 1RM) from the local
  workouts history and write a Claude-narrated markdown report. Idempotent per calendar month." —
  expand it to cover the exact baseline/current period definitions, the three metrics (best single
  set, total volume, Epley-estimated 1RM), and the null-weight/reps skip rule.

## Done when
