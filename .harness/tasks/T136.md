Make the Backlog "Mark reviewed" action **durable and remote** — today
`POST /api/backlog/:id/reviewed` does a field-scoped atomic write of `reviewed` INTO
`.harness/TASKS.json` but never commits it, so the flag only reaches GitHub if the loop's
next `mark_done` happens to sweep it into an unrelated status commit, and a working-tree
reset on the box silently loses it. Fix this with the design the owner chose (interview

a disjoint git path from everything the loop commits, merges are always clean. Do all of:
