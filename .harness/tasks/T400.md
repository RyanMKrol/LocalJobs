## Overview

The "⚙︎ Local Jobs" brand link in the dashboard header now wraps onto two lines at
normal desktop width. This is a direct regression from T323 (the Admin nav tab
addition): giving `<nav>` an inline `width: '100%'` starves its flex sibling
`.brand` of space inside the `header.site .inner` flex row, since `.inner` has no
`flex-wrap` at desktop widths. Fix it so the brand always renders on one line while
preserving T323's original mobile fix (the nav's six links wrapping onto extra rows
within their own box, rather than the page overflowing horizontally).

## Do

- In `dashboard/app/globals.css`, update the `header.site .brand` rule (currently
  around line 79: `font-weight: var(--heading-weight, 700); font-size: 16px; color:
  var(--text); font-family: var(--font-display); overflow-wrap: anywhere;`) to add
  `flex-shrink: 0;` and `white-space: nowrap;`. This guarantees `.brand` keeps its
  natural (shrink-to-fit) width inside the `header.site .inner` flex row, so the
  sibling `<nav>` — which still declares its own inline `width: '100%'` — shrinks to
  take only the REMAINING space instead of starving `.brand`, at every viewport
