## Overview
The owner wants to 'certify' a whole workflow (e.g. places, stocks-sync) as reviewed-and-settled, distinct from the harness's own per-task reviewed/done overlays, so they stop re-reviewing the same workflow's output every session. This unit adds the underlying data + API: a plain boolean 'certified' flag on a workflow, toggle-only (no auto-invalidation, no code-seeded default to reconcile against), mirroring the existing `enabled`/`notify_enabled` toggle shape already on the `workflows` table.
