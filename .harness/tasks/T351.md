## Overview

The "✎ Edit limits" button (and, in edit mode, the "✓ Save" / "Cancel" pair) in the last column of
the Integrations page's service tables sits too close to the table's right edge. A quick screenshot
check during shaping this task confirmed it's worse than cosmetic in the edit-mode case: at a normal
1440px desktop width, the "Cancel" button is actually CLIPPED (rendered as "Cal…") by the surrounding
`.panel`'s `overflow: hidden`, because the action `<td>` forces `whiteSpace: 'nowrap'` on two
side-by-side buttons in a column that's only 12% of the table's width. Fix it by giving the action
column a bit more room AND letting its button(s) wrap onto a second line when the column is still too
narrow — the owner's own suggested approach — rather than overflowing/clipping.

2. **Let the action-column button(s) wrap instead of overflow.** Add rules scoped to
   `.services-table`'s last column in `dashboard/app/globals.css` (near the existing
   `.services-table { table-layout: fixed; }` rule around line 93):
   ```css
   /* Action column (Edit limits / Save+Cancel) — never let a button's label force horizontal
      overflow past the panel's clipped edge; wrap onto a second line instead if it doesn't fit. */
   .services-table td:last-child { text-align: right; }
   .services-table td:last-child .btn { white-space: normal; text-align: center; }
   ```
   This overrides `.btn`'s default `white-space: nowrap` (globals.css ~line 114) ONLY for buttons in
   this column, so "✎ Edit limits" can break onto two lines ("✎ Edit" / "limits") instead of
   overflowing when the column is narrow (e.g. at a viewport in between the 1440px desktop case and
   the 640px mobile breakpoint, where `table-layout: auto` doesn't yet kick in).

   actual clipping bug. Replace the inline `whiteSpace: 'nowrap'` `<td>` with a plain `<td>` wrapping
   the two buttons in a flex container that can wrap onto two lines:
   ```tsx

     <span style={{ display: 'inline-flex', flexWrap: 'wrap', gap: 6, justifyContent: 'flex-end' }}>

       <button className="btn secondary" onClick={() => setEditing(null)} disabled={busy}>Cancel</button>
     </span>
   </td>
   ```
   (The `.services-table td:last-child { text-align: right }` rule from step 2 keeps this
   right-aligned like the rest of the column; `justifyContent: 'flex-end'` keeps the buttons
   right-aligned within their own wrap container too.)
