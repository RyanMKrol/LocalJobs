## Overview
stocks-sync gain-alert breach notifications are marked 'notified' one stage too early AND the push result is ignored, so a failed or interrupted alert is silently lost for months. stocks-watch writes the `<account:ticker>::notified` episode row the instant it detects a fresh breach, but the push happens one stage later in stocks-notify, which ignores the pushFn { ok, error } return and logs 'notification sent' unconditionally. If ntfy is down or the run is cancelled between the two stages, the alert never arrives yet the episode row exists, so that position never re-alerts until it drops below 30% and re-crosses. Fix: move the episode-marking into stocks-notify AFTER a verified-ok push, and throw on failure — matching the movies/missing-tv-seasons pattern. Strictly read-only alerting, no broker mutation.
