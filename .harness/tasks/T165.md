## Do

Table rows **jitter** (their content visibly shifts) when you hover into them, which is jarring.
Root cause: under the joyful themes (T142), `tbody tr:hover` gets `transform: translateY(-2px)`
(`dashboard/app/globals.css` ~lines 551–553) — a hover-LIFT. A `transform` on a table ROW (`<tr>`)
reflows / sub-pixel-jitters as the cursor enters (table rows don't transform cleanly), so the row's
content jumps. The base highlight `tbody tr:hover { background: var(--panel-2) }` is fine — only the
transform jitters.

Fix:
- **Remove the `translateY(-2px)` hover-lift from table rows** (`tbody tr:hover` in the joyful-theme
  block). KEEP the hover-lift on the elements where it works well — the `.statcard` and `.dag-node`
  cards — just not on `<tr>`.
- Give table rows a **non-layout-shifting** hover instead: the background change (keep it), and if
  more feedback is wanted, a subtle `box-shadow` inset or a left-accent — anything that does NOT
  move the row's content. No `transform`, no border-WIDTH change on hover (those reflow a row).
- This applies under the joyful themes; the default theme already just changes the background
  (which is fine and unaffected). Keep the reduce-motion path working (it already disables the
  transform).
- (The "beige" highlight is the joyful theme's `--panel-2` — that's intentional; only the jitter
  needs fixing. A gentler hover background is acceptable but optional.)
- Mobile-responsive.

## Done when

Hovering a table row no longer jitters/shifts its content — the `translateY` hover-lift is removed
from `tbody tr` (it stays on `.statcard`/`.dag-node` cards); rows still get a smooth, non-reflowing
hover highlight (background, optionally a subtle inset shadow); `npm --prefix dashboard run build`
passes and `node dashboard/scripts/mobile-check.mjs` stays green.
