## Overview
The listening-digest workflow currently writes ONE markdown file per monthly run — a current-month (period=1month) top-albums/top-tracks digest from Last.fm. This task adds a SECOND markdown file to the same run, covering the trailing 3 months (period=3month), reusing the exact same fetch/filter/render pipeline a second time with a different period, output filename, and ledger key. No new job, stage, or workflow member is added — both files are produced by the existing single `lastfm-digest` job/stage in one run.
