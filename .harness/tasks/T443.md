## Overview

Add the actual mutating step to the `plex-language-fix` workflow: for every file the scan stage flagged
as needing a change, set Plex's default audio (and, when needed, subtitle) stream via Plex's own
official `PUT /library/parts/<id>` endpoint. The owner has explicitly said they do NOT want to
manually approve every run â€” this runs unattended on the workflow's schedule â€” so this task must also
build in the safety net the owner already validated live (a Butler-triggered server backup right before
applying, plus a recorded before/after log a separate undo script can replay), since nobody will be
reviewing each run by hand.

## Do

**Depends on the `plex-language-scan` stage** (a separate, already-built task) which writes
`data/out/language-scan.json` â€” an array of per-file entries each with a `status`
(`'change' | 'already-correct' | 'no-match'`), a `partId`, `currentAudio`/`currentSubtitle`, and
`proposedAudio`/`proposedSubtitle` (see `src/workflows/plex-language-fix/types.ts`'s `FileEntry`).
There is **no `'ambiguous'` status** â€” the owner has explicitly said to remove any ambiguous carve-out;
every `'change'` entry (including what would previously have been a channel-count tie) gets applied the
same way, using the best-judgment pick the scan stage already resolved.

There is a **gitignored, repo-root scratch reference** at `/plex-language-fix/apply.ts`, `mutate.ts`,
and `undo.ts` â€” read these for the validated approach (they were dry-run tested against 1,758 real
proposed changes but never actually applied live), but **rebuild the logic properly inside this
workflow's own files** rather than importing/wrapping the scratch script.

1. **Add a Plex stream-selection PUT helper.** The scratch `mutate.ts`'s `plexPutStreams(partId,
   audioStreamId, subtitleStreamId)` calls `PUT /library/parts/<partId>?audioStreamID=<id>&
   subtitleStreamID=<id>&allParts=1` (the SAME documented endpoint Plex's own clients call when a user
   manually picks a track) â€” port this into **`src/core/plex-client.ts`** as an exported
   `plexPutStreams` function (not workflow-local): it's a Plex-generic mutating capability like `plexGet`
   is a Plex-generic read, so it belongs with the other shared Plex primitives rather than being
   duplicated inside this one workflow, in case a future workflow needs it too. Reuse the existing
   `resolvePlexHost()` + the module's scoped `insecureAgent` (self-signed cert) â€” don't add a second TLS
   bypass.
2. **Also port a Butler-backup trigger** into `src/core/plex-client.ts` (e.g. `triggerButlerBackup()`),
   calling `POST /butler/BackupDatabase` with the Plex token â€” this was validated live by the exploration
   session (produces a real dated backup within about a minute). This is the safety net that stands in
   for a human reviewing each run.
3. **`stages/apply.job.ts` + `stages/apply.ts`** (+ `stages/apply.test.ts`) in
   `src/workflows/plex-language-fix/`:
   - Reads `data/out/language-scan.json`. Filters to `status === 'change'` entries with a usable
     `proposedAudio.streamId` (skip + warn on anything malformed, mirroring the scratch `apply.ts`'s
     `skipped` counter).
   - **Before the FIRST real PUT of the run**, calls `triggerButlerBackup()` once (log the result; a
     failed backup trigger should log a clear WARN but not block the apply â€” the per-file undo log
     below is the primary safety net, Butler is the secondary one, matching the exploration's own
     framing).
   - For each qualifying file, calls `plexPutStreams(partId, proposedAudio.streamId,
     proposedSubtitle?.streamId)` and records the outcome. **Idempotent, always-record semantics** (mirror
     `stocks-sync`'s `stocks-watch` T300 pattern, referenced in root `CLAUDE.md`): call `markWorkItem`
     for every `'change'` entry processed this run â€” `'success'` with `detail` describing what was
     applied (`{ name, path: <applied-log path>, format: 'json' }` pointing at the per-run log below, per
     the "success detail must describe what it produced" convention), or `'failed'` with the error â€” so a
     quiet run with zero eligible changes doesn't look identical to a run that actually processed
     changes. Key each row by `partId` (a Plex Part id is globally unique on the server).
   - Writes a **per-run applied-changes log** to `data/out/applied-log-<ISO-timestamp>.json` (or
     appends to a single running log â€” your call, but it MUST record, per file: `partId`, `file` path,
     `beforeAudio`/`afterAudio`/`beforeSubtitle`/`afterSubtitle` (each `{ streamId, label }`), `outcome`
     (`'applied' | 'failed'`), and a timestamp â€” this is the exact data the undo script below needs to
     revert a file, self-contained (carries both before AND after, so undo never has to cross-reference
     the scan file).
   - Log verbosely (every file applied/failed/skipped, a final summary: attempted/applied/failed/skipped
     counts, whether the Butler backup fired).
4. **`contracts.ts`** â€” extend with the gate for the `plex-language-scan â†’ plex-language-apply` edge.
   Reuse the SAME `plexLanguageScanContract()` factory from task 1 as both the scan job's `produces` and
   this job's `consumes` (the established convention in this repo when both sides of an edge assert the
   same shape â€” see root `CLAUDE.md`'s gate-collapse note).
5. **Add a manual, NEVER-scheduled undo script**, `scripts/plex-language-undo.ts` (top-level `scripts/`,
   alongside this repo's other one-off admin scripts like `scripts/backfill-service-usage.ts`) â€” reads
   the most recent applied-log file (or one passed by path/flag), and for each `'applied'` entry, calls
   `plexPutStreams` with the recorded `beforeAudio`/`beforeSubtitle` to revert it, writing its own
   `undo-results` log. Default dry-run, `--apply` flag to actually revert (mirroring the scratch
   `undo.ts`'s safety default). This is NOT a workflow stage/job â€” it is a manually-invoked recovery tool,
   run by the owner directly (`tsx scripts/plex-language-undo.ts [--apply] [path]`), matching how this
   repo's other manual admin scripts work.
6. **Wire the new stage into the workflow.** Update `plex-language-fix.workflow.ts`'s `jobs` array to add
   `{ job: 'plex-language-apply', dependsOn: ['plex-language-scan'] }`.
7. Update root `CLAUDE.md`'s `plex-language-fix` bullet (added in the scan task) to describe the apply
   stage, the Butler-backup + per-file-undo safety net, and the "no ambiguous carve-out â€” the owner
   explicitly chose full automation over per-run manual sign-off" decision. Update the workflow's own
   `CLAUDE.md` too, and document `scripts/plex-language-undo.ts` in whichever doc lists this repo's
   manual admin scripts.
