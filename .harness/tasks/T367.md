## Overview

`projects-sync` (`src/jobs/projects-sync/`) has 2 stages — `github-sync → project-summarize` — and
currently has NO validation gate between them (this is the specific workflow flagged by the owner
after noticing a real workflow run with no gate at that boundary). Add a real `ArtifactContract` at
the boundary, mirroring the existing `places`/`perfumes` pattern.

## Do

Read root `CLAUDE.md`'s "Validation gates between workflow stages" section first, then
`src/core/types.ts` (`ArtifactContract`/`ArtifactShape`/`GateResult`/`ExpectationResult`) and
`src/core/dag.ts`'s `deriveGates`. Use `src/jobs/places/contracts.ts` as your primary worked example
(specifically `normalizedPlacesContract` — closest analog: a single JSON array hand-off with a
non-empty requirement, since a projects catalog with zero repos genuinely means nothing to
summarize). Give it a proper `shape` + per-expectation `checks[]` + `sample`, not a bare stub.

**Create `src/jobs/projects-sync/contracts.ts`** with one factory function:

**`projectsCatalogContract(file = projectsSyncConfig.catalogPath)`** — key `'projects-catalog'`.
Validates `data/out/projects.json` (written by `github-sync.ts` at
`src/jobs/projects-sync/stages/github-sync.ts:118`): a plain JSON **array** of `CatalogEntry` records
(`src/jobs/projects-sync/stages/github-sync.ts:36-45`):
`{ repoId: string, name: string, fullName: string, description: string, url: string, language: string, topics: string[], pushedAt: string, defaultBranch: string }`.
Expectations: file exists + parses as JSON; top-level value is an array; the array is **non-empty**
(unlike `stocks-sync`, an empty catalog genuinely means `project-summarize` has nothing to do — this
mirrors places' `normalizedPlacesContract`, not stocks-sync's leniency); every entry has a non-empty
string `repoId` and a non-empty string `name` (the two fields `project-summarize` keys its work off).

`projectsSyncConfig.catalogPath` is already exported from `src/jobs/projects-sync/config.ts`.

**Wire the contract into the 2 stage jobs:**
- `src/jobs/projects-sync/stages/github-sync.job.ts` — add `produces: [projectsCatalogContract()]`.
- `src/jobs/projects-sync/stages/project-summarize.job.ts` — add `consumes: [projectsCatalogContract()]`.

Import from `../contracts.js` (relative from `stages/`, same pattern as
`src/jobs/perfumes/stages/fetch.job.ts` importing `../contracts.js`).
