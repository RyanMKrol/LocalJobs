## Overview
The Integrations page's service table currently prints a small 'timeout: ... · source: ...' line directly under each service's description text in the leftmost cell, and it visually reads as part of the description even though it's a separate element. Remove that line so the description cell shows only the service name/pills and its description text.

## Do
In dashboard/app/integrations/page.tsx, inside the main services table body's Service column <td> (the block containing `<div className="muted" style={{ fontSize: 12 }}>{s.description}</div>`), delete the immediately-following `<div className="muted mono service-meta" style={{ fontSize: 11 }}>...</div>` that renders `timeout: {...}` and the optional ` · source: {s.rate_limit_source}` text. Do not touch the existing dedicated read-only 'Timeout (ms)' column (rendered when not editing, around where `s.timeout_ms` is shown in its own <td>) or the editable Timeout input (shown when editing that row) — both already display the same numeric timeout value elsewhere in the row and are unaffected. Do not touch the ConsumersPanel popover's 'Rate limit provenance' section, which is the other/remaining place `rate_limit_source` is shown. If the `.service-meta` CSS class becomes unused elsewhere in the dashboard after this removal, it's fine to leave the class definition in globals.css (out of scope) — only remove the JSX usage in this file.
