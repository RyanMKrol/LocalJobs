## Overview
The plex-space-saver workflow's Output section renders its disk-size breakdown as a raw JSON dump because no renderer is registered for its custom 'size-table' output format. Add a dedicated renderer that draws one proportional horizontal bar per title so relative disk usage is obvious at a glance.

## Do
In dashboard/app/components/OutputRenderer.tsx, add a new `SizeTableOutputBody` renderer and register it in the `OUTPUT_RENDERERS` dispatch table (~lines 128-132) under the key `'size-table'` (the T458 'add a new form's renderer HERE, once' convention). The renderer receives the standard `{ content: string; truncated?: boolean }` props (same shape as MarkdownOutputBody/JsonOutputBody/RawOutputBody). It must `JSON.parse(content)` into the `SizeBreakdownFile` shape (see src/workflows/plex-space-saver/types.ts — `{ items: SizeBreakdownItem[] }` where each `SizeBreakdownItem = { title, year: number|null, type: 'movie'|'show', bytes: number, human: string }`, already sorted biggest-first) and render one horizontal bar per item, in the given biggest-first order, with bar width proportional to that item's `bytes` relative to the largest item's `bytes` (guard against divide-by-zero when the max is 0). Each bar shows the item's `title` and its human-readable `human` size label. On a JSON parse failure (or missing/empty `items`), fall back to `RawOutputBody` so the renderer never throws or renders blank — mirror the try/catch fallback pattern JsonOutputBody already uses. Add the bar-chart CSS classes to dashboard/app/globals.css (follow the existing `.md-*` / `.empty-state-panel` idiom already in that file; do NOT introduce a new styling system, and keep it mobile-safe within the phone-width media block). The exact all-items-vs-top-N behaviour and the visual-style choices (bar orientation, movie/show color distinction, showing the year) are pending owner answers in the paired questions file — apply whatever the owner decides; if still unanswered at build time, default to: render ALL items, horizontal bars ranked biggest-first top-to-bottom, single bar color, title only (no year).
