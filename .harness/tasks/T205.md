## Do

PROTOTYPE a unified **"joined output" view** for workflow detail pages — like the
movie-recommendations recommendation cards today, generalized so every workflow can present its
output in one consolidated view. This is deliberately an **experimentation/chooser** task: build
**up to 3 distinct options** for the owner to compare (different layout libraries / component
frameworks AND/OR different visual styles), behind a switch so all options are viewable live, then a
paired review task (T206) picks the winner and a follow-up (T207) hardcodes it.

The output view, in every option, must solve the core problem the owner called out:
- **Consolidated output per workflow.** Surface a workflow's produced items (e.g. movie recs, perfume
  profiles, place profiles, missing TV seasons) as a unified card/list view on the workflow detail
  page — not just the existing per-run Input→Output table.
- **De-dupe by a stable key, NOT "latest".** Output can repeat across runs (e.g. the same movie
  recommendation next month) and must NOT show duplicates. Each option must de-dupe by a stable
  per-item key (e.g. the recommended film's tmdb id, a perfume id, a place CID, a `<show>::S<season>`
  pair) rather than naively taking the latest run. Prototype a per-workflow dedupe-key convention
  (how a workflow declares the stable key for its output items) — even a rough version is fine for the
  prototype; the goal is to SEE the approaches, not finalize the framework.

Build the 3 options behind a toggle (persisted to localStorage, mirroring the existing gate-style
selector pattern in `dashboard/app/ui.tsx`) so the owner can switch between them live on a workflow
page. Scope: `dashboard/app/components/**`, `dashboard/app/workflows/[name]/page.tsx`,
`dashboard/app/lib/api.ts`, and any minimal `src/api/server.ts` endpoint needed to feed the view; a
dashboard-side npm dependency (a card/layout/gallery library) is acceptable for an option. Keep it
mobile-responsive and theme-variable-driven. Document the options in the task worklog so the reviewer
(T206) knows what each one is.

## Done when

- A workflow detail page shows a unified "joined output" view with **at least 2–3 switchable options**
  (different libraries/styles), selectable live via a persisted toggle, each rendering the workflow's
  output as a consolidated card/list view.
- Every option **de-dupes by a stable per-item key** (not latest-run), so repeated output across runs
  appears once; the prototype demonstrates a per-workflow way to declare that key for at least 2
  example workflows (e.g. movie-recommendations + one other).
- The options are documented (what each is — library/style) in `.harness/worklog/T205.md` for the
  reviewer. `npm --prefix dashboard run build`, `npx tsc --noEmit`, `npm test`, and
  `node dashboard/scripts/mobile-check.mjs` are green.
