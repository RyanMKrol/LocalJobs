## Do

Improve the **quality** and **quantity** of the movies workflow's automated recommendations
(built in T146 — the 8 branch recommenders + the TMDB-verified merge stage). Two changes:

1. **Quality threshold — recommend only well-rated films.** When the merge stage verifies each
   suggestion against TMDB (it already resolves a tmdb id), also read its **`vote_average`** and
   **drop anything below ~7.0**. To stop an obscure film with a handful of votes sneaking through on
   a fluke rating, ALSO require a sensible **`vote_count` floor** (e.g. ≥ 50–100) so the rating is
   meaningful — a 7.0 with 3 votes doesn't count. Make the threshold (`7.0`) and the vote floor
   env-overridable in `src/jobs/movies/config.ts` with those defaults.

2. **Target at least 15 recommendations.** The current output is thin (the merge dedups/verifies
   down to fewer than wanted). Aim for **≥ 15** final recommendations. Achieve it by (a) asking each
   branch for MORE candidates (raise the per-branch ask, e.g. ~8–10 each so there's headroom after
   filtering), and (b) a bounded **top-up loop** in the merge: after the first pass, if fewer than
   the target (15) survive the dedup + owned-filter + not-previously-recommended + the new quality
   threshold, re-prompt the branches for ADDITIONAL suggestions (excluding everything already
   collected / owned / previously recommended) and merge again — repeat up to a small bounded number
   of rounds, then stop. Claude CLI is free, so a couple of extra rounds is fine; keep it bounded so
   it can't loop forever. If it genuinely can't reach 15 quality picks (rare), output what it has and
   log why.
   - Make the target count (`15`) env-overridable too.

Preserve the T146 design: **stratified** sampling (so it doesn't collapse to the dominant genre) and
the **balanced output** (cap per genre) so hitting ≥15 doesn't make the list one-flavoured. The
quality threshold + the target apply to the recommendations only — the deterministic franchise gaps
are untouched.

Tests (hermetic, no live LLM/TMDB): assert the merge **drops** a sub-7.0 / low-vote-count suggestion
and **keeps** a ≥7.0 well-voted one; assert the top-up loop requests more when under target and stops
at the cap; assert the target + threshold are config-driven; assert output stays genre-balanced.

## Done when
