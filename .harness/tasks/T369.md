## Overview

`stock-digest` (`src/jobs/stock-digest/`) has 2 stages ‚Äî `stock-sector-lookup ‚Üí stock-digest-build` ‚Äî
and currently has NO validation gate between them. Add a real `ArtifactContract` at the boundary,
mirroring the existing `places`/`perfumes` pattern ‚Äî with one important twist: this boundary is
**deliberately optional** (see below), so the contract must be lenient in a way none of the other
3 gap workflows' contracts are.

## Do

Read root `CLAUDE.md`'s "Validation gates between workflow stages" section first, then
`src/core/types.ts` (`ArtifactContract`/`ArtifactShape`/`GateResult`/`ExpectationResult`) and
`src/core/dag.ts`'s `deriveGates`. Use `src/jobs/places/contracts.ts` as your structural template
(factory function + `fromChecks` idiom), but NOT its strictness ‚Äî see the leniency requirement below.

**Critical design constraint ‚Äî read this before writing the contract.** `stock-sector-lookup`
(`src/jobs/stock-digest/stages/stock-sector-lookup.ts`) resolves each ticker's industry via the
Finnhub API, gated behind `FINNHUB_API_KEY`. Per root `CLAUDE.md`: *"unset key soft-skips the whole
stage (clear WARN, no crash), which just means `stock-digest-build`'s diversification section is
omitted that run."* This means `data/out/sectors.json` may legitimately **not exist at all**, or
exist but be an **empty object**, on a perfectly healthy run (Finnhub not configured is a supported,
documented configuration, not an error state). `readSectorMap` in
`src/jobs/stock-digest/stages/stock-sector-lookup.ts` already treats a missing/unparseable file as
`{}` ‚Äî mirror that tolerance in the contract. **The gate must NOT fail a run just because
`FINNHUB_API_KEY` is unset** ‚Äî that would turn an intentionally-optional feature into a hard failure,
which is a regression, not an improvement. So:

**Create `src/jobs/stock-digest/contracts.ts`** with one factory function:

**`stockSectorsContract(file = sectorsJsonPath)`** ‚Äî key `'stock-sectors'`. Validates
`data/out/sectors.json` (written by `stock-sector-lookup.ts` at
`src/jobs/stock-digest/stages/stock-sector-lookup.ts:126`): a `SectorMap`
(`src/jobs/stock-digest/stages/stock-sector-lookup.ts:27`) ‚Äî `Record<ticker: string, string | null>`,
a plain JSON **object** (NOT an array, unlike the other 3 gap workflows), mapping ticker ‚Üí resolved
Finnhub industry name (or `null` if unresolved for that ticker).
Expectations, in order of leniency:
1. **Missing file ‚Üí `ok: true`** (not a violation) ‚Äî this is the expected shape when
   `FINNHUB_API_KEY` is unset; `sample`/`detail` should say something like "sectors.json not present
   ‚Äî sector lookup is optional (FINNHUB_API_KEY unset or no tickers resolved yet)".
2. **File present but not valid JSON ‚Üí `ok: false`** (a real corruption, unlike "absent").
3. **File present and valid JSON but the top-level value is an array (not an object) ‚Üí `ok: false`**
   (wrong shape ‚Äî a real format drift, e.g. Finnhub's response shape changed and leaked through).
4. **File present, valid JSON object (including `{}`) ‚Üí `ok: true`** ‚Äî an empty object is a
   legitimate "no tickers resolved yet" state, not a violation.
5. IF the object is non-empty, spot-check that values are either `string` or `null` (not some other
   type) ‚Äî a genuine per-entry shape check without requiring non-emptiness anywhere.

`sectorsJsonPath` is already exported from `src/jobs/stock-digest/config.ts`.

**Wire the contract into the 2 stage jobs:**
- `src/jobs/stock-digest/stages/stock-sector-lookup.job.ts` ‚Äî add `produces: [stockSectorsContract()]`.
- `src/jobs/stock-digest/stages/stock-digest-build.job.ts` ‚Äî add `consumes: [stockSectorsContract()]`.

Import from `../contracts.js` (relative from `stages/`, same pattern as
`src/jobs/perfumes/stages/fetch.job.ts` importing `../contracts.js`). Do NOT add a contract for
`stock-digest-build`'s cross-workflow read of `stocks-sync`'s `portfolio.json` ‚Äî per root
`CLAUDE.md`'s `deriveGates` note, "a consumed key with no producing upstream is an external input, not
a gate," and the two workflows are explicitly NOT DAG-linked (no cross-workflow `dependsOn` exists in
this framework) ‚Äî that read is out of scope for this task.

**Add a test file** `src/jobs/stock-digest/contracts.test.ts` mirroring `src/core/contracts.test.ts`'s
structure (synthetic fixtures in a temp dir, NO live Finnhub call): cover ALL of the pass cases above
‚Äî missing file, `{}`, and a well-formed non-empty object ‚Äî and the fail cases (invalid JSON, an
array instead of an object, a value that's neither string nor null). Also add a test that builds the
real `stock-digest` job refs (or a synthetic 2-node chain) via `buildDag` and asserts `deriveGates`
produces exactly 1 gate for the `stock-sector-lookup‚Üístock-digest-build` edge.
