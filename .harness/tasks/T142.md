## Do

Build a live, evaluatable set of "joyful" dashboard THEME + FONT options behind switchers so
the owner can mix-and-match and narrow down a direction. Direction (interview 2026-06-23):
joyful, bright, pixely/cute like the owner's "BASKET" app — cream/paper background with a
subtle dot grid, a pixel display font, soft rounded cards, emoji in joyful places, a
signature little **yellow-spark flash** on success, and gentle interactivity. This is an
EVALUATION AID — mirror the existing gate/caret/output-style choosers exactly: options live
behind a small persisted switcher (localStorage), apply live across ALL pages, and a
FOLLOW-UP (gated by review task T143) will hardcode the winners and remove the switchers +
unused options.

⚠️ TWO HARD CONSTRAINTS:
- **Density:** unlike BASKET's sparse list, the dashboard is dense — run tables, streaming
  logs, the gate DAG, stat grids. Every theme/font must keep that data LEGIBLE (pixel/retro
  fonts go on brand/headings only, never on body/table/log text).
- **Mobile:** the owner accesses this REMOTELY FROM A PHONE. Every theme, font and accent
  must work at ~402px with no horizontal overflow, legible small text, and a reachable
  switcher — `node dashboard/scripts/mobile-check.mjs` stays green and it's spot-checked at
  phone width.

Build:
1. **Theme switcher — SIX themes (2 dark + 4 bright), independent of the font axis.** The UI
   is already CSS-variable-driven (`:root` in `globals.css`: `--bg`, `--panel`, `--panel-2`,
   `--border`, `--text`, `--muted`, `--accent`, `--green`, `--red`, `--yellow`, `--grey`,
   `--amber`). Each theme is a full vibe package overriding those vars (palette, background +
   optional texture/dot-grid, card radius/shadow, accent, status colours) applied via
   `html[data-theme="…"]` over `:root`. Suggested set (refine freely): **4 bright** — Pixel
   Picnic (BASKET-like cream + dot grid + pixel display + soft green/peach + emoji), Candy
   Bright (white + multi-colour pink/teal/purple, big rounded cards), Sunny 8-bit
   (amber/retro arcade + sparkles), Soft Pastel (airy light pastels); **2 dark** — Joyful
   Dark (dark base, bright accents + emoji + sparkle) and a second distinct dark (e.g.
   Midnight Neon / synthwave — dark with vivid neon accents). All six must keep
   tables/logs/DAG/stat-tiles legible.
2. **Font switcher — 5–10 options, each a DISPLAY+BODY PAIR.** A characterful display font
   for brand/headings + a READABLE body font for dense tables/logs (so pixel/retro never
   lands on body text). Candidates: Silkscreen / Pixelify Sans / VT323 / Press Start 2P
   (display-only) paired with Nunito / Quicksand / Fredoka / Baloo 2 / system-ui (body), plus
   a couple of characterful monos (Space Mono, JetBrains Mono). Load fonts efficiently
   (`next/font/google` in `layout.tsx`, or self-hosted) with sensible fallbacks and no
   blocking FOUC. Expose `--font-display` / `--font-body` (+ `--font-mono`) vars swapped via
   `html[data-font="…"]`; wire brand/headings to `--font-display`, body/tables/logs to
   `--font-body`.
3. **Joyful accents baked into EACH theme** (not a separate axis — owner's choice), each
   demoing in context: the signature yellow-spark / sparkle flash on a run succeeding (and/or
   on hover), hover-lift on cards/rows, animated progress-bar fills, stat-tile count-ups,
   emoji status badges (✅ ❌ ⏳ ✨), and friendlier empty states ("nothing stuck — nice ✨").
   Tasteful + performant.
4. **Global "reduce motion / minimal emoji" toggle** (persisted): defaults to honouring
   `prefers-reduced-motion`; when on, it dampens/disables animations and swaps emoji for plain
   glyphs/text — for accessibility and taste.
5. **Switcher UI**: a small, unobtrusive control (header or a compact settings popover)
   offering Theme · Font · Reduce-motion, persisted via `use…()`-style hooks (mirror
   `useGateStyle` in `ui.tsx`), applied to `document.documentElement` (`data-theme` /
   `data-font` + a class). Add an inline pre-paint script in `layout.tsx` to set these from
   localStorage BEFORE first paint (no theme flash). The DEFAULT (untouched) must remain the
   current dark + system-font look so nothing regresses.
6. Confine CSS to `dashboard/app/globals.css` (theme var-sets, font vars, accents, keyframes),
   the switcher/hooks to `dashboard/app/ui.tsx`, font loading + the pre-paint script to
   `dashboard/app/layout.tsx`, and any font deps to `dashboard/package.json`.

Note in the worklog that a follow-up (gated by T143) hardcodes the chosen theme + font +
motion preference and removes the switchers + unused themes/fonts.
