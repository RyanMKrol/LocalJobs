## Overview

With `StageIoPanel` now the default for every workflow (T386), this task proves it against the two remaining trickier DAG shapes: genuine PARALLEL FAN-OUT (many sibling stages hanging off one predecessor — e.g. `movie-recommendations`' 8 recommender branches, or `tv-recommendations`' equivalent) and the `outputJob` MANIFEST-OVERRIDE case (`stocks-sync`, whose true DAG terminal stage `stocks-notify` never records `work_items` rows — see root `CLAUDE.md`'s "outputJob override (T348)" note and `src/jobs/stocks-sync/stocks-sync.workflow.ts`).

## Do

Add or extend `/stage-io` fixtures in `dashboard/scripts/_dashboard-harness.mjs` for:
- `movie-recommendations` (or `tv-recommendations`) — a workflow whose DAG has ONE stage (e.g. a snapshot stage) with MULTIPLE direct dependents in the same wave (the 8 recommender branches). Confirm each branch's own stage tab shows only ITS OWN inputs/outputs (the shared predecessor's rows as input, that ONE branch's rows as output) — not every sibling branch's rows bleeding into each other. The existing `movieRecsMembers`/`movieRecsWorkflowRun` fixtures in that file are the starting point; add the `/stage-io?job=<branch>` responses needed.
- `stocks-sync` — confirm the "Overall" tab (from T385) correctly shows `stocks-snapshot`'s rows as Outputs (the `outputJob` override), NOT an empty result from the true terminal `stocks-notify`, matching what T384 already made the BACKEND do — this task is about proving the DASHBOARD renders it correctly end to end, not re-testing the backend logic. The existing `stocksMembers`/`workflowIoStocks`/`stocksWorkflowRun` fixtures are the starting point.

If either shape reveals an actual bug in `StageIoPanel` (`dashboard/app/components/StageIoLists.tsx`) — e.g. a tab rendering the wrong stage's rows, or the Overall tab not respecting the override in the UI even though the API returns it correctly — fix it as part of this task; that's why it's in scope.

## Done when

- New/updated `/stage-io` fixtures exist for a parallel-fan-out workflow and for `stocks-sync`, each with a corresponding `PAGES` entry in `_dashboard-harness.mjs` if one doesn't already point at a suitable run.
- `npm --prefix dashboard run build`, `node dashboard/scripts/mobile-check.mjs` (all pages), and `node dashboard/scripts/visual-check.mjs` all pass.
- You look at the resulting screenshots for the fan-out workflow's run page and for `stocks-sync`'s run page — confirm (a) each parallel branch's stage tab shows only that branch's own rows, and (b) the "Overall" tab for `stocks-sync` shows `stocks-snapshot`'s ledger rows as Outputs, not an empty state. Record what you saw in this task's worklog entry.
- If a bug was found and fixed, note it explicitly in the worklog (what broke, what the fix was) — a sampled audit will check this task didn't just add fixtures without actually verifying correctness.
