## Overview

`stocks-sync` (`src/jobs/stocks-sync/`) has 3 stages — `stocks-snapshot → stocks-watch → stocks-notify`
— and currently has NO validation gates between any of them (confirmed: no `contracts.ts`, no
`produces`/`consumes` anywhere under the folder). Add real `ArtifactContract`s at both stage
boundaries, mirroring the existing `places`/`perfumes` pattern.

## Do

Read root `CLAUDE.md`'s "Validation gates between workflow stages" section first, then
`src/core/types.ts` (`ArtifactContract`, `ArtifactShape`, `GateResult`, `ExpectationResult` —
lines ~50-111) and `src/core/dag.ts`'s `deriveGates` for the mechanism. Use
`src/jobs/places/contracts.ts` as your primary worked example to mirror exactly — same
`readJson`/`fromChecks` idiom, same "factory function taking an optional path param, defaulting to
the real config path, for unit-test fixture injection" shape, same `shape: ArtifactShape` +
per-expectation `checks[]` + `sample` richness. Do not write bare `check(): { ok: true }` stubs here
— these are real, well-known JSON artifacts with a clear shape, so give them proper shape + non-empty
(where appropriate) checks like places/perfumes do.

**Create `src/jobs/stocks-sync/contracts.ts`** with two factory functions:

1. **`stocksPortfolioContract(file = stocksSyncConfig.portfolioJsonPath)`** — key `'stocks-portfolio'`.
   Validates `data/out/portfolio.json` (written by `stocks-snapshot.ts` at
   `src/jobs/stocks-sync/stages/stocks-snapshot.ts:130`): a plain JSON **array** (NOT wrapped in an
   object, unlike places) of `NormalizedPosition` records (`src/jobs/stocks-sync/stages/stocks-snapshot.ts:34-41`):
   `{ ticker: string, quantity: number, averageBuyPrice: number, currentPrice: number, currentValue: number, account: 'invest'|'isa' }`.
   Expectations: file exists + parses as JSON; top-level value is an array; IF non-empty, every entry
   has a string `ticker` and an `account` that is `'invest'` or `'isa'`. **Do NOT require the array to
   be non-empty** — a portfolio with zero open positions (everything sold) is a legitimate state, not
   a data-format violation; an empty array should be `ok: true` with a `sample` noting "0 positions".

2. **`stocksFreshBreachesContract(file = stocksSyncConfig.freshBreachesJsonPath)`** — key
   `'stocks-fresh-breaches'`. Validates `data/out/fresh-breaches.json` (written by `stocks-watch.ts`
   at `src/jobs/stocks-sync/stages/stocks-watch.ts:127`): a plain JSON **array** of `BreachLine`
   records (`src/jobs/stocks-sync/stages/stocks-watch.ts:22-27`):
   `{ ticker: string, account: 'invest'|'isa', gain: number, averageBuyPrice: number, currentPrice: number }`.
   Expectations: file exists + parses as JSON; top-level value is an array; IF non-empty, every entry
   has a string `ticker` and a numeric `gain`. **Critically: an EMPTY array is the normal, expected,
   healthy state on most runs** (most runs have zero fresh breaches) — never treat emptiness as a
   violation; only a missing file or a non-array value is a violation.

`stocksSyncConfig` (both paths) is already exported from `src/jobs/stocks-sync/config.ts`.

**Wire the contracts into the 3 stage jobs:**
- `src/jobs/stocks-sync/stages/stocks-snapshot.job.ts` — add `produces: [stocksPortfolioContract()]`.
- `src/jobs/stocks-sync/stages/stocks-watch.job.ts` — add `consumes: [stocksPortfolioContract()]` and
  `produces: [stocksFreshBreachesContract()]`.
- `src/jobs/stocks-sync/stages/stocks-notify.job.ts` — add `consumes: [stocksFreshBreachesContract()]`.

Import both factories from `../contracts.js` (relative from `stages/`, same as
`src/jobs/perfumes/stages/fetch.job.ts` imports `../contracts.js`).

**Add a test file** `src/jobs/stocks-sync/contracts.test.ts` mirroring `src/core/contracts.test.ts`'s
structure (synthetic fixtures in a temp dir via `mkdtempSync`/`writeFileSync`, NO live Trading212
call): cover both contracts' pass cases (well-formed array, empty array — must be `ok: true`), and
fail cases (missing file, invalid JSON, non-array top-level, a malformed entry). Also add at least
one test that builds a small `Dag` via `buildDag` over the real `stocks-sync` job refs (or a
synthetic 3-node linear chain) and asserts `deriveGates` produces exactly 2 gates matching the
`stocks-snapshot→stocks-watch` and `stocks-watch→stocks-notify` edges — the direct regression guard
that this specific workflow now derives gates.
