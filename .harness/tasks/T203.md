## Do

Add a reusable, per-workflow **"Clear output data / reset to run from scratch"** action, surfaced on
the workflow detail page. This is the productised, repeatable, single-workflow version of the one-time
all-workflows script in T111 — distinct from it: this one lives in the dashboard, targets ONE
workflow, and is meant to be used whenever the owner wants to re-run that workflow fresh (e.g. the
manual missing-tv-seasons reset already done by hand).

Behaviour — clear OUTPUT, preserve INPUT:
- **Clear** (scoped to the chosen workflow's member jobs): the `work_items` ledger rows + the
  `work_item_runs` rows for those jobs; the member `runs` + `run_logs`; the `workflow_runs` +
  `workflow_run_logs` for that workflow; and the workflow's OUTPUT artifacts under each member job's
  `data/out/**`.
- **Preserve (must NOT touch):** `data/raw/**` inputs, `data/chrome-profile/`, `.env`, the definition
  tables (`jobs`/`workflows`/`services`), and user settings (service limit overrides, workflow
  enabled/schedule/concurrency overrides). (Mirror T111's clear/preserve lists, scoped to one
  workflow. Do NOT clear `service_usage` — that's a cross-workflow spend meter, not this workflow's
  output.)

Shape:
- **Store.** Add a `resetWorkflowOutput(workflowName)` (or similar) in `src/db/store.ts` that performs
  the scoped DB deletes in one transaction (resolve member jobs via `getWorkflowJobs`).
- **API.** Add a mutating `POST /api/workflows/:name/reset-output` behind the same loopback/token guard
  as the other POST routes; it runs the store reset AND removes the member jobs' `data/out/**` files
  (path-confined to each job's own `data/out`, like the existing output-reading endpoints — never
  touch `data/raw`). Refuse if the workflow currently has an active run (avoid racing the executor).
- **UI.** A button in a clearly-marked "danger"/management area of the workflow detail page
  (`dashboard/app/workflows/[name]/page.tsx`) that triggers it behind an **explicit confirmation**
  (typed confirm or a clear confirm dialog naming the workflow and what will be cleared/preserved).

Update CLAUDE.md/README to document the new endpoint + action and its clear/preserve contract.

## Done when

- `POST /api/workflows/:name/reset-output` clears the named workflow's ledger + run/usage history/logs
