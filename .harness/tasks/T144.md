## Do

Build the **Plex new-seasons audit workflow** — a scheduled workflow that scans the owner's
Plex TV library, checks TMDB for newly-released *complete* seasons the owner doesn't own, and
pushes a digest notification. Fully designed + validated live in T108 (see
`.harness/worklog/T108.md`): GUID-based matching, ~35s over 621 shows, 54 real hits. Use the
owner's public PlexUtils repo as a *reference* for the Plex/TMDB endpoints, but don't copy it
wholesale (it's hand-written, XML-based, and fuzzy-matches). This is a **public example
workflow with private data/tokens** (like places/perfumes): code is committed; tokens live in
`.env` and library data in `data/`, both gitignored.

### Connectivity (config + a gotcha)
- Env (read via `process.env`; document NAMES + placeholders in `.env.example`):
  `PLEX_HOST` (e.g. `https://192.168.1.x:32400`), `PLEX_API_TOKEN`, optional `PLEX_MACHINE_ID`,
  `TMDB_API_TOKEN` (Bearer; accept legacy `TVDB_API_TOKEN` as fallback), and `PLEX_TV_SECTION`
  (default `5`).
- Plex uses a **self-signed cert** → use a scoped `https.Agent({ rejectUnauthorized: false })`
  for Plex requests ONLY (never set global `NODE_TLS_REJECT_UNAUTHORIZED`).
- **Request JSON** from Plex (`Accept: application/json`). Section items come under
  `MediaContainer.Metadata` (NOT `Directory` — that's the XML shape). The sections list IS
  under `Directory`.
- Host resolution: try `PLEX_HOST`; if unreachable, optionally LAN-scan the local /24 for a
  server on `:32400` matching `PLEX_MACHINE_ID` (PlexUtils' `scanForPlexHost` is a reference) —
  nice-to-have; a clear "Plex unreachable, set PLEX_HOST" error is an acceptable v1 fallback.

### Stages (3) — private folder `src/jobs/plex/`
1. **`plex-tv-snapshot`** — GET `/library/sections/<PLEX_TV_SECTION>/all?includeGuids=1` (shows
   + GUIDs + ratingKey) and the flat episode read `/library/sections/<id>/all?type=4`; compute
   each show's **highest owned regular season** = max episode `parentIndex` (>0) grouped by
   `grandparentRatingKey`. Emit per show `{ title, year, tmdbId (from tmdb:// guid), ratingKey,
   highestOwnedSeason }` to `data/out/`. **Re-scans fresh every run** (no skip-if-done).
2. **`tmdb-season-check`** — for each show with a `tmdbId`: GET TMDB `/tv/{id}` → `status` +
   `seasons[]`. Highest **aired** regular season = max `season_number>0` with `air_date` ≤ now.
   If > `highestOwnedSeason`, for each candidate season N (owned+1..tmdbAired) GET
   `/tv/{id}/season/{N}` and keep it ONLY if **complete** — every episode has an `air_date` and
   the latest is ≤ now (excludes still-airing seasons). **Include ended/canceled shows** (don't
   skip — revivals). Match is ALWAYS by GUID; a show with no tmdbId is flagged "unverifiable",
   never guessed. Route TMDB calls through a rate-limited **`tmdb` service** (free; jittered
   pacing as a good citizen, ~10–15/s). Emit per show its list of complete missing seasons to
   `data/out/`. Re-checks fresh every run.
3. **`plex-seasons-notify`** — gather all `(show, completeMissingSeason)` pairs. For each, check
   the `work_items` ledger keyed `(plex-seasons-notify, "<tmdbId>::S<season>")`: those NOT
   already done are newly-detected. Send **ONE digest push** (`core/notifier.ts`) listing the
   newly-detected seasons ("📺 N new season(s): Futurama S8–S10, Bridgerton S4, …"), mark each
   notified `(show,season)` done so it's never repeated, and write a markdown report artifact
   under `data/out/` recording `detail.markdown` (T110, so the dashboard can preview it). **First
   run = one big digest of the whole current backlog**; if nothing new, send nothing. The ledger
   here is a **"notified" log, not a work-done log** (only this stage uses it to dedup).

### Framework shape (important — this workflow inverts the usual pattern)
- **Do NOT declare `inputKeys()`** → the workflow is not "limitable" (inputs are discovered, not
  a static file); scheduled-only + unlimited.
- All stages key by `tmdbId`, so `root_key = item_key` naturally (no lineage args).
- Snapshot/check **recompute every run** (no skip-if-done); only notify uses the ledger.
- Record notify ledger entries only for **actionable** shows (with missing seasons) so the IO
  panel highlights those, not 600+ "up to date" rows.

### Service, schedule, privacy
- `tmdb` `ServiceDefinition` (rate-limited, `paid:false`). Place where services live (`src/services/`
  if T044 has landed, else alongside the workflow — the registry discovers either).
- `*.workflow.ts` schedule: **weekly**, e.g. `0 9 * * 1`.
- Privacy: `data/` stays gitignored (`src/jobs/**/data/`). Tokens only in `.env`. `.env.example`
  gets the var NAMES + placeholders only.

### Tests (hermetic — NO live Plex/TMDB)
Synthetic fixtures for: Plex section JSON (shows w/ guids), flat-episode JSON, TMDB `/tv` +
`/season` responses. Cover: highest-owned-season from episodes (exclude season 0); GUID
extraction; highest-aired (exclude future/season-0); **complete-season filter** (a season whose
last episode airs in the future is excluded; a fully-aired one included — model Futurama S1–7
owned vs S1–10 aired → S8,9,10, and a mid-airing season excluded); **notify dedup + digest** (a
`(show,season)` already in the ledger isn't re-notified; new ones aggregate into one digest);
**ended show NOT skipped**. Live end-to-end verification is the OWNER's (copy the 4 env vars from
PlexUtils/.env into local-jobs/.env, run once) — the agent's verification is the fixtures.

## Done when

A scheduled (weekly) `plex` workflow exists under `src/jobs/plex/` that: snapshots Plex section
5 by GUID, checks TMDB by `tmdb://` id for **complete** missing seasons (including for ended
shows), and pushes ONE digest notification of newly-detected missing seasons (first run = the
full backlog), deduped per `(show, season)` via the `work_items` ledger; it declares no
`inputKeys()` (not limitable) and re-scans every run. Tokens/library data stay private
(`.env` + gitignored `data/`); env var names are in `.env.example`. Hermetic fixture tests cover
the season math, the complete-season filter, the ended-not-skipped case, and the notify
dedup/digest; `npx tsc --noEmit` and `npm test` are green. (Live end-to-end run is the owner's,
after provisioning the env.)
