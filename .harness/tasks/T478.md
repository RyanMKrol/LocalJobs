## Overview
The service_cache table (T451) has no way to be cleared short of a manual DB edit. The owner wants a way to bust it from the dashboard, explicitly kept SEPARATE from the existing 'Delete all workflow output' danger-zone action on /admin — that action must remain scoped to work-item ledgers/run-history/output files only, never touching the response cache.

## Do
Add a new function `clearServiceCache(serviceName?: string): number` to src/db/store.ts that deletes rows from `service_cache` (all rows if no serviceName given, else scoped to that service) and returns the count deleted. Add a new mutating endpoint (e.g. `POST /api/cache/clear` accepting an optional `{ serviceName }` body) in src/api/server.ts, behind the same loopback/token guard as every other mutating endpoint, calling `clearServiceCache` and returning `{ ok: true, cleared: N }`. Create a NEW dashboard page at dashboard/app/admin-cache/page.tsx (a distinct route from /admin, linked from the main admin page or the nav) showing: a summary of current service_cache row counts (add a lightweight read query + a small `GET /api/cache` endpoint if needed to show counts per service), and a 'Clear all cached responses' button (with a lighter single window.confirm() safety net, consistent with this session's admin-page confirmation-friction ideas) that calls the new clear endpoint and refreshes the count. Add a client helper in dashboard/app/lib/api.ts. Update dashboard/scripts/_dashboard-harness.mjs's PAGES list per the living-artifact rule. Update root CLAUDE.md's file map / ports table if a new route is added.

## Done when
A new dashboard page exists at a distinct route from /admin, showing service_cache row counts and a 'Clear all cached responses' action that calls a new mutating `POST /api/cache/clear` endpoint (loopback/token guarded like every other mutating endpoint) which deletes matching service_cache rows and returns the count cleared; the existing 'Delete all workflow output' action on /admin is verified UNCHANGED (does not touch service_cache); dashboard build + mobile-check + visual-check pass showing the new page renders correctly; npx tsc --noEmit and npm test are green including new endpoint/store tests; root CLAUDE.md is updated to list the new page/route.
