## Overview
Survey every Claude-using job/workflow's current model + effort and decide, per job, whether it can move to a cheaper model or lower effort without unacceptable quality loss. Quality tolerance differs sharply between the creative recommender fan-outs and the mechanical narration/summary jobs, so this is an owner judgment call, not a unilateral downgrade.

## Do
List each Claude-using job with its env-configurable model + effort and default: movies recommenders (MOVIES_RECS_MODEL, default claude-sonnet-5); tv-recs recommenders (TV_RECS_MODEL, default claude-sonnet-4-6); workouts-sync narration (WORKOUTS_PROGRESS_CLAUDE_MODEL, default claude-sonnet-5); stock-digest narration (STOCK_DIGEST_CLAUDE_MODEL default claude-sonnet-5, STOCK_DIGEST_CLAUDE_EFFORT default medium); projects-sync repo summaries (PROJECTS_SYNC_CLAUDE_MODEL default claude-sonnet-5, PROJECTS_SYNC_CLAUDE_EFFORT default medium); perfumes (PERFUMES_MODEL_FIND/PARSE default claude-sonnet-4-6, PERFUMES_MODEL_BUILD default claude-opus-4-8, via perfumes/claude.ts); claude-warmer (hardcoded claude-haiku-4-5, already cheapest — exclude). Confirm listening-digest and places do NOT use Claude. Cheaper tiers available: claude-haiku-4-5, then claude-sonnet-5 at low/medium/high effort (see .harness/config/facets.json ladder). For each job record a decision to keep or downgrade, naming the target cheaper model/effort. Author a dependent buildable follow-up to change the confirmed defaults (a src/workflows/** config change).

## Done when
A recorded per-job decision (in the worklog) of which Claude-using jobs switch to which cheaper model/effort and which stay unchanged, and a dependent buildable follow-up authored to change the confirmed ones' defaults.
