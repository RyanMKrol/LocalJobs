## Do

The markdown-preview popover that opens from a workflow run's output (the `MarkdownModal`
in `dashboard/app/workflow-runs/[id]/page.tsx`) renders a perfume profile's YAML
frontmatter as a key-value header (`<dl className="md-fm">`), and the key/value columns are
MISALIGNED — values start at different x-positions depending on each key's length. This is
purely a RENDERING issue (the generated markdown frontmatter is valid YAML — do NOT change
the perfumes generation/template). Root cause: each `.md-fm-row` is its OWN flex container
with `.md-fm-key { flex: 0 0 auto }`, so the key sizes to its own text per row and values
never share a column. Fix the alignment AND tidy two related presentation rough-edges in the
same component (interview 2026-06-23: alignment + presentation polish). Do all of:

1. **Align the columns.** Make `.md-fm` a real two-column grid with a SHARED key column so
   every value lines up — e.g. `.md-fm { display: grid; grid-template-columns: max-content
   1fr; column-gap: 12px; row-gap: 3px; }` with `.md-fm-row { display: contents; }` so the
   `<dt>`/`<dd>` become grid items that align across all rows (this is the pattern the
   existing `.kv` blocks already use). Keep the panel's background/border/padding. At
   phone width (~402px) the long keys (`FRAGRANTICA_STATUS`, `COMMUNITY_RATING`,
   `FRAGRANTICA_URL`) must not cause horizontal overflow — stack each pair to a single
   column under the mobile breakpoint (mirror the `.kv` mobile rules already in the
   `@media (max-width: 640px)` block).

2. **Render array values readably.** Frontmatter array values currently show as raw JSON —
   `["green", "woody", "aromatic", …]`. In the renderer, detect an array-shaped value
   (try `JSON.parse`; if it yields an array of primitives) and render it as a clean
   comma-separated list (or small inline chips) instead of the raw bracketed/quoted string.
   Fall back to the raw string if it doesn't parse as an array. Keep it XSS-safe (no raw
   HTML — render as text/elements, same as today).

3. **Hide empty / null fields.** Omit rows whose value is effectively empty — an empty
   string, `null`, `~`, or an empty array `[]` (e.g. the `RATING null` and blank `SOURCES`
   rows in the current output). Don't render a key with nothing useful beside it.

Keep everything else about the popover unchanged (the react-markdown body, title, close
behaviour, truncation note). Confine changes to the frontmatter parsing/rendering in
`dashboard/app/workflow-runs/[id]/page.tsx` and `.md-fm*` styling in
`dashboard/app/globals.css`.
