## Overview
Cache the Trading212 portfolio fetch for 22 hours. Read-only per the repo's non-negotiable broker
rule (brokers are GET-only). The 22h TTL is chosen so the daily scheduled runs (>22h apart) always
fetch fresh, while intra-day / dev re-runs are served from cache. Outcome of the T479 caching-adoption
decision; the caching mechanism already exists (T451/T476).

## Do
1. Set `cacheTtlMs: 79_200_000` (22 hours) on the `trading212` `ServiceDefinition` in
   `src/services/trading212.service.ts`.
2. Add a stable `cacheKey` to BOTH `callService('trading212', ...)` portfolio-fetch READ sites:
   - `src/workflows/stock-digest/stages/stock-portfolio-fetch.ts`
   - `src/workflows/stocks-sync/stages/stocks-fetch.ts`
   Key it by the account being fetched: `` `t212:portfolio:${account}` `` (e.g. `invest` / `isa`).
3. Cache the read (portfolio GET) only — never any mutating call (the repo forbids broker mutations
   entirely; there are none here, and none may be introduced).

## Done when
- `cacheTtlMs: 79_200_000` is set on the `trading212` service def.
- Both portfolio-fetch sites pass a stable `t212:portfolio:<account>` cacheKey.
- `npx tsc --noEmit` and `npm test` are green.
- Verified a repeated portfolio fetch for the same account within 22h is served from `service_cache`
  (hermetic scratch-DB check) rather than a second live call.
