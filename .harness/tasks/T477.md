## Overview
T474 (already a real task) wraps every direct plexGet call in these 6 workflows in callService('plex', () => ...). idea-11-a (a dependency of this task) adds a 3-hour configurable cache TTL to the plex service. This task wires a cacheKey into every genuinely idempotent Plex READ call so it actually engages the existing service_cache mechanism with that 3-hour window — helping the concrete case of multiple Plex-reading workflows triggered back-to-back (e.g. the admin 'Run all workflows' button).

## Do
Prerequisites: T474 and idea-11-a must already be merged. For each of the following READ call sites, add a `cacheKey` to the `callService('plex', ..., { cacheKey })` options object, deriving the key from the request path (and any query params that vary the response) so two calls hitting the SAME Plex resource within the 3-hour TTL reuse the cached response — e.g. `cacheKey: `plex:${path}`` — following the plex-language-fix/stages/resolve.ts precedent (`cacheKey: 'tmdb-language:${d.type}:${d.tmdbId}'`) for the *shape* of a descriptive, collision-safe key: (1) missing-tv-seasons/stages/snapshot.ts — the whole-TV-section shows listing AND the flat episode listing calls; (2) tv-recs/stages/tv-snapshot.ts — its TV section snapshot call; (3) movies/stages/snapshot.ts — its movie-library snapshot call; (4) plex-space-saver/stages/scan.ts — the movies/shows/episodes section-listing calls; (5) plex-language-fix/lib.ts — the section-listing and per-title/per-episode metadata READ calls only (do NOT touch plex-language-fix/stages/apply.ts's plexPutStreams/triggerButlerBackup calls — mutating, must never be cached); (6) plex-profiles/stages/build.ts — the movies/shows/episodes section listings AND the per-title metadata detail call.
